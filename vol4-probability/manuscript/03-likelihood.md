# 第3章 尤度と最尤推定 — 「もっともらしさ」を最大化する

第1章で、不確かさを数にする言葉——確率——を手に入れました。第2章では、分布を期待値と分散という2つの数に要約できるようになりました。振り返ると、ここまでの問いには共通の形があります。**分布を知っている側から、データの出方を予言する**という形です。この分布に従うなら、次の単語はどれくらいの確率で「猫」か。この分布からサンプリングし続けたら、平均はどこに落ち着くか。いつも、分布が先、データが後でした。

ところが、第3巻から持ち越している宿題は、逆の形をしています。あのスパムフィルタで手元にあったのは、データ(ラベル付きのメール200件)だけでした。知りたかったのは、良いパラメータ $\mathbf{w}, b$ の値。つまり**データが先、パラメータが後**です。

この章では、問いの向きを反転させます。データからパラメータを当てる——この逆向きの問いに答える原理を作り、章の終わりには、その原理が第3巻の「学習 = 損失最小化」という世界観と1本の式で合流するところまで行きます。第3巻エピローグの宿題3「分類の正しい損失関数とは何か」に答えるための原理が、この章で手に入ります。

## 3.1 逆向きの問い: データからパラメータを当てる

問題は、原理がむき出しになるくらい小さくしたい。そこでこの章の主役は、確率の話で最も古くからある道具——コインにします。遠回りに見えますが、そうではありません。「表か裏か」という2択(1 か 0)の構造は、第3巻のスパム判定のラベル(スパム = 1、通常 = 0)とまったく同じです。第1章の「次の単語当て」も、「次の単語は『猫』か、そうでないか」と問い直せば2択です。コインで原理を仕留めれば、そのまま持ち帰れます。

表(1)が出る確率を $\theta$(シータ)と書きます。裏(0)が出る確率は、全部足すと1という規則(第1章)から $1 - \theta$ です。この、結果が2択しかない最小の確率分布には名前がついています。**ベルヌーイ分布**(Bernoulli distribution)です。パラメータは $\theta$ のただ1つ。$\theta = 0.5$ なら歪みのないコイン、$\theta = 0.9$ なら表ばかり出るイカサマコインです。

ここまでは順方向の話です。では、逆向きの問いを出します。

> 目の前に、歪んでいるかもしれないコインがあります。$\theta$ の値は見えません。10回投げたら、こうなりました。
>
> 表, 表, 裏, 表, 表, 表, 裏, 表, 裏, 表 (表7回、裏3回)
>
> $\theta$ はいくつだと思いますか?

ほとんどの人が「0.7」と答えます。あなたもそうでしょう。では、続けて聞きます。**なぜ 0.7 なのでしょうか。**

「10回中7回だから」——それは答えをもう一度言っただけで、理由になっていません。「表の割合で当てる」という手続きがなぜ正しいのか、どんな意味で正しいのかを、まだ何も説明できていないのです。

コインの $\theta$ 1個なら、説明できなくても直観で済みます。しかし第3巻のスパムフィルタはパラメータが3個($\mathbf{w}$ が2個と $b$)で、もう「見ればわかる」とはいきませんでした。原論文のモデルは約6500万個です。直観は通用しません。だから、コインで効いた直観を、**どんなパラメータにも適用できる手続き**に変換しておく必要があります。第3巻第3章で「いい感じ」という直観を損失という数に変換したのと、同じ作業です。

手続き化の第一歩は、いつもどおり、問いを数式に翻訳することです。

## 3.2 尤度関数: パラメータの関数として見た「データの出やすさ」

まず、慣れた順方向の計算から入ります。もし $\theta$ を知っていたら、さっきの10回の結果が出る確率は計算できるはずです。コイン投げの各回は独立なので、同時確率は積になります(第1章)。

<!-- TODO(consistency): 第1章で「独立なら同時確率は積」を導入している前提。第1章原稿の執筆時に突き合わせる -->

$$P(\text{表},\text{表},\text{裏},\ldots,\text{表}) = \theta \cdot \theta \cdot (1-\theta) \cdots \theta = \theta^7 (1-\theta)^3$$

表が出た回はそれぞれ $\theta$ を、裏が出た回はそれぞれ $1-\theta$ を1個ずつ掛ける。掛け算は順序を入れ替えられるので、表の分をまとめて $\theta^7$、裏の分をまとめて $(1-\theta)^3$ です。

さて、ここからが本章でいちばん大事な一歩です。出来上がった式をよく見てください。

$$\theta^7 (1-\theta)^3$$

この式に入っている文字は $\theta$ だけです。データ(表7回、裏3回)は、もう起きてしまった事実として、指数の 7 と 3 に焼き付いています。つまりこの式は、**$\theta$ を入れると数がひとつ出てくる、$\theta$ の関数**として読めます。

$$L(\theta) = \theta^7 (1-\theta)^3$$

同じ式に、2通りの読み方があることになります。

- **$\theta$ を固定して、データの側を動かして読む**: 「このコインから、このデータが出る確率」。第1章からずっとやってきた読み方です。
- **データを固定して、$\theta$ の側を動かして読む**: 「この**起きてしまったデータ**は、$\theta$ がいくつのコインだったとしたら、どれくらい出やすかったか」。

後者の読み方をするとき、この関数を**尤度関数**(likelihood function)、その値を**尤度**(likelihood)と呼びます。「尤」は「もっともらしい」と読む字です。$L(\theta)$ は、$\theta$ という仮説ひとつひとつに付けられた、データに照らした「もっともらしさの点数」です。

既視感があるはずです。第3巻第3章で、まったく同じ反転をやりました。MSE の式を「データは定数、パラメータが変数」と読み替えて、損失をパラメータの関数 $L(w, b)$ という地形にした、あの反転です。確率の世界でも、同じ反転が同じ働きをします。

実際に数を入れてみましょう。

| $\theta$ | $L(\theta) = \theta^7(1-\theta)^3$ |
|---|---|
| 0.3 | 0.000075 |
| 0.5 | 0.00098 |
| 0.7 | **0.00222** |
| 0.9 | 0.00048 |

$\theta = 0.7$ の点数がいちばん高い。歪みのないコイン(0.5)の2倍以上、極端なイカサマコイン(0.9)の4倍以上です。グラフに描くと、$L(\theta)$ は両端($\theta = 0$ と $1$)でゼロに潰れ、0.7 にただ1つの頂上を持つ、なだらかな山になります。

数を読むときの注意を2つ。第一に、値そのもの(0.00222)の小ささに意味はありません。10回の結果の並び方は $2^{10} = 1024$ 通りもあるので、どんな $\theta$ でも、特定の1つの並びの確率は小さくなります。尤度は**絶対値ではなく、比較**で使う数です。第二に、$L(\theta)$ は $\theta$ についての確率分布では**ありません**。$\theta$ の全候補にわたって足し合わせても1になる義務がない、ただの点数表です。確率の公理(第1章)はデータの側にかかっていて、$\theta$ の側にはかかっていないからです。

頂上の位置は、数値でも確かめられます(本章のコード片はつなげると1本のスクリプトで、`code/ch03/likelihood_shape.py` に置いてあります)。

```python
import numpy as np

theta = np.linspace(0.001, 0.999, 999)  # (999,) θ の候補を 0.001 刻みで敷き詰める
L = theta**7 * (1 - theta)**3           # (999,) 各候補の尤度

assert np.isclose(theta[np.argmax(L)], 0.7)  # 山の頂上は θ = 0.7
```

ここまで来れば、「$\theta$ はいくつだと思いますか」という曖昧な問いを、数学の問いに翻訳できます。

> **最尤推定**(maximum likelihood estimation): 尤度 $L(\theta)$ を最大にする $\theta$ を、推定値として採用する。

「データからパラメータを当てる」が、「関数の最大化」という見慣れた問題に変わりました。推定値には帽子を付けて $\hat{\theta}$ と書きます(第3巻の $\hat{y}$ と同じ、「これは正解そのものではなく推定だ」という印です)。

最大化なら、やり方は知っています。第2巻第2章で「谷底や頂上では傾きが 0」を学びました。$L(\theta)$ は変数1個の式ですから、傾き 0 を手で解きにいきましょう——と言いたいところですが、その前に、この式が**積**でできていることから来る困りごとを2つ、正直に挙げておきます。

## 3.3 対数尤度: 積を和に変える

**困りごと1: 数値が消える。**データ10個で、頂上の高さがすでに 0.00222 でした。データが1個増えるたびに、1以下の数がもう1つ掛かるので、尤度は急速に小さくなっていきます。1000個ならどうなるか。確かめるのが早い。

```python
rng = np.random.default_rng(42)
p = rng.uniform(0.1, 0.9, size=1000)  # (1000,) 確率のつもりの数を1000個

assert np.prod(p) == 0.0              # 1000個の積は 0.0 ぴったり: 情報が完全に消えた
```

積が「およそ 0」ではなく「**0.0 ぴったり**」になっています。本当の答えはおよそ $10^{-364}$ で(後で確かめます)、0 ではありません。しかしコンピュータの小数(倍精度浮動小数点数)が表せるいちばん小さい正の数はおよそ $10^{-308}$ で、それを下回った瞬間、値は 0 に丸められます。**アンダーフロー**(underflow)と呼ばれる現象です。0 になってしまえば、どの $\theta$ の尤度が大きいかという比較そのものができません。そして機械学習の世界で、データ1000個は「極小」の部類です。これは致命傷です。

**困りごと2: 微分がつらい。**第2巻第1章で積の微分ルールを学びましたが、あれは因子2つの積でした。1000個の因子の積を微分すると、「999個の積」を1000本並べて足した式が出てきます。手で書く気は失せますし、計算する機械にも優しくありません。一方、もしこれが**和**だったなら——「和の微分は微分の和」(第2巻第1章)で、項ごとにばらばらに微分できたはずです。

つまり、欲しい道具はこうなります。**積を和に変える変換。ただし、山の頂上の場所を動かさないこと。**

あつらえたような道具が、高校数学にあります。対数です。使う性質は3つだけなので、ここで復習を完結させておきます。

**対数の復習(ここだけで自己完結します)。** $a^x$ という指数の式を逆から読み、「$a$ を何乗したら $M$ になるか」という数を $\log_a M$ と書いて、$a$ を底(てい)とする $M$ の**対数**(logarithm)と呼びます。$2^3 = 8$ なので $\log_2 8 = 3$ です。

**性質1(積を和に)**: $\log MN = \log M + \log N$。
理由は指数法則そのものです。$a^m \times a^n = a^{m+n}$——数を掛け算すると、肩の数は**足し算**になる。対数は「肩の数を取り出す」装置なので、掛け算の世界が足し算の世界に翻訳されます。$8 \times 16 = 128$ を底2の肩で見れば、$3 + 4 = 7$ です。

**性質2(累乗を係数に)**: $\log M^k = k \log M$。$M$ を $k$ 回掛けたものに性質1を $k$ 回使っただけです。

**性質3(順序を保つ)**: $M < N$ ならば $\log M < \log N$(底が1より大きいとき)。対数は単調増加で、数の大小関係を一切壊しません。

底には何を使うか。機械学習では、**ネイピア数** $e = 2.71828\ldots$ を底にした対数——**自然対数**(natural logarithm)だけを使います。$e$ という中途半端に見える数がなぜ特別なのかは次節でわかります(微分が驚くほど短くなる、という理由です)。以後この本で $\log$ と書いたら、断りなく底は $e$ です。NumPy の `np.log` も自然対数です。

感覚合わせに、値をいくつか。$\log 1 = 0$($e^0 = 1$ だから)。$\log e = 1$。$\log(1/e) = -1$。1より小さい正の数の対数は負になり、$M$ を 0 に近づけると $\log M$ は $-\infty$ へ沈んでいきます。つまり、**0と1の間の数——確率——の対数は、必ず負**です。これは次章以降ずっと使う感覚なので、ここで体に入れておいてください。

では、尤度に適用します。$L(\theta) = \theta^7(1-\theta)^3$ の対数を取ると、性質1で積が和に割れ、性質2で指数が係数に降りてきます。

$$\log L(\theta) = 7 \log \theta + 3 \log(1-\theta)$$

これを**対数尤度**(log-likelihood)と呼び、$\ell(\theta)$ と書きます。積だった尤度が、たった2項の和になりました。一般に、データが $n$ 個に増えても、対数尤度は「各データの $\log$ 確率を $n$ 個**足す**」だけです。困りごと2は消えました。

そして性質3(単調)が、いちばん大事な保証をくれます。**$L(\theta)$ を最大にする $\theta$ と、$\ell(\theta)$ を最大にする $\theta$ は同じ**です。対数は大小を保つので、山の頂上の「場所」は動きません。変わるのは標高の目盛りだけです。頂上の高さは $L(0.7) = 0.00222$ から $\ell(0.7) = \log 0.00222 \approx -6.11$ に変わりますが(負の数になるのは、確率の対数だから当然です)、頂上は 0.7 のままです。

```python
assert np.argmax(np.log(L)) == np.argmax(L)  # 対数を取っても頂上の場所は同じ
```

困りごと1も、これで消えます。さっき 0.0 に消えた1000個の積を、対数の世界でやり直すと——

```python
log_lik = np.sum(np.log(p))   # 積の対数 = 対数の和

assert np.isfinite(log_lik)   # 今度は何事もなく計算できる
print(log_lik)                # -838.98...
```

積の世界では 0 に消えた量が、対数の世界では $-838.98$ という、ごく普通の大きさの数として扱えています。$e^{-838.98} \approx 10^{-364}$、先ほど「本当の答え」と言ったのはこの数です。1以下の数の掛け算は奈落に向かって**指数的に暴走**しますが、その対数の足し算は1項ごとに**一定の歩幅でしか進まない**からです。

積を和に。暴走を行進に。そして頂上はそのまま。役者が揃いました。微分しましょう。

## 3.4 コイン投げ(ベルヌーイ)の最尤推定を手で解く

せっかくなので、一般の形で解きます。コインを $n$ 回投げ、表が $k$ 回出たとします。対数尤度は

$$\ell(\theta) = k \log \theta + (n-k) \log(1-\theta)$$

頂上では傾きが 0(第2巻第2章)。なので $\ell'(\theta) = 0$ を解きます。そのために必要な微分の道具は、実は1つだけ——「$\log \theta$ の微分」です。第2巻で微分したのは多項式だけでしたから、ここで新しい事実を1つ、道具箱に追加します。

$$(\log x)' = \frac{1}{x}$$

底が $e$ の対数は、微分するとこれだけになります。底が $e$ 以外だと余計な定数が掛かります——前節で「$e$ は特別」と言った理由がこれです。この本の流儀どおり証明はせず、数値で検算します。第2巻第1章の中心差分で $x = 2$ での傾きを測ると、$\bigl(\log(2+h) - \log(2-h)\bigr)/2h = 0.50000\ldots$($h = 10^{-5}$)。確かに $1/2$ です。$x = 4$ なら $0.25$。「$x$ が大きいほど対数の曲線は寝ていく。その寝方が正確に $1/x$」という事実です(この検算も `code/ch03/likelihood_shape.py` に入れてあります)。

もう1つ、第2項の $\log(1-\theta)$ には連鎖律(第2巻第5章)を使います。外側の $\log$ の微分で $\dfrac{1}{1-\theta}$、内側 $(1-\theta)$ の微分で $-1$。掛けて $-\dfrac{1}{1-\theta}$ です。

では微分します。和は項ごとに、定数 $k$ と $n-k$ は係数としてそのまま(どちらも第2巻第1章)。

$$\ell'(\theta) = \frac{k}{\theta} - \frac{n-k}{1-\theta}$$

これを 0 と置いて解きます。分母を払うため、両辺に $\theta(1-\theta)$ を掛けます($0 < \theta < 1$ なのでこれは 0 ではありません)。

$$\begin{aligned}
k(1-\theta) - (n-k)\,\theta &= 0 \\
k - k\theta - n\theta + k\theta &= 0 \\
k - n\theta &= 0 \\
\hat{\theta} &= \frac{k}{n}
\end{aligned}$$

——これだけです。最尤推定の答えは、**表が出た回数の割合**。10回中7回なら $\hat{\theta} = 7/10 = 0.7$。3.2 の表で、そしてグリッドの `argmax` で見た頂上 0.7 は、偶然ではありませんでした。

(傾き 0 の点が、谷底ではなく頂上である保証はあるのか——第2巻第2章で、傾き 0 だけでは両者を区別できないと学びました。今回は両端の様子で判定できます。$\theta \to 0$ では $k \log \theta$ の項が、$\theta \to 1$ では $(n-k)\log(1-\theta)$ の項が $-\infty$ へ沈むので、$\ell(\theta)$ は両端が奈落です。両端が奈落で、内部に傾き 0 の点がただ1つなら、そこは頂上しかありえません。ただしこの議論は $k$ が 0 でも $n$ でもないときの話です。端の場合に何が起きるかは、演習の問4で扱います。)

この結果を、少しだけ眺めていたいと思います。

3.1 であなたは「0.7 だと思う。10回中7回だから」と答え、私は「それは理由になっていない」と言いました。いま、理由が付きました。**「頻度で当てる」というあの直観は、『もっともらしさを最大化せよ』という原理から、微分で導出される結論だった**のです。直観が、手続きから「出てくる」。この気持ちよさは、原理が信頼に足ることの何よりの証拠です。コインの世界では直観と原理が同じ答えを出す——だから、直観の届かない6500万パラメータの世界には、原理の側だけを持っていけばいいのです。

もうひとつ、いま自分がやったことの「形」を見てください。パラメータの関数を1つ立てて、微分して、傾き 0 を解いた。第2巻第6章で直線のパラメータを当てたときと、寸分違わぬ段取りです。確率の話をしていたはずなのに、着地した先の景色は最適化でした。これは偶然ではありません。最後の節で、この2つの世界を正式に接続します。

## 3.5 視点の合流: 「尤度最大化」も「損失最小化」も同じ形 — 負の対数尤度

第3巻で確立した世界観を、もう一度声に出します。**モデルはパラメータ付き関数。学習は損失の最小化。**損失 $L(w, b)$ という地形の谷底を、勾配降下法で探すのでした。

一方、この章で手に入れた原理は、**もっともらしさ $\ell(\theta)$ の最大化**。山の頂上探しです。

谷底探しと、頂上探し。下りと、登り。向きが逆なだけで、同じ営みに見えます。実際、2つを合流させるのに必要な操作は、$-1$ を掛けることだけです。$\ell(\theta)$ を最大にすることと、$-\ell(\theta)$ を最小にすることは同じです。山に $-1$ を掛ければ上下が反転して谷になり、頂上だった場所が、そのまま谷底になる。

そこで、この反転した量に名前を付けます。

$$\mathrm{NLL}(\theta) = -\ell(\theta) = -\sum_{i=1}^{n} \log p(x_i;\, \theta)$$

ここで $p(x_i; \theta)$ は「パラメータ $\theta$ のもとで、$i$ 番めのデータ $x_i$ が出る確率」、$\sum$ はデータ全体にわたる和です。この量を**負の対数尤度**(negative log-likelihood)、略して **NLL** と呼びます。

この量の「資格」を点検してください。

- **データは定数、パラメータの関数である**(第3巻第3章で損失に要求した条件そのもの)
- **小さいほど良い**(尤度が大きい=もっともらしいほど、NLL は小さい)
- **データ1個ごとの項 $-\log p(x_i; \theta)$ の和でできている**(だから平均にもでき、ミニバッチ(第3巻第5章)にも切り分けられる)
- **微分できる**(この章で、現に微分しました)

負の対数尤度は、**損失関数の資格を完全に満たしています**。第3巻の MSE と同じ席に座れる量です。

つまり、こういうことです。第3巻の世界の住人から見ると、この章の出来事は「新しい損失関数がひとつ手に入った」と要約できます。しかも、出自が MSE とは違います。MSE は「符号を消したい、大外れを重く罰したい」という使い勝手から**選ばれた**損失でした(第3巻第3章で「なぜ正しいかの正当化は第4巻で」と先送りしたことを覚えているでしょうか)。NLL はそうではありません。**「観測されたデータを最ももっともらしくするパラメータを選べ」という確率の原理から、導出されて出てきた**損失です。選んだのではなく、出てきた。

ここまで来ると、第3巻エピローグに積まれた3つの宿題の解け方が、ほとんど見えています。

1. シグモイドは、なぜあの形でなければならないのか
2. シグモイドが出す 0〜1 の数は、何と解釈すべきなのか
3. 分類の正しい損失関数とは何か

宿題2の答えの候補は、第1章からずっと匂っていました——あの 0〜1 は**確率**だ、と。そして、もし宿題2がそう解けるなら、つまりモデルの出力が「ラベルが 1 である確率」なら、宿題3にはこの章の原理が機械的に答えます。**モデルがデータに割り当てた確率の、負の対数尤度を損失にすればいい。**ラベルは 0 か 1 の2択——ベルヌーイ分布の尤度なら、この章でもう書き下せるようになっています。

残る作業は3つです。シグモイドの出力を「確率」として正当化すること(宿題1・2)。ベルヌーイの NLL を損失として書き下し、勾配を手で出すこと。そして、**あの死んだ勾配が生き返るのを、第3巻と同じデータでこの目で確かめる**こと。次章で、宿題を全部、机から下ろします。ちなみに、こうして導出される損失の一族には、原論文にも登場する有名な名前が付いているのですが——その名前を知るのは第5章です。

## まとめ

- **尤度** $L(\theta)$ は、「データの出る確率」の式を**データ固定・パラメータ可変**で読み替えた、パラメータのもっともらしさの点数。第3巻第3章の「損失はパラメータの関数」と同じ視点の反転
- **対数は積を和に変える**($\log MN = \log M + \log N$)。尤度の積は対数尤度の和になり、数値はアンダーフローから救われ、微分は項ごとにできるようになる。単調性により、頂上の場所は動かない
- ベルヌーイ分布の最尤推定を微分で解くと $\hat{\theta} = k/n$。**「頻度で当てる」という直観は、最尤原理から導出される結論**だった
- 尤度の最大化は、**負の対数尤度(NLL)の最小化**と同じ。NLL は損失関数の資格を完全に満たし、確率の世界(本章)と第3巻の「学習 = 損失最小化」の世界がここで合流する
- 新しい微分の道具 $(\log x)' = 1/x$ を道具箱に追加した(次章、log loss の勾配計算で主役になります)

**ラスボスとの距離**: 論文 Section 5.4 の "perplexity" も、label smoothing が施される損失も、すべて「モデルの良さを $\log$ 確率で測る」という本章の物差しの上に立っています。その損失の本名(第5章)まで、あと2章です。

**問1**(対数の足慣らし) $\log 2 \approx 0.693$、$\log 3 \approx 1.099$ とします($\log$ は自然対数)。電卓を使わず、この2つの値と対数の性質だけから、$\log 8$、$\log 12$、$\log 1.5$ を求めてください。

**問2**(最尤推定を微分で解く) コインを20回投げたら、表が5回出ました。対数尤度 $\ell(\theta)$ を書き下し、微分して傾き 0 を解き、$\hat{\theta}$ を求めてください。3.4 の一般式 $k/n$ に代入するのではなく、$k=5,\ n=20$ の数字で最初から手を動かすこと。

**問3**(対数のありがたみ) 対数を取らずに、$L(\theta) = \theta^7(1-\theta)^3$ を第2巻の積の微分ルールで直接微分し、頂上が 3.4 の答えと一致することを確認してください。ヒント: 微分した式から共通因数 $\theta^6(1-\theta)^2$ をくくり出すと、残りが見やすくなります。

**問4**(端で起きること) コインを3回投げたら、3回とも表でした($n = k = 3$)。対数尤度は $\ell(\theta) = 3 \log \theta$ です。微分して傾き 0 を解こうとすると、何が起きますか。$\hat{\theta}$ はいくつになりますか。最後に、この推定値を一言で批評してください。

<summary>略解</summary>

**問1** $\log 8 = \log 2^3 = 3 \log 2 \approx 2.079$。$\log 12 = \log(4 \cdot 3) = 2\log 2 + \log 3 \approx 2.485$。$\log 1.5 = \log(3/2) = \log 3 - \log 2 \approx 0.405$。最後の式は「割り算は引き算になる」($\log(M/N) = \log M - \log N$)を使っており、これは性質1を $N \cdot (M/N) = M$ に適用すれば出てきます。

**問2** $\ell(\theta) = 5 \log \theta + 15 \log(1-\theta)$。微分して $\ell'(\theta) = \dfrac{5}{\theta} - \dfrac{15}{1-\theta}$。これを 0 と置き、$\theta(1-\theta)$ を掛けて $5(1-\theta) = 15\theta$、つまり $5 = 20\theta$ で $\hat{\theta} = 1/4$。表の頻度 $5/20$ と一致します。

**問3** 積のルールと連鎖律で
$$L'(\theta) = 7\theta^6(1-\theta)^3 + \theta^7 \cdot 3(1-\theta)^2 \cdot (-1) = \theta^6(1-\theta)^2\,\{7(1-\theta) - 3\theta\} = \theta^6(1-\theta)^2\,(7 - 10\theta)$$
$0 < \theta < 1$ でこれが 0 になるのは $7 - 10\theta = 0$、つまり $\hat{\theta} = 0.7$ で、3.4 と一致します。積のルール・連鎖律・因数分解を総動員してこの手数——対数を取れば2項の和の微分で済んだことと比べてください。因子が1000個になっても同じ方法を選びたいか、想像してみると、3.3 の「積を和に」のありがたみが体感できます。

**問4** $\ell'(\theta) = 3/\theta$ は $0 < \theta < 1$ でつねに正で、**傾き 0 の解は区間の内部に存在しません**。$\ell(\theta)$ は単調増加なので、もっともらしさは右端で最大、つまり $\hat{\theta} = 1$ です(3.4 の一般式 $k/n = 3/3$ とも一致)。批評: たった3回の観測から「このコインは裏が永遠に出ない」と断言する推定になっています。最尤推定は手元のデータを全面的に信頼する原理なので、データが少ないと、このように極端な値へ振り切れることがあります。
