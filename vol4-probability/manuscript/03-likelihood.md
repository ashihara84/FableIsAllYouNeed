これまでの問いは**分布を知っている側から、データの出方を予言する**形でした。分布が先、データが後です。ところが第3巻から持ち越した宿題は逆。スパムフィルタで手元にあったのはデータ(ラベル付きメール200件)だけで、知りたいのは良いパラメータ $\mathbf{w}, b$。つまり**データが先、パラメータが後**です。

この章では問いの向きを反転させます。データからパラメータを当てる原理を作り、章末ではその原理が第3巻の「学習 = 損失最小化」と1本の式で合流する。第3巻エピローグの宿題3「分類の正しい損失関数とは何か」に答える原理が、ここで手に入ります。

## 3.1 逆向きの問い: データからパラメータを当てる

原理がむき出しになるよう問題を小さくしたい。そこで主役はコインにします。「表か裏か」という2択(1 か 0)の構造は、第3巻のスパム判定のラベル(スパム = 1、通常 = 0)とまったく同じです。第1章の「次の単語当て」も「『猫』か、そうでないか」と問い直せば2択。コインで原理を仕留めれば、そのまま持ち帰れます。

表(1)が出る確率を $\theta$(シータ)と書きます。裏(0)が出る確率は、全部足すと1という規則(第1章)から $1 - \theta$。この、結果が2択しかない最小の確率分布を**ベルヌーイ分布**(Bernoulli distribution)と呼びます。パラメータは $\theta$ ただ1つ。$\theta = 0.5$ なら歪みのないコイン、$\theta = 0.9$ なら表ばかり出るイカサマコインです。

逆向きの問いを出します。

> 目の前に、歪んでいるかもしれないコインがあります。$\theta$ の値は見えません。10回投げたら、こうなりました。
>
> 表, 表, 裏, 表, 表, 表, 裏, 表, 裏, 表 (表7回、裏3回)
>
> $\theta$ はいくつだと思いますか?

ほとんどの人が「0.7」と答えます。では**なぜ 0.7 なのか。**「10回中7回だから」は答えをもう一度言っただけで、「表の割合で当てる」という手続きがなぜ、どんな意味で正しいのかを説明できていません。

コインの $\theta$ 1個なら直観で済みます。しかし第3巻のスパムフィルタはパラメータ3個($\mathbf{w}$ が2個と $b$)で「見ればわかる」とはいかず、原論文のモデルは約6500万個。だから、コインで効いた直観を**どんなパラメータにも適用できる手続き**に変換しておく必要があります。第3巻第3章で「いい感じ」という直観を損失という数に変換したのと同じ作業で、第一歩は問いを数式に翻訳することです。

## 3.2 尤度関数: パラメータの関数として見た「データの出やすさ」

まず順方向から。$\theta$ を知っていれば10回の結果が出る確率は計算できます。各回は独立なので同時確率はただの積です(第1章「独立なら、同時確率は積」)。

$$P(\text{表},\text{表},\text{裏},\ldots,\text{表}) = \theta \cdot \theta \cdot (1-\theta) \cdots \theta = \theta^7 (1-\theta)^3$$

表が出た回はそれぞれ $\theta$ を、裏が出た回はそれぞれ $1-\theta$ を掛け、順序を入れ替えて表の分を $\theta^7$、裏の分を $(1-\theta)^3$ とまとめます。

ここが本章でいちばん大事な一歩です。この式に入っている文字は $\theta$ だけ。データ(表7回、裏3回)は、もう起きた事実として指数の 7 と 3 に焼き付いています。つまりこの式は、**$\theta$ を入れると数がひとつ出てくる、$\theta$ の関数**として読めます。

$$L(\theta) = \theta^7 (1-\theta)^3$$

同じ式に2通りの読み方があります。

- **$\theta$ を固定し、データ側を動かす**: 「このコインから、このデータが出る確率」。第1章からの読み方です。
- **データを固定し、$\theta$ 側を動かす**: 「この**起きてしまったデータ**は、$\theta$ がいくつのコインだったら、どれくらい出やすかったか」。

後者の読み方をするとき、この関数を**尤度関数**(likelihood function)、その値を**尤度**(likelihood)と呼びます。「尤」は「もっともらしい」と読む字。$L(\theta)$ は、$\theta$ という仮説ひとつひとつに付けた、データに照らした「もっともらしさの点数」です。

第3巻第3章でまったく同じ反転をやりました。MSE を「データは定数、パラメータが変数」と読み替えて損失を $L(w, b)$ という地形にした、あの反転です。確率の世界でも同じ反転が同じ働きをします。実際に数を入れてみましょう。

| $\theta$ | $L(\theta) = \theta^7(1-\theta)^3$ |
|---|---|
| 0.3 | 0.000075 |
| 0.5 | 0.00098 |
| 0.7 | **0.00222** |
| 0.9 | 0.00048 |

$\theta = 0.7$ の点数がいちばん高い。歪みのないコイン(0.5)の2倍以上、極端なイカサマコイン(0.9)の4倍以上です。$L(\theta)$ は両端($\theta = 0$ と $1$)でゼロに潰れ、0.7 にただ1つの頂上を持つ、なだらかな山になります。

数を読むときの注意を2つ。第一に、値そのもの(0.00222)の小ささに意味はありません。10回の並び方は $2^{10} = 1024$ 通りあるので、どんな $\theta$ でも特定の1つの並びの確率は小さくなる。尤度は**絶対値ではなく比較**で使う数です。第二に、$L(\theta)$ は $\theta$ についての確率分布では**ありません**。$\theta$ の全候補にわたって足しても1になる義務がない、ただの点数表です。確率の公理(第1章)はデータの側にかかっていて、$\theta$ の側にはかかっていないからです。

頂上の位置は数値でも確かめられます。

```python
import numpy as np

theta = np.linspace(0.001, 0.999, 999)  # (999,) θ の候補を 0.001 刻みで敷き詰める
L = theta**7 * (1 - theta)**3           # (999,) 各候補の尤度

assert np.isclose(theta[np.argmax(L)], 0.7)  # 山の頂上は θ = 0.7
```

全文と動作確認は `code/ch03/likelihood_shape.py`(`python3` で全 assert 通過)。

ここまで来れば、「$\theta$ はいくつだと思いますか」という曖昧な問いを、数学の問いに翻訳できます。

> **最尤推定**(maximum likelihood estimation): 尤度 $L(\theta)$ を最大にする $\theta$ を、推定値として採用する。

「データからパラメータを当てる」が「関数の最大化」に変わりました。推定値には帽子を付けて $\hat{\theta}$ と書きます(第3巻の $\hat{y}$ と同じ、「正解そのものではなく推定だ」という印です)。

最大化のやり方は知っています。第2巻第2章で「谷底や頂上では傾きが 0」を学びました。$L(\theta)$ は変数1個ですから傾き 0 を手で解く——その前に、この式が**積**でできていることから来る困りごとを2つ挙げます。

## 3.3 対数尤度: 積を和に変える

**困りごと1: 数値が消える。**データ10個で頂上の高さがすでに 0.00222。データが1個増えるたび1以下の数が掛かるので、尤度は急速に小さくなります。1000個ならどうなるか。

```python
rng = np.random.default_rng(42)
p = rng.uniform(0.1, 0.9, size=1000)  # (1000,) 確率のつもりの数を1000個

assert np.prod(p) == 0.0              # 1000個の積は 0.0 ぴったり: 情報が完全に消えた
```

積が「およそ 0」ではなく「**0.0 ぴったり**」です。本当の答えはおよそ $10^{-364}$ で 0 ではありません。しかしコンピュータの小数(倍精度浮動小数点数)が表せるいちばん小さい正の数はおよそ $10^{-308}$ で、それを下回った瞬間 0 に丸められます。**アンダーフロー**(underflow)です。0 になればどの $\theta$ の尤度が大きいかという比較ができません。機械学習でデータ1000個は「極小」の部類ですから、これは致命傷です。

**困りごと2: 微分がつらい。**第2巻第1章の積の微分ルールは因子2つの積でした。1000個の因子の積を微分すると「999個の積」を1000本並べて足した式が出ます。もしこれが**和**だったなら、「和の微分は微分の和」(第2巻第1章)で項ごとにばらばらに微分できたはずです。

欲しい道具はこうなります。**積を和に変える変換。ただし山の頂上の場所を動かさないこと。**あつらえたような道具が高校数学にあります。対数です。使う性質は3つだけなので、ここで復習を完結させます。

**対数の復習(ここだけで自己完結します)。** $a^x$ を逆から読み、「$a$ を何乗したら $M$ になるか」を $\log_a M$ と書いて、$a$ を底(てい)とする $M$ の**対数**(logarithm)と呼びます。$2^3 = 8$ なので $\log_2 8 = 3$ です。

**性質1(積を和に)**: $\log MN = \log M + \log N$。
理由は指数法則そのもの。$a^m \times a^n = a^{m+n}$——数を掛けると肩の数は**足し算**になる。対数は「肩の数を取り出す」装置なので、掛け算の世界が足し算の世界に翻訳されます。$8 \times 16 = 128$ を底2の肩で見れば $3 + 4 = 7$ です。

**性質2(累乗を係数に)**: $\log M^k = k \log M$。$M$ を $k$ 回掛けたものに性質1を $k$ 回使っただけです。

**性質3(順序を保つ)**: $M < N$ ならば $\log M < \log N$(底が1より大きいとき)。対数は単調増加で、数の大小関係を一切壊しません。

底は機械学習では**ネイピア数** $e = 2.71828\ldots$ を底にした**自然対数**(natural logarithm)だけを使います。$e$ が特別な理由は次節でわかります(微分が驚くほど短くなる)。以後この本で $\log$ と書いたら底は $e$ です。NumPy の `np.log` も自然対数です。

感覚合わせに値をいくつか。$\log 1 = 0$($e^0 = 1$)。$\log e = 1$。$\log(1/e) = -1$。1より小さい正の数の対数は負になり、$M$ を 0 に近づけると $\log M$ は $-\infty$ へ沈みます。つまり**0と1の間の数——確率——の対数は、必ず負**。次章以降ずっと使う感覚なので、ここで体に入れておいてください。

では尤度に適用します。$L(\theta) = \theta^7(1-\theta)^3$ の対数を取ると、性質1で積が和に割れ、性質2で指数が係数に降りてきます。

$$\log L(\theta) = 7 \log \theta + 3 \log(1-\theta)$$

これを**対数尤度**(log-likelihood)と呼び、$\ell(\theta)$ と書きます。積だった尤度がたった2項の和になりました。データが $n$ 個に増えても、対数尤度は「各データの $\log$ 確率を $n$ 個**足す**」だけ。困りごと2は消えました。

そして性質3(単調)が、いちばん大事な保証をくれます。**$L(\theta)$ を最大にする $\theta$ と、$\ell(\theta)$ を最大にする $\theta$ は同じ**です。対数は大小を保つので山の頂上の「場所」は動かない。変わるのは標高の目盛りだけで、頂上の高さは $L(0.7) = 0.00222$ から $\ell(0.7) = \log 0.00222 \approx -6.11$ に変わりますが(確率の対数だから負)、頂上は 0.7 のままです。

```python
assert np.argmax(np.log(L)) == np.argmax(L)  # 対数を取っても頂上の場所は同じ
```

困りごと1も消えます。さっき 0.0 に消えた1000個の積を、対数の世界でやり直すと——

```python
log_lik = np.sum(np.log(p))   # 積の対数 = 対数の和

assert np.isfinite(log_lik)   # 今度は何事もなく計算できる
print(log_lik)                # -838.98...
```

積の世界で 0 に消えた量が、対数の世界では $-838.98$ という普通の大きさの数として扱えています。$e^{-838.98} \approx 10^{-364}$、先ほど「本当の答え」と言ったのはこの数です。1以下の掛け算は奈落へ**指数的に暴走**しますが、その対数の足し算は1項ごとに**一定の歩幅でしか進まない**からです。

積を和に。暴走を行進に。そして頂上はそのまま。役者が揃いました。微分しましょう。

## 3.4 コイン投げ(ベルヌーイ)の最尤推定を手で解く

一般の形で解きます。コインを $n$ 回投げ、表が $k$ 回出たとします。対数尤度は

$$\ell(\theta) = k \log \theta + (n-k) \log(1-\theta)$$

頂上では傾きが 0(第2巻第2章)なので $\ell'(\theta) = 0$ を解きます。必要な微分の道具は1つだけ——「$\log \theta$ の微分」です。第2巻で微分したのは多項式だけでしたから、新しい事実を道具箱に追加します。

$$(\log x)' = \frac{1}{x}$$

底が $e$ の対数は微分するとこれだけ。底が $e$ 以外だと余計な定数が掛かる——前節で「$e$ は特別」と言った理由がこれです。証明はせず数値で検算します。第2巻第1章の中心差分で $x = 2$ での傾きを測ると $\bigl(\log(2+h) - \log(2-h)\bigr)/2h = 0.50000\ldots$($h = 10^{-5}$)。確かに $1/2$、$x = 4$ なら $0.25$。「$x$ が大きいほど対数の曲線は寝ていき、その寝方が正確に $1/x$」という事実です(この検算も `code/ch03/likelihood_shape.py` に入れてあります)。

第2項の $\log(1-\theta)$ には連鎖律(第2巻第5章)を使います。外側 $\log$ の微分で $\dfrac{1}{1-\theta}$、内側 $(1-\theta)$ の微分で $-1$。掛けて $-\dfrac{1}{1-\theta}$ です。

では微分します。和は項ごとに、定数 $k$ と $n-k$ は係数としてそのまま(どちらも第2巻第1章)。

$$\ell'(\theta) = \frac{k}{\theta} - \frac{n-k}{1-\theta}$$

これを 0 と置いて解きます。両辺に $\theta(1-\theta)$ を掛けて分母を払います($0 < \theta < 1$ なのでこれは 0 ではありません)。

$$\begin{aligned}
k(1-\theta) - (n-k)\,\theta &= 0 \\
k - k\theta - n\theta + k\theta &= 0 \\
k - n\theta &= 0 \\
\hat{\theta} &= \frac{k}{n}
\end{aligned}$$

——これだけです。最尤推定の答えは**表が出た回数の割合**。10回中7回なら $\hat{\theta} = 7/10 = 0.7$。3.2 の表で、そしてグリッドの `argmax` で見た頂上 0.7 は偶然ではありませんでした。

(傾き 0 の点が谷底ではなく頂上である保証はあるか——第2巻第2章で、傾き 0 だけでは両者を区別できないと学びました。今回は両端で判定できます。$\theta \to 0$ では $k \log \theta$ が、$\theta \to 1$ では $(n-k)\log(1-\theta)$ が $-\infty$ へ沈むので、$\ell(\theta)$ は両端が奈落です。両端が奈落で内部に傾き 0 の点がただ1つなら、そこは頂上しかありえません。ただしこの議論は $k$ が 0 でも $n$ でもないときの話で、端の場合は演習の問4で扱います。)

この結果を少し眺めます。3.1 であなたは「0.7 だと思う。10回中7回だから」と答え、私は「理由になっていない」と言いました。いま理由が付きました。**「頻度で当てる」というあの直観は、『もっともらしさを最大化せよ』という原理から微分で導出される結論だった**のです。コインの世界では直観と原理が同じ答えを出す。だから、直観の届かない6500万パラメータの世界には、原理の側だけを持っていけばいい。

もうひとつ、いまやったことの「形」を見てください。パラメータの関数を1つ立て、微分して、傾き 0 を解いた。第2巻第6章で直線のパラメータを当てたときと寸分違わぬ段取りです。確率の話だったはずが、着地した景色は最適化でした。偶然ではありません。最後の節で、この2つの世界を正式に接続します。

## 3.5 視点の合流: 「尤度最大化」も「損失最小化」も同じ形 — 負の対数尤度

第3巻で確立した世界観をもう一度。**モデルはパラメータ付き関数。学習は損失の最小化。**損失 $L(w, b)$ という地形の谷底を勾配降下法で探すのでした。一方この章で手に入れた原理は**もっともらしさ $\ell(\theta)$ の最大化**、山の頂上探しです。

谷底探しと頂上探し。向きが逆なだけで同じ営みです。合流に必要な操作は $-1$ を掛けることだけ。$\ell(\theta)$ を最大にすることと $-\ell(\theta)$ を最小にすることは同じです。山に $-1$ を掛ければ上下が反転して谷になり、頂上だった場所がそのまま谷底になります。

この反転した量に名前を付けます。

$$\mathrm{NLL}(\theta) = -\ell(\theta) = -\sum_{i=1}^{n} \log p(x_i;\, \theta)$$

ここで $p(x_i; \theta)$ は「パラメータ $\theta$ のもとで $i$ 番めのデータ $x_i$ が出る確率」、$\sum$ はデータ全体の和です。この量を**負の対数尤度**(negative log-likelihood)、略して **NLL** と呼びます。

この量の「資格」を点検します。

- **データは定数、パラメータの関数である**(第3巻第3章で損失に要求した条件そのもの)
- **小さいほど良い**(尤度が大きい=もっともらしいほど、NLL は小さい)
- **データ1個ごとの項 $-\log p(x_i; \theta)$ の和でできている**(だから平均にもでき、ミニバッチ(第3巻第5章)にも切り分けられる)
- **微分できる**(この章で現に微分しました)

負の対数尤度は**損失関数の資格を完全に満たしています**。第3巻の MSE と同じ席に座れる量です。

第3巻の世界の住人から見れば、この章の出来事は「新しい損失関数がひとつ手に入った」と要約できます。しかも出自が MSE とは違う。MSE は「符号を消したい、大外れを重く罰したい」という使い勝手から**選ばれた**損失でした(第3巻第3章で「なぜ正しいかの正当化は第4巻で」と先送りしたのを覚えているでしょうか)。NLL は**「観測されたデータを最ももっともらしくするパラメータを選べ」という確率の原理から導出されて出てきた**損失です。選んだのではなく、出てきた。

ここまで来ると、第3巻エピローグに積まれた3つの宿題の解け方がほとんど見えています。

1. シグモイドは、なぜあの形でなければならないのか
2. シグモイドが出す 0〜1 の数は、何と解釈すべきなのか
3. 分類の正しい損失関数とは何か

宿題2の答えの候補は第1章からずっと匂っていました——あの 0〜1 は**確率**だ、と。もし宿題2がそう解けるなら、つまりモデルの出力が「ラベルが 1 である確率」なら、宿題3にはこの章の原理が機械的に答えます。**モデルがデータに割り当てた確率の、負の対数尤度を損失にすればいい。**ラベルは 0 か 1 の2択——ベルヌーイ分布の尤度なら、この章でもう書き下せます。

残る作業は3つ。シグモイドの出力を「確率」として正当化すること(宿題1・2)。ベルヌーイの NLL を損失として書き下し、勾配を手で出すこと。そして**あの死んだ勾配が生き返るのを、第3巻と同じデータでこの目で確かめる**こと。次章で宿題を全部、机から下ろします。ちなみに、こうして導出される損失の一族には原論文にも登場する有名な名前が付いていますが——その名前を知るのは第5章です。

## まとめ

- **尤度** $L(\theta)$ は、「データの出る確率」の式を**データ固定・パラメータ可変**で読み替えた、パラメータのもっともらしさの点数。第3巻第3章の「損失はパラメータの関数」と同じ視点の反転
- **対数は積を和に変える**($\log MN = \log M + \log N$)。尤度の積は対数尤度の和になり、数値はアンダーフローから救われ、微分は項ごとにできるようになる。単調性により、頂上の場所は動かない
- ベルヌーイ分布の最尤推定を微分で解くと $\hat{\theta} = k/n$。**「頻度で当てる」という直観は、最尤原理から導出される結論**だった
- 尤度の最大化は、**負の対数尤度(NLL)の最小化**と同じ。NLL は損失関数の資格を完全に満たし、確率の世界(本章)と第3巻の「学習 = 損失最小化」の世界がここで合流する
- 新しい微分の道具 $(\log x)' = 1/x$ を道具箱に追加した(次章、log loss の勾配計算で主役になります)

**ラスボスとの距離**: 論文 Section 5.4 の "perplexity" も、label smoothing が施される損失も、すべて「モデルの良さを $\log$ 確率で測る」という本章の物差しの上に立っています。その損失の本名(第5章)まで、あと2章です。

## 演習

**問1**(対数の足慣らし) $\log 2 \approx 0.693$、$\log 3 \approx 1.099$ とします($\log$ は自然対数)。電卓を使わず、この2つの値と対数の性質だけから、$\log 8$、$\log 12$、$\log 1.5$ を求めてください。

**問2**(最尤推定を微分で解く) コインを20回投げたら、表が5回出ました。対数尤度 $\ell(\theta)$ を書き下し、微分して傾き 0 を解き、$\hat{\theta}$ を求めてください。3.4 の一般式 $k/n$ に代入するのではなく、$k=5,\ n=20$ の数字で最初から手を動かすこと。

**問3**(対数のありがたみ) 対数を取らずに、$L(\theta) = \theta^7(1-\theta)^3$ を第2巻の積の微分ルールで直接微分し、頂上が 3.4 の答えと一致することを確認してください。ヒント: 微分した式から共通因数 $\theta^6(1-\theta)^2$ をくくり出すと、残りが見やすくなります。

**問4**(端で起きること) コインを3回投げたら、3回とも表でした($n = k = 3$)。対数尤度は $\ell(\theta) = 3 \log \theta$ です。微分して傾き 0 を解こうとすると、何が起きますか。$\hat{\theta}$ はいくつになりますか。最後に、この推定値を一言で批評してください。

<summary>略解</summary>

**問1** $\log 8 = \log 2^3 = 3 \log 2 \approx 2.079$。$\log 12 = \log(4 \cdot 3) = 2\log 2 + \log 3 \approx 2.485$。$\log 1.5 = \log(3/2) = \log 3 - \log 2 \approx 0.405$。最後の式は「割り算は引き算になる」($\log(M/N) = \log M - \log N$)を使っており、これは性質1を $N \cdot (M/N) = M$ に適用すれば出てきます。

**問2** $\ell(\theta) = 5 \log \theta + 15 \log(1-\theta)$。微分して $\ell'(\theta) = \dfrac{5}{\theta} - \dfrac{15}{1-\theta}$。これを 0 と置き、$\theta(1-\theta)$ を掛けて $5(1-\theta) = 15\theta$、つまり $5 = 20\theta$ で $\hat{\theta} = 1/4$。表の頻度 $5/20$ と一致します。

**問3** 積のルールと連鎖律で
$$L'(\theta) = 7\theta^6(1-\theta)^3 + \theta^7 \cdot 3(1-\theta)^2 \cdot (-1) = \theta^6(1-\theta)^2\,\{7(1-\theta) - 3\theta\} = \theta^6(1-\theta)^2\,(7 - 10\theta)$$
$0 < \theta < 1$ でこれが 0 になるのは $7 - 10\theta = 0$、つまり $\hat{\theta} = 0.7$ で、3.4 と一致します。積のルール・連鎖律・因数分解を総動員してこの手数——対数を取れば2項の和の微分で済んだことと比べてください。因子が1000個になっても同じ方法を選びたいか、想像してみると、3.3 の「積を和に」のありがたみが体感できます。

**問4** $\ell'(\theta) = 3/\theta$ は $0 < \theta < 1$ でつねに正で、**傾き 0 の解は区間の内部に存在しません**。$\ell(\theta)$ は単調増加なので、もっともらしさは右端で最大、つまり $\hat{\theta} = 1$ です(3.4 の一般式 $k/n = 3/3$ とも一致)。批評: たった3回の観測から「このコインは裏が永遠に出ない」と断言する推定になっています。最尤推定は手元のデータを全面的に信頼する原理なので、データが少ないと、このように極端な値へ振り切れることがあります。
