# 第6章 softmax — 多クラスへ

> [目次](../TOC.md) ・ [← 前の章](05-entropy.md) ・ [次の章 →](07-variance-of-dot-products.md)

第4章で第3巻からの宿題は片づきました。シグモイドの出力を確率と解釈し、尤度から log loss を導出すると、勾配は $(y − t)x$ というきれいな形になり、死んでいた学習が動き出した。第5章では、その log loss が cross-entropy の特殊例であり、「尤度最大化」と「cross-entropy 最小化」が同じだと判明しました。分類の正しい損失関数は、もう手の中にあります。

——ただし、いまのところ**2クラス限定**です。

この巻の例題は第1章からずっと「次の単語当て」でした。本物の言語モデルは2択ではなく、語彙にある数万の単語から次の1語を選ぶ——選択肢が $K$ 個ある分類です。シグモイドは数を1個しか出せません。$K$ 個の選択肢に確率を配る道具が、ひとつ足りないのです。

その道具が、ラスボスの式(1)の真ん中に鎮座する $\mathrm{softmax}(QK^T/\sqrt{d_k})$ の **softmax** です。この章を終えると、式(1)の未読部分は $\sqrt{d_k}$ ただひとつになります。

## 6.1 2クラスから K クラスへ: 「スコアの列を確率分布に変える」関数が欲しい

第4章のロジスティック回帰では、線形(第1巻第6章の `X @ w + b`)がスコアを**1個**出し、シグモイドがそれを 0〜1 に写して「クラス1の確率」$p$ とした。クラス0の確率は $1 - p$ で自動的に決まる。2クラスなら数は1個で足ります。

では $K$ クラス——$K = 3$ でメールを「通常・スパム・メルマガ」に仕分けるなら、出力すべきは3つの確率です。1個では足りません。

モデル側の拡張は、もう持っている道具でできます。第1巻第6章の linear は $W$ の列数を増やすだけで出力を並べられました。$\mathbf{x}$ `(d,)` に対して $W$ `(d, K)` と $\mathbf{b}$ `(K,)` を使えば

$$\mathbf{z} = \mathbf{x} W + \mathbf{b} \quad \texttt{(K,)}$$

と、クラスごとのスコアが $K$ 個並んだ列が出てきます。スコア $z_k$ は「クラス $k$ らしさの点数」です(この列は慣用で **ロジット(logits)** とも呼ばれ、以後の巻でも顔を出します)。

問題は、この列が**確率分布ではない**ことです。第1章のルールは「0以上で、全部足すと1」。一方スコアはただの実数の列で、負の値もあれば、和が1になる理由もありません。第4章でシグモイドが「1個のスコア → 1個の確率」の橋だったように、今度は「スコアの列 → 確率分布」の橋が欲しい。これがこの章の需要です。

橋に求める条件はこうです。

1. **すべての成分が正**になること(確率のルールその1)
2. **和が1**になること(確率のルールその2)
3. **スコアの大小関係を保つ**こと(点数が高いクラスほど高い確率に)
4. **滑らかに微分できる**こと(勾配で学習するため。第3巻エピローグの教訓です)

素朴な案「和で割れば1になるのだから $z_k / \sum_j z_j$ でいい」は、条件1で即死します。スコアに負の値が混ざれば負の「確率」が生まれ、和がゼロに近ければ割り算ごと吹き飛びます。

なら、先に**全部を正に変えてから**和で割ればいい。負の実数も正に写し、大小関係を保ち、どこでも滑らかな関数を、私たちはひとつ知っています。指数関数 $e^z$ です。常に正、単調増加、微分しても形が変わらない。おまけに log と打ち消し合う——この最後の性質が、第3章で対数尤度を選んだことと噛み合って、この章の後半で効いてきます。

## 6.2 softmax の定義と性質: exp で正に、和で割って1に。最大値シフトによる数値安定化

形式化します。スコアの列 $\mathbf{z}$ `(K,)` に対し、**softmax** は次の成分を持つベクトル `(K,)` を返す関数です。

$$\mathrm{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$

やっていることは2手だけ。**exp で正にして、和で割って1にする。**

小さな例で。$\mathbf{z} = (2, 1, 0)$ なら、$e^2 = 7.389$、$e^1 = 2.718$、$e^0 = 1$、和は $11.107$。割ると

$$\mathrm{softmax}(2, 1, 0) = (0.665,\ 0.245,\ 0.090)$$

全部正で、足すと1で、点数の順位どおり。条件1〜3が満たされ、exp と割り算だけなので条件4(微分可能)も満たします。

名前の由来です。スコアから「最大の成分だけを選ぶ」と、出力は (1, 0, 0) のような勝者総取りの配分になります(いわば hardmax)。softmax はその柔らかい版で、最大スコアのクラスに最も厚く配りつつ、2位以下にも確率を残します。「最大値選びの、柔らかい(soft)版」だから softmax です。

地味ですが後で命綱になる性質をひとつ。**全成分に同じ数 $c$ を足しても、softmax の結果は変わりません。**

$$\mathrm{softmax}(\mathbf{z} + c) = \mathrm{softmax}(\mathbf{z})$$

分子は $e^{z_i + c} = e^c \cdot e^{z_i}$、分母も各項が $e^c$ 倍で、分子分母の $e^c$ が約分されて消えます。softmax が見ているのはスコアの**差**だけで、目盛りの原点は気にしないのです。

### 実装して、壊す

定義をそのままコードにします(この節と6.5節のコードは1本のスクリプトで、`code/ch06/stable_softmax.py` にあります。`python3` で全 assert 通過)。核心は素朴版の定義と、それが大きなスコアで壊れる観測です。

```python
def softmax_naive(z):
    """定義をそのまま書いた softmax。あとで壊れるところを観測するための版。"""
    e = np.exp(z)
    return e / e.sum()


z = np.array([2.0, 1.0, 0.0])
p = softmax_naive(z)
assert np.all(p > 0) and np.isclose(p.sum(), 1.0)  # softmax([2,1,0]) = [0.6652 0.2447 0.09]
```

手計算と一致します。では壊します。スコアを大きくすると——

```python
z_big = np.array([1000.0, 1001.0, 1002.0])
p_naive = softmax_naive(z_big)   # → [nan nan nan]
assert np.isnan(p_naive).any()
```

答えは確率分布ではなく `[nan nan nan]`——「数ではない(not a number)」が3つ並びます。

犯人は $e^{1000}$ です。float64 に入る最大の数はおよそ $1.8 \times 10^{308}$、exp で言えば $e^{709}$ あたりが天井です。$e^{1000}$ はそれを突き抜けて無限大(`inf`)に化けます。これを**オーバーフロー(overflow)**と呼びます。`inf / inf` の答えが `nan` で、一度生まれると以後の計算をすべて汚染します。学習の途中で損失が突然 `nan` になる——機械学習の実装で最も古典的な事故のひとつです。

「スコアが1000なんて極端では?」と思うかもしれませんが、第7章で見るように、内積で作ったスコアは次元が大きいほど膨らみます。これは現実に起きる事故です。

### 最大値シフトで直す

直し方は、さきほどのシフト不変性が教えてくれます。全成分から同じ数を引いても答えは変わらないのですから、**最大値を引いてから** exp すればよいのです。

```python
def softmax(z):
    """数値安定化版。最大値を引いてから exp する(答えは naive 版と同じ)。"""
    z = np.asarray(z, dtype=float)
    e = np.exp(z - z.max())  # 最大成分が 0 → exp は高々 1 → オーバーフロー不可能
    return e / e.sum()
```

最大値を引くと、シフト後の最大成分はちょうど0、他はすべて負。exp の値は高々 $e^0 = 1$ で、オーバーフローは原理的に起きません。$(1000, 1001, 1002)$ の softmax は $(-2, -1, 0)$ の softmax として計算され、$(0.090, 0.245, 0.665)$ という真っ当な分布が返ります。数学の上では何も変えていない——変えたのは計算の**手順**だけです。これが**数値安定化(numerical stabilization)**で、以後すべての巻の softmax はこの版を使います(逆方向の事故、exp が0に潰れるアンダーフローも起きえますが、分母は最大成分の $e^0 = 1$ が支えるので、分布が `nan` に壊れることはありません)。`stable_softmax.py` ではシフト不変性と naive 版との一致を assert で確認しています。

最後に、シグモイドとの関係です。$K = 2$ で片方のスコアを0に固定すると

$$\mathrm{softmax}(z, 0)_1 = \frac{e^z}{e^z + e^0} = \frac{1}{1 + e^{-z}} = \mathrm{sigmoid}(z)$$

**シグモイドは、softmax の $K=2$ の特殊例だったのです**(`stable_softmax.py` で複数の $z$ について assert)。第5章で「log loss は cross-entropy の特殊例」と判明したのと同じ構図が、モデル側でも起きました。

## 6.3 softmax + cross-entropy の勾配もきれい: (y − t)

橋は架かりました。スコアの列 $\mathbf{z}$ が確率分布 $\mathbf{y} = \mathrm{softmax}(\mathbf{z})$ `(K,)` になります。次は損失です。

その前に正解の表し方を決めます。$K$ クラスの正解は「クラス2」のような整数ラベルですが、これを長さ $K$ のベクトルに書き直します。正解クラスの位置だけ1、残りは0——$K=3$ で正解がクラス1なら $\mathbf{t} = (0, 1, 0)$ `(K,)`。これを **one-hot(one-hot encoding)** と呼びます。

one-hot は単なる符号化ではありません。$\mathbf{t}$ は0以上で和が1——**それ自体が確率分布**です。「正解クラスに全確率を置いた、一切迷いのない分布」。正解がこう書けるなら、第5章の道具がそのまま使えます。予測分布 $\mathbf{y}$ と正解分布 $\mathbf{t}$ のズレを測る損失——cross-entropy です。

$$L = -\sum_{k=1}^{K} t_k \log y_k$$

ここで one-hot の威力が出ます。$t_k$ は正解クラス $c$ 以外すべて0なので、$K$ 項の和が1項に潰れて

$$L = -\log y_c$$

「**正解クラスに割り当てた確率の、負の対数**」。第3章の言葉で言えば負の対数尤度そのもので、第5章5.5節の「尤度最大化 = cross-entropy 最小化」が $K$ クラスでもそっくり成り立っています。

では勾配です。第4章では sigmoid + log loss のスコアへの勾配が $(y - t)$ になりました。softmax + cross-entropy では——

$$\frac{\partial L}{\partial z_j} = y_j - t_j \qquad \text{つまり} \qquad \frac{\partial L}{\partial \mathbf{z}} = \mathbf{y} - \mathbf{t} \quad \texttt{(K,)}$$

**今度も $(y - t)$ です。** クラスが何個に増えても、スコアへの勾配は「予測した分布 − 正解の分布」。第4章の結果は、この一般形の $K=2$ の断面でした。

<details><summary>導出(付録: 読み飛ばしても先へ進めます)</summary>

**手順1: softmax 自身の微分。** $y_k = e^{z_k} / \sum_j e^{z_j}$ を $z_j$ で微分すると(第2巻の商の微分で、$k = j$ と $k \neq j$ を別々に計算すると)、次の1本にまとまります。

$$\frac{\partial y_k}{\partial z_j} = y_k(\delta_{kj} - y_j)$$

$\delta_{kj}$ はクロネッカーのデルタという記号で、$k = j$ のとき1、それ以外は0です。

**手順2: 連鎖律(第2巻第5章)で合成。** $\partial L / \partial y_k = -t_k / y_k$ なので

$$\frac{\partial L}{\partial z_j} = \sum_k \left(-\frac{t_k}{y_k}\right) y_k (\delta_{kj} - y_j) = -\sum_k t_k (\delta_{kj} - y_j) = -t_j + y_j \sum_k t_k = y_j - t_j$$

最後の等号は one-hot の性質 $\sum_k t_k = 1$ です。途中、$y_k$ が分母と分子で約分されて消えました。これが「exp(モデル側)と log(損失側)の打ち消し合い」の正体です。

</details>

パラメータへの勾配は、あとは連鎖律の流れ作業です。スコアが $Z = XW + \mathbf{b}$($X$ `(n, d)`、$W$ `(d, K)`)で作られているので、$n$ 件の平均損失に対して

$$\frac{\partial L}{\partial W} = \frac{1}{n} X^T (Y - T) \quad \texttt{(d, K)}$$

($Y$ `(n, K)` は予測確率を、$T$ `(n, K)` は one-hot を行に積んだ行列)。1サンプルで見れば「$(\mathbf{y} - \mathbf{t})$ に入力 $\mathbf{x}$ を掛けたもの」——第4章の $(y - t)\,\mathbf{x}$ と寸分違わぬ形です。並べてみましょう。

| | モデルの出口 | 損失 | スコアへの勾配 |
|---|---|---|---|
| 第4章(2クラス) | sigmoid | log loss | $y - t$ |
| 本章($K$ クラス) | softmax | cross-entropy | $\mathbf{y} - \mathbf{t}$ |

このきれいさは見た目だけではありません。勾配が「予測 − 正解」なら、**大きく間違えるほど大きな勾配が出ます**。第3巻エピローグで観測した sigmoid + MSE の勾配死は、ここでは原理的に起きません。いちばん反省すべき点が、いちばん大声で文句を言う。学習にとって正しい世界です。

そしてこれは偶然ではありません。確率のルールに合わせて exp で分布を作り(本章)、尤度から log で損失を導いた(第3〜5章)。**exp と log は逆関数同士**ですから、勾配で打ち消し合うのは仕組まれた再会です。損失を「導出」したご褒美が、この勾配です。

## 6.4 [コード] 多クラス分類(softmax回帰)をフルスクラッチ実装

道具が揃ったので組み立てます。linear でスコア、softmax で確率、cross-entropy で損失、勾配は $(Y - T)$——この一式は **softmax回帰(softmax regression)** と呼ばれます(多項ロジスティック回帰という別名も)。この節のコードは全体で1本のスクリプトで、`code/ch06/softmax_regression.py` にあります(`python3` で全 assert 通過)。

新しいのは、6.2節の softmax を「行列の各行に適用する」形に書き直す点だけです($n$ 件をまとめて処理するため。最大値シフトも**行ごと**)。部品はこうです。

```python
def softmax_rows(Z):
    """(n, K) の各行に数値安定化版 softmax(6.2)を適用する。"""
    Z_shift = Z - Z.max(axis=1, keepdims=True)  # 行ごとの最大値シフト
    E = np.exp(Z_shift)
    return E / E.sum(axis=1, keepdims=True)


def forward(X, W, b):
    """モデル: スコア Z = X @ W + b を softmax で確率に。→ (n,K)"""
    return softmax_rows(X @ W + b)


def gradients(X, Y, T):
    """6.3 の結果: スコアへの勾配は (Y - T) / n。あとは第2巻の連鎖律で W と b へ。"""
    n = X.shape[0]
    dZ = (Y - T) / n         # (n, K)
    grad_W = X.T @ dZ        # (d, K)
    grad_b = dZ.sum(axis=0)  # (K,)
    return grad_W, grad_b
```

(`one_hot` と `cross_entropy` も同スクリプトにあります。)データは第3巻エピローグのスパム判定を3クラスに拡張したもの——通常メール・スパム・メルマガ、特徴量は同じ2つ(怪しい単語の出現量、感嘆符の多さ)。各クラスのガウス分布から100件ずつ、計300件を生成し、one-hot 化が確率分布になっていること(各行ちょうど1つだけ1、1の位置=正解)を assert で確認します。散布図を描けば(第3巻エピローグ図E.1の要領)、左下に通常、右上にスパム、左上にメルマガの3つの塊が見えます。

学習の前に、6.3節の主張を検算します。導出を信じるのではなく、**数値微分(第2巻)と突き合わせる**のです。各パラメータを $\pm\varepsilon$ 動かして損失の差分を取り、解析勾配 $(Y - T)$ 由来の値と比べる。

```python
# --- 勾配チェック: 解析勾配 (Y - T) を数値微分(第2巻)と突き合わせる ---
grad_W, grad_b = gradients(X, forward(X, W, b), T)
eps = 1e-6
for i in range(W.shape[0]):
    for j in range(W.shape[1]):
        W_pos = W.copy(); W_pos[i, j] += eps
        W_neg = W.copy(); W_neg[i, j] -= eps
        num = (cross_entropy(forward(X, W_pos, b), T)
               - cross_entropy(forward(X, W_neg, b), T)) / (2 * eps)
        assert np.isclose(grad_W[i, j], num, rtol=1e-4, atol=1e-8)
```

assert は通ります。紙の上の導出と機械の差分計算が、$W$ の6成分すべてで一致しました。この「勾配チェック」は、第5巻で自動微分を自作するときの命綱になります。

あとは第3巻と同じ4拍子(forward → loss → gradient → update)を300ステップ回すだけです。実行するとこう表示されます。

```
loss: 1.2104 → 0.0589, accuracy: 0.977
```

損失は 1.21 から 0.059 へ一直線に下がり、300件中293件を正しく仕分けています(初期損失が $\log 3 \approx 1.10$ の近くなのは偶然ではありません。学習前のモデルはほぼ「3択を当てずっぽう」で、その cross-entropy が $-\log\frac{1}{3} = \log 3$ だからです)。スクリプトは損失が1桁以上下がること・accuracy ≥ 0.95 を assert します。勾配は最後まで死にませんでした。2クラスの宿題(第4章)に続き、$K$ クラスの分類も、導出した損失で解けました。

## 6.5 温度: 分布を尖らせる・なだらかにする(√d_k への前振り)

最後に softmax の「つまみ」をひとつ。この巻の残りの伏線——$\sqrt{d_k}$——に直結します。

6.2節で softmax はスコアの差だけを見る(シフト不変)と確認しました。では、スコアを**定数倍**したら? $(2, 1, 0)$ と $(20, 10, 0)$ は順位こそ同じですが、softmax を通すと前者は $(0.665, 0.245, 0.090)$、後者はほぼ $(1, 0, 0)$。**softmax は、スコアの目盛りの大きさに敏感なのです。**

この敏感さを逆手に取って、つまみとして整備したものが**温度(temperature)**です。スコアを温度 $T$ で割ってから softmax に入れます。

$$\mathrm{softmax}_T(\mathbf{z}) = \mathrm{softmax}\!\left(\frac{\mathbf{z}}{T}\right)$$

(この $T$ は転置の $T$ とも one-hot の行列 $T$ とも無関係。統計力学から借りた名前で、コードでは `tau` と書いて区別します。)

実験で体感します(`stable_softmax.py` の後半。温度を上げるほどエントロピーが単調増加すること・順位不変を assert)。

```
tau= 0.1: [1. 0. 0.]  entropy=0.000
tau= 0.5: [0.867 0.117 0.016]  entropy=0.441
tau= 1.0: [0.665 0.245 0.09 ]  entropy=0.832
tau= 2.0: [0.506 0.307 0.186]  entropy=1.020
tau=10.0: [0.367 0.332 0.301]  entropy=1.095
```

読み取れることは3つです。

- **温度を下げる($T < 1$)と、分布は尖ります。** $T \to 0$ の極限では勝者総取りの one-hot、つまり6.2節の hardmax に行き着きます
- **温度を上げる($T > 1$)と、分布はなだらかになります。** $T \to \infty$ では差が押し潰されて一様分布($1/3$ ずつ)に近づきます
- 第5章のエントロピーで測ると、この変化は単調です。温度は「分布の不確かさを連続的に調整するつまみ」です。なお、どの温度でも**順位は変わりません**($z/T$ は単調変換なので argmax は不変)

さて、この形をどこかで見ていませんか。ラスボスの式(1)です。

> *"... we compute the dot products of the query with all keys, divide each by √d_k, and apply a softmax function ..."*
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1
>
> 訳: クエリとすべてのキーの内積を計算し、それぞれを √d_k で割ってから、softmax 関数を適用する。

スコア($QK^T$ の各成分)を、定数($\sqrt{d_k}$)で**割ってから** softmax に入れる——式(1)の $\sqrt{d_k}$ は、まさしく**温度**です。論文は softmax を温度 $\sqrt{d_k}$ で運転しているのです。

なぜ温度調節が必要なのか、予感はもうあるはずです。内積で作ったスコアは次元 $d_k$ が大きいほど膨らみます(見積もる道具——和の分散——は第2章で仕込みました)。膨らんだスコアを softmax に入れると分布は one-hot 近くまで尖り、尖り切った softmax の勾配は——第3巻エピローグのシグモイドと同じ病気で——死にます。スコアがどれくらい膨らむのか、なぜ割る数が $2$ でも $d_k$ でもなく $\sqrt{d_k}$ なのか。この伏線は次の第7章で回収します。

温度にはもうひとつ先の楽しみがあります。訓練済みの言語モデルに文章を**生成**させるとき、次の単語を softmax の分布からサンプリングしますが、その際この温度つまみで「堅実な出力(低温)」と「多彩な出力(高温)」を調整します。チャットAIの設定にある temperature の正体がこれです。実物は第8巻5章で回収します。

## まとめ

- **softmax** はスコアの列 `(K,)` を確率分布に変える橋。「exp で正に、和で割って1に」の2手。シグモイドは $K=2$ の特殊例だった
- 素朴な実装は大きなスコアで $e^z$ が**オーバーフロー**し `nan` を返す。**最大値シフト**(シフト不変性が保証)で直すのが数値安定化の定石
- 正解の **one-hot** 表現は「正解クラスに全確率を置いた分布」。one-hot との cross-entropy は $-\log y_c$、つまり負の対数尤度に一致する
- softmax + cross-entropy のスコアへの勾配は $\mathbf{y} - \mathbf{t}$。第4章の $(y-t)\,\mathbf{x}$ の一般形で、勾配死は起きない。きれいさの源は exp と log の打ち消し合い
- **温度** $T$ で割ると分布を尖らせ($T<1$)・なだらかに($T>1$)できる。式(1)の $\sqrt{d_k}$ は温度そのもの(第7章で回収)。生成時のサンプリング調整にも使う(第8巻5章で回収)

**ラスボスとの距離**: 式(1) $\mathrm{softmax}(QK^T/\sqrt{d_k})V$ のうち、ついに softmax が読めました。残る未読は $\sqrt{d_k}$ ——「なぜこの値で割るのか」だけです。次章で回収します。

## 演習

**問1(one-hot と cross-entropy の関係)** 3クラス分類のモデルが、あるサンプルに予測分布 $\mathbf{y} = (0.7, 0.2, 0.1)$ を出しました。正解はクラス1(one-hot で $\mathbf{t} = (0, 1, 0)$)です。

1. cross-entropy $L = -\sum_k t_k \log y_k$ を定義どおり3項書き下して計算してください
2. その値が $-\log y_1$(正解クラスの確率の負の対数)と一致することを確認してください
3. $K = 2$、$\mathbf{t} = (1-t, t)$、$\mathbf{y} = (1-y, y)$ とおくと、この cross-entropy が第4章の log loss $-\{t \log y + (1-t)\log(1-y)\}$ そのものになることを式で確認してください

<details><summary>略解</summary>

1. $L = -(0 \cdot \log 0.7 + 1 \cdot \log 0.2 + 0 \cdot \log 0.1) = -\log 0.2 \approx 1.609$。$t_k = 0$ の項は $\log y_k$ の値によらず消えます
2. 残るのは正解クラスの項だけなので $L = -\log y_1 = -\log 0.2$。一致します。one-hot との cross-entropy とは「正解に何%賭けていたか」だけを問う損失です
3. $-\sum_k t_k \log y_k = -\{(1-t)\log(1-y) + t \log y\}$。第4章4.2節の log loss と同一です。第5章の「log loss は cross-entropy の特殊例」を one-hot の言葉で言い直したことになります

</details>

**問2(手計算とシフト不変性)** 次の softmax を、電卓なしの手計算で求めてください。

1. $\mathrm{softmax}(1, 1, 1)$ と $\mathrm{softmax}(5, 5, 5)$
2. $\mathrm{softmax}(100, 101, 102)$ を、6.2節の安定版の手順(最大値シフト)で「どんな計算に帰着するか」まで書き下してください(exp の数値自体は求めなくて構いません)

<details><summary>略解</summary>

1. どちらも $(1/3, 1/3, 1/3)$。成分が全部同じなら $e^z$ も全部同じで、和で割れば均等割りです。$(1,1,1)$ と $(5,5,5)$ が同じ答えになるのは、シフト不変性($c=4$ を足しただけ)からも当然です
2. 最大値 102 を引いて $\mathrm{softmax}(-2, -1, 0)$ に帰着します。つまり $(e^{-2}, e^{-1}, 1)$ を和 $e^{-2} + e^{-1} + 1$ で割ったもの。素朴にやれば $e^{100}$ 級の巨大数が3つ現れるところが、高々1の数3つの計算になりました

</details>

**問3(コード: 温度は順位を変えない)** 6.4節の学習済みモデルのスコア `Z = X @ W + b` に温度を導入し、`softmax_rows(Z / tau)` の argmax で予測したときの accuracy を `tau = 0.1, 1.0, 10.0` で比べてください。実行前に結果を予想してから確かめること。

<details><summary>略解</summary>

3つとも accuracy は**完全に同じ**(0.977)になります。温度はスコアの単調変換なので、各行の argmax は変わりません。温度が変えるのは「どれを1位にするか」ではなく「1位にどれだけ確率を集中させるか」。だから argmax で答えだけ取る分類では温度は無意味で、分布から**サンプリングする**生成(第8巻5章)や勾配の流れ(第7章)で初めて意味を持ちます

</details>
