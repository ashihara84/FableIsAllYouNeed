前章で、第3巻からの宿題が解けました。シグモイドの出力を「確率」と読み、ベルヌーイ分布の尤度を最大化する。そこから負の対数尤度として log loss が導出され、MSE では死んでいた勾配が log loss なら生きていることも確認しました。

分類の正しい損失は手に入りました。では、この巻はこれで終わりでしょうか。まだ読めない言葉が残っています。巻頭に掲げたラスボスのひとつを、もう一度見ます。

> *"We employed label smoothing of value ε_ls = 0.1. This hurts perplexity, as the model learns to be more unsure..."*

>
> 訳: 私たちは値 ε_ls = 0.1 の label smoothing を用いた。モデルがより「自信を持たない」ように学習するため、これは perplexity を悪化させるが……

label smoothing。perplexity。そしてこの文の背後には、書かれてこそいませんが、**cross-entropy** という損失関数が前提として横たわっています。Transformer の訓練で実際に最小化されるのは、この cross-entropy です。

前章で分類の損失を尤度から導出した私たちには、奇妙な疑問が湧きます。論文の言う cross-entropy は、それとは別物なのか、それとも同じものなのか。答えを先に言います。**同じもの**です。前章の log loss は、cross-entropy という大きな概念の特殊例(2クラスの場合)だった——これがこの章の最後で判明し、その瞬間が本巻の副題「分類の損失関数を求めて」の終着点になります。

そのために、まず「情報」という数になっていなかったものを数にします。出発点は意外なほど日常的な感覚——「驚き」です。

## 5.1 情報量 = 驚きの量: −log p という定義の妥当性

「情報の量」を数で測れるでしょうか。

「明日、太陽が昇る」と聞いて驚く人はいません。ほぼ確実とわかっていることを聞いても得るものがないからです。一方「6月の東京に雪が降った」と聞けば驚きます。めったに起きないことが起きたと知ったとき、私たちは大きな情報を受け取っています。言葉でも同じで、「吾輩は」の次に「猫」が来ても驚きませんが「冷蔵庫」なら驚きます。**起きる確率が低い出来事ほど、それが起きたと知ったときの情報は大きい。** 情報の量とは驚きの量である——これがこの節の出発点です。

確率 $p$ で起きる出来事の驚きを $I(p)$ と書き、$I$ が満たすべき条件を挙げます。驚きという感覚から、次の2つは譲れません。

**性質1: 稀なほど驚く。** $p$ が小さいほど $I(p)$ は大きい。そして確実な出来事($p = 1$)の驚きは $0$ です。起きるとわかっていたことが起きても、何も学ばないからです。

**性質2: 独立な驚きは足せる。** 互いに独立な出来事 A と B の知らせを同時に受け取ったときの驚きは、それぞれの驚きの合計であってほしい。「コインで表」と「サイコロで6」を同時に聞いた驚きは、別々に聞いた驚きの和になるべきです。

性質2を式にします。第1章で見たように、独立な出来事が両方起きる確率は積 $p_A \, p_B$ でした。だから性質2はこう書けます。

$$I(p_A \, p_B) = I(p_A) + I(p_B)$$

**確率の積を驚きの和に変える関数**——これは第3章で、尤度の積を扱いやすい和に変えるために使った道具と同じです。対数です。$\log(ab) = \log a + \log b$ がここでもう一度主役になります。

ただし $\log p$ そのままでは向きが逆です。$p$ が小さいほど $\log p$ は大きな負の数になるので、符号を反転させます。こうして情報量の定義に到達します。

確率 $p$ で起きる出来事の**情報量(information content)**、すなわち驚きの量を、次で定義します。

$$I(p) = -\log p$$

性質1と性質2を満たすことはすぐ確かめられます。$p = 1$ なら $I = -\log 1 = 0$。$p$ が小さくなるほど $-\log p$ は大きくなります。そして $-\log(p_A p_B) = -\log p_A - \log p_B$ なので、独立な驚きは足せます。

実は、この2つの性質(と「$p$ を少し変えたら驚きも少しだけ変わる」連続性)を満たす関数は対数の定数倍しかないことが証明できます。つまり $-\log p$ は便利な選択ではなく**本質的に唯一の選択**です。証明はこの本の流儀に従って行わず、「驚きの2つの性質を式にしたら対数しか残らなかった」という納得で先に進みます。

対数の底は何でもよく、底を変えても全体が定数倍されるだけです。底を $2$ にしたときの単位を**ビット(bit)**、自然対数(底 $e$)のときの単位を**ナット(nat)**と呼びます。手計算では底 $2$ がきりがよいのでこの章の数値例は底 $2$ で進め、機械学習の実装では第3章の対数尤度と同じく自然対数を使います。

小さな数値で感覚をつかみます。

| 出来事の確率 $p$ | 情報量 $-\log_2 p$ |
|---|---|
| $1$(確実) | $0$ ビット |
| $1/2$(コインの表) | $1$ ビット |
| $1/8$ | $3$ ビット |
| $1/1024$ | $10$ ビット |

コイン投げの結果を知ることがちょうど $1$ ビット。確率 $1/1024$ の珍事を知ることは、コイン投げ $10$ 回分の驚きに相当します。「2分の1になるたびに1ビット増える」というスケール感を覚えておいてください。

## 5.2 エントロピー: 驚きの期待値。分布の「不確かさ」

情報量は「出来事1つ」の驚きでした。次は分布全体についての問いです。

「次の単語当て」(第1章からの定番)では、文脈によって当てやすさがまるで違います。「吾輩は」の次はほぼ「猫」で決まりですが、「今日は」の次は何でもありえます。この**分布そのものが持つ不確かさ**を、1つの数で表せないでしょうか。

材料は揃っています。分布の各出来事には情報量 $-\log p_k$ がつき、それぞれ確率 $p_k$ で起きる。「確率つきの数を1つに要約する」道具は、第2章の期待値でした。そこで**驚きの期待値**を取ります。

$K$ 個の出来事を持つ分布 $p = (p_1, \dots, p_K)$ の**エントロピー(entropy)**を、次で定義します。

$$H(p) = \sum_{k=1}^{K} p_k \cdot (-\log p_k) = -\sum_{k=1}^{K} p_k \log p_k$$

意味は「この分布から1回サンプルしたとき、平均してどれだけ驚くか」です。次の単語が「猫」「犬」「鳥」のどれかとして、3語の小さな世界で計算します(底は $2$)。

**例1: ほぼ決まっている。** $p = (1, 0, 0)$、つまり必ず「猫」。

$$H = -1 \log_2 1 = 0 \ \text{ビット}$$

驚きはゼロ。結果を聞く前から答えがわかっている分布です。なお $p_k = 0$ の項は $0 \cdot \log 0$ という不定形になりますが、「起きないことには驚きようがない」ので $0$ と扱う慣習です($p \log p$ は $p \to 0$ で $0$ に近づくので、この慣習は連続性とも合っています)。

**例2: 完全に五分(三分)。** $p = (1/3, 1/3, 1/3)$。

$$H = -3 \cdot \tfrac{1}{3} \log_2 \tfrac{1}{3} = \log_2 3 \approx 1.585 \ \text{ビット}$$

**例3: 偏りがある。** $p = (1/2, 1/4, 1/4)$。

$$H = \tfrac{1}{2} \log_2 2 + \tfrac{1}{4} \log_2 4 + \tfrac{1}{4} \log_2 4 = 0.5 + 0.5 + 0.5 = 1.5 \ \text{ビット}$$

例2と例3を見比べると、同じ3択でも偏りがあると不確かさは小さくなります($1.5 < 1.585$)。一般に、$K$ 択のエントロピーは**一様分布のとき最大で $\log K$、1点に集中したとき最小で $0$** です(証明はせず、両端の例と章末のコードで納得します)。

エントロピーは「分布の予測しにくさ」のメーターです。言語モデルの世界では、このメーターの指数を取った **perplexity** という量が「平均して何択に迷っているか」を表す指標として使われ、本巻の第7章で論文の "This hurts perplexity" を読むときに登場します。ここでは「エントロピーの親戚が論文に出てくる」とだけ覚えておいてください。

## 5.3 cross-entropy: 本当の分布 p を、自分の分布 q で符号化する損

ここまでは分布が1つの話でした。しかし私たちが本当に扱いたい状況には、分布が**2つ**あります。

世界が単語を生成する**本当の分布 $p$**。そして、モデルが「次はこれが来るはず」と予想する**モデルの分布 $q$**。前章のシグモイドの出力は、まさにこの $q$ でした。問いはこうです——**$q$ が $p$ からどれだけ「悪い」かを数で測れないか。** 測れたら、その数を損失として最小化すればよいはずです。

驚きの言葉で考えます。$q$ を信じている人は、単語 $k$ を見たとき $-\log q_k$ だけ驚きます。自分の予想 $q_k$ が小さい単語ほど強く驚くわけです。ところが単語が実際に出てくる頻度を決めるのは本人の予想ではなく世界の分布 $p$ です。つまりこの人の**平均の驚き**は、「$p$ で重みづけた $-\log q$ の期待値」になります。

これを **cross-entropy(交差エントロピー)** と呼びます。

$$H(p, q) = -\sum_{k=1}^{K} p_k \log q_k$$

エントロピー $H(p)$ との違いは1か所だけ。$\log$ の中身が $p_k$ ではなく $q_k$ になっている——**世界は $p$ で動くのに、驚き方は $q$ で決まる**。2つの分布が「交差」しているから cross-entropy です。

名前の由来にも一言。情報理論では、稀な単語に長い符号、頻出単語に短い符号を割り当てて通信を圧縮します。最適な符号の長さが情報量 $-\log p_k$ に対応し、$H(p,q)$ は「分布を $q$ だと思い込んで符号を設計した人が、本当は $p$ の世界で払う平均の符号長」を表します。思い込みで設計したぶんだけ、通信は長く、損になります。

数値で見ます。本当の分布と、少しずれた予想を用意します。

$$p = \left(\tfrac{1}{2}, \tfrac{1}{4}, \tfrac{1}{4}\right), \qquad q = \left(\tfrac{1}{4}, \tfrac{1}{2}, \tfrac{1}{4}\right)$$

$q$ は、本当は一番よく出る「猫」を $1/4$ と過小評価し、「犬」を $1/2$ と過大評価しています。cross-entropy を計算すると、

$$H(p, q) = \tfrac{1}{2} \log_2 4 + \tfrac{1}{4} \log_2 2 + \tfrac{1}{4} \log_2 4 = 1 + 0.25 + 0.5 = 1.75 \ \text{ビット}$$

一方、前節の例3で $H(p) = H(p, p) = 1.5$ ビットでした。正しい分布を知っていれば平均 $1.5$ ビットの驚きで済むところを、ずれた $q$ を信じているせいで $1.75$ ビット驚いている。**$0.25$ ビットが「予想のずれ」で余分に払っている損**です。

重要な事実を2つ述べておきます。

1. **$H(p, q) \ge H(p)$ が常に成り立ち、等号は $q = p$ のときだけ**です。どんな予想 $q$ も、正解 $p$ 自身より平均の驚きを小さくすることはできません。この不等式の証明もこの本では行わず、章末のコードでランダムな分布を1000組作って確認します。
2. **cross-entropy は対称ではありません**。$H(p, q)$ と $H(q, p)$ は一般に別の値です。どちらの分布が「世界」でどちらが「予想」かによって、意味も値も変わります(演習3で実際に確かめます)。

事実1が言っているのは、cross-entropy は $q = p$ で最小になるということ。つまり、**$q$ を動かして $H(p,q)$ を最小化すれば、$q$ は $p$ に向かう**。「$q$ の悪さを測る数」という冒頭の問いへの答えです。

## 5.4 KL divergence: cross-entropy − entropy。分布間の「ズレ」

前節の $1.75$ ビットは2つの成分に分かれていました。

- $1.5$ ビット: 正解を知っていても避けられない驚き。**世界そのものの不確かさ**($H(p)$)
- $0.25$ ビット: 予想がずれているせいで余分に払う驚き。**モデルのせいの損**

「モデルの悪さ」を純粋に測りたいなら、世界のせいの分を引いた後半だけを取り出すのが筋です。この差を **KL divergence(KLダイバージェンス、Kullback–Leibler divergence)**と呼びます。

$$D_{\mathrm{KL}}(p \,\|\, q) = H(p, q) - H(p)$$

定義どおり引き算すると、和が1つにまとまって次の形にもなります。

$$D_{\mathrm{KL}}(p \,\|\, q) = -\sum_k p_k \log q_k + \sum_k p_k \log p_k = \sum_{k=1}^{K} p_k \log \frac{p_k}{q_k}$$

先ほどの例なら $D_{\mathrm{KL}}(p \,\|\, q) = 1.75 - 1.5 = 0.25$ ビット。性質は cross-entropy の2つの事実から直ちに従います。

- $D_{\mathrm{KL}}(p \,\|\, q) \ge 0$。常にゼロ以上($H(p,q) \ge H(p)$ だから)
- $D_{\mathrm{KL}}(p \,\|\, q) = 0$ となるのは $q = p$ のときだけ
- 対称ではない: $D_{\mathrm{KL}}(p \,\|\, q) \ne D_{\mathrm{KL}}(q \,\|\, p)$ が一般に成り立つ

「ゼロ以上で、一致したときだけゼロ」——距離によく似ています。ただし対称性がないので数学の意味での距離ではありません。だから divergence(隔たり)という控えめな名前がついています。私たちは「**分布間のズレ**を測る物差し」と呼ぶことにします。

学習にとって決定的に重要な観察を1つ。

$$H(p, q) = H(p) + D_{\mathrm{KL}}(p \,\|\, q)$$

と書き直すと、本当の分布 $p$ が固定なら $H(p)$ はただの定数です。ということは、**$q$ について cross-entropy を最小化することと KL divergence を最小化することは、まったく同じ問題**です。cross-entropy の最小化とは「モデルの分布を、本当の分布にズレなく重ねにいくこと」にほかなりません。損失関数の正体が、分布合わせのゲームとして見えてきました。

なお、この KL divergence は本シリーズでもう一度、意外な場所で再登場します。論文 Section 5.4 の **label smoothing**——巻頭で掲げた "the model learns to be more unsure" の文——を実装する第8巻です。そこでは「正解の分布をわざと少しなだらかにしてから、モデルの分布とのズレを測る」という形でこの道具を使います。第8巻で回収します。

## 5.5 導出のクライマックス: 尤度最大化 = cross-entropy 最小化

道具が揃いました。この巻最大の山場です。

このシリーズでは証明や導出は原則として行わず、直観と数値例とコードで代えてきました。しかし**この導出だけは例外**です。前章で尤度から導いた log loss と、この章で驚きから定義した cross-entropy——出自のまったく違う2つが同じものだという事実は、Transformer の訓練すべての土台であり、途中を飛ばさず自分の手でたどる価値があります。各ステップの式変形はどれも小さなものです。

**ステップ0: 役者を再掲する。** 前章の設定を思い出します。データは $N$ 個の組 $(\mathbf{x}_n, t_n)$ で、正解ラベルは $t_n \in \{0, 1\}$。モデルはシグモイドの出力

$$y_n = \sigma(\mathbf{w} \cdot \mathbf{x}_n + b)$$

を「クラス $1$ である確率」として出します($\mathbf{w}$, $\mathbf{x}_n$ はともに `(d,)` のベクトル、$y_n$ はスカラー)。ベルヌーイ分布として書けば、モデルが主張する確率は

$$q(t \mid \mathbf{x}_n) = \begin{cases} y_n & (t = 1) \\ 1 - y_n & (t = 0) \end{cases}$$

です。そして前章で、データ全体の尤度から出発し、

$$L = \prod_{n=1}^{N} y_n^{t_n} (1 - y_n)^{1 - t_n} \tag{5.1}$$

対数を取って符号を反転した負の対数尤度

$$E = -\log L = \sum_{n=1}^{N} \ell_n, \qquad \ell_n = -\bigl[\, t_n \log y_n + (1 - t_n) \log(1 - y_n) \,\bigr] \tag{5.2}$$

を log loss と呼んだのでした。第3章5節で確認したとおり、$\log$ は単調増加で符号反転は最大と最小を入れ替えるだけなので、**尤度 $L$ の最大化と損失 $E$ の最小化は同じ問題**です。ここまでは前章の復習です。

**ステップ1: データ点を1つだけ見つめる。** 和の中身 $\ell_n$、つまりデータ点1個分の損失に注目します。これから、この $\ell_n$ を情報理論の言葉に翻訳します。

**ステップ2: 正解を「分布」として書き直す。** ここがこの導出の急所です。正解ラベル $t_n$ は $0$ か $1$ かの数ですが、これを無理やり、クラス $\{1, 0\}$ の上の確率分布として読み替えます。

$$p^{(n)} = (\,\underbrace{t_n}_{\text{クラス1の確率}},\ \underbrace{1 - t_n}_{\text{クラス0の確率}}\,)$$

これが本当に確率分布であることを確かめてください。各成分は $0$ 以上で、和は $t_n + (1 - t_n) = 1$。そして $t_n$ は $0$ か $1$ しか取らないので、$p^{(n)}$ は $(1, 0)$ か $(0, 1)$ のどちらか——**正解のクラスに全確率を置き、不正解に確率ゼロを置く分布**です。「答えを知っている世界」を表す、迷いのない分布だと言えます。

同じ形で、モデルの予想も分布として並べます。

$$q^{(n)} = (\, y_n,\ 1 - y_n \,)$$

こちらは $(0.7, 0.3)$ のような、迷いを含んだ分布です。

**ステップ3: 2つの分布の cross-entropy を計算する。** 正解の分布 $p^{(n)}$ と予想の分布 $q^{(n)}$ が揃ったので、5.3節の定義どおり cross-entropy を計算します。クラスは $1$ と $0$ の2つなので、和は2項です。

$$H(p^{(n)}, q^{(n)}) = -\sum_{k \in \{1, 0\}} p^{(n)}_k \log q^{(n)}_k = -\bigl[\, t_n \log y_n + (1 - t_n) \log(1 - y_n) \,\bigr] \tag{5.3}$$

右辺をよく見てください。**式 (5.2) の $\ell_n$ と、一字一句同じ**です。

$$\ell_n = H(p^{(n)}, q^{(n)}) \tag{5.4}$$

前章の log loss は、新しい何かではありませんでした。**正解の分布とモデルの分布の cross-entropy を、クラス数2の場合について書き下したもの**——それが log loss の正体です。

**ステップ4: データ全体に戻す。** 式 (5.4) を式 (5.2) の和に戻せば、

$$E = \sum_{n=1}^{N} H(p^{(n)}, q^{(n)}) \tag{5.5}$$

負の対数尤度とは、全データ点にわたる cross-entropy の合計でした。ステップ0で確認した「尤度最大化 = $E$ 最小化」とつなげば、この巻のクライマックスに到達します。

$$\text{尤度の最大化} \;=\; \text{負の対数尤度の最小化} \;=\; \text{cross-entropy の最小化} \tag{5.6}$$

**ステップ5(仕上げ): KL の言葉でも読む。** さらにもう一歩進めます。正解の分布 $p^{(n)}$ は $(1,0)$ か $(0,1)$、つまり1点に集中した分布なので、5.2節の例1よりエントロピーはゼロです: $H(p^{(n)}) = 0$。すると 5.4節の分解 $H(p,q) = H(p) + D_{\mathrm{KL}}(p \,\|\, q)$ から、

$$H(p^{(n)}, q^{(n)}) = D_{\mathrm{KL}}(p^{(n)} \,\|\, q^{(n)}) \tag{5.7}$$

損失はそのまま「ズレ」です。つまり学習とは、**データ点ひとつひとつで、モデルの迷いを含んだ分布 $q^{(n)}$ を、正解に全賭けした分布 $p^{(n)}$ に近づけていく営み**だった——確率の言葉で始めても、驚きの言葉で始めても、同じ山頂に出ました。

導出は以上です。何が起きたかを整理します。

| 視点 | 言葉 | やっていること |
|---|---|---|
| 第3章(確率) | 尤度の最大化 | データが最も出やすいパラメータを選ぶ |
| 第4章(損失) | log loss の最小化 | 負の対数尤度を坂下りで減らす |
| 第5章(情報) | cross-entropy の最小化 | 正解を見たときの平均の驚きを減らす |
| 第5章(情報) | KL divergence の最小化 | モデルの分布を正解の分布に重ねる |

4つの行は、すべて**同一の最適化問題**です。論文が "cross-entropy" や "label smoothing" と書くとき、その裏ではこの同一性が黙って使われています。私たちはもう、その黙っている部分を自分の手で埋められます。

最後に次章への扉を開けておきます。この導出のどこにも「クラスが2つであること」の本質的な制約はありませんでした。正解の分布を $K$ 個の成分で書き、モデルの分布も $K$ 個並べれば、式 (5.3) の和が $K$ 項になるだけで、同じ導出がそのまま通ります。足りないのは、**$K$ 個のスコアを確率分布に変える、シグモイドの多クラス版**だけ。それが次章の主役、論文の式 (1) に堂々と鎮座する softmax です。

### 数値で検算する

手計算した数値とステップ3の「一致」は、コードで検算できます。核心は3つの定義関数と、それらが本文の数値を再現する `assert` 群です。

```python
def entropy(p, base=2.0):
    p = np.asarray(p, dtype=float)
    mask = p > 0
    return -np.sum(p[mask] * np.log(p[mask])) / np.log(base)

def cross_entropy(p, q, base=2.0):
    p, q = np.asarray(p, dtype=float), np.asarray(q, dtype=float)
    mask = p > 0
    return -np.sum(p[mask] * np.log(q[mask])) / np.log(base)

def kl(p, q, base=2.0):
    return cross_entropy(p, q, base) - entropy(p, base)

# 本文の数値例(5.3 / 5.4)
p = np.array([0.5, 0.25, 0.25]); q = np.array([0.25, 0.5, 0.25])
assert np.allclose(cross_entropy(p, q), 1.75)   # 余分な損を含む 1.75 ビット
assert np.allclose(kl(p, q), 0.25)              # 予想のずれ 0.25 ビット

# 5.5 ステップ3: log loss = 2クラスの cross-entropy(式 5.4)
for t in (0, 1):
    for y in (0.1, 0.5, 0.8, 0.99):
        log_loss = -(t * np.log(y) + (1 - t) * np.log(1 - y))
        assert np.allclose(log_loss, cross_entropy([t, 1 - t], [y, 1 - y], base=np.e))
```

全文(エントロピーの両端の値、$H(p,q) \ge H(p)$ をランダムな分布1000組で確認する検算、非対称性の検算を含む)と動作確認は `code/ch05/entropy_check.py`(`python3` で全 assert 通過)を参照してください。最後のループが、ステップ3の一致 $\ell_n = H(p^{(n)}, q^{(n)})$ の数値版です。$t$ と $y$ の組み合わせをいくつ変えても、log loss と2クラス cross-entropy はぴったり一致します。

## まとめ

- 情報量は驚きの量 $I(p) = -\log p$。「稀なほど驚く」「独立な驚きは足せる」の2つの性質を式にすると、対数しか残らない
- エントロピー $H(p) = -\sum_k p_k \log p_k$ は驚きの期待値で、分布の不確かさを測る。1点集中で $0$、一様分布で最大 $\log K$
- cross-entropy $H(p, q) = -\sum_k p_k \log q_k$ は「世界は $p$ で動くのに、驚き方は $q$ で決まる」ときの平均の驚き。常に $H(p,q) \ge H(p)$ で、等号は $q = p$ のみ
- KL divergence $D_{\mathrm{KL}}(p \,\|\, q) = H(p,q) - H(p)$ は分布間のズレ。$p$ が固定なら、cross-entropy 最小化と KL 最小化は同じ問題(label smoothing の形で第8巻に再登場)
- **尤度最大化 = cross-entropy 最小化**。前章の log loss は、正解の分布とモデルの分布の cross-entropy の2クラス特殊例だった(式 5.4)

**ラスボスとの距離**: 論文 Section 5.4 の "label smoothing" が前提とする cross-entropy が読めるようになりました。式 (1) の残りは softmax——次章で仕留めます。

## 演習

**演習1**: 4語の語彙の上の分布 $p = (1/2, 1/4, 1/8, 1/8)$ のエントロピーを底 $2$ で手計算してください。また、同じ4語の一様分布 $(1/4, 1/4, 1/4, 1/4)$ のエントロピーと比べて、どちらが大きいか確認してください。

<details>
<summary>略解</summary>

$$H(p) = \tfrac{1}{2} \log_2 2 + \tfrac{1}{4} \log_2 4 + \tfrac{1}{8} \log_2 8 + \tfrac{1}{8} \log_2 8 = 0.5 + 0.5 + 0.375 + 0.375 = 1.75 \ \text{ビット}$$

一様分布は $H = \log_2 4 = 2$ ビット。偏りのある $p$ の方が小さい($1.75 < 2$)。一様分布が同じ語彙数での最大値、という 5.2 節の事実と整合します。

</details>

**演習2**: 本当の分布 $p = (1/2, 1/4, 1/4)$、モデルの分布 $q = (1/8, 3/4, 1/8)$ について、$H(p, q)$ と $D_{\mathrm{KL}}(p \,\|\, q)$ を底 $2$ で計算してください($\log_2 3 \approx 1.585$ を使ってよい)。本文の例の $q = (1/4, 1/2, 1/4)$ と比べて、ズレは大きくなったでしょうか。

<details>
<summary>略解</summary>

$$H(p, q) = \tfrac{1}{2} \log_2 8 + \tfrac{1}{4} \log_2 \tfrac{4}{3} + \tfrac{1}{4} \log_2 8 = 1.5 + \tfrac{1}{4}(2 - \log_2 3) + 0.75 \approx 2.354 \ \text{ビット}$$

$H(p) = 1.5$ ビット(本文の例3)なので、

$$D_{\mathrm{KL}}(p \,\|\, q) = H(p,q) - H(p) \approx 0.854 \ \text{ビット}$$

本文の例ではズレは $0.25$ ビットでした。今回の $q$ は、最頻の単語(確率 $1/2$)をわずか $1/8$ と見積もる、より大きく外れた予想なので、KL も $0.854$ ビットと大きくなります。物差しとして妥当な挙動です。

</details>

**演習3**: $p = (1/2, 1/2)$、$q = (1/4, 3/4)$ について、$D_{\mathrm{KL}}(p \,\|\, q)$ と $D_{\mathrm{KL}}(q \,\|\, p)$ をそれぞれ底 $2$ で計算し、一致しないことを確かめてください($\log_2 3 \approx 1.585$)。

<details>
<summary>略解</summary>

向き1: $H(p) = 1$、$H(p, q) = \tfrac{1}{2}\log_2 4 + \tfrac{1}{2}\log_2 \tfrac{4}{3} = 1 + \tfrac{1}{2}(2 - \log_2 3) \approx 1.2075$。よって

$$D_{\mathrm{KL}}(p \,\|\, q) \approx 0.2075 \ \text{ビット}$$

向き2: $H(q) = \tfrac{1}{4}\log_2 4 + \tfrac{3}{4}\log_2 \tfrac{4}{3} = 0.5 + \tfrac{3}{4}(2 - \log_2 3) \approx 0.8113$。$H(q, p) = \tfrac{1}{4}\log_2 2 + \tfrac{3}{4}\log_2 2 = 1$。よって

$$D_{\mathrm{KL}}(q \,\|\, p) = 1 - 0.8113 \approx 0.1887 \ \text{ビット}$$

$0.2075 \ne 0.1887$ で、確かに非対称です。だから KL は「距離」ではなく「ズレ」と呼ぶのでした。

</details>

**演習4**: あるデータ点で正解は $t = 0$、モデルの出力は $y = 0.9$ でした(間違ったクラスに自信満々の状態です)。このデータ点の log loss を自然対数で計算し、同じ値を「正解の分布 $p$ とモデルの分布 $q$ の cross-entropy」として書き直してください。

<details>
<summary>略解</summary>

log loss は $\ell = -[\,0 \cdot \log 0.9 + 1 \cdot \log 0.1\,] = -\log 0.1 \approx 2.303$ ナット。

分布で書くと、正解の分布 $p = (t, 1-t) = (0, 1)$、モデルの分布 $q = (y, 1-y) = (0.9, 0.1)$ で、

$$H(p, q) = -[\,0 \cdot \log 0.9 + 1 \cdot \log 0.1\,] = -\log 0.1 \approx 2.303$$

一致します。正解の分布が片側に全確率を置いているため、和のうち生き残るのは「正解クラスにモデルが割り当てた確率の $-\log$」の1項だけ。間違いに自信を持つほど($1 - y = 0.1$ が小さいほど)損失は跳ね上がる——前章で見た log loss の挙動が、cross-entropy の言葉でそのまま読み直せます。

</details>
