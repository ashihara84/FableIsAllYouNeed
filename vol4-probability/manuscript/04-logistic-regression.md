第3巻の最後で、私たちは宿題を3つ残したまま巻を閉じました。

1. シグモイドは、なぜあの形でなければならないのか
2. シグモイドが出す 0〜1 の数は、何と解釈すべきなのか
3. 分類の正しい損失関数とは何か

このとき「ある1本の糸をたぐると、3つが同時にほどける」と予告しました。その糸は、もう手の中にあります。第1章で確率の言葉を、第3章で尤度と負の対数尤度を手に入れたからです。この章で3つの宿題をすべて回収し、最後には第3巻エピローグとまったく同じデータ・同じ初期値——学習が凍りついたあの $\mathbf{w} = (-8, -8)$ ——から、学習が動き出すところを自分のコードで確認します。

記号の約束をひとつ。第3巻では正解ラベルを $y$ と書きましたが、この章からは**モデルの出力を $y$、正解ラベルを $t$(target)**と書き分けます(ニューラルネットの文献の慣例で、以後の巻もこれに従います)。一方、コードの変数名はシリーズ規約どおりラベルが `y` のままで、出力は `p` と書きます。つまり**数式の $(y - t)$ はコードの `(p - y)`** に対応します。混乱しやすいので、コードの該当行にもコメントを入れてあります。

## 4.1 シグモイドの正体 — 出力を「確率」として正当化する

第3巻エピローグでは、シグモイド関数

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

を「実数を 0〜1 に潰す、都合のいい滑らかな関数」という**道具**として使いました。出てきた 0〜1 の数が何なのかは、語る言葉がなく答えられませんでした。

いまは違います。0 以上で、全部足すと 1 になり、「どれくらい〜らしいか」を表す数の体系——第1章の確率です。そこでモデルの出力をこう読み替えます。

$$y = \sigma(\mathbf{x} \cdot \mathbf{w} + b) = P(t = 1 \mid \mathbf{x})$$

出力 $y$ は「このメール $\mathbf{x}$ を見たとき、ラベルが 1(スパム)である条件付き確率」。モデルは答えを言い切る機械ではなく、**自信を口にする機械**だと捉え直すのです。$y = 0.93$ なら「93% スパムだと思う」、$y = 0.5$ なら「五分五分」。これが宿題2の答えです。

この読み替えは言葉遊びではありません。出力が確率なら、第3章の物語——**データからパラメータを当てるには、データの尤度を最大化すればよい**——に分類問題がそのまま乗ります。これが本章の背骨で、次の節で実行します。

その前に宿題1(なぜこの形か)にも短く答えます。線形モデル $z = \mathbf{x} \cdot \mathbf{w} + b$ はどんな実数でも出せるのに、確率 $p$ は 0〜1 にしか住めません。そこで確率を一度「比」に直します。$\frac{p}{1-p}$ を**オッズ(odds)**と呼ぶと(「スパム 9 : 非スパム 1」のような比です)、オッズは 0 から無限大を動き、その対数 $\log \frac{p}{1-p}$ は**実数全体**を動きます。「線形モデルの出力 $z$ はこの対数オッズ(log-odds)なのだ」と約束して $p$ について解くと

$$\log \frac{p}{1-p} = z \;\Longrightarrow\; \frac{p}{1-p} = e^z \;\Longrightarrow\; p = \frac{e^z}{1 + e^z} = \frac{1}{1 + e^{-z}}$$

シグモイドが出てきました。「たまたま 0〜1 に潰す関数」ではなく、**対数オッズから確率への翻訳機**だったのです。ただしこの導出は由来の説明で、主役はあくまで「出力を確率と解釈し、尤度を最大化する」という機能のほうです。この線形 + シグモイドのモデルを**ロジスティック回帰(logistic regression)**と呼びます。「回帰」という名ですが、やることは分類です(歴史的な命名のいたずらで、定番の混乱ポイントです)。

## 4.2 ベルヌーイ尤度 → 負の対数尤度 = log loss

宿題3、分類の正しい損失関数です。第3巻では MSE を「回帰でうまくいったから」と検品せずに持ち込み、痛い目を見ました。今回は逆に、損失を**選ぶ**のではなく確率の原理から**導出**します。

材料は第3章のコイン投げです。表(1)が確率 $p$ で出るコイン——ベルヌーイ分布——のラベル $t \in \{0, 1\}$ の確率は、1本の式に畳めるのでした。

$$P(t) = p^t (1-p)^{1-t}$$

$t=1$ なら $p$、$t=0$ なら $1-p$。指数が場合分けのスイッチとして働く、あの書き方です。

ロジスティック回帰は、データ1件ごとに「表の出る確率」を $y_i = \sigma(\mathbf{x}_i \cdot \mathbf{w} + b)$ と申告するコイン職人だと思えます。手元の $n$ 件のラベル $t_1, \dots, t_n$ がこの申告どおり観測される確率——**尤度**——は、第3章どおり積になります。

$$L(\mathbf{w}, b) = \prod_{i=1}^{n} y_i^{t_i} (1 - y_i)^{1 - t_i}$$

あとは第3章の手順そのままです。対数で積を和に変え、符号を反転して最大化を最小化に読み替え、データ数で割って平均にする。出てくるのが負の対数尤度、この形には **log loss** という名前がついています。

$$L_{\text{log}} = -\frac{1}{n} \sum_{i=1}^{n} \Bigl[ t_i \log y_i + (1 - t_i) \log (1 - y_i) \Bigr]$$

この式は誰かが「良さそうだから」と発明したものではなく、**「出力は確率である」と認めた瞬間に、尤度最大化から自動的に導出された**ものです。MSE のときのような「分類でも正しいか確かめていない」という後ろめたさが、今回はありません。

挙動を1件分だけ読んでおきます。正解が $t = 1$ の項は $-\log y$。$y = 0.9$ なら損失 $0.105$、$y = 0.5$ なら $0.693$。そして第3巻の顕微鏡で見た「自信満々に間違えている」点 $y = 0.000005$ では $-\log(0.000005) \approx 12.2$ です。MSE は $0 < y < 1$ ゆえ $(y-t)^2 < 1$ で、どれだけ壊滅的に間違えても1件あたり 1 で頭打ちでした。log loss は**自信を持って間違えるほど際限なく損失が膨らみます**。いちばん反省すべき点に大きな声が割り当てられている——この性格の違いが、次の節で勾配の違いとして表に出ます。

## 4.3 勾配を手で出す — sigmoid + log loss の勾配は驚くほどきれい (y − t)・x

損失が決まったら勾配です。第3巻エピローグでは数値微分に逃げましたが、今回は第2巻第5章の連鎖律で手導出します。この章でいちばん気持ちのいい計算なので、ぜひ紙の上で追ってください。データ1件分の計算の鎖はこうつながっています。

$$\mathbf{w} \;\longrightarrow\; z = \mathbf{x} \cdot \mathbf{w} + b \;\longrightarrow\; y = \sigma(z) \;\longrightarrow\; L = -\bigl[ t \log y + (1-t) \log(1-y) \bigr]$$

**部品1: シグモイドの微分。** 使う事実は1つ、指数関数は微分しても形が変わらない、$(e^z)' = e^z$ です(第2巻では出番のなかった公式ですが、$e$ という数の存在理由そのものです。疑わしければ演習2で数値微分と突き合わせます)。$u = 1 + e^{-z}$ と置くと $\sigma = u^{-1}$ で、連鎖律から

$$\sigma'(z) = -u^{-2} \cdot (-e^{-z}) = \frac{e^{-z}}{(1 + e^{-z})^2} = \underbrace{\frac{1}{1+e^{-z}}}_{\sigma(z)} \cdot \underbrace{\frac{e^{-z}}{1+e^{-z}}}_{1 - \sigma(z)}$$

つまり $\sigma'(z) = \sigma(z)\bigl(1 - \sigma(z)\bigr) = y(1-y)$。シグモイドの傾きは自分の出力だけで書けます。この $y(1-y)$ という因子の顔を覚えておいてください。**4.5 で犯人として再登場します。**

**部品2: 損失の $y$ による微分。** $(\log y)' = 1/y$(第3章で対数尤度を微分したときの相棒)を使って

$$\frac{\partial L}{\partial y} = -\frac{t}{y} + \frac{1-t}{1-y} = \frac{-t(1-y) + (1-t)y}{y(1-y)} = \frac{y - t}{y(1-y)}$$

分母に $y(1-y)$ が現れました。$y$ が 0 や 1 に近い——モデルが断言している——とき、この微分は巨大になります。log loss の「青天井」の正体です。

**合体。** 連鎖律で2つの部品をつなぎます。

$$\frac{\partial L}{\partial z} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial z} = \frac{y - t}{y(1-y)} \cdot y(1-y) = y - t$$

部品2の分母 $y(1-y)$ と、部品1のシグモイドの傾き $y(1-y)$ が、**ぴったり相殺して消えました**。あれだけ込み入っていた鎖の答えが、「出力ひく正解」。あとは $\partial z/\partial \mathbf{w} = \mathbf{x}$、$\partial z/\partial b = 1$ を掛け、$n$ 件で平均するだけです。

$$\frac{\partial L_{\text{log}}}{\partial \mathbf{w}} = \frac{1}{n} \sum_{i=1}^{n} (y_i - t_i)\, \mathbf{x}_i, \qquad \frac{\partial L_{\text{log}}}{\partial b} = \frac{1}{n} \sum_{i=1}^{n} (y_i - t_i)$$

この形、見覚えがあるはずです。第3巻第4章で導出した線形回帰 + MSE の勾配は $\frac{2}{n}\sum r_i \mathbf{x}_i$、「残差×入力の平均」でした。sigmoid + log loss の勾配は、定数 2 すらない $(y-t)\,\mathbf{x}$ ——**同じ骨格**です。シグモイドという非線形を1枚はさんだのに、勾配はむしろ簡素になりました。大事なのはきれいさより意味です。$y - t$ は間違いの大きさそのもの。**大きく外すほど強く引き戻される**という、回帰で学習を支えたあの性質が、分類の世界に帰ってきたのです。

NumPy では shape の規律(第1巻)どおり、$X$ `(n, 2)`、出力 `p` `(n,)`、ラベル `y` `(n,)` に対して `grad_w = X.T @ (p - y) / n` で `(2,)` が出ます。

## 4.4 [コード] ロジスティック回帰をフルスクラッチ実装 — 第3巻エピローグと同じデータで学習が進む

理論は揃いました。宿題の回収に行きます。

実験の流儀を先に宣言します。**第3巻エピローグの実験条件を、1か所だけ残してすべて凍結します。**データ生成は乱数の種(42)も生成順も同一、モデル(線形 + シグモイド)も同一、学習率 0.5・2000 ステップ・問題の初期値 $\mathbf{w} = (-8, -8),\ b = 0$ も同一。**変えるのは損失関数だけ**、MSE → log loss です。これで何かが変われば、その手柄は損失関数だけのものです。

コード全文と動作確認は `code/ch04/logistic_regression.py`(`python3` で全 assert 通過)。核心は損失と勾配です。

```python
def log_loss(X, y, w, b):
    p = np.clip(predict(X, w, b), 1e-12, 1.0 - 1e-12)  # p がちょうど 0/1 に丸まると log が無限大に飛ぶため
    return -np.mean(y * np.log(p) + (1.0 - y) * np.log(1.0 - p))


def grad_log_loss(X, y, w, b):
    """4.3 で手導出した解析形: (出力 − 正解) を入力で重みづけて平均。数式の (y − t) はここの (p − y)"""
    p = predict(X, w, b)
    grad_w = X.T @ (p - y) / len(y)   # (2,)
    grad_b = np.mean(p - y)
    return grad_w, grad_b
```

`log_loss` の `np.clip` は、`p` が浮動小数点でちょうど 0 や 1 に丸まると `log` が無限大に飛ぶため、ごくわずか内側に避難させる安全弁です(理論の式には変更なし)。データ・モデル(`sigmoid`, `predict`, `accuracy`)・MSE 版・第3巻 E.3 の `numerical_grad`(中心差分)はファイルに収めてあり、まず**手導出の `grad_log_loss` が数値微分と一致することを assert で検算**しています(第2巻第1章以来の習慣)。訓練ループ `train_logloss` も第3巻と同じ4拍子・同じ lr・同じ steps で、勾配だけ解析形(4.3)を使います。MSE 版 `train_mse` は第3巻 E.3 の `train`(勾配は数値微分のまま)の忠実な再現です。

## 4.5 なぜ MSE では勾配が死んだのか — 勾配式の比較で種明かし

宿題は回収しました。しかし第3巻の謎には「なぜ」が残っています。第3巻では数値微分を使ったので勾配は実測値としてしか見えず、**式の形は隠れたまま**でした。いまなら 4.3 の道具で、その式を初めて書き下せます。

MSE をシグモイドに通した損失 $L = \frac{1}{n}\sum (y_i - t_i)^2$ を、4.3 とまったく同じ鎖で微分します。変わるのは部品2だけで、$\partial L/\partial y = 2(y-t)$。部品1のシグモイドの傾き $y(1-y)$ を掛けると、1件分の勾配の芯は

$$\frac{\partial L_{\text{MSE}}}{\partial z} = 2\,(y - t)\cdot y(1-y)$$

2つの損失の勾配を並べます。これがこの章の種明かしです。

| 損失 | $\partial L / \partial z$(1件分) |
|---|---|
| log loss | $y - t$ |
| MSE | $2\,(y - t)\cdot y(1-y)$ |

違いはただ1つ、MSE には $y(1-y)$ という因子が**残っている**ことです。この因子は $y = 0.5$ で最大値 $0.25$、$y$ が 0 や 1 に近づくと 0 に潰れます。つまりモデルが断言を始めた瞬間、その断言が正しかろうが間違っていようが、勾配は一律に握り潰される。「自信を持って間違えている」点では $|y - t| \approx 1$ と間違いは最大なのに、$y(1-y) \approx 0$ がそれを揉み消すのです。

数字で確かめます。実験4の点では $y = 0.000005$ なので $y(1-y) \approx 5.2 \times 10^{-6}$。勾配の芯は $2 \times (-1) \times 5.2 \times 10^{-6} \approx -1.0 \times 10^{-5}$——実験4(そして第3巻 実験3)で実測された `grad_b = -1.0e-05` と一致します。$\mathbf{x} \approx (2.05, 2.00)$ を掛ければ `grad_w = [-2.2e-05, -2.1e-05]` も再現。第3巻で「故障でしょうか」と眺めた不可解な実測値は、すべてこの式が出していた値でした。スクリプトの末尾では、この解析形が数値微分と一致することも assert しています。

```python
# === 4.5 の種明かし: MSE + sigmoid の勾配の解析形(第3巻では数値微分の裏に隠れていた式) ===
def grad_mse_sigmoid(X, y, w, b):
    """∂L/∂z = 2 (y − t) · y(1 − y) を入力で重みづけて平均。y(1 − y) が犯人"""
    p = predict(X, w, b)
    dz = 2.0 * (p - y) * p * (1.0 - p)   # (n,)
    grad_w = X.T @ dz / len(y)
    grad_b = np.mean(dz)
    return grad_w, grad_b
```

(全文と、穏当な点・飽和した点の両方で数値微分との一致を確認する assert、顕微鏡の1点での $y(1-y) \approx 5.2 \times 10^{-6}$ の表示は `code/ch04/logistic_regression.py`。)

では、なぜ log loss はこの因子に殺されないのでしょうか。4.3 の導出を思い出してください。log loss の $\partial L/\partial y$ は分母に $y(1-y)$ を持っていました。シグモイドが勾配を $y(1-y)$ 倍に握り潰すちょうどそのとき、log loss は損失を $\frac{1}{y(1-y)}$ 倍の急勾配にして対抗する。掛け合わせると相殺して $y - t$ だけが残る。これは幸運な偶然ではありません。$\log$ は $e$ の逆関数——log loss はシグモイドの中の $e^{-z}$ をほどくために生まれたような形をしています。**確率(シグモイド)と損失(負の対数尤度)が、同じ尤度の世界から来た部品どうしだから噛み合う**のです。MSE は回帰の世界からの借り物で、この噛み合わせを持っていませんでした。

これで第3巻 E.4 の問いに完全な答えが書けます。二乗誤差が分類で間違っている理由は、(1) 確率の原理から導出されない場当たりな選択であること、(2) シグモイドの傾き $y(1-y)$ を打ち消す仕組みがなく、モデルが「自信を持って間違える」領域で勾配が消滅すること。正しい損失は、尤度から導出された log loss です。

ひとつ予告を。「出力が飽和して勾配が死ぬ」というこの病気は、シグモイド固有のものではありません。第7章で、論文の $\sqrt{d_k}$ がまさに**同じ病気の予防策**だったことがわかります。第3巻エピローグの宿題は、ラスボスの心臓部まで一直線につながっていたのです。

## まとめ

- シグモイドの出力は条件付き確率 $P(t=1 \mid \mathbf{x})$ と解釈できる。形自体も、線形モデルの出力を対数オッズとみなすことから導かれる(宿題1・2の回収)
- 出力を確率と認めると、ベルヌーイ尤度の最大化から分類の損失が**導出**される: 負の対数尤度 = log loss(宿題3の回収)
- sigmoid + log loss の勾配は $\partial L/\partial z = y - t$。途中の $y(1-y)$ が厳密に相殺し、「大きく外すほど強く引き戻される」が分類でも成立する
- 第3巻エピローグと同一のデータ・初期値 $\mathbf{w}=(-8,-8)$・学習率・ステップ数で、損失を log loss に替えるだけで学習が進む(8ステップで正解率 0.5% → 99.5%)
- MSE の勾配には因子 $y(1-y)$ が残り、自信を持って間違えるほど勾配が潰れる。第3巻の「死んだ勾配」の実測値はこの式で完全に再現できる

**ラスボスとの距離**: 論文が当然のように前提する「分類モデルの損失」を、選ぶのではなく導出できるようになりました。第5章でこの log loss に本当の名前が付き、Section 5.4 の "label smoothing" が立っている地面まであと一歩です。

**問3(grad_b はなぜほぼ 0 だったか)** 実験2の開始時、`grad_w = [-2.04, -2.02]` と大きいのに `grad_b = -0.00` でした。4.3 の式 $\partial L/\partial b = \frac{1}{n}\sum (y_i - t_i)$ を使って理由を説明してください。

<details><summary>略解</summary>

$\mathbf{w} = (-8,-8)$ のモデルは判定が全件裏返っているので、通常メール(正解 0)にはほぼ 1 を出して $y_i - t_i \approx +1$、スパム(正解 1)にはほぼ 0 を出して $y_i - t_i \approx -1$。両クラスは 100 件ずつなので和はほぼ打ち消されます。「全体を一律にかさ上げする」係の $b$ には、上げる理由と下げる理由が同数あって動けないのです。一方 $\partial L/\partial \mathbf{w} = \frac{1}{n}\sum (y_i - t_i)\,\mathbf{x}_i$ は誤差に $\mathbf{x}_i$ が掛かるため、$+1 \times (-2,-2)$ と $-1 \times (+2,+2)$ が**同じ向き**(負)に揃い、大きな勾配になります。直線の高さではなく傾きを直せ、と勾配が正しく指差しているわけです。
