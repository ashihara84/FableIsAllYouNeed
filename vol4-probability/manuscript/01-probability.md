# 第1章 確率の言葉 — 不確かさを数にする

第3巻のエピローグで、私たちは机の上に宿題を3つ置いたまま、巻を閉じました。

1. シグモイドは、なぜあの形でなければならないのか
2. シグモイドが出す 0〜1 の数は、何と解釈すべきなのか
3. 分類の正しい損失関数とは何か

そして「3つはひとつながりで、ある1本の糸をたぐると同時にほどける」とだけ言い残しました。もう名指しを避ける理由はありません。その糸の名前は、**確率**です。

ただし、この章でいきなり宿題が片づくわけではありません。いま私たちが困っているのは、もっと手前です。「0〜1 の数を確率と解釈する」と言おうにも、確率という言葉の読み書きをまだ知らないのです。序章で掲げたラスボスを思い出してください。

> *"...assume that the components of q and k are independent random variables with mean 0 and variance 1."*
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1, 脚注4
>
> 訳: q と k の成分が、平均0・分散1の独立な確率変数であると仮定する。

random variables——確率変数。論文は、この言葉を定義せずに使います。読者全員が読めるはずの、基礎語彙だからです。この章の仕事は、その基礎語彙の習得です。確率とは何か、分布とは何か、縦棒の入った $P(\cdot \mid \cdot)$ という記号はどう読むのか。地味に見えますが、この巻の残り全部と、第6巻以降の言語モデルの話は、すべてこの章の言葉で書かれます。

## 1.1 動機: 分類モデルの出力は「答え」ではなく「自信」であるべき

第3巻エピローグのスパムフィルタに、もう一度だけ戻ります。シグモイドを通したモデルは、メール1通ごとに 0〜1 の数を1つ出力するのでした。あのとき私たちは「0.5 以上ならスパムと判定する」という約束だけ決めて、数そのものの意味は宙づりにしました。

宙づりのまま、2通のメールを見比べてみましょう。1通めへの出力は 0.93、2通めへの出力は 0.51。約束に従えば、判定はどちらも同じ「スパム」です。しかし、この2つの 0〜1 の数が同じことを言っていないのは明らかです。0.93 は「ほぼ間違いなくスパム」、0.51 は「どちらとも言いがたいが、強いて言えばスパム寄り」。判定(ラベル)に潰してしまうと消えてしまうこの差にこそ、価値があります。実際、あのデータには2つの塊の境目に落ちた「運の悪い1通」がいました。入力の特徴量だけからでは正解が決まりきらないメールは、現実にはいくらでもあります。そういう入力に対して断定を返すモデルより、「わからない」を「わからない」と正直に申告するモデルの方が、信用できます。

似た数を、あなたは毎日見ています。降水確率です。「降水確率 70%」は「雨が降る」という答えではありません。「降る方に 70 の自信、降らない方に 30 の自信」という、**自信の配分の申告**です。優れた予報とは、当たり外れの言い切りがうまい予報ではなく、この申告が誠実な予報のことです。分類モデルの出力も、これと同じものであるべきです。出力は「答え」ではなく「自信」。

そう宣言した瞬間に、新しい問いが生まれます。「自信」を数で表すとは、どういう作法に従うことなのでしょうか。0.93 と 0.51 を足したり比べたりしてよい根拠はどこにあるのでしょうか。この作法の体系が**確率(probability)**です。第3巻では、あの 0〜1 の数を「都合よく潰された出力」としか呼べませんでした。この章で確率の言葉を獲得し、第4章で「シグモイドの出力は確率の名に値する」ことを示します。宿題の回収はそこからです。

## 1.2 確率の基本ルール: 全部足すと1、0以上

確率の教科書は、ふつう集合論と公理系から始まります。私たちは始めません。cross-entropy への最短経路に必要なルールは、たった2つだからです。

まず素材を決めます。**起こりうる結果のリスト**を用意します。コイン投げなら {表, 裏}、サイコロなら {1, 2, 3, 4, 5, 6}。リストの条件は、どれか1つが必ず起き、2つが同時には起きないことです。そのリストの各結果に、自信の度合いを数で割り振ります。この数が確率で、守るべきルールは次の2つだけです。

1. **どの確率も 0 以上**
2. **全部足すと 1**

ルール1の端、確率 0 は「絶対に起きない」という申告です。反対の端、確率 1 は「必ず起きる」。その間の数が、自信の濃淡を表します。

ルール2は、一見すると恣意的な約束に見えます。なぜ合計は 1 でなければならないのでしょうか。これは「リストのどれかは必ず起きる」という前提の数値版です。手持ちの自信の総量を 1 と決めて、それを結果たちに**配り分ける**、と考えてください。配り方が1点に偏っていれば確信、均等にばらけていれば「まったく見当がつかない」。歪みのないコインなら表と裏に 0.5 ずつ、歪みのないサイコロなら各目に $1/6$ ずつ。$1/6$ を6個足せば 1 に戻ります。

最後に、確率の「作り方」を決めておきます。哲学的にはいくつも流儀がある話題ですが、この本では一貫してひとつの作り方しか使いません。**数えて、割る。**データの中である結果が起きた回数を、全体の回数で割った割合を、その結果の確率とします。回数は 0 以上なので、ルール1は自動的に満たされます。各結果の回数を全部足せば全体の回数に戻るので、ルール2も自動的に満たされます。「数えて割る」は、2つのルールを破りようがない製法なのです。次の節から、この製法を実際に動かします。

## 1.3 確率変数と離散分布 — 例は一貫して「次の単語当て」

確率の教科書の例題といえばサイコロとコインですが、この本の主役は別にいます。こんなゲームを考えてください。

> the cat sat on the ____

空欄に入る単語は何でしょうか。

確かなことは言えません。しかし、あなたは無力でもありません。"mat" なら本命という顔をするでしょうし、"fish" なら「ありえる」、"of" なら「それはない」と即座に却下するはずです。つまりあなたの頭の中には、**候補の単語ごとに自信の濃淡を割り振った表**がすでにあります。前節のルールで言えば、起こりうる結果のリストは「単語の一覧」、そこに配り分けられた自信が確率。この表を作って当てるゲームが「次の単語当て」です。

なぜサイコロではなく単語なのか。このシリーズの行き先が、まさにこのゲームだからです。文章の続きの単語を当てる機械を**言語モデル(language model)**と呼びます。本格的な登場は第6巻ですが、この巻の確率の例題は最初から一貫して「次の単語当て」で進めます。サイコロとコインには、形がきれいな脇役として時々手伝ってもらうだけにします。

道具立てを2つ、ここで形式化します。1つめは**確率変数(random variable)**です。「空欄に入る単語」を $W$ と書きます。$W$ は変数ですが、プログラムの変数とは様子が違うので注意してください。代入済みの値はどこにもありません。決まっているのは「取りうる値のリスト」と「各値への確率の配分」だけで、値そのものは観測するたびに(文の続きを実際に見るたびに、サイコロを振るたびに)決まります。**値の代わりに確率の表を持っている変数**——それが確率変数です。論文の脚注4にあった "random variables" は、この意味です。

2つめは**確率分布(probability distribution)**です。確率変数が持っている「値と確率の対応表」のことを分布と呼びます。$W$ のように取りうる値が数えあげられる(単語は1個、2個と数えられます)場合の分布を、特に**離散分布(discrete distribution)**と呼びます。記法はこうです。

$$P(W = \text{mat}) = 0.25$$

読み方は「確率変数 $W$ の値が mat になる確率は 0.25」。$P$ は probability の頭文字です。

実物を1つ、作ってみましょう。前節の製法どおり「数えて割る」ためには、数える対象の文章の集まりが要ります。文章の集まりを**コーパス(corpus)**と呼びます。本物のコーパスは何十億語という規模ですが、原理は4文でも同じなので、手で数えきれる大きさで行きます。

```
the cat sat on the mat
the cat ate the fish
the dog sat on the mat
the dog chased the cat
```

ゲームを決めます。この22語(数えてみてください。6 + 5 + 6 + 5 です)から1語を無作為に指す。指された単語が確率変数 $W$ です。各単語の出現回数を数えて22で割れば、$W$ の分布が出来上がります。

| 単語 | 回数 | 確率 |
|---|---|---|
| the | 8 | $8/22 \approx 0.364$ |
| cat | 3 | $3/22 \approx 0.136$ |
| sat | 2 | $2/22 \approx 0.091$ |
| on | 2 | $2/22 \approx 0.091$ |
| mat | 2 | $2/22 \approx 0.091$ |
| dog | 2 | $2/22 \approx 0.091$ |
| ate | 1 | $1/22 \approx 0.045$ |
| fish | 1 | $1/22 \approx 0.045$ |
| chased | 1 | $1/22 \approx 0.045$ |
| **合計** | **22** | **1** |

ルールを点検しましょう。どの確率も 0 以上(ルール1)、合計は $22/22 = 1$(ルール2)。そして、サイコロとの違いに注目してください。サイコロの分布は各目 $1/6$ の真っ平らな表でしたが、この分布は the に大きく偏っています。分布の形が、コーパスの癖をそのまま写し取っているのです。この「1単語の出現確率の表」には**ユニグラム(unigram)**という名前がついています。

ただし、この表には大きな限界があります。冒頭のゲームを思い出してください。"the cat sat on the ____" の本命は mat だ、とあなたは即答しました。ところがユニグラムの表で mat は $2/22$、cat の $3/22$ より下です。表のとおりに賭けるなら本命は the になってしまいますが、"on the the" はどう考えても変です。あなたの直感は、この表にない情報——**直前の単語が何か**——を使っています。文脈を知ったとき、賭けはどう変わるべきか。それを書くための道具が、次の節です。

## 1.4 条件付き確率と同時確率

文脈を扱うには、単語を1つずつではなく、**隣り合う2語のペア**で数えます。さきほどのコーパスから、文ごとにペアを抜き出します(文と文は別の話なので、文をまたぐペアは作りません)。1文めなら (the, cat), (cat, sat), (sat, on), (on, the), (the, mat) の5ペア。4文ぶん合わせて $5 + 4 + 5 + 4 = 18$ ペアです。

ゲームを更新します。18ペアから1つを無作為に選ぶ。確率変数は今度は2つで、ペアの前半を「前の単語 $C$」(context のつもりです)、後半を「次の単語 $W$」と呼びます。

さて、知りたかったのは「直前が the だとわかっているとき、次は何か」でした。作り方は、前節の製法に一手間足すだけです。**絞って、数えて、割る。**まず18ペアのうち、前半が the のペアだけに絞ります。数えると8ペアあります。その8ペアの中で後半を数えると、cat が3回、dog が2回、mat が2回、fish が1回。8で割れば表の完成です。

| 次の単語 $W$ | 回数(前が the の8ペア中) | 確率 |
|---|---|---|
| cat | 3 | $3/8 = 0.375$ |
| dog | 2 | $2/8 = 0.25$ |
| mat | 2 | $2/8 = 0.25$ |
| fish | 1 | $1/8 = 0.125$ |
| **合計** | **8** | **1** |

このように「〜だとわかっているとき」という条件の下で作り直した確率を、**条件付き確率(conditional probability)**と呼びます。記法には縦棒を使います。

$$P(W = \text{cat} \mid C = \text{the}) = \frac{3}{8}$$

縦棒の読み方を、ここでしっかり決めておきます。**縦棒の右は、わかっていること(手がかり)。左は、当てたいこと。**英語では縦棒を given と読みます——「$C$ が the だと given(与えられた)ときの、$W$ が cat の確率」。右と左は対等ではないので、入れ替えると別物になります。$P(W \mid C)$ と $P(C \mid W)$ の取り違えは確率の事故の定番ですが、この本で必要になるのは読み書きまでなので、「縦棒の右は手がかり」とだけ体に入れてください。

表をもう一度見てください。合計は 1。つまり、絞って作り直した表も、それ自体が2つのルールを満たす立派な確率分布です。ここが今日いちばん大事な風景です。**文脈ごとに、分布が1つずつある。**「the の次」の分布、「cat の次」の分布、「on の次」の分布(これは演習で数えます)——文脈が変われば、賭けの表ごと変わる。ユニグラムの1枚きりの表では書けなかった「文脈を見て賭けを変える」が、条件付き確率なら書けるのです。

もう1つだけ道具を足します。条件を付けるのではなく、「前が the で、**かつ**、次が cat」というペアそのものの確率も考えられます。これを**同時確率(joint probability)**と呼び、カンマで書きます。

$$P(C = \text{the},\ W = \text{cat}) = \frac{3}{18}$$

18ペア中、(the, cat) は3つなので $3/18$ です。カンマは「かつ」と読みます。

条件付き確率と同時確率は、無関係な2つの道具ではありません。並べてみます。「前が the」は18ペア中8つなので $P(C = \text{the}) = 8/18$。すると

$$\frac{8}{18} \times \frac{3}{8} = \frac{3}{18}$$

つまり

$$P(C = \text{the},\ W = \text{cat}) = P(C = \text{the}) \times P(W = \text{cat} \mid C = \text{the})$$

が成り立っています。偶然ではありません。左辺は「18個から (the, cat) を直接探す」、右辺は「まず18個を『前が the』の8個に絞り、その中で cat を探す」——同じ場所への、一段絞り込みを挟んだ二段の道のりです。一般に

$$P(A,\ B) = P(A) \times P(B \mid A)$$

が成り立ち、これを**掛け算の規則(multiplication rule)**と呼びます。この巻で条件付き確率について使う一般法則は、実質これ1つです(教科書ならここからベイズの定理へ進むところですが、私たちの経路には不要なので進みません。扱わない宣言は序章 0.3 のとおりです)。

掛け算の規則には、覚えておくべき特別な場合が1つあります。$A$ を知っても $B$ の見通しがまったく変わらない——つまり $P(B \mid A) = P(B)$ が成り立つ——とき、$A$ と $B$ は**独立(independent)**であるといいます。このとき掛け算の規則は

$$P(A,\ B) = P(A) \times P(B)$$

と、条件なしの積に単純化されます。コインを2回投げるとき、1回目の表裏は2回目の見通しを変えませんから、2回とも表の確率は $\frac{1}{2} \times \frac{1}{2}$。この「独立なら、同時確率はただの積」は、第3章(コイン投げ10回の確率)と第7章(ラスボス脚注4の "independent random variables")で主役級の働きをします。逆に、いま数えてきた「次の単語当て」は独立の**反例**であることにも注意してください。直前の単語を知ると次の単語の見通しは大きく変わる——変わるからこそ、言語モデルという商売が成り立つのです。

ひとつ、正直に断っておくべき細部があります。いまの計算で $P(C = \text{the}) = 8/18$ と書きましたが、前節のユニグラムでは $P(W = \text{the}) = 8/22$ でした。同じ the なのに分母が違うのは、**ゲームが違う**からです。前節は22語から1語を指すゲーム、この節は18ペアから1ペアを選ぶゲームで、文末の単語(mat, fish, cat)は「ペアの前半」には一度もなれません。確率は常に「どういう試行の話か」とセットで決まります。分母が何かわからなくなったら、ゲームの設定まで戻ってください。

仕上げに、この章の記法でいちばん書きたかった式を書きます。文脈は直前の1語に限る必要はありません。手がかりが「これまでの単語ぜんぶ」でも、縦棒の右に並べるだけで記法は同じです。

$$P(W = \text{mat} \mid \text{the cat sat on the})$$

「the cat sat on the まで読んだとき、次が mat である確率」。冒頭のゲームであなたの頭の中にあった自信の濃淡は、まさにこの条件付き分布でした。そして、任意の文脈を受け取ってこの分布を返す機械が言語モデルです——この一行が読み書きできることが、第6巻への入場券になります。ついでに言えば、掛け算の規則を繰り返し使うと文全体の確率も「条件付き確率の積」に分解できるのですが、それは言語モデルを本格的に扱う第6巻でのお楽しみです。

第3巻の宿題にも、半歩だけ近づきました。スパムフィルタのあの 0〜1 の数に、私たちが読ませたかった意味は、いまや記号で書けます。

$$P(\text{スパム} \mid \text{メールの特徴量})$$

「このメールの特徴量を見たという条件の下での、スパムである確率」。ただし、シグモイドの吐く数がこの名に**値する**かどうかは、まだ示していません。それが第4章の仕事です。

## まとめ

- 分類モデルの出力は「答え」ではなく「自信」であるべき。自信を数にする作法が確率で、第3巻エピローグの 0〜1 の数に与えたい名前も確率(正当化は第4章)
- 確率のルールは2つだけ: **どれも 0 以上、全部足すと 1**。この本での確率の製法は一貫して「数えて、割る」で、この製法は2つのルールを自動的に満たす
- 確率変数は「値の代わりに確率の表を持つ変数」、その表が確率分布。例題はサイコロではなく「次の単語当て」で、コーパスを数えて実物の分布を作った
- 条件付き確率 $P(W \mid C)$ は「絞って、数えて、割る」。文脈ごとに分布が1つずつあり、どれも合計1の立派な分布。同時確率とは掛け算の規則 $P(A, B) = P(A)\,P(B \mid A)$ でつながる
- $P(\text{次の単語} \mid \text{これまでの単語})$ が読み書きできるようになった。これが言語モデル(第6巻)の骨組み

**ラスボスとの距離**: 脚注4の "random variables"(確率変数)が読めるようになりました。残る "mean 0 and variance 1" は、第2章で手に入ります。

<details><summary>略解</summary>

(a) $P(W = \text{chased} \mid C = \text{dog})$。dog が前半になるペアは (dog, sat) と (dog, chased) の2つなので、$2$ ペア中 $1$ つで $1/2$。 (b) 「前の単語が cat だとわかっているときの、次の単語が sat である確率」。cat は3回出てきますが、4文めの cat は文末なのでペアの前半になれません。cat が前半のペアは (cat, sat) と (cat, ate) の2つで、答えは $1/2$。「cat は3回だから分母は3」としないのが落とし穴です。分母は常に「条件に合う試行(ここではペア)の数」——ゲームの設定に戻って数えてください。

</details>

**演習3** 本文のコーパスについて、(a) ユニグラム分布と (b) 「the の次に来る単語」の条件付き分布を数えるコードを書いてください。本文の表と一致すること、そして作ったどの分布も「0 以上・全部足すと 1」を満たすことを `assert` で確認すること。

<details><summary>略解</summary>

```python
# 第4巻 第1章 演習3 略解: 小さなコーパスで単語の出現確率を数える
# 本文 1.3(ユニグラム)と 1.4(「the」の次の条件付き分布)の手計算を、
# コードで数え直して assert で一致確認する。確率の2つのルール(0以上・全部足すと1)が
# 「数えて割る」から自動的に満たされることも確認する。
from collections import Counter

import numpy as np

# --- コーパス: 本文 1.3 と同じ4文 ---
corpus = [
    "the cat sat on the mat",
    "the cat ate the fish",
    "the dog sat on the mat",
    "the dog chased the cat",
]

# --- 1.3 ユニグラム: P(W = w) = (w の出現回数) / (総単語数) ---
words = []
for sentence in corpus:
    words.extend(sentence.split())

total = len(words)
counts = Counter(words)
P = {w: c / total for w, c in counts.items()}

print(f"総単語数: {total}語, 語彙サイズ: {len(counts)}語")
print("単語の出現確率(ユニグラム):")
for w, p in sorted(P.items(), key=lambda kv: -kv[1]):
    print(f"  P(W = {w:6s}) = {counts[w]:2d}/{total} = {p:.3f}")

# 本文 1.3 の表と一致するか
assert total == 22
assert counts["the"] == 8 and np.isclose(P["the"], 8 / 22)
assert counts["cat"] == 3 and np.isclose(P["cat"], 3 / 22)
assert counts["mat"] == 2 and np.isclose(P["mat"], 2 / 22)
# 確率のルール: どの値も0以上、全部足すと1
assert all(p >= 0 for p in P.values())
assert np.isclose(sum(P.values()), 1.0)

# --- 1.4 隣り合う2語のペアを数える(文をまたぐペアは作らない) ---
pairs = []
for sentence in corpus:
    ws = sentence.split()
    for i in range(len(ws) - 1):
        pairs.append((ws[i], ws[i + 1]))

n_pairs = len(pairs)
pair_counts = Counter(pairs)
prev_counts = Counter(prev for prev, nxt in pairs)
assert n_pairs == 18


def next_word_distribution(context):
    """条件付き分布 P(W | C = context) を「絞って、数えて、割る」で作る"""
    followers = Counter(nxt for prev, nxt in pairs if prev == context)
    return {nxt: c / prev_counts[context] for nxt, c in followers.items()}


P_next = next_word_distribution("the")
print('\n「the」の次に来る単語の条件付き分布:')
for w, p in sorted(P_next.items(), key=lambda kv: -kv[1]):
    print(f"  P(W = {w:4s} | C = the) = {p:.3f}")

# 本文 1.4 の表と一致するか
assert np.isclose(P_next["cat"], 3 / 8)
assert np.isclose(P_next["dog"], 1 / 4)
assert np.isclose(P_next["mat"], 1 / 4)
assert np.isclose(P_next["fish"], 1 / 8)
# 条件付き分布も、それ自体が1つの確率分布(0以上・全部足すと1)
assert all(p >= 0 for p in P_next.values())
assert np.isclose(sum(P_next.values()), 1.0)
# どの文脈で絞っても、同じ2つのルールが成り立つ
for context in prev_counts:
    dist = next_word_distribution(context)
    assert all(p >= 0 for p in dist.values())
    assert np.isclose(sum(dist.values()), 1.0)

# --- 1.4 同時確率と掛け算の規則: P(C, W) = P(C) × P(W | C) ---
P_joint = pair_counts[("the", "cat")] / n_pairs  # 「前が the かつ 次が cat」のペアの割合
P_prev_the = prev_counts["the"] / n_pairs        # 「前が the」のペアの割合
print("\n掛け算の規則の検算:")
print(f"  P(C=the, W=cat)          = 3/18 = {P_joint:.4f}")
print(f"  P(C=the) × P(W=cat|C=the) = {P_prev_the:.4f} × {P_next['cat']:.4f}"
      f" = {P_prev_the * P_next['cat']:.4f}")

assert np.isclose(P_joint, 3 / 18)
assert np.isclose(P_joint, P_prev_the * P_next["cat"])

print("\nok: すべての assert を通過しました")
```

このスクリプトは `code/ch01/word_probabilities.py` にも置いてあります。最後の for ループに注目してください。**どの**文脈で絞っても「0 以上・足して 1」が成り立つのは、条件付き分布もまた「数えて割る」で作られているからです。製法がルールを保証する、という 1.2 の話のコードによる確認になっています。

</details>

**演習4** 掛け算の規則 $P(A, B) = P(A) \times P(B \mid A)$ を、本文とは別のペアで検算します。$P(C = \text{dog},\ W = \text{chased})$ を、(a) 18ペアから直接数える方法と、(b) $P(C = \text{dog})$ と $P(W = \text{chased} \mid C = \text{dog})$ の積として計算する方法の両方で求め、一致することを確かめてください。

<details><summary>略解</summary>

(a) 18ペア中 (dog, chased) は1つなので $1/18$。 (b) $P(C = \text{dog}) = 2/18$、演習2より $P(W = \text{chased} \mid C = \text{dog}) = 1/2$。積は $\dfrac{2}{18} \times \dfrac{1}{2} = \dfrac{1}{18}$ で一致します。「直接探す」と「絞ってから探す」が同じ答えになる——掛け算の規則は、この当たり前の数え方の言い換えです。

</details>
