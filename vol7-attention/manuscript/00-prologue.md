# 序章 装備の点検

いつもの儀式から始めますが、今回は様子が違います。

これまでの6巻は、論文の**一部**を掲げてきました。第1巻は式(1)の行列構造、第2巻は学習率のスケジュール、第4巻は softmax と $\sqrt{d_k}$、第6巻は Introduction と Background。毎回どこかを切り取って「今は読めない」と確認し、巻を通して道具を揃え、巻末に読めるようになったことを確かめる——その繰り返しでした。

この巻のラスボスは、**論文そのもの**です。もう切り取りません。Abstract から、アーキテクチャ図(図1)、式(1)・式(2)・式(3)、Table 1 まで。論文の心臓部である Section 3 と Section 4 の全文を、セクション順に1行も飛ばさずに読みます。

論文全体を引用することはできないので、代表として、論文の主張をいちばん凝縮した一文を掲げます。

> *"We propose a new simple network architecture, the Transformer, based solely on attention mechanisms, dispensing with recurrence and convolutions entirely."*
> — Vaswani et al., "Attention Is All You Need", Abstract
>
> 訳: 「我々は Transformer という新しい単純なネットワークアーキテクチャを提案する。それは attention 機構のみに基づき、再帰と畳み込みを完全に排している。」

もう一文、本丸である Section 3 の入り口に立つ文も掲げます。終章で、この文の前にもう一度立つことになります。

> *"The Transformer follows this overall architecture using stacked self-attention and point-wise, fully connected layers for both the encoder and decoder, shown in the left and right halves of Figure 1, respectively."*
> — 同論文, Section 3
>
> 訳: Transformer はこの全体アーキテクチャに従い、encoder と decoder の両方を、積み重ねた self-attention と位置ごとの全結合層で構成する。それぞれ図1の左半分と右半分に示す。

単語はおそらく全部知っているはずです。それでも "stacked self-attention" の stack とは何か、"point-wise" とは何の位置のことか——文として正確に読めるかというと、まだ怪しい。その「怪しさ」の正体こそ、0.1 で点検する残りの空白です。

宣言の仕方も逆になります。これまでは「今は読めない。この巻を終えると読める」と宣言してきました。今回は——

**もう、装備は全部ある。**

この巻に新しい数学はひとつも登場しません。内積も softmax も分散も誤差逆伝播も embedding も、すべて第1〜6巻で手に入れました。第6巻の終章では、attention が解決した3つの痛みを自分の訓練時間と勾配ノルムで体感し、「RNN は足枷では?」というところまで来ています。

つまりこの巻は、新しい武器を「習う」巻ではありません。手持ちの装備を1つずつ取り出して論文の対応する行に当てはめ、**本当に読めることを確認しながら進む**巻です。戦い方が変わります。

——とはいえ「装備は全部ある」はかなり強い主張です。鵜呑みにせず、出発前に点検しましょう。それがこの序章の仕事です。

最初の3つはいずれも Section 3.2——attention のセクション——の中にあり、しかも新しい**数学**はどこにもありません。masking は「softmax の前に $-\infty$ を足す」仕掛けで、softmax 自体は第4巻のもの。Multi-Head は「行列を分割して attention を $h$ 回やる」整形の話で、行列の束の扱いは第1巻6.4でやりました。3つの使い方は「$Q, K, V$ をどこから持ってくるか」という配線の話です。つまりこの3つは、未知の部品ではなく、**既知の部品の組み立て方**です。

総点検の結論。**読めずに残っているのは、Section 3.2 の組み立てと、Section 3.5 だけ。** 「装備は全部ある」は誇張ではありませんでした。あとは論文の前に立って、確認しながら進むだけです。

## 0.2 精読の作法

この巻の第1章から第8章は、すべて同じ型で進みます。

1. **原文**: 論文の該当セクションを原文で掲げる(英語 → 和訳)
2. **逐行読解**: 1文ずつ、どの巻のどの装備で読めるかを確かめながら読む
3. **単体実装**: そのセクションが記述する部品を、その部品**だけ**で実装する
4. **テスト**: 実装が論文の記述と一致することを assert で確かめる
5. **対応表**: 「論文の式番号・主張 ↔ 自分のコードのファイル・行」の対応表を章末に置く

精読の巻に、なぜコードとテストが毎章あるのか。「読めたつもり」という状態があるからです。目で追って和訳ができて頷ける——それでも、いざ実装すると手が止まることがあります。softmax をどの軸に適用するのか。mask は割る前か後か。$W^O$ を掛けるのはどのタイミングか。原文はすべて答えを書いていますが、読み飛ばしても「読めたつもり」にはなれてしまう。

だからこの巻では、**コードを理解の検算機にします**。第1巻の序章で交わした「コードが書けるなら、その概念を理解している」という約束の、これが最終形です。実装が論文と食い違えばテストが落ち、テストが落ちたらまだ読めていない行がある。読解とテストを往復しながら、1章につき論文の1セクションを「読めたつもり」から「読めた」に変えていきます。

各章末の対応表は、終章で1枚に合流します。「論文の式・主張 ↔ 自分のコードのファイル・行」の完全対応表——本シリーズの卒業証書の半分です(残り半分は、第8巻の「組み立てて訓練できた」という事実です)。

## 0.3 実装の約束

実装に入る前に、道具と規格を約束します。第8巻で効いてくる約束なので、ここに明文化します。

**道具は、NumPy と第5巻の自作 autograd です。** 第5巻4章で書いたスカラー版 `micrograd.py` と、第5巻5.3でテンソル対応に拡張した `tensor_autograd.py` を import して使います。PyTorch は使いません。完成品のライブラリで組むと「論文が読めたから書けた」のか「ライブラリが揃っていたから動いた」のか区別がつかなくなるからです。自分で backward まで書いた道具だけで組み上がるなら、それは正真正銘、読めた証拠です(PyTorch は第8巻2章で解禁し、そのとき自作版との対応表で橋を渡します)。

**各部品は、独立にテスト可能に作ります。** attention、multi-head、FFN、embedding、positional encoding——それぞれを `code/chNN/` に1ファイルずつ置き、単体で実行して assert が通る状態を保ちます。部品どうしを序章の段階で繋ぐことはしません。理由は次巻にあります。第8巻1章は、この巻の部品を**そのまま** import してフル Transformer を組み立てます。部品の単体テストが揃っていれば、組み立てて動かないとき、疑う場所は「繋ぎ目」だけに絞れます。単体テストは、未来の自分への装備です。

**入出力の shape を、厳密に決めます。** 部品を独立に作って後から繋ぐ計画は、インターフェースが揃っていて初めて成立します。この巻の全部品は、次の規格に従います。

- 系列を流れるデータの基本形は `(seq, d_model)`。バッチ処理では先頭に1軸足した `(batch, seq, d_model)` になりますが、**全部品は軸が1本増えてもそのまま動く**ように作り、各章のテストで確認します
- すべての sub-layer(attention も FFN も)は、受け取った shape と**同じ shape を返す**。入出力が同じ形だからこそ $x + \mathrm{Sublayer}(x)$ という residual が足せる——この規格が論文自身の設計であることを、第2章で原文から読み取ります
- multi-head の内部だけ、一時的に `(h, seq, d_k)`(バッチ付きなら `(batch, h, seq, d_k)`)の「束」に整形する(第4章)。外から見れば入口も出口も基本形のまま
- 実装の節では、関数の初出時に必ず引数と返り値の shape を併記する。第1巻3章から続けてきた shape の規律を、この巻では**部品間の契約**に格上げします

最後にひとつ、これまで通りの約束を。乱数は `default_rng(42)` で固定し、本文中のコードは全文掲載・全文実行可能、検算は `assert` で完結させます。
