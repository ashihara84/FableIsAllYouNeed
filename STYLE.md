## 文体

- **です・ます調が基調**。講義の語り口。一人称は「私たち」、読者への呼びかけは「あなた」(多用しない)
- ただし**体言止め・常体終止を、リズムのために公認する**: 列挙や畳みかけ(「例外を足せば、業者が表記を変える。ルールを足す。また回避される。」)、要点の言い切り(「引き算は $(-1)$ 倍の足し算にすぎない。それだけ。」)では、です・ますに戻すために文を膨らませない。目安は1段落に1回程度——地の文の背骨はあくまで です・ます で、体言止めはアクセントとして使う
- 1つの段落で1つのこと。長い段落より短い段落を重ねる
- 問いを先に立ててから答える(「なぜ〜でしょうか」→ 説明)。需要駆動の原則を文章レベルでも守る: **新しい道具は、困ってから出す**
- 比喩は1概念につき1つまで。比喩を重ねない
- 用語の初出は英語を併記する: 「内積(dot product)」。2回目以降は日本語のみ
- 「簡単です」「自明です」は禁止。読者が詰まる場所を著者が先回りして言語化する

## 簡潔さの規律(冗長を出さない)

> ターゲット読者(入社1〜2年目プログラマ)に「まわりくどい/同じ説明が何度も出る」と評価された反省から、全巻で次を守る。意図的反復(巻頭巻末のラスボス再読など、下記で明示するもの)は残し、無自覚な冗長を出さない。

- **現在地確認は短く**: 章冒頭の「前章まででできること/今の困りごと」は2〜3文(または前提ボックス1つ)まで。前巻全体の振り返りはしない。「直前に何ができて、今何に困っているか」だけを書く
- **結論ファースト**: 各節は要点を先に出してから展開する。「問いを立ててから答える」型は使ってよいが、1つの節で「〜でしょうか」の問い→答えのピンポンは1回程度に抑える
- **言い換え反復をしない**: 同じ主張を角度を変えて何度も言い直さない。念押しは1回。「最短測地線」「変わらない」等のキーフレーズの再唱もしない
- **メタ語りは予算制**: 本の方法論(最短経路/需要駆動/網羅しない宣言)の説明は各巻の序章で一度。本編の各章で繰り返さない
- **儀式的前口上・過剰な呼びかけを書かない**: 「思い出してください」「安心してください」「では、〜してみましょう」式のクッションは最小限に

5. **演習**: 2〜4問。各問に「略解」を付す(別ファイルにしない、章末に畳んで置く)

分量目安(数式・表・コードスニペット・演習・略解をすべて含む `wc -m` ベース。2026-06 の全巻圧縮後の実測分布に基づく):

## 数式・記法

- 数式は LaTeX(`$...$` / `$$...$$`)
- 行列は大文字 $X, W$、ベクトルは小文字太字 $\mathbf{x}$、スカラーは小文字 $a$
- **shape 注釈を必ず書く**(第1巻3.2で導入するシリーズ共通の規律): 行列・ベクトルの初出時に `(行数, 列数)` を併記。例: $X$ `(n, d)`
- 論文の記号(Q, K, V, d_k, d_model, W^Q …)は論文の表記に厳密に合わせる
- 証明は原則しない。直観・小さな数値例・コードでの確認で代える(例外は TOC に明記された導出のみ: 4巻5.5など)

## ラスボス引用の書式

論文の原文引用は引用ブロックで、英語原文 → 和訳の順:

> *"Attention(Q, K, V) = softmax(QK^T / √d_k) V"*
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1
>
> 訳: (和訳をここに)

巻頭(序章)と巻末(終章)で**同一の引用**を使うこと。終章では「巻頭では読めなかったが、いま読める」ことを明示的に確認する。

論文引用ブロックの**全文再掲は巻頭・巻末のみ**。本編の各章で論文に触れるときは引用ブロックを貼り直さず、「§5 の training(第N章)」のように1行で参照する。「ラスボス」という語の使用も巻あたり数回までを目安にする。

## コード規約

- Python 3.9 互換(型ヒントは `list[int]` 等の新記法を避ける)。NumPy(`import numpy as np`)。第8巻2章以降のみ PyTorch 解禁
- 乱数は必ず `rng = np.random.default_rng(42)` で固定
- 変数名はシリーズ共通: データ `X`, `y`、パラメータ `W`, `b`、学習率 `lr`、損失 `loss`、勾配 `grad_W` 形式
- 動作確認は `assert`(数値比較は `np.allclose`)。「実行すると一致が確認できる」形で完結させる
- **本文には、説明に必要な最小限のスニペットだけを載せる**。概念を教えるのに要る数行を示し、逐行は本文で語る。同じ処理を何度も全文で再掲しない
- **全文実行可能な完成プログラムは `code/chNN/` に .py として置き、本文からはパスで参照する**(例:「全文と動作確認は `code/ch01/numpy_intro.py`。`python3` で全 assert を通過する」)。本文に40行級の完成プログラムを丸ごと貼らない。検証は .py 側で担保する
- すべての .py は `python3 ファイル名.py` で assert が通ること(本文がコードを全文掲載しなくなったぶん、.py の網羅性と実行可能性が品質保証の本体になる)
- 本文スニペットと .py は乖離させない。スニペットは .py の一部の正確な抜粋にする
- matplotlib は**コード掲載のみ**(執筆環境に未導入のため実行検証外)。図は `図N.M:` のキャプション付きで、何が見えるはずかを本文で必ず言語化する
