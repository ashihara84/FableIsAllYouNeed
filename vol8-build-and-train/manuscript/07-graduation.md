## 7.1 通読 — Abstract から Conclusion まで、止まらずに

論文を印刷するか、画面に開いてください。15ページ。これから、Abstract から Section 7 Conclusion まで、頭から順に通り抜けます。各巻の終章でやってきた「再戦」は、いつも論文の一部分との再戦でした。今日はその総決算——相手は全文です。

確認することは、第7巻の終章と同じく、1つだけです。**どこで立ち止まるか。** 立ち止まらなければ、卒業です。

**Abstract。** "dispensing with recurrence and convolutions entirely"——再帰と畳み込みを完全に捨てる、という宣言。この一文の過激さは、捨てられた側を知らないと読めません。第6巻で RNN を自分で実装し、並列化できない痛み・長距離依存に弱い痛みを実測した私たちは、これが強がりではなく設計判断だと知っています。第7巻第1章で逐行読解済み。"superior in quality while being more parallelizable and requiring significantly less time to train" の3点セットの根拠は、いまや全部この下のセクションに紐づけて言えます。

**1 Introduction。** "This inherently sequential nature precludes parallelization within training examples"——逐次性が並列化を妨げる。第6巻第5章で、RNN の hidden state が前の時刻の計算を待つしかないことをコードの構造として見ました。読めるどころか、「うちの RNN もそうでした」と相槌が打てる段落です(第7巻第1章)。

**2 Background。** 畳み込みで逐次性を消そうとした先行研究では、離れた位置同士を結ぶのに距離に応じた段数が要る——という整理。第6巻第7章で attention が「どの位置へも1ホップ」であることを見た私たちには、この節は Transformer の家系図として読めます(第7巻第1章)。

**3 Model Architecture。** 私たちが第7巻を丸ごと使って精読し、全部品を単体実装した本丸です。3.1 の stack と $\mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$(第5巻第6章の配管)、3.2 の attention、3.3 の FFN(第5巻の MLP に "position-wise" の一語を足したもの)、3.4 の embedding と出口の softmax(第6巻第3章と第4巻第6章)、3.5 の positional encoding(第7巻第7章)。

そして 3.2.1 に、あの式があります。

$$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

第1巻の序章で、私たちはこの式の前に立ち、「おそらく、何ひとつ読めないはずです」と言われました。$Q$ とは何か、肩の $T$ は何か、なぜ $\sqrt{d_k}$ で割るのか——あのとき、読めない箇所を指さすことすらできませんでした。第1巻終章で shape が読めるようになり、第4巻で softmax と $\sqrt{d_k}$ が埋まり、第6巻で Q・K・V という言葉の意味がわかり、第7巻で実装し、この巻で訓練しました。8巻かけて、のべ6回この式の前に立っています。

7回目の今日、初めて、**立ち止まりません。** 歩く速さを変えずに、式(1)の前を通り過ぎます。素通りできることが、こんなに感慨深いとは思わなかったはずです。

**4 Why Self-Attention / Table 1。** 層あたりの計算量・逐次操作数・最大経路長の3列の表。第7巻第8章で全行を自分で検算しました。第6巻で体感した「3つの痛み」が、そのまま表の3列になっていることも知っています。

**5 Training。** この巻の前半そのものです。5.1 のデータと "batched together by approximate sequence length" は第2章で実装し(BPE は第6巻第2章)、5.2 のハードウェアと訓練日数は第1章の実測と第6章のコスト見積もりで重みがわかり、5.3 の Adam と warmup の式は第4章で実装して「warmup を切ると壊れる」ことまで実験し、5.4 の dropout は第5巻 6.4、label smoothing は第3章で実装しました。"This hurts perplexity, as the model learns to be more unsure" の一文を、第3章の演習で自分のモデルの出力分布で確かめた人は、ここでにやりとできるはずです。

**6 Results。** Table 2 の BLEU と training cost は第6章で読み方を揃えました。"beam search with a beam size of 4" は第5章で実装済み。Table 3 のアブレーションは、各行が1〜7巻のどの議論に対応するかの地図を第6章で作りました——head 数(第7巻第4章)、$d_k$(第4巻第7章)、dropout(第5巻)、学習版 PE(第7巻第7章)。あなたが第5章の演習でやった自分のアブレーションの、本家版です。6.3 の構文解析への一般化は概観で読みます。ここは深追いしないと第6章で決めました——立ち止まらないことと、すべてを掘ることは、別だからです。

## 7.3 旅程 — 通った道と、通らなかった道

卒業式なので、入学の日を思い出しましょう。

すべては1行から始まりました。

```python
Y = X @ W + b
```

第1巻は、この1行の部品——ベクトル、内積、行列積、転置——を揃える巻でした。第2巻で、$W$ を機械が自分で見つける仕組みの土台として、微分と勾配降下法を手に入れました。第3巻で「モデル = パラメータ付き関数、学習 = 損失最小化」という世界観と4拍子の訓練ループを確立し、巻末で分類問題に敗北しました。第4巻がその敗北に答えて cross-entropy を導出し、ついでに softmax と $\sqrt{d_k}$ を式(1)に埋めました。第5巻で backprop を理解して autograd を自作し、residual・layer norm・dropout という配管を揃えました。第6巻で言語をベクトルにし、RNN の痛みを体感して「attention だけでいい」というタイトルの意味を理解しました。第7巻で論文の Section 3・4 を逐行精読し、全部品を実装しました。そしてこの第8巻で、組み立て、訓練し、評価しました。

`X @ W + b` から Transformer まで。振り返れば一本道ですが、それは偶然ではありません。**一本道になるように、最初から測地線を引いてあった**からです。

そして卒業式では、通った道と同じくらい、**通らなかった道**を誇るべきです。

このシリーズには、固有値と固有ベクトルが出てきませんでした。行列式も、逆行列も(第3巻で名前だけ登場して、通らないと宣言されました)。ベイズ統計が出てきませんでした。SVM も、決定木も、k-means も。CNN は Background の「捨てられた側」としてしか出てきませんでした。強化学習も出てきませんでした。どれも機械学習の正規の教養であり、どれも素晴らしい道具です。

それらを省いたのは、知らなくてよいからではありません。**この論文への最短測地線の上にないから**です。第1巻の序章はこう宣言していました——「この本は、機械学習の教科書ではありません」。網羅性はこの本の美徳ではない、とも。8巻を終えたいま、この宣言を検証できます。固有値を知らないことが原因で立ち往生した章が、1つでもあったでしょうか。ありませんでした。省いたものはすべて、「いつか必要になるかもしれない」だけのものでした。そして必要は、最後まで来ませんでした。

これは「学ばなくてよかった」という話ではなく、**「いつ学んでもよくなった」**という話です。あなたはいま、1本の論文を完全に読み切るとはどういうことかを知っています。次にベイズが必要になったら——たとえば不確実性を測る仕事に出会ったら——そのときのあなたは、ベイズへの測地線を自分で引けるはずです。通らなかった道は、消えていません。地図の上で、出発点だけが変わりました。

もうひとつ、需要駆動の順序も最後まで守られました。確率より先に機械学習をやったのは、cross-entropy に困ってから確率を出すためでした。PyTorch を第8巻まで我慢したのは、自作スタックでは訓練が終わらないと体感してから出すためでした。8巻を通して、**困る前に出てきた道具は、ひとつもなかった**はずです。新しい道具が出てきたとき、あなたはいつも、それが解決する痛みを先に知っていました。
