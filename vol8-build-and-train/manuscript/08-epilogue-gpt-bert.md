# 最終章(エピローグ) その後の世界 — GPT と BERT への分岐

> [目次](../TOC.md) ・ [← 前の章](07-graduation.md)

卒業式は終わりました。論文はアブストラクトから最後まで通読され、論文読解マップに「まだ」の行は残っていません。あなたが組み立てた Transformer は訓練を終えて動いています。本としては、もう閉じてよいはずです。

それでもこの短いエピローグを置くのには理由があります。終章で、あなたは論文の Conclusion をこう読み終えました。

> *"We are excited about the future of attention-based models and plan to apply them to other tasks."*
> — Vaswani et al., "Attention Is All You Need", Section 7 Conclusion
>
> 訳: 私たちは attention に基づくモデルの未来に期待しており、他のタスクへ適用していく計画である。

2017年のこの一文は、その後、著者たちの想像も超える規模で現実になりました。第1巻の序章にはこうありました——「あなたが今日どこかで AI と会話したなら、その向こう側で動いていたのは、ほぼ間違いなくこの論文の子孫です」。8巻前のあなたはこの文を信じるしかありませんでしたが、いまのあなたは**検証できます**。

このエピローグはその答え合わせです。論文と、あなたがいま使っている LLM とのあいだの数年間に何が起きたのかを駆け足で眺めます。新しい数式もコードも出てきません。そして驚くべきことに、**新しい部品もほとんど出てきません**。これから見る「その後の世界」は、あなたがこの8冊で作った部品の、組み替えと増量でできています。

物語は論文の翌年、2018年から始まります。この年、図1のアーキテクチャ図が——encoder と decoder が並ぶあの図が——**真っ二つに切られました**。

半分にして、何になるのか。1つずつ見ていきます。

## 8.1 encoder だけ使う: BERT(穴埋めで事前学習)— 概観

まず左半分です。encoder を単体で取り出すと、何をする機械になるでしょうか。

第7巻2章で確認したとおり、encoder はトークン列を受け取り、**各位置の「文脈込み」ベクトルの列**を返す機械です。self-attention に mask はなく、すべての位置がすべての位置を見られます。翻訳の中では「入力文を読む係」でした。

「読む係」を単体で取り出す価値ははっきりしています。文を分類したい、似た文を検索したい、文中の固有名詞を拾いたい——言語の「理解」系のタスクはどれも、まず**各単語の意味を文脈込みで捉えたベクトル**が欲しいのです。それは第6巻3章の埋め込みが目指したものの完成形であり、encoder の出力そのものです。

問題は**どうやって訓練するか**です。タスクごとにラベル付きデータを人手で作るのは高くつきます(第6巻1.3の「理由2」の裏返し)。あのとき私たちは「次の単語当てなら、文章そのものが問題集になり、教師がタダで手に入る」と確認しました。では encoder にも次の単語当てをさせればよいのでは?

ところが、これが**成立しません**。encoder の self-attention には causal mask がないからです。第7巻3.5で causal mask を入れたのは、位置 $t$ の出力に $w_{t+1}$ 以降の情報が流れ込むのを防ぐためでした。mask のない encoder では未来が全部見えています。「次の単語を当てよ」と言っても答えが入力に書いてある。カンニングが構造的に防げないのです。

2018年に発表された **BERT**(Bidirectional Encoder Representations from Transformers)の答えは鮮やかでした。**モデルを変えるのではなく、問題の方を変える。** 入力トークンの一部(約15%)を `[MASK]` という特殊トークンに置き換えて隠し、「そこに元々あったトークンは何か」を当てさせます。おなじみの穴埋め問題です。この事前学習タスクを **masked language model** と呼びます(置き換え方には細かい工夫がありますが、概観なので深追いしません)。

これなら教師はタダのままです。隠す前の文がそのまま正解になるからです。出力側の形もよく知っているものです。隠された位置のベクトルに linear + softmax を重ねた $|V|$ クラス分類(第7巻6章の出力 head と同じ形)で、損失は cross-entropy(第4巻)。新しい数学は何もありません。

ここで用語の混同に先回りしておきます。**masked language model の「mask」と、attention の「mask」は、名前が同じだけの別物です。** 住所が違います。

- **attention の mask**(第7巻3.5)は、モデルの**内部**にあります。score 行列に $-\infty$ を足し、softmax 後の重みを 0 にする計算上の仕掛けでした。
- **BERT の `[MASK]`** は、**入力データの側**にあります。トークン列の一部を特殊トークンに置き換える操作で、仲間は BOS / EOS / PAD(第2章2.4)です。score 行列には指一本触れません。

むしろ BERT は、attention 側の causal mask を**使わない**ことが本質です。穴を正確に埋めるには、穴の**左右両方**の文脈を見たい。「彼は____を飲んだ」の空欄は左だけでは絞れませんが、右に「酔った」と続けば一気に絞れます。全方向が見える encoder は、穴埋めにとって欠陥ではなく最高の素材でした。BERT の B が Bidirectional(双方向)なのは、この設計の宣言です。

こうして穴埋めだけで訓練された BERT は、言語の「理解」系タスクの土台として、発表後数年の自然言語処理を席巻しました(土台をタスクごとに少しだけ追加訓練して使う二段構えの一般論は 8.3 で扱います)。一方で BERT に**できないこと**もはっきりしています。文章の生成です。穴埋め機は、左から右へ文章を書き続ける機械ではありません。

書く方の半分は、もう一方の道が引き受けます。

## 8.2 decoder だけ使う: GPT(第6巻1.3「次を当てる」の伏線回収 — 言語モデルそのもの)— 概観

次は右半分、decoder です。単体で取り出すと、1か所だけ困ったことが起きます。

decoder の真ん中の層、cross-attention です。第7巻5.3で確認したとおり、あの層の K と V は **encoder の出力**から来ていました。encoder を捨てた今、矢印の出どころがありません。相手のいない層は、層ごと取り除きます。

残ったものを並べます。埋め込みと positional encoding(第7巻6・7章)、causal mask 付き self-attention(第7巻3.5・5章)、FFN(第7巻6章)、それを N 段積んで、最後に linear + softmax の出力 head。——全部、第7巻で単体実装してテストし、第1章で組み上げた部品です。**新しい部品はゼロです。**

この機械は何をするのか。causal mask があるので各位置は自分より過去しか見られません。そして各位置で次のトークンの分布を出します。つまりこの機械の仕様は、

$$P(w_t \mid w_{<t})$$

——文脈を受け取り、次のトークンの条件付き分布を返す。**言語モデルの定義、そのもの**です(第6巻1.1)。decoder だけの Transformer は新しい種類のモデルではありません。第6巻で n-gram から始めて RNN へと作り続けてきた、あの言語モデルの系譜の最新型です。

ここで、シリーズで一番遠くに張った伏線を回収します。第6巻1.3で私たちは作業仮説を立てました——「次の単語を十分うまく当てられる機械は、言語がわかっている(とみなす)」。そしてあの節の最後にこう予告しました。**この仮説を文字どおり極限まで推し進めたらどうなるか。モデルをとことん大きくし、世界中の文章を集め、ただひたすら「次の単語当て」だけを訓練し続けたら——その答えは GPT という名前で世界に知られることになり、回収は第8巻の最終章である**、と。いま、その場所に立っています。

2018年に発表された **GPT**(Generative Pre-trained Transformer)は、まさにその賭けでした。アーキテクチャの新規性はほとんどありません。図1の右半分から cross-attention を抜いただけです。新しかったのは**賭け金の大きさ**です。第6巻1.3の理由2(文章そのものが問題集になる)により訓練データに原理的な上限はなく、理由1(当てる精度の要求に天井がない)によりモデルを大きくする意味が尽きません。GPT とその後継たちはこの2つの理由を文字どおりに受け取り、データとモデルをひたすら増やし続けました。その先に何が起きたかは 8.3 で見ます。

BERT と並べると、2つの道の関係がきれいに見えます。どちらも出発点は同じ、「教師がタダの問題で事前学習したい。しかしカンニングは防ぎたい」でした。**BERT は問題の方を変えました**——穴埋めにして、そのかわり両方向を見せる。**GPT は問題を守りました**——次の単語当てのまま、causal mask で未来に目隠しをする。「隠す」が入力データ側の操作か score 行列側の操作か。8.1 で区別した2つの mask が、そのままこの分岐の見取り図です。

最後に足元を確認してください。GPT の訓練の中身は、teacher forcing の1トークンずらし(第3章)、cross-entropy、Adam と warmup(第4章)、温度付きサンプリングでの生成(第5章)。**すべて、この巻であなたが縮小版 Transformer で実際にやったこと**です。あなたが第5章で回した訓練ループと GPT の訓練ループの違いは、本質的には1つ、規模だけです。

## 8.3 ここから先の地図 — スケール則、事前学習+ファインチューニング、そして読者が今使っているLLMへ

2018年の分岐点から現在までを、各1段落の見取り図で眺めます。ここから先は「この本の外の世界」です。1つ1つを深追いはしません。地図として持ち帰ってください。

**スケール則(scaling laws)。** 第5章で、あなたは loss 曲線が下っていくのを眺めました。では、モデルを10倍、データを10倍、計算量を10倍にしたら loss はどこまで下がるのか。2020年頃までに、この問いには驚くほど整然とした答えがあることがわかりました。言語モデルの loss は、モデルサイズ・データ量・計算量のべき乗則に沿って、**事前に予測できる形で**下がり続けるのです。GPT は約1億、その次の世代は15億、さらに次は1750億パラメータへと2年で千倍以上に膨らみましたが、これは闇雲な巨大化ではなく「大きくすればどれだけ良くなるかが読める」という法則に裏打ちされた投資でした。当てる精度の要求に天井がないという第6巻1.3の理由1は、ここで「下げ続けられる loss」という実測のグラフになったのです。

**事前学習+ファインチューニング(pre-training + fine-tuning)。** BERT と GPT が共通して確立したのは、訓練を二段に分ける作法です。まず教師がタダの問題(穴埋め、または次の単語当て)で巨大コーパスを使って一度だけ高価な訓練をする——これが**事前学習**。次にその重みを初期値として、目の前のタスクの少量データで軽く追加訓練する——これが**ファインチューニング**。第3巻で確立した「学習=損失最小化」の世界観は、どちらの段でも一切変わりません。変わったのは**出発点**です。乱数で初期化した $W$ から坂を下るのではなく、言語をすでに知っている $W$ から下り始める。第3巻では数千個のデータで数千個のパラメータを学びましたが、初期値が良ければ、数千個のデータで数億個のパラメータを**調整**できるのです。

**指示チューニングと RLHF。** 次の単語を当てる機械は「ありそうな続き」を書く機械であって、「役に立つ応答」を書く機械ではありません。質問を入力すると、答えではなく**似た質問の列挙**を続けることすらあります(訓練データに質問リストが多ければ、それが「ありそうな続き」だからです)。この溝を埋めるのが訓練の最終段です。**指示チューニング(instruction tuning)**は「指示と模範応答」のペアを集めてファインチューニングし、指示には応答で続けるものだと教えます。**RLHF**(reinforcement learning from human feedback)は、モデルの応答を人間に比較してもらい、その好みを再現する報酬モデルを学び、報酬が高くなる方へモデルを調整します。強化学習はこのシリーズが意図的に通らなかった道なので(第1巻序章の宣言どおり)中身には立ち入りません。1つだけ言えるのは、ここでも forward → loss → backward → update の4拍子(第3章)は変わらず、変わるのは「何を損失とするか」の設計だけ、ということです。

**そして、読者がいま使っている LLM へ。** ここまでの段落を積むと、ChatGPT や Claude のようなチャット AI の正体が部品表として書き下せます。BPE 系のトークナイザ(第6巻2章)。埋め込み(第6巻3章)。causal mask 付き self-attention と FFN の積み重ね、つまり decoder-only Transformer(第7巻+本章8.2)。それを Adam(第4章)で、世界規模のコーパスの「次の単語当て」にかける(事前学習)。スケール則を頼りに桁違いに大きく作り、指示チューニングと RLHF で応答の作法を仕込み、推論時には温度付きサンプリング(第5章)で1トークンずつ生成する——。あなたがチャット欄にメッセージを送るたび、その向こうで起きているのは、第5章であなたが観察したあの計算の、桁違いの規模での反復です。第1巻序章の「ほぼ間違いなくこの論文の子孫です」という一文は、もう信じる必要はありません。**読めるはずです。** この数年で増えたのは魔法ではなく、規模と、訓練の段数でした。

もちろん、この地図には白い部分がたくさん残っています。スケール則はなぜ成り立つのか。RLHF の中身はどうなっているのか。——その白地図を塗る旅は、いつか別のシリーズの仕事になるかもしれません。

## 8.4 読者への最後の課題: 自分の nanoGPT を育てる

この本の演習は各章の章末にありました。シリーズ最後の課題だけは、章末ではなくここに、略解なしで置きます。**読む課題ではなく、作る課題だからです。**

8.2 を読み返すと気づくはずです。decoder-only の言語モデル——GPT と同じ形のモデル——を作るのに必要な部品のうち、**あなたが持っていないものは1つもありません**。在庫を確認しましょう。

| 部品 | どこで作ったか |
|---|---|
| トークナイザ(BPE) | 第6巻2章(文字レベルで始めるなら、語彙=出現文字の集合で代用可) |
| 埋め込み + positional encoding | 第7巻6・7章 |
| causal mask 付き self-attention + FFN のブロック | 第7巻3〜5章(cross-attention は使わない) |
| 訓練ループ(1トークンずらし + cross-entropy) | 第3章 |
| Adam + warmup | 第4章 |
| 生成(greedy / 温度付きサンプリング) | 第5章 |
| 物差し(loss・perplexity) | 第6巻1章・第6章 |

しかも、この巻で作った翻訳モデルより**簡単**です。encoder がないので組むのは片側だけ。cross-attention がないので、mask と入力の組み合わせに悩む場所が減ります。対訳コーパスも要りません——テキストが1本あればよく、そこから固定長の窓を切り出せば長さは自動的に揃い、第2章で苦労した padding の管理もほぼ消えます。

課題の名前にした **nanoGPT** は、Andrej Karpathy が公開している、数百行で書かれた decoder-only GPT の教育的実装の名前です(敬意を込めて借りました)。8巻前のあなたには呪文の羅列だったそのコードが、いまのあなたには既知の部品のリストとして読めます。ただし読むのは後の楽しみにしてください。**まず、自分の部品で組むこと。** それがこの課題です。

最初の一歩は小さく踏んでください。

1. **コーパス**: 好きな文章を1つ選びます。青空文庫の長編1作、童話集、あなた自身が書きためた文章——数百KBもあれば十分です。「自分が読みたい文章」を選ぶこと。生成結果を眺める時間がいちばん長くなるからです。
2. **トークン化**: まずは文字レベルで。出現した文字の集合がそのまま語彙です。BPE(第6巻2章)への差し替えは2周目の楽しみに取っておきます。
3. **モデル**: 第5章で訓練した縮小版 Transformer から、encoder と cross-attention を消します。層数4、$d_{model} = 128$ 程度の小さな構成から(パラメータ数の見積もりは第1章の検算がそのまま使えます)。
4. **訓練**: 第3章のループがほぼそのまま動きます。入力と正解を1トークンずらすのも同じです。文字レベルの極小構成なら GPU がなくても回ります(第2章で約束した逃げ道です)。
5. **観察**: 数百ステップごとに必ず生成させてください。最初はでたらめな文字の羅列、やがて単語らしき塊、そして文らしきもの——モデルが言語を獲得していく過程は、loss 曲線の数字よりずっと雄弁です。
6. **育てる**: 物足りなくなったら、コーパスを増やす、文字レベルを BPE に替える、層と幅を増やす。変更は1つずつ、効果は perplexity で測る——第5章の演習でやった、あのアブレーションの作法で。

うまくいかないときは第5章5.4に戻ってください。loss が下がらない、発散する、同じ語を繰り返す——decoder-only になっても、犯人の顔ぶれ(mask 漏れ、学習率、ずらし忘れ)は同じです。

第1巻の序章で、この本は2つの約束をしました。この論文を1行も飛ばさずに読めるようになること。そして、論文に書かれたものを自分の手でゼロから実装し、訓練できるようになること。1つ目は終章の通読で果たされました。2つ目は、いままさに、あなたの手元で果たされようとしています。

第6巻1章で「次の単語当て」というゲームに初めて出会ったとき、私たちはこう言いました——この地味なゲームは、そこまで続く一本道の入口だ、と。その一本道の上に、`X @ W + b` があり、坂を下る方法があり、cross-entropy があり、attention があり、この論文があり、そしていま、あなたの nanoGPT がいます。

最後の指示です。訓練を仕掛けて loss が下がり始めるのを確かめたら、しばらくそれを眺めてください。8冊ぶんの部品が、全部そこで動いています。

それから、本を閉じてください。続きを書くのは、あなたと、あなたのモデルです。
