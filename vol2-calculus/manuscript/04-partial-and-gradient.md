# 第4章 偏微分と勾配 — 多変数の坂

前の章で、私たちは勾配降下法を手に入れました。$x \leftarrow x - \eta\, f'(x)$ ——たった1行のこのアルゴリズムは、傾きの逆向きに小さく動くことを繰り返して、解けない方程式の最小値ににじり寄ってくれます。learning rate $\eta$ の選び方ひとつで収束も振動も発散もすること、止めどきの決め方も、実験で確かめました。

ところが、思い出してほしいことがあります。この巻の出発点は「第1巻の `Y = X @ W + b` の、$W$ の中身はどう決まるのか?」という問いでした。$W$ は行列です。中身は数が1個ではありません。第1巻第6章のおもちゃの線形レイヤーでさえ十数個、論文の Transformer なら数千万個の数が詰まっています。

私たちの勾配降下法は、いまのところ変数1個の関数 $f(x)$ しか扱えません。変数が数千万個は遠すぎるので、まず**2個**にしてみましょう。実は、それで十分です。1個から2個に増えるときに起きることがすべてで、2個から数千万個へは同じ話の繰り返しになります。この章では、変数が2個になった瞬間に「傾き」という言葉が壊れること、そしてそれを修理する道具——偏微分と勾配——を手に入れます。

## 4.1 変数が2個になる: 偏微分 = 他を止めて1変数ずつ

この章の実験台として、次の関数を使い続けます。

$$f(x, y) = x^2 + 3y^2$$

入力が $x$ と $y$ の2個、出力は1個の数です。グラフを思い浮かべるなら、地面の各点 $(x, y)$ の上に高さ $f(x, y)$ を立てた、3次元の曲面になります。形は「お椀」です。$f$ は2乗の和なので必ず 0 以上で、最小値は $x = 0,\ y = 0$ のときの 0。そこから離れるほど、どの向きでも高くなります。

ただし、まん丸のお椀ではありません。いくつか値を見てみます。

$$f(1, 0) = 1, \qquad f(0, 1) = 3$$

原点から同じ距離 1 だけ離れたのに、$x$ 方向では高さ 1、$y$ 方向では高さ 3。$3y^2$ の係数 3 のせいで、**$y$ 方向の坂のほうが急**なのです。このお椀は、$x$ 方向に間延びした楕円形をしています。

さて、この $f$ を前の章のアルゴリズムにかけたい。そのためには $f'$ が要ります。ここで手が止まります。**$f'(x, y)$ とは何でしょうか。** $x$ で微分するのか、$y$ で微分するのか。そもそもいま見たとおり、この曲面の「傾き」は進む方向によって違います($x$ 方向よりも $y$ 方向のほうが急でした)。傾きを1個の数で言い表すことが、もうできないのです。

困りました。しかし、新しい微分を発明する必要はありません。手持ちの1変数の微分を、**2回使う**だけでよいのです。

やり方はこうです。まず $y$ を「ただの定数」だと思って固定してしまいます。たとえば $y = 1$ に釘付けにすると、$f(x, 1) = x^2 + 3$ となり、これは $x$ だけの1変数関数です。第1章のルールで微分できます。定数 3 は消えて、答えは $2x$。$y$ をどの値に固定しても $3y^2$ は定数として消えるので、結果はいつでも $2x$ です。

この「$y$ を止めて $x$ で微分したもの」を、$f$ の $x$ についての**偏微分(partial derivative)**と呼び、こう書きます。

$$\frac{\partial f}{\partial x} = 2x$$

記号 $\partial$ は、1変数のときの $d$ を丸めた形で、「パーシャル」や「ラウンド」と読まれます。$\dfrac{\partial f}{\partial x}$ 全体で「他の変数は止めてあります。動かしたのは $x$ だけです」という宣言です。「偏った微分」という名前のとおり、全方向のうち $x$ 方向だけに偏って傾きを測っています。

今度は逆に $x$ を釘付けにして、$y$ で微分します。$x^2$ が定数として消えて、

$$\frac{\partial f}{\partial y} = 6y$$

具体的な点で値を出してみましょう。点 $(3, 1)$ では、

$$\frac{\partial f}{\partial x} = 2 \times 3 = 6, \qquad \frac{\partial f}{\partial y} = 6 \times 1 = 6$$

読み方はそのまま1変数のときと同じです。点 $(3, 1)$ に立って $x$ 方向に微小に動くと、$f$ は「進んだ距離あたり 6」の率で増える。$y$ 方向に動いても、率は 6。それだけのことです。

手計算は、検算する習慣でした。第1章で作った数値微分 $\dfrac{f(x+h) - f(x-h)}{2h}$ は、ここでもそのまま使えます。「他の変数を止める」をコードに翻訳すると、**動かしたい変数にだけ $\pm h$ を足す**になります。

```python
# 第2巻 第4章 4.1: 偏微分を数値微分で検算する(第1章の習慣の2変数版)
import numpy as np


def f(x, y):
    return x ** 2 + 3 * y ** 2


h = 1e-5
x0, y0 = 3.0, 1.0

df_dx = (f(x0 + h, y0) - f(x0 - h, y0)) / (2 * h)  # y を止めて x だけ動かす
df_dy = (f(x0, y0 + h) - f(x0, y0 - h)) / (2 * h)  # x を止めて y だけ動かす

print(df_dx)  # 6.000000... ← 手計算の 2 * 3.0 と一致
print(df_dy)  # 6.000000... ← 手計算の 6 * 1.0 と一致
assert np.allclose(df_dx, 2 * x0)
assert np.allclose(df_dy, 6 * y0)
```

新しい微分ルールはひとつも増えていないことに注意してください。偏微分は「他を定数扱いする」という**構えの工夫**であって、微分そのものは第1章の道具のままです。変数が10個でも数千万個でも同じです。1個ずつ釘付けを外しては、1変数の微分をする。それを変数の個数だけ繰り返せばよいのです。

## 4.2 勾配ベクトル ∇f: 偏微分を束ねたもの

点 $(3, 1)$ での測量の結果、手元に 6 と 6 という**2つの数**が残りました。変数が増えれば、偏微分の個数も増えます。この数の集まりを、どう持ち歩けばよいでしょうか。

数を並べたもの——この言い方に聞き覚えがあるはずです。第1巻第1章で、私たちはそれを**ベクトル**と名付けました。第1巻のベクトルが、ここで再登場します。あのときベクトルに乗せたのは「単語の意味」でした。今度は乗せる中身が変わります。**各方向の傾き**です。

偏微分を束ねたベクトルを、$f$ の**勾配(gradient)**と呼び、$\nabla f$ と書きます。

$$\nabla f(x, y) = \left( \frac{\partial f}{\partial x},\ \frac{\partial f}{\partial y} \right)$$

shape は `(2,)`、変数の個数と同じ長さのベクトルです。記号 $\nabla$ は「ナブラ」と読みます(逆三角形の形をした楽器が語源とされる、ただの記号です。怖がる必要はありません)。私たちの実験台なら、

$$\nabla f(x, y) = (2x,\ 6y)$$

勾配の大事な性質は、**点ごとに違うベクトルが出てくる**ことです。いくつか計算してみます。

| 立っている点 | $\nabla f$ |
|---|---|
| $(3, 1)$ | $(6,\ 6)$ |
| $(1, 0)$ | $(2,\ 0)$ |
| $(0, -1)$ | $(0,\ -6)$ |
| $(0, 0)$ | $(0,\ 0)$ |

つまり $\nabla f$ は「点を入れるとベクトルが返ってくる関数」です。お椀の上のどこに立っているかで、足元の傾き具合は違う——その当たり前を、そのまま式にした形になっています。

表の最後の行に注目してください。最小値の点 $(0, 0)$ では、勾配が**零ベクトル**になっています。第2章で「最小値では傾きが 0」を確認しましたが、あれの多変数版がこれです。**最小値では、どの方向の偏微分も 0。すなわち $\nabla f = \mathbf{0}$。** 谷底では、どっちを向いても坂が消えているのです。

ところで、ベクトルには成分の並び以上の顔がありました。第1巻では、ベクトルを「向きと大きさを持つ矢印」としても扱いました。では、$\nabla f(3, 1) = (6, 6)$ を矢印として地面に描いたとき、その**向き**は何を意味しているのでしょうか。実は、ここにこの章でいちばん大事な事実が隠れています。

## 4.3 等高線と勾配: 勾配は最も急な登り方向 → 逆向きに下る

その事実を見るために、まず曲面の描き方を変えます。3次元の鳥瞰図は、紙の上では歪んで読みにくい。代わりに、山の地図の知恵を借ります。**等高線(contour line)**——同じ高さの点を結んだ線です。

私たちのお椀なら、高さ $c$ の等高線は $x^2 + 3y^2 = c$ を満たす点の集まりで、$y$ 方向に潰れた楕円になります。$y$ 方向の坂が急だということは、少し進むだけで高さが変わるということなので、等高線の**間隔が詰まる**——地図の読み方と同じです。

等高線の上に、各点の勾配ベクトルを矢印として重ねた図を描いてみます。描画コードは次のとおりです(matplotlib が必要です。図を描くコードはこの本では掲載のみで、写経しなくても先へ進めます)。

```python
# 第2巻 第4章 図4.1: 等高線と勾配ベクトル(掲載のみ。matplotlib が必要です)
import matplotlib.pyplot as plt
import numpy as np


def f(x, y):
    return x ** 2 + 3 * y ** 2


xs = np.linspace(-3.5, 3.5, 200)
ys = np.linspace(-2.0, 2.0, 200)
X_grid, Y_grid = np.meshgrid(xs, ys)
Z = f(X_grid, Y_grid)

fig, ax = plt.subplots(figsize=(7, 4))
cs = ax.contour(X_grid, Y_grid, Z, levels=[0.3, 1, 2.5, 5, 9, 14], colors="gray")
ax.clabel(cs, fmt="%.1f")

# いくつかの点で勾配ベクトル (2x, 6y) を矢印として重ねる
points = [(3, 1), (-2, 1), (0, 1.5), (1.5, -0.5), (-2.5, -1)]
for px, py in points:
    ax.quiver(px, py, 2 * px, 6 * py,
              angles="xy", scale_units="xy", scale=8, color="tab:blue")

ax.set_ylabel("y")
ax.set_aspect("equal")
plt.show()
```

**図4.1**: $f(x, y) = x^2 + 3y^2$ の等高線(灰色の楕円、数字は高さ)と、5つの点での勾配ベクトル(青い矢印)。

この図には、3つのことが見えるはずです。

1. 矢印はどれも**外向き**——高い方を指している
2. 等高線が詰まっている場所(原点から $y$ 方向に離れた場所)ほど、矢印が**長い**
3. どの矢印も、その点を通る等高線と**直角**に交わっている

「外向きで、急な場所ほど長い」。まるで勾配ベクトルが「最も急な登り方向と、その急さ」を表しているかのようです。実際、そのとおりなのです。**勾配 $\nabla f$ の向きは、その点で $f$ が最も急に増える方向。長さは、その方向での増える率。**

ただし、図を眺めて「そうらしい」で済ませるのは、この本の流儀ではありません。かといって厳密な証明もしません(これも流儀です)。代わりに、数値実験で追い詰めましょう。実験のデザインはこうです。**点 $(3, 1)$ に立ち、長さ 1 の方向ベクトルを15度刻みで24本用意して、それぞれの方向に微小な一歩を踏み出し、$f$ がいくら増えたかを総当たりで比べる。**「最も急な登り方向」が本当に勾配の向きなら、増分の優勝者は $\nabla f(3,1) = (6, 6)$ の向き——ちょうど45度——になるはずです。

スクリプト全体は次のとおりです(前半の数値検算は4.1節の再掲です。`code/ch04/steepest_direction.py` として保存してあります)。

```python
# 第2巻 第4章 4.1〜4.3: 偏微分の数値検算 → 「最も急な方向」を総当たりで探す実験
import numpy as np


def f(x, y):
    """この章の実験台: x方向に間延びしたお椀"""
    return x ** 2 + 3 * y ** 2


def grad_f(x, y):
    """手計算した勾配: ∇f = (2x, 6y)"""
    return np.array([2 * x, 6 * y])


# --- 4.1 偏微分を数値微分で検算する(第1章の習慣の2変数版) ---
h = 1e-5
x0, y0 = 3.0, 1.0

df_dx = (f(x0 + h, y0) - f(x0 - h, y0)) / (2 * h)  # y を止めて x だけ動かす
df_dy = (f(x0, y0 + h) - f(x0, y0 - h)) / (2 * h)  # x を止めて y だけ動かす

assert np.allclose(df_dx, 2 * x0)  # 手計算: ∂f/∂x = 2x
assert np.allclose(df_dy, 6 * y0)  # 手計算: ∂f/∂y = 6y
assert np.allclose(grad_f(x0, y0), np.array([6.0, 6.0]))

# --- 4.3 いろんな方向に微小に動いて、f の増分を比べる ---
step = 1e-4
results = []
for deg in range(0, 360, 15):
    theta = np.deg2rad(deg)
    d = np.array([np.cos(theta), np.sin(theta)])  # 長さ1の方向ベクトル
    delta = f(x0 + step * d[0], y0 + step * d[1]) - f(x0, y0)
    results.append((deg, delta))
    print(f"{deg:3d}度: Δf = {delta:+.3e}")

best_deg, best_delta = max(results, key=lambda t: t[1])
worst_deg, worst_delta = min(results, key=lambda t: t[1])
print("増分が最大の方向:", best_deg, "度 / 最小の方向:", worst_deg, "度")

# 勾配 (6, 6) の向きは45度。最大は45度、最小はその正反対の225度のはず
assert best_deg == 45
assert worst_deg == 225

# 増分は内積で予言できる: Δf ≈ step × (∇f・d)
g = grad_f(x0, y0)
for deg, delta in results:
    theta = np.deg2rad(deg)
    d = np.array([np.cos(theta), np.sin(theta)])
    assert np.allclose(delta, step * np.dot(g, d), atol=1e-6)

print("ok: すべての assert を通過しました")
```

`np.cos` と `np.sin` で作っている `d` は、角度 `deg` の向きを指す長さ 1 のベクトルです(第1巻第2章の単位ベクトル。長さを 1 に揃えるのは、方向だけを比べたいからです)。実行結果から、出力を一部抜き出します。

```
  0度: Δf = +6.000e-04
 45度: Δf = +8.485e-04
 90度: Δf = +6.000e-04
135度: Δf = +2.000e-08
180度: Δf = -6.000e-04
225度: Δf = -8.485e-04
増分が最大の方向: 45 度 / 最小の方向: 225 度
```

優勝は予想どおり**45度**——勾配 $(6, 6)$ の向きです。読みどころは他にもあります。最下位(最も急な**下り**)は、その正反対の**225度**。そして45度から90度ずれた135度では、増分がほぼ 0(表示の $2 \times 10^{-8}$ は一歩の幅を有限にしたことによる残りかすで、`step` を小さくするほど 0 に近づきます)。増えも減りもしない方向とは、**等高線に沿って歩く方向**です。図4.1で見た「勾配は等高線と直角」が、数値でも現れました。

なぜこうなるのか、種明かしをしましょう。証明ではなく、見当のつけ方です。方向 $\mathbf{d} = (d_x, d_y)$(長さ1)に幅 $h$ の一歩を踏むと、$x$ は $h\, d_x$ だけ、$y$ は $h\, d_y$ だけ動きます。一歩が微小なら、それぞれの変数の寄与は「変化率 × 動いた量」で見積もれて、足し合わせると

$$\Delta f \approx \frac{\partial f}{\partial x} \cdot h\, d_x + \frac{\partial f}{\partial y} \cdot h\, d_y = h \left( \nabla f \cdot \mathbf{d} \right)$$

最後の変形で現れた $\nabla f \cdot \mathbf{d}$ は——**内積**です。第1巻第2章で、内積は「2つのベクトルの相性の採点」だと学びました。つまりこの式はこう読めます。**一歩あたりの増分は、進む方向と勾配との相性で決まる。**相性が最大になるのは同じ向きを向いたとき($\cos\theta = 1$)、最小は正反対($\cos\theta = -1$)、直交なら 0。これがそのまま、実験の45度・225度・135度に対応します。スクリプト末尾の assert は、24方向すべてでこの予言 $\Delta f \approx h\,(\nabla f \cdot \mathbf{d})$ が成り立つことを機械的に確かめたものです。

整理しましょう。

- $\nabla f$ の向き = **最も急な登り方向**
- $-\nabla f$ の向き = **最も急な下り方向**
- $\nabla f$ と直交する方向 = 等高線に沿う方向(増えも減りもしない)

私たちは下りたいのでした。だから使うのは $-\nabla f$ です。1変数のとき「$f'(x)$ の逆符号に動く」だったものが、多変数では「$\nabla f$ の逆**向き**に動く」になる。符号が向きに格上げされただけで、発想は同じです。

ひとつだけ、先回りして注意しておきます。「最も急な下り方向」は「最小値へまっすぐ向かう方向」と**同じとは限りません**。勾配が知っているのは足元の傾きだけで、谷底の場所は知らないのです。これがどういうことかは、この後の図4.2と演習2で実際に目にします。

## 4.4 [コード] 2変数関数の最小値探しを可視化(等高線 + 降下の軌跡)

道具が揃いました。第3章のアルゴリズムを多変数に拡張します。現在地をベクトル $\mathbf{p} = (x, y)$ `(2,)` と書くと、

$$\mathbf{p} \leftarrow \mathbf{p} - \eta\, \nabla f(\mathbf{p})$$

ベクトルの引き算とスカラー倍——どちらも第1巻第1章の演算ですから、NumPy ならこのまま1行で書けます。実装しましょう(`code/ch04/gradient_descent_2d.py`)。

```python
# 第2巻 第4章 4.4: 2変数の勾配降下 — 軌跡を記録し、手計算の予言と照合する
import numpy as np


def f(x, y):
    return x ** 2 + 3 * y ** 2


def grad_f(x, y):
    return np.array([2 * x, 6 * y])


def gradient_descent_2d(start, lr, n_steps):
    """点 (x, y) を勾配の逆向きに動かし続け、通った点をすべて記録する"""
    p = np.array(start, dtype=float)  # 現在地 (2,)
    trajectory = [p.copy()]
    for _ in range(n_steps):
        p = p - lr * grad_f(p[0], p[1])  # 更新はこの1行。第3章と同じ形
        trajectory.append(p.copy())
    return np.array(trajectory)  # (n_steps + 1, 2)


start = (3.0, 1.5)
lr = 0.1
n_steps = 30
traj = gradient_descent_2d(start, lr, n_steps)

print("出発点 :", traj[0])
print(" 3歩目 :", traj[3])
print("10歩目 :", traj[10])
print("30歩目 :", traj[30])

# --- 検証1: f の値は1歩ごとに必ず減っている ---
values = np.array([f(px, py) for px, py in traj])
assert np.all(np.diff(values) < 0)

# --- 検証2: 軌跡は手計算の予言と一致する(x は1歩ごとに0.8倍、y は0.4倍) ---
k = np.arange(n_steps + 1)
assert np.allclose(traj[:, 0], 3.0 * 0.8 ** k)
assert np.allclose(traj[:, 1], 1.5 * 0.4 ** k)

# --- 検証3: 最小値 (0, 0) のすぐそばまで来ていて、そこでの勾配はほぼ零ベクトル ---
assert np.allclose(traj[-1], np.zeros(2), atol=1e-2)
assert np.linalg.norm(grad_f(traj[-1][0], traj[-1][1])) < 0.1

print("ok: すべての assert を通過しました")
```

第3章のコードとの違いは、現在地が数からベクトルになったことと、通った点をぜんぶ `trajectory` に記録していることだけです(あとで図に描くためです。shape は `(n_steps + 1, 2)`——「点のベクトルを行に積んだ行列」という第1巻第3章の形になっています)。

検証2が、このスクリプトの肝です。実は今回の $f$ では、**軌跡を1歩も実行せずに手計算で予言できます**。更新式に $\nabla f = (2x, 6y)$ と $\eta = 0.1$ を代入してみると、

$$x \leftarrow x - 0.1 \times 2x = 0.8x, \qquad y \leftarrow y - 0.1 \times 6y = 0.4y$$

つまり、1歩ごとに $x$ 座標は 0.8 倍、$y$ 座標は 0.4 倍になるだけです。$k$ 歩目の位置は $(3.0 \times 0.8^k,\ 1.5 \times 0.4^k)$ と閉じた式で書けてしまう。検証2の assert は、ループが刻んだ30歩がこの予言と全点一致することを確かめています。実行すると、すべて通過します。

```
出発点 : [3.  1.5]
 3歩目 : [1.536 0.096]
10歩目 : [3.22122547e-01 1.57286400e-04]
30歩目 : [3.71382012e-03 1.72938226e-12]
```

予言の式は、軌跡の**形**も教えてくれます。$y$ は1歩で 0.4 倍——3歩で約 0.06 倍と、あっという間に潰れます。$x$ は1歩で 0.8 倍——30歩かけてようやく 0.004 倍と、のんびりです。出力でも、3歩目にはもう $y$ がほぼ消えているのに、$x$ はまだ半分残っていますね。ということは、点はまず**斜めに急降下して谷底($x$ 軸)に乗り、それから谷底に沿って原点へにじり寄る**はずです。

それを目で確かめるのが、本巻のハイライト図です。

```python
# 第2巻 第4章 図4.2: 等高線 + 降下の軌跡(掲載のみ。matplotlib が必要です)
import matplotlib.pyplot as plt
import numpy as np


def f(x, y):
    return x ** 2 + 3 * y ** 2


def grad_f(x, y):
    return np.array([2 * x, 6 * y])


# 軌跡の計算(gradient_descent_2d.py と同じ)
p = np.array([3.0, 1.5])
traj = [p.copy()]
for _ in range(30):
    p = p - 0.1 * grad_f(p[0], p[1])
    traj.append(p.copy())
traj = np.array(traj)

# 等高線
xs = np.linspace(-3.5, 3.5, 200)
ys = np.linspace(-2.0, 2.0, 200)
X_grid, Y_grid = np.meshgrid(xs, ys)
Z = f(X_grid, Y_grid)

fig, ax = plt.subplots(figsize=(7, 4))
cs = ax.contour(X_grid, Y_grid, Z, levels=[0.05, 0.3, 1, 2.5, 5, 9, 14], colors="gray")
ax.clabel(cs, fmt="%.2f")
ax.plot(traj[:, 0], traj[:, 1], "o-", color="tab:red", markersize=4, label="降下の軌跡")
ax.plot(0, 0, "k*", markersize=12, label="最小値 (0, 0)")
ax.set_xlabel("x")
ax.set_ylabel("y")
ax.set_aspect("equal")
ax.legend()
plt.show()
```

**図4.2**: $f(x, y) = x^2 + 3y^2$ の等高線と、点 $(3.0,\ 1.5)$ から出発した勾配降下の軌跡($\eta = 0.1$、30歩)。赤い点が1歩ごとの現在地。点は等高線を直角に横切りながら左下へ急降下し、数歩で谷底($x$ 軸)に乗ったあとは、谷底に沿って最小値 $(0, 0)$ へ向かう。最小値に近づくほど、点の間隔(=歩幅)はひとりでに縮んでいく。

この図は、言葉を尽くして読む価値があります。上から順に。

**踏み出す向き。**各ステップを見ると、点はその場の等高線に対して直角に踏み出しています。4.3節の「$-\nabla f$ は等高線と直交する最急の下り」が、軌跡の一歩一歩に刻まれているのです。

**軌跡の曲がり方。**軌跡は出発点から最小値へまっすぐ向かっていません。最初の数歩は $y$ 方向の急な坂に引っ張られて斜め下へ落ち、$y$ が潰れてからは進路を変えて $x$ 軸沿いを進みます。4.3節の終わりに注意したとおり、勾配降下は**ゴールの方角ではなく、足元の最急方向**に従うからです。それでも谷底をたどって、ちゃんと最小値に着きます。

**歩幅の変化。**出発直後、等高線が混んでいる場所では1歩が大きく、原点に近づくと点の間隔がどんどん詰まります。$\eta$ は 0.1 のまま一度も変えていないのに、です。歩幅は $\eta \times \|\nabla f\|$ で決まり、谷底に近づくほど勾配そのものが小さくなる——第3章で1変数のグラフに見た「自動ブレーキ」が、地図の上では点の間隔として見えるのです。

この1枚が、この巻の表題「学習とは坂を下ること」の絵姿です。この先、変数が2個から数千万個になっても、等高線が描けなくなるだけで、起きていることはこの図と同じです。第6章では動かす相手が $x, y$ からパラメータ $W$ に変わり、第3巻以降のすべての「学習」は、この赤い点列の高次元版になります。

## まとめ

- 偏微分は「他の変数を止めて、1変数ずつ微分」: $\dfrac{\partial f}{\partial x}$。新しい微分ルールは不要で、第1章の道具がそのまま使える。検算も第1章の数値微分(動かす変数だけ $\pm h$)でできる
- 偏微分を束ねたベクトルが勾配 $\nabla f$ `(2,)`。第1巻のベクトルが「各方向の傾きの運び手」として再登場した。勾配は点ごとに違い、最小値では零ベクトルになる(「最小値では傾き 0」の多変数版)
- 微小な一歩の増分は $\Delta f \approx h\,(\nabla f \cdot \mathbf{d})$ ——進む方向と勾配の**内積(相性)**で決まる。だから $\nabla f$ は最も急な登り方向、$-\nabla f$ は最も急な下り方向、等高線とは直交する(証明ではなく、図4.1と24方向の総当たり実験で確認した)
- 多変数の勾配降下は $\mathbf{p} \leftarrow \mathbf{p} - \eta\, \nabla f(\mathbf{p})$。1変数と同じたった1行。ただし最急の下りはゴールへの直行方向とは限らず、軌跡は谷底に乗ってから曲がる(図4.2)
- 歩幅 $\eta \|\nabla f\|$ は谷底に近づくほどひとりでに縮む。$\eta$ 固定でも収束していく仕組みは1変数のときと同じ

**ラスボスとの距離**: 論文が当たり前のように使う **gradient(勾配)**という言葉が読めるようになりました。Section 5.3 の learning rate が「歩幅」なら、勾配は「向きと急さ」——更新の相棒の正体です。残るは、その歩幅を step ごとに変える理由(第6章・終章)。

## 演習

**演習1** 次の2つの関数について、$\dfrac{\partial f}{\partial x}$ と $\dfrac{\partial f}{\partial y}$ を手計算してください。(a) はさらに $\nabla f(1, 2)$ を、(b) は勾配が零ベクトルになる点を求めてください。

$$\text{(a)}\ f(x, y) = x^2 y + 3y \qquad \text{(b)}\ f(x, y) = (x - 1)^2 + 2(y + 3)^2$$

<details><summary>略解</summary>

(a) $y$ を止めると $x^2 y$ は「定数 × $x^2$」なので $\dfrac{\partial f}{\partial x} = 2xy$。$x$ を止めると $x^2 y + 3y$ は「定数 × $y$ + 定数 × $y$」なので $\dfrac{\partial f}{\partial y} = x^2 + 3$。よって $\nabla f(1, 2) = (2 \times 1 \times 2,\ 1 + 3) = (4,\ 4)$。

(b) $\dfrac{\partial f}{\partial x} = 2(x - 1)$、$\dfrac{\partial f}{\partial y} = 4(y + 3)$。両方が同時に 0 になるのは $(x, y) = (1, -3)$。これはこの関数の最小値の点です(2乗の和なので、そこで最小値 0 を取ります)。

</details>

**演習2** 本文の $f(x, y) = x^2 + 3y^2$ について、点 $(3, 1)$ に立っているとします。(a) 最小値 $(0, 0)$ へまっすぐ向かう方向のベクトルをひとつ挙げてください。(b) 最も急な下り方向 $-\nabla f(3, 1)$ を求めてください。(c) 両者は同じ向きですか? 違うなら、どんな関数なら一致するか考えてみてください(ヒント: 等高線がまん丸な場合)。

<details><summary>略解</summary>

(a) 原点へ向かう方向は $(0, 0) - (3, 1) = (-3, -1)$(の正のスカラー倍ならどれでも正解)。(b) $-\nabla f(3, 1) = -(6, 6) = (-6, -6)$、つまり $(-1, -1)$ の向き。(c) 同じ向きではありません。$(-3, -1)$ と $(-1, -1)$ は別の方向です。勾配降下は谷底の場所を知らず、足元で最も急な下りを選ぶだけなので、$y$ 方向の急な坂に余計に引っ張られるのです。等高線がまん丸な関数、たとえば $f = x^2 + y^2$ なら $\nabla f = (2x, 2y)$ が位置ベクトルの定数倍になるので、$-\nabla f$ は常に原点(最小値)を直接指します。図4.2の軌跡が曲がるのは、お椀が楕円だからです。

</details>

**演習3** 関数 $f(x, y) = 4x^2 + y^2$ を、出発点 $(1, 2)$、$\eta = 0.1$、20歩で勾配降下します。コードを実行する**前に**、次の順で軌跡を予想してください。(a) $\nabla f$ を手計算する。(b) 更新式に代入して「$x$ は1歩ごとに何倍、$y$ は何倍」を求める。(c) 軌跡はどんな形になるか——本文の図4.2と比べて、どちらの軸に先に乗るか——を予想する。最後にコードで確認してください。

<details><summary>略解</summary>

(a) $\nabla f = (8x,\ 2y)$。(b) $x \leftarrow x - 0.1 \times 8x = 0.2x$、$y \leftarrow y - 0.1 \times 2y = 0.8y$。(c) 今度は $x$ のほうが1歩で 0.2 倍と速く潰れるので、本文とは逆に、**先に $y$ 軸に乗り、$y$ 軸に沿って原点へ下りる**軌跡になるはずです。確認コード:

```python
# 第2巻 第4章 演習3 略解: 軌跡の予想をコードで確認する
import numpy as np


def grad_f(x, y):
    return np.array([8 * x, 2 * y])


p = np.array([1.0, 2.0])
traj = [p.copy()]
for _ in range(20):
    p = p - 0.1 * grad_f(p[0], p[1])
    traj.append(p.copy())
traj = np.array(traj)

k = np.arange(21)
assert np.allclose(traj[:, 0], 1.0 * 0.2 ** k)  # x は1歩ごとに0.2倍
assert np.allclose(traj[:, 1], 2.0 * 0.8 ** k)  # y は1歩ごとに0.8倍
print(" 5歩目:", traj[5])    # [3.2000e-04 6.5536e-01] — x はもうほぼ 0
print("20歩目:", traj[20])   # [1.0486e-14 2.3058e-02] — y 軸に沿って原点へ
```

5歩目で $x$ はほぼ消え、$y$ はまだ 0.66 残っています。予想どおり、先に $y$ 軸へ落ちてから $y$ 軸沿いに下る軌跡です。

</details>

**演習4** 演習3と同じ関数・同じ出発点で、$\eta$ だけを 0.3 に上げると何が起きるでしょうか。演習3(b) と同じ計算で「$x$ は何倍、$y$ は何倍」を求めてから予想し、コードで確認してください。

<details><summary>略解</summary>

$x \leftarrow x - 0.3 \times 8x = -1.4x$、$y \leftarrow y - 0.3 \times 2y = 0.4y$。$y$ の倍率は 0.4 で順調に収束しますが、$x$ の倍率は $-1.4$——符号が毎歩反転し、絶対値は 1.4 倍ずつ膨らみます。つまり **$x$ だけが谷を飛び越して反対側へ、を繰り返しながら発散**します(20歩後の $x$ は $(-1.4)^{20} \approx 837$)。第3章で見た「$\eta$ が大きすぎると発散」が、多変数では**一部の変数だけに**起きうるのです。$\eta$ は最も急な方向(ここでは $4x^2$ の $x$ 方向)に合わせて選ばざるを得ない——この窮屈さは、勾配降下の弱点のひとつとして第6章で再登場します。

```python
# 第2巻 第4章 演習4 略解: ηを上げると x だけが発散する
import numpy as np


def grad_f(x, y):
    return np.array([8 * x, 2 * y])


p = np.array([1.0, 2.0])
traj = [p.copy()]
for _ in range(20):
    p = p - 0.3 * grad_f(p[0], p[1])
    traj.append(p.copy())
traj = np.array(traj)

k = np.arange(21)
assert np.allclose(traj[:, 0], (-1.4) ** k)     # x は振動しながら発散
assert np.allclose(traj[:, 1], 2.0 * 0.4 ** k)  # y はおとなしく収束
print("20歩目:", traj[20])  # [8.3668e+02 2.1990e-08]
```

</details>
