# 序章 ラスボスとの対面

## 0.1 第1巻の到達点 — $QK^T$ は読めた。だが $W$ の中身はどう決まる?

第1巻の終章で、私たちは論文の式(1)を端から端まで読み切りました。$Q$ も $K$ も $V$ も正体不明の文字ではなく、$QK^T$ は「query と key の類似度の総当たり表」として読め、shape を声に出して確かめれば配管は一度も詰まりませんでした。ただ、棚卸し表に最大の謎がひとつ残りました。

**$W$ の中身は、どう決まるのか。**

第1巻を通して $W$ はずっと乱数のままで、乱数の $W_i^Q$ では $QW_i^Q$ は「でたらめな役割への変換」にすぎません。論文の Transformer が翻訳できるのは $W$ が「良い値」になっているからです。その成分は論文の標準設定だけでも数千万個あり、各つまみをどちらに、どれだけ回せば「良い値」に近づくのか、総当たりでは宇宙の寿命があっても足りません。

第1巻の終章では答えの形だけ予告しました。いまの $W$ の「悪さ」を1つの数で測り、その数が最も急に減る方向へ全部のつまみを少しずつ同時に回す——これを何万回も繰り返す。霧の中の山で、足元の傾きだけを頼りに谷底へ降りる方法です。

では「最も急に減る方向」とは数学の言葉で何で、「少しずつ」の「少し」は誰がどう決め、歩幅は同じままでよいのでしょうか。この巻はその問いに答えます。道具の名前は**微分(differentiation)**、そして**勾配(gradient)**。これを手に入れると、論文の中で今まで素通りするしかなかった一角が読めます。まずはその一角——この巻のラスボス——と対面しましょう。

## 0.2 ラスボスの掲示

論文のセクション 5.3 "Optimizer" に、こんな式と一文があります。

> $$lrate = d_{model}^{-0.5} \cdot \min(step\_num^{-0.5},\ step\_num \cdot warmup\_steps^{-1.5})$$
> *"We used the Adam optimizer with β1 = 0.9, β2 = 0.98 and ε = 10^{−9}."*
> — Vaswani et al., "Attention Is All You Need", Section 5.3

英文を訳すと、「私たちは β1 = 0.9、β2 = 0.98、ε = 10⁻⁹ の Adam オプティマイザを使用した」。

第1巻を終えたあなたなら、まったく歯が立たないわけではありません。$d_{model}$ はモデル内部のベクトルの次元数で論文では 512、$d_{model}^{-0.5}$ が $1/\sqrt{512}$ のことだ、というところまでは読めます。

しかし、そこから先が読めません。$lrate$ とは何の率なのか。$step\_num$ が式に入っている以上、この $lrate$ は訓練の進行とともに**値が変わっていく**らしい——なぜ変える必要があるのか。$\min$ は何と何の小さい方を競わせているのか。$warmup\_steps$——「準備運動のステップ数」とは穏やかでない名前ですが、訓練に準備運動が要るとはどういうことか。指数の $-1.5$ がどこから来たのかも、2行目の Adam、β1、β2、ε も、手がかりすらありません。

読めなくて構いません。**今は読めない。それを確認するためにここに掲げました。**

この巻を終えたとき、あなたは **learning rate とは何か、なぜ step ごとに変えるのか——つまり warmup の意図**が読めるようになっています。具体的には:

- $lrate$(learning rate)が「坂を下る歩幅」であり、勾配降下法の中で唯一、人間が決めなければならない数であること
- 歩幅が大きすぎると何が起き、小さすぎると何が起きるか(自分で実験して観察済みになります)
- $\min$ の中身が「最初は歩幅を徐々に広げ(warmup)、その後は徐々に狭める(decay)」という2つの戦略の切り替えであること
- そのような歩幅のスケジュールが、なぜ理にかなっているのか

一方で、**Adam の中身(β1、β2、ε)は第8巻まで読めないまま残ります。** Adam は勾配降下法の改良版で、その改良を語るには、この巻で手に入れる素の勾配降下法の弱点を実際の訓練で体感している必要があるからです。第6章の最後で弱点の入口だけ覗き、本格的な対面は第8巻に預けます。読めないものを読めないと言えることも、第1巻で学んだとおり立派な前進です。

## 0.3 この巻のゴール — 「学習 = 坂を下ること」を道具にする

この巻のテーマは**微分と最適化**です。やることはひとつの像に集約されます。

**学習とは、坂を下ることである。**

霧の中の山の比喩を、この巻では数学の言葉に翻訳し、動くアルゴリズムにします。部品は6つ、各章にひとつずつです。

まず「足元の傾き」を1つの数として取り出す道具——**微分**です(第1章)。次に「谷底では傾きが 0 になる」という事実と、にもかかわらず「傾き = 0 を方程式として解く」作戦がほとんどの関数で通用しない現実を確認します(第2章)。解けないなら反復で近づくしかなく、ここで本巻の主役**勾配降下法(gradient descent)**が登場します。アルゴリズム本体はたった1行で、その中にラスボスの式の主語 learning rate が住んでいます(第3章)。

後半は、この道具を $W$ の世界に近づけます。変数が複数になったときの傾き——**偏微分**と**勾配ベクトル**(第4章。第1巻のベクトルが再登場します)。入れ子の関数を機械的に微分する**連鎖律**(第5章。ここで作る小さな計算グラフは、第5巻で自作する autograd の原型です)。そして最終章で視点を転換します。「$x$ を動かす」のではなく「$W$ を動かす」——入力は固定し、パラメータの方を調整する、本シリーズ全体で最も重要な一段です(第6章)。その第6章で、学習の途中で歩幅を変える実験をします。最初は大股、あとは小股——それがそのまま、ラスボスの式の $\min$ の正体です。

第1巻と同じく、コードは理解の検算機です。手で微分した結果を数値微分で検算し、learning rate を変えて収束と発散を自分の目で観察する——「式が読める」と「動かして確かめた」を常にセットにします。

## 0.4 この巻で扱わないこと

微分積分の教科書には、積分、ε-δ論法による極限の厳密な定義、テイラー展開——といった目次が並びます。

**この巻には、どれも出てきません。**

省く理由は第1巻と同じで、Transformer の訓練に登場しないからです。

- **積分**は微分と対をなす道具ですが、勾配降下法は微分だけで動き、論文のどの式にも積分記号は現れません
- **ε-δ論法**は極限を厳密にする道具です。必要なのは「$h$ を小さくすると傾きがある値に近づく」という手触りまでで、それは数値実験で十分に得られます(第1章)
- **テイラー展開の一般論**も同様で、「関数を多項式で近似する」発想の断片は第1章の数値微分に含まれますが、一般論として展開する必要はこの旅では生じません

「いつか必要になるかもしれないものを念のため学んでおく」ことはしない——第1巻の原則を守ります。ここで省いたものが原因で後の巻で立ち往生することはないように設計してあります。
