# 第1章 微分 — 変化の速さを1つの数にする

第1巻の終章で、私たちは大きな宿題を持ち帰りました。ラスボスの式は、配管図としてはもう読めます。$Q$ も $K^T$ も、shape の流れも追えます。しかし、配管の中身——$W$ という行列の成分——は、最初から最後まで**乱数のまま**でした。乱数の $W$ では、どれほど立派な配管を組んでも、出てくるのはでたらめです。
 
$W$ の成分は、論文の標準設定だけでも数千万個あります。数千万個のつまみを、どちらに、どれだけ回せば「良く」なるのか。総当たりで試す時間は宇宙にも残っていません。これがこの第2巻で挑む問題です。

この章では、問題を思い切り小さくして始めます。つまみは**1個だけ**。その1個についてさえ、「増やすべきか、減らすべきか、どれくらい敏感に効くのか」を答えるのは、実は自明ではありません。この問いに**たった1つの数**で答える道具——それが微分です。

## 1.1 動機: 「W をどっちに動かせば良くなるか」を知りたい

具体的な状況を作りましょう。第1巻第6章の線形層 `X @ W + b` から、考えうる限り一番小さい部品を切り出します。入力は数1個で $x = 2$。パラメータも数1個で $w$。バイアスは省略して、出力は $y = wx = 2w$ です。そして「出力は 6 になってほしい」という目標があるとします。なぜそんな目標が降ってくるのか——データが目標を連れてくる仕組み——は第3巻の主題なので、いまは天下りで構いません。

いまの $w$ は 1 だとします。出力は 2、目標は 6、ずれは $-4$ です。ずれの符号を消して「悪さ」を測るために、2乗しておきます:

$$f(w) = (2w - 6)^2$$

$w = 1$ なら $f(1) = (-4)^2 = 16$。この $f$ を、この章では「**悪さの点数**」と呼ぶことにします(この点数に正式な名前が付くのは第3巻です)。大事なのは、$f$ が **$w$ の関数になっている**ことです。つまみ $w$ を回すと、点数が動きます。

さて、問いです。**$w$ を 1 から少しだけ増やすと、$f$ は増えるでしょうか、減るでしょうか。**

試せばわかります。$f(1.1) = (2.2 - 6)^2 = 14.44$。16 より小さい。減りました。では増やす方向が正解です——確かに、つまみが1個ならこれで何の問題もありません。しかし本番のつまみは数千万個です。1個ずつ「ちょい回し」して様子を見ると、つまみ1個の判定ごとにネットワーク全体の計算をやり直すことになります。数千万回の再計算で、ようやく全つまみの「回す向き」が1回分わかる。これを何万歩も繰り返すのですから、論外です。

それでも、この素朴な「ちょい回して様子を見る」の中に、欲しいものの正体ははっきり見えています。私たちが知りたいのは:

- $w$ を増やしたら $f$ は増えるのか、減るのか(**符号**)
- どれくらい敏感に動くのか(**大きさ**)

この2つを兼ね備えた1つの数です。いわば、$w$ の現在地における $f$ の「変化の速さ」。この章のゴールは、それを試行錯誤ではなく、**関数の式から直接、計算で取り出せる**ようになることです(数千万個分を一斉に手に入れる方法は第4章で「勾配」として登場します。まずは1変数からです)。

## 1.2 平均変化率から瞬間の傾きへ

道具を作る間は、いちばん素朴な曲線 $f(x) = x^2$ を相手にします(1.1 の $f(w)$ には、道具が揃ってから戻ってきます)。いま $x = 3$ にいるとして、「この地点での $f$ の変化の速さ」とは何でしょうか。

誰でも思いつく測り方がひとつあります。**2点とって、変化量を割る**ことです。$x = 3$ から $x = 3 + h$ まで動いたとき、

$$\frac{f(3+h) - f(3)}{h}$$

を計算する。これを**平均変化率**(average rate of change)と呼びます。グラフの上では、2点を結ぶ直線の傾きです。$h = 1$ なら $(16 - 9)/1 = 7$。

ところが、この値は $h$ の選び方に依存します。$h = 2$ なら $(25 - 9)/2 = 8$ で、さっきと違う数が出ます。「$x = 3$ における変化の速さ」とただ1つの数で言いたいのに、どの $h$ を選ぶべきか、決め手がありません。

決め手がないなら、実験です。$h$ をどんどん小さくしていったら、平均変化率はどうなるでしょうか。

| $h$ | $\dfrac{f(3+h) - f(3)}{h}$ |
|---|---|
| 1 | 7 |
| 0.1 | 6.1 |
| 0.01 | 6.01 |
| 0.001 | 6.001 |
| 0.0001 | 6.0001 |

明らかに、**6 に吸い寄せられています**。負の側から近づけても同じです。$h = -0.1$ なら $(8.41 - 9)/(-0.1) = 5.9$、$h = -0.01$ なら $5.99$。下からも 6 に向かっています。

この「$h$ を 0 に近づけたとき、値が吸い寄せられていく先」を**極限**(limit)と呼び、$\lim$ という記号で書きます。本書では極限を、いまやったとおりの「数値実験の行き先」という直観のまま使います。極限には ε-δ 論法という厳密な定義があり、それは解析学という分野の土台石ですが、Transformer への最短経路には乗っていないため扱いません。「$h$ を縮めていくと、ある1つの値に好きなだけ近づく。その値のこと」——この理解で、第8巻まで一度も困りません。

これで「$x$ における変化の速さ」が定義できます:

$$f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}$$

この値 $f'(x)$ を $x$ における**微分係数**と呼び、$x$ を動かしたときの対応 $x \mapsto f'(x)$ を関数とみなしたものを**導関数**(derivative)と呼びます。導関数を求める操作が「**微分する**(differentiate)」です。$f'$ のほかに $\frac{df}{dx}$ という記法もあります。本物の分数ではありませんが、「$x$ のちょい変化分($dx$)に対して $f$ がその何倍変わるか($df$)」と読める良い記法で、第5章の連鎖律で真価を発揮します。

グラフの上での意味も確認しておきましょう。$h$ を縮めていくと、2点を結ぶ直線は、その点でグラフに寄り添ってひとつの直線に落ち着きます。これが**接線**(tangent line)で、$f'(x)$ はその傾きです。曲線を虫眼鏡でどんどん拡大すると、ほとんど直線に見えてきます。その「局所的にはただの直線」の傾きが微分係数だ、と言ってもかまいません。

$f(x) = x^2$ なら、この極限は数値実験に頼らず「手で」取れます:

$$\frac{(x+h)^2 - x^2}{h} = \frac{2xh + h^2}{h} = 2x + h \;\xrightarrow{\;h \to 0\;}\; 2x$$

$h$ で約分できてしまえば、あとは $h$ に 0 を近づけるだけです。$f'(x) = 2x$。$x = 3$ で 6——数値実験の行き先と一致しました。

最後に、$f'(x)$ の**読み方**を整理します。ここがこの章の収穫物です。

- $f'(x) > 0$: その地点で $f$ は増加中。$x$ を増やすと $f$ は増える
- $f'(x) < 0$: 減少中。$x$ を増やすと $f$ は**減る**
- 大きさは敏感さ: $f'(3) = 6$ とは「$x = 3$ 付近で $x$ を微小に動かすと、$f$ はその約6倍動く」という意味

1.1 の問いは、こう翻訳されました。「$w$ をどっちに動かせば良くなるか」=「$f'(w)$ の符号を知りたい」。では $f(w) = (2w - 6)^2$ の導関数は何でしょうか。定義の極限を毎回手で取るのは、できなくはありませんが面倒です。機械的に微分するためのルールが欲しくなります。

## 1.3 微分の基本ルール — 多項式を機械的に微分する

最初に方針を断っておきます。これから並べるルールを、私たちは証明しません(第1巻からの流儀です)。代わりに、定義に立ち返る計算を2つだけ自分の手でやってパターンをつかみ、残りは直観で納得します。そして次の 1.4 で、**どんなルールの適用結果も機械で検算できる装置**を作ります。証明の代わりに検算——これがこの本の安全網です。

**ルール1: 定数の微分は 0。** $f(x) = c$(定数)なら $f'(x) = 0$ です。グラフは水平な直線で、傾きはどこでも 0。定義で見ても $f(x+h) - f(x) = c - c = 0$ なので、割る前から 0 です。

**ルール2: べきの微分。** $x^2 \to 2x$ は 1.2 で確かめました。$x^3$ でも同じ計算をしてみます。$(x+h)^3 = x^3 + 3x^2h + 3xh^2 + h^3$ なので、

$$\frac{(x+h)^3 - x^3}{h} = 3x^2 + 3xh + h^2 \;\xrightarrow{\;h \to 0\;}\; 3x^2$$

$x^2 \to 2x^1$、$x^3 \to 3x^2$。肩の数が前に降りて、肩が1つ減る。このパターンは一般に成り立ちます:

$$(x^n)' = n\,x^{n-1}$$

$x^4 \to 4x^3$、$x^{100} \to 100x^{99}$。$x = x^1$ なら $1 \cdot x^0 = 1$ ——$y = x$ は入力と出力が直結しているので、変化の速さが常に1倍、というもっともな結果です。

**ルール3: 定数倍と和はそのまま通る。**

$$(c f)' = c f', \qquad (f + g)' = f' + g'$$

定数倍: グラフを縦に3倍に引き伸ばせば、どの地点の傾きも3倍になります。和: $f$ と $g$ の変化分は、それぞれ勝手に発生して足し合わさるだけです。

この3つで、**多項式はすべて微分できます**。項ごとにべきのルールを使い、結果を足すだけです。

$$f(x) = 3x^2 + 2x + 1 \;\Longrightarrow\; f'(x) = 6x + 2$$

**ルール4: 積のルール。** 掛け算になっている関数には専用のルールが要ります。

$$(fg)' = f'g + fg'$$

「微分の積 $f'g'$」では**ない**ことに注意してください。なぜこの形になるのか、直観だけ示します。縦の長さ $f(x)$、横の長さ $g(x)$ の長方形を思い浮かべます。$x$ を $h$ だけ動かすと、縦は約 $f'h$、横は約 $g'h$ だけ伸びます。面積の増分は「右に増えた細長い帯 $f \cdot g'h$」と「上に増えた帯 $f'h \cdot g$」と「角の小さな欠片 $f'h \cdot g'h$」の合計です。角の欠片は $h^2$ サイズなので、$h$ で割ってから $h \to 0$ とすると消えます。残った2枚の帯を $h$ で割ったものが $f'g + fg'$ です。

例で確かめます。$(x^2 + 1)(x - 2)$ を積のルールで微分すると $2x \cdot (x - 2) + (x^2 + 1) \cdot 1 = 3x^2 - 4x + 1$。一方、先に展開すると $x^3 - 2x^2 + x - 2$ で、項ごとに微分しても $3x^2 - 4x + 1$。同じ答えです。

道具が揃ったので、1.1 の宿題を解きます。$f(w) = (2w-6)^2$ は展開すれば多項式です:

$$f(w) = 4w^2 - 24w + 36 \;\Longrightarrow\; f'(w) = 8w - 24$$

($(2w-6)(2w-6)$ に積のルールを使っても、$2(2w-6) + (2w-6) \cdot 2 = 8w - 24$ で同じです。)

$w = 1$ を入れると $f'(1) = -16$。**負**です。つまり、$w$ を増やすと悪さの点数は減る——1.1 で「ちょい回し」して得た結論と同じ答えが、今度は**試さずに、式から**出ました。しかも大きさ 16 という敏感さの情報付きです。数千万個のつまみへの道は、まだ遠いですが開きました。

ついでにもうひとつ観察を。$f'(w) = 8w - 24$ は $w = 3$ で 0 になります。傾きゼロ。そして $f(3) = (6-6)^2 = 0$ で、そこは悪さの点数が最小になる地点です。「傾きが 0 になる場所」と「最小の場所」の関係——これが次の第2章の主題です。

なお、$(2w-6)^2$ をいちいち展開せず、「外側の2乗」と「内側の $2w-6$」に分けて一発で微分する近道があります。合成関数の**連鎖律**といい、第5章で手に入れます。連鎖律こそ深層学習の心臓部なのですが、需要が生まれるまで取っておきます。

## 1.4 [コード] 数値微分 — 微分の検算機を作る

ルールは揃いました。しかし、ルールの適用は人間の仕事です。人間は間違えます。符号を落とす、肩の数を下げ忘れる、積のルールでうっかり $f'g'$ と書く。間違った微分は間違った方向につまみを回させるので、被害は静かで深刻です。**自分の微分が正しいか、機械に検算させる装置**が欲しくなります。

実は、その装置の設計図はすでに持っています。1.2 の数値実験——「$h$ に小さい値を入れて平均変化率を計算する」——は、そのままプログラムになります:

$$\frac{f(x+h) - f(x)}{h} \quad (h \text{ は小さい数})$$

これを**数値微分**(numerical differentiation)と呼び、この形は特に**前進差分**(forward difference)と呼ばれます($x$ から前方 $x+h$ を見るからです)。

ただし、実装する前にもう一工夫します。1.2 の表をもう一度見てください。$h = 0.1$ のとき、右側から測ると 6.1。左側($h = -0.1$)から測ると 5.9。真の値 6 を挟んで、**上下にほぼ対称にずれています**。それなら、両側から測って平均してしまえばいい:

$$\frac{6.1 + 5.9}{2} = 6$$

ずれが打ち消し合いました。この「右と左の平均」を式として整理すると:

$$\frac{1}{2}\left(\frac{f(x+h) - f(x)}{h} + \frac{f(x) - f(x-h)}{h}\right) = \frac{f(x+h) - f(x-h)}{2h}$$

途中で $f(x)$ が消えてくれるのが小気味よいところです。両脇の2点だけで計算できるこの形を**中心差分**(central difference)と呼びます。どれだけ得かは、このあと実験で測ります。

まず装置の本体です。中心差分を実装し、1.3 のルールで手計算した導関数(以後**解析解**と呼びます)と突き合わせます。題材は $f(x) = x^3 - 2x$、解析解は $f'(x) = 3x^2 - 2$ です。

```python
# 第2巻 第1章 1.4: 数値微分(中心差分)を実装し、解析解と突き合わせる
import numpy as np


def numerical_diff(f, x, h=1e-5):
    """中心差分による数値微分: (f(x+h) - f(x-h)) / 2h"""
    return (f(x + h) - f(x - h)) / (2 * h)


def f(x):
    return x ** 3 - 2 * x


def f_prime(x):
    return 3 * x ** 2 - 2  # 1.3 のルールで手計算した微分


for x in [-1.0, 0.0, 0.5, 2.0]:
    approx = numerical_diff(f, x)
    exact = f_prime(x)
    print(f"x = {x:5.2f}   数値微分 = {approx:.10f}   解析解 = {exact:.10f}")
    assert np.allclose(approx, exact)

print("ok: 4点すべてで数値微分と解析解が一致しました")
```

`numerical_diff` は**関数そのものを引数に取る**ことに注意してください。Python では関数も値として渡せるので、どんな $f$ に対しても同じ1行で傾きが測れます。実行結果はこうなります:

```
x = -1.00   数値微分 = 1.0000000001   解析解 = 1.0000000000
x =  0.00   数値微分 = -1.9999999999   解析解 = -2.0000000000
x =  0.50   数値微分 = -1.2499999999   解析解 = -1.2500000000
x =  2.00   数値微分 = 10.0000000002   解析解 = 10.0000000000
ok: 4点すべてで数値微分と解析解が一致しました
```

小数第9位あたりまで一致しています。手で求めた $3x^2 - 2$ が正しいことを、定義側から計算した機械が裏書きしてくれました。比較に `np.allclose` を使うのは第1巻からの作法です——数値微分は近似なので、「十分近ければ合格」がちょうどよい判定になります。

では、その「近さ」はどれくらいで、$h$ をどう選べばよいのでしょうか。そして中心差分は前進差分よりどれだけ得なのでしょうか。$x = 2$(解析解 10)に固定し、$h$ を振りながら両者の誤差を測る実験を足したものが完成版です(`code/ch01/numerical_diff.py` として保存してあります。前半は再掲):

```python
# 第2巻 第1章 1.4: 数値微分(中心差分)を実装し、手計算の微分(解析解)と突き合わせる
import numpy as np


def numerical_diff(f, x, h=1e-5):
    """中心差分による数値微分: (f(x+h) - f(x-h)) / 2h"""
    return (f(x + h) - f(x - h)) / (2 * h)


def f(x):
    return x ** 3 - 2 * x


def f_prime(x):
    return 3 * x ** 2 - 2  # 1.3 のルールで手計算した微分


# --- 突き合わせ: 数値微分 ≈ 解析解 になっているか ---
for x in [-1.0, 0.0, 0.5, 2.0]:
    approx = numerical_diff(f, x)
    exact = f_prime(x)
    print(f"x = {x:5.2f}   数値微分 = {approx:.10f}   解析解 = {exact:.10f}")
    assert np.allclose(approx, exact)

print("ok: 4点すべてで数値微分と解析解が一致しました")


# --- h を変えて誤差を観察: 前進差分 vs 中心差分 ---
def forward_diff(f, x, h):
    """前進差分: (f(x+h) - f(x)) / h"""
    return (f(x + h) - f(x)) / h


x = 2.0
exact = f_prime(x)  # 10.0

hs = [1e-1, 1e-2, 1e-3, 1e-5, 1e-8, 1e-11, 1e-13]
errors_central = {}

print()
print("      h     前進差分の誤差   中心差分の誤差")
for h in hs:
    err_fwd = abs(forward_diff(f, x, h) - exact)
    err_ctr = abs(numerical_diff(f, x, h) - exact)
    errors_central[h] = err_ctr
    print(f"  {h:.0e}      {err_fwd:.2e}       {err_ctr:.2e}")

# h = 1e-5 のとき中心差分はほぼ厳密(誤差 1e-8 未満)
assert errors_central[1e-5] < 1e-8
# h を小さくしすぎると、丸め誤差でかえって悪化する
assert errors_central[1e-13] > errors_central[1e-5]

print()
print("ok: すべての assert を通過しました")
```

誤差の表は、こうなります:

```
      h     前進差分の誤差   中心差分の誤差
  1e-01      6.10e-01       1.00e-02
  1e-02      6.01e-02       1.00e-04
  1e-03      6.00e-03       1.00e-06
  1e-05      6.00e-05       1.99e-10
  1e-08      6.08e-08       1.05e-07
  1e-11      8.27e-07       8.27e-07
  1e-13      7.99e-03       7.99e-03
```

この表には読みどころが3つあります。

**1つめ: 中心差分の圧勝。** 表の上半分で、$h$ を 1/10 にするたびに、前進差分の誤差は 1/10 に、中心差分の誤差は **1/100** になっています(前進差分は $h$ に比例、中心差分は $h^2$ に比例して誤差が縮む、と言えます)。「上下対称のずれが打ち消し合う」という工夫は、1桁どころか別次元の精度を連れてきました。$h = 10^{-3}$ の時点で、中心差分はすでに誤差 $10^{-6}$ です。

**2つめ: 小さくしすぎると壊れる。** $h = 10^{-8}$ から下では、誤差はむしろ**増えて**います。極限の定義に従えば $h$ は小さいほど真の値に近づくはずなのに、なぜでしょうか。原因は数学ではなくコンピュータ側にあります。浮動小数点数の精度は有限で、$h$ が極端に小さいと $f(x+h)$ と $f(x-h)$ はほとんど同じ値になります。**ほとんど同じ数同士の引き算は、有効数字を大量に失います**(桁落ちと呼ばれる現象です)。そのうえ、失った後の値を微小な $2h$ で割るので、傷が拡大されます。$h = 10^{-13}$ では誤差 $10^{-2}$ ——せっかくの検算機が、小数第2位までしか信用できなくなっています。

**3つめ: だから「ちょうどよい小ささ」を使う。** 数学の誤差は $h$ が小さいほど減り、計算機の誤差は $h$ が小さいほど増える。最良の $h$ は中間にあります。この実験では $h = 10^{-5}$ 付近が谷底で、誤差はおよそ $10^{-10}$。`numerical_diff` のデフォルトを `h=1e-5` にしたのはこのためです。コード末尾の2つの assert は、この観察(谷底ではほぼ厳密・小さすぎると悪化)をそのまま検査に翻訳したものです。

## まとめ

- 微分係数 $f'(x)$ は、平均変化率 $\frac{f(x+h)-f(x)}{h}$ の $h \to 0$ での行き先(極限)。グラフでは接線の傾き。極限は「数値実験の行き先」という直観で使う
- $f'$ の符号は方向、大きさは敏感さ。「$w$ をどっちに動かせば悪さが減るか」の答えは「$f'(w)$ の符号の逆向き」——これがこの巻全体を駆動する読み方
- 基本ルール: $(x^n)' = nx^{n-1}$、定数は 0、和と定数倍はそのまま通り、積は $(fg)' = f'g + fg'$。多項式はこれですべて微分できる
- 中心差分 $\frac{f(x+h)-f(x-h)}{2h}$ は前進差分より精度の階級が上($h^2$ 比例)。ただし $h$ は小さいほど良いのではなく、丸め誤差との兼ね合いで $h \approx 10^{-5}$ 程度が谷底
- 「解析解と数値微分を突き合わせる」習慣は、第5巻の autograd の単体テストの原型(第5巻で回収)

```python
# 第2巻 第1章 演習3 略解: 手で微分した結果を数値微分(中心差分)で検算する
import numpy as np


def numerical_diff(f, x, h=1e-5):
    """中心差分による数値微分: (f(x+h) - f(x-h)) / 2h"""
    return (f(x + h) - f(x - h)) / (2 * h)


# (手で求めた関数, その導関数) のペア。演習1の3問
cases = [
    ("(a) x^4 - 3x^2 + 5", lambda x: x ** 4 - 3 * x ** 2 + 5,
     lambda x: 4 * x ** 3 - 6 * x),
    ("(b) (x^2 + 1)(x - 2)", lambda x: (x ** 2 + 1) * (x - 2),
     lambda x: 3 * x ** 2 - 4 * x + 1),
    ("(c) (3w + 1)^2", lambda w: (3 * w + 1) ** 2,
     lambda w: 18 * w + 6),
]

for name, f, f_prime in cases:
    for x in [-2.0, -0.5, 0.0, 1.0, 3.0]:
        assert np.allclose(numerical_diff(f, x), f_prime(x))
    print(f"{name}: 5点すべてで一致")

print("ok: 手の微分はすべて正しい")
```
