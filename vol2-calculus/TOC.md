## [第1章 微分 — 変化の速さを1つの数にする](manuscript/01-derivative.md)

- 1.1 動機: 「W をどっちに動かせば良くなるか」を知りたい
- 1.2 平均変化率から瞬間の傾きへ(極限は直観+数値で。厳密さより手触り)
- 1.3 微分の基本ルール: 多項式、積、和(必要最小限のみ)
- 1.4 **[コード]** 数値微分: (f(x+h) − f(x−h)) / 2h を実装し、解析解と突き合わせる
  - この「数値微分と照合する」習慣は第5巻の autograd テストでそのまま使う
- 演習: 手で微分 → 数値微分で検算

## [第2章 最小値と傾き — 解けない方程式だらけの世界](manuscript/02-minima.md)

- 2.1 最小値では傾きが 0(図で)
- 2.2 「傾き = 0 を解けばいい」が通用しない関数が世の中のほとんど
- 2.3 だから反復で近づく、という発想の転換(この巻の主役の登場)
- 演習: 解析的に解ける例・解けない例の仕分け

## [第3章 勾配降下法(1変数)— 坂を下るアルゴリズム](manuscript/03-gradient-descent-1d.md)

- 3.1 アルゴリズム: x ← x − η・f'(x)。たった1行
- 3.2 learning rate η の役割: 小さすぎると遅い、大きすぎると発散
- 3.3 **[コード]** 1変数の勾配降下を実装し、η を変えて収束・振動・発散を観察(図で記録)
- 3.4 止めどき: 収束判定、ステップ数
- 演習: 凸でない関数で局所解にハマる体験

## [第4章 偏微分と勾配 — 多変数の坂](manuscript/04-partial-and-gradient.md)

- 4.1 変数が2個になる: 偏微分 = 他を止めて1変数ずつ
- 4.2 勾配ベクトル ∇f: 偏微分を束ねたもの(第1巻のベクトルがここで再登場)
- 4.3 等高線と勾配: 勾配は最も急な登り方向 → 逆向きに下る
- 4.4 **[コード]** 2変数関数の最小値探しを可視化(等高線 + 降下の軌跡)。本巻のハイライト図
- 演習: 勾配を手計算 → 軌跡を予想 → コードで確認

## [第5章 連鎖律 — 合成関数を機械的に微分する](manuscript/05-chain-rule.md)

- 5.1 関数を入れ子にする: f(g(x)) は第1巻5.3「変換の合成」の微分版
- 5.2 連鎖律: 外の微分 × 中の微分。掛け算で伝わっていく
- 5.3 計算グラフ: 計算を「ノードと辺」で描くと、微分は辺をたどる掛け算になる
- 5.4 **[コード]** 小さな計算グラフを実装し、前向き計算と微分の伝播を書く
  - これが第5巻 backprop / autograd の原型であることを予告
- 演習: グラフを描いて連鎖律で微分(手と紙が主役の章)

## [第6章 勾配降下法の実践 — パラメータ付き関数を調整する](manuscript/06-fitting-parameters.md)

- 6.1 「x を動かす」から「W を動かす」へ: 入力は固定、動かすのはパラメータ(視点の転換。本シリーズ最重要の一段)
- 6.2 **[コード]** パラメータ2〜3個の関数をデータに合わせて勾配降下で調整する(損失関数という言葉はまだ使わない。第3巻で命名する)
- 6.3 learning rate を途中で変える: 最初は大股、あとは小股(warmup / decay の直観)
- 6.4 勾配降下の弱点メモ: 谷でのジグザグ等(モーメンタム・Adam は第8巻、と予告のみ)
- 演習: η スケジュールを変えて収束を比較

## [終章 ラスボス再戦 — learning rate の式が読める](manuscript/07-boss-rematch.md)

- 7.1 Section 5.3 の式を再掲: step_num が小さいうちは lrate が増え(warmup)、その後 1/√step で減る — 6.3 でやったことそのもの
- 7.2 読めるようになった部分: learning rate、step、warmup の意図
- 7.3 まだ読めない部分の棚卸し: Adam の中身(第8巻)、そもそも何を最小化するのか=損失関数(次巻)
- 7.4 次巻予告: 「学習する」とは何か — ついに機械学習に入る
