# 第2章 内積 — 「似ている」を数にする

前の章で、私たちはベクトルという部品を手に入れました。数を並べれば、単語のような「数ではないもの」さえ表せるかもしれない——そういう無茶な計画が始まっています。足し算とスカラー倍もできるようになりました。

ところが、いまの私たちには、まだ決定的にできないことがあります。2つのベクトルを並べて、「この2つは**似ているか?**」と問うことです。

なぜそれを問いたいのでしょうか。論文のタイトルを思い出してください。"Attention Is All You Need"——注意こそすべて。文章の中のある単語が、他のどの単語に注意を向けるべきか。それを決めるには、単語と単語の「関連の強さ」を測る必要があります。関連の強さ、似ている度合い、相性——呼び方は何でも構いませんが、要するに**2つのベクトルから1つの数を作って、その大小で「似ている」を比べたい**のです。

この章で手に入れる道具は、たった1つ。**内積**です。定義は驚くほど短く、しかしこの演算こそが、ラスボスの式の心臓部に座っています。

## 2.1 内積の定義: 掛けて、足す。それだけ

具体例から始めましょう。前の章の計画に沿って、3つの単語を無理やり3次元のベクトルにしてみます。各成分は順に「生き物らしさ」「ふわふわ度」「速さ」を0〜9で採点したものだと思ってください(採点は筆者の独断です。まともな作り方は第6巻でやります)。

$$\mathbf{猫} = (4,\ 4,\ 1), \qquad \mathbf{犬} = (4,\ 3,\ 2), \qquad \mathbf{車} = (0,\ 1,\ 5)$$

直観的には、猫と犬は似ていて、猫と車は似ていません。この直観を、数の操作で再現できるでしょうか。

やってみるのは、こういう操作です。**対応する成分同士を掛けて、全部足す。**

猫と犬なら:

$$4 \times 4 + 4 \times 3 + 1 \times 2 = 16 + 12 + 2 = 30$$

猫と車なら:

$$4 \times 0 + 4 \times 1 + 1 \times 5 = 0 + 4 + 5 = 9$$

猫と犬は 30、猫と車は 9。大きい方が「似ている」だとすれば、私たちの直観と一致しています。ついでに犬と車も計算すると $0 + 3 + 10 = 13$ で、やはり猫と犬のペアが頭ひとつ抜けています。

この「掛けて、足す」が**内積(dot product)**です。形式化しておきましょう。$n$ 次元のベクトル $\mathbf{a} = (a_1, \ldots, a_n)$ と $\mathbf{b} = (b_1, \ldots, b_n)$ に対して、

$$\mathbf{a} \cdot \mathbf{b} = a_1 b_1 + a_2 b_2 + \cdots + a_n b_n = \sum_{i=1}^{n} a_i b_i$$

と書きます。記号は中黒の点「$\cdot$」。英語名 dot product の dot はこの点のことです。定義から、すぐに確認できることが3つあります。

1. **結果はスカラー**です。ベクトルとベクトルを入れると、ベクトルではなく**1つの数**が出てきます。「2つのベクトルから1つの数を作りたい」という冒頭の要求に、形の上ではぴったり合っています
2. **次元が同じベクトル同士でしか定義されません**。3次元と4次元では、対応する成分の組が作れないからです
3. **順序を入れ替えても同じ値**です: $\mathbf{a} \cdot \mathbf{b} = \mathbf{b} \cdot \mathbf{a}$。掛け算と足し算しか使っていないので、当然そうなります

ところで、なぜこの操作が「似ている」の検出になるのでしょうか。和の中身 $a_i b_i$ を1項ずつ見るとわかります。ある成分で**両方の値が大きい**と、その項は大きな正の貢献をします。猫と犬は「生き物らしさ」と「ふわふわ度」が揃って大きいので、$16 + 12$ という大口の貢献が積み上がりました。猫と車にはそういう成分がひとつもありません。さらに、成分に負の値を許すと、片方が正で片方が負——つまり**その性質について正反対**——の成分は、負の貢献として合計を引き下げます。内積とは、いわば全成分の一致・不一致の集計なのです。

これで「似ている」が数になった——と言いたいところですが、実はこの定義にはひとつ落とし穴があります。それを見つけるために、まず寄り道に見える道具を1つ用意します。ベクトルの「長さ」です。

## 2.2 ノルム(長さ)と単位ベクトル

内積は2つのベクトルから数を作る演算ですが、2つの引数に**同じベクトル**を入れたら何が出てくるでしょうか。

$$\mathbf{a} \cdot \mathbf{a} = a_1^2 + a_2^2 + \cdots + a_n^2$$

各成分の2乗の和です。これがどこかで見た形だと感じたら、2次元で確かめてみてください。$\mathbf{a} = (3, 4)$ なら $\mathbf{a} \cdot \mathbf{a} = 9 + 16 = 25$。原点から点 $(3, 4)$ までの距離は、三平方の定理で $\sqrt{3^2 + 4^2} = \sqrt{25} = 5$。つまり $\mathbf{a} \cdot \mathbf{a}$ の平方根は、矢印としてのベクトルの**長さ**そのものです。

この長さを**ノルム(norm)**と呼び、$\|\mathbf{a}\|$ と書きます。

$$\|\mathbf{a}\| = \sqrt{\mathbf{a} \cdot \mathbf{a}} = \sqrt{a_1^2 + \cdots + a_n^2}$$

3次元以上では矢印を直接描けませんが、定義はそのまま使えます。先ほどの単語ベクトルなら $\|\mathbf{猫}\| = \sqrt{16 + 16 + 1} = \sqrt{33} \approx 5.74$、$\|\mathbf{犬}\| = \sqrt{29} \approx 5.39$、$\|\mathbf{車}\| = \sqrt{26} \approx 5.10$ です。

もうひとつ、長さに関する用語を導入します。長さがちょうど1のベクトルを**単位ベクトル(unit vector)**と呼びます。どんなベクトルも(零ベクトルを除いて)、自分のノルムで割れば単位ベクトルになります:

$$\hat{\mathbf{a}} = \frac{\mathbf{a}}{\|\mathbf{a}\|}$$

割るといってもスカラー倍の一種で、$\frac{1}{\|\mathbf{a}\|}$ を掛けるだけです。この操作を**正規化(normalization)**と呼びます。長さの情報を捨てて、**向きの情報だけを残す**操作だと言えます。

なぜ長さを捨てたい場面があるのでしょうか。それが、先ほど予告した内積の落とし穴です。

## 2.3 角度とコサイン類似度 — 内積の幾何的意味

単語の動物園に、新しい仲間を入れてみます。

$$\mathbf{ゾウ} = (9,\ 1,\ 3)$$

生き物らしさは満点級、ふわふわはしておらず、走ればそこそこ速い。さて、内積で「猫に似ているのはどちらか」を判定してみましょう。

$$\mathbf{猫} \cdot \mathbf{犬} = 30, \qquad \mathbf{猫} \cdot \mathbf{ゾウ} = 36 + 4 + 3 = 43$$

内積の判定は「猫はゾウに似ている」。……本当でしょうか。猫と犬は3成分ともそこそこ揃っているのに、ゾウは「ふわふわ度」も「速さ」も猫とずれています。それでも勝ててしまったのは、ゾウの第1成分 9 が**飛び抜けて大きい**からです。実際、$\|\mathbf{ゾウ}\| = \sqrt{91} \approx 9.54$ で、他のベクトルの倍近い長さがあります。

ここに内積の正体が隠れています。内積は「向きの一致」だけでなく「ベクトルの大きさ」にも反応するのです。**長いベクトルは、向きが多少ずれていても内積を稼げてしまう。**

このことをきちんと言い当てるのが、内積のもうひとつの顔——幾何的な公式です。2つのベクトルを原点から伸びる矢印として描いたとき、その間の角度を $\theta$ とすると、

$$\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\| \, \|\mathbf{b}\| \cos\theta$$

が成り立ちます。「掛けて足す」というそろばん勘定と、「長さと角度」という幾何が、同じ値を指すのです。この等式の証明はしませんが(この本の流儀です)、2次元の例で本当に成り立つか検算しておきましょう。$\mathbf{a} = (1, 1)$、$\mathbf{b} = (2, 0)$ とします。定義どおりに計算すると $\mathbf{a} \cdot \mathbf{b} = 1 \times 2 + 1 \times 0 = 2$。一方、$\mathbf{a}$ は45度方向、$\mathbf{b}$ は真横なので $\theta = 45°$、長さは $\sqrt{2}$ と $2$。幾何の側は $\sqrt{2} \times 2 \times \cos 45° = \sqrt{2} \times 2 \times \frac{1}{\sqrt{2}} = 2$。一致しました。

この公式を読むと、内積は3つの量の積に分解できます。**両者の長さ × 向きの一致度($\cos\theta$)**。$\cos\theta$ の挙動を思い出しておくと:

- 向きが完全に一致($\theta = 0°$)なら $\cos\theta = 1$
- 直角($\theta = 90°$)なら $\cos\theta = 0$。このとき内積は、長さによらず必ず 0 です。**内積 0 = 直交**という対応は、今後も繰り返し使います
- 正反対($\theta = 180°$)なら $\cos\theta = -1$。内積は負になります

ゾウが勝った理由も、この分解で説明がつきます。$\cos\theta$ では負けていても、$\|\mathbf{ゾウ}\|$ の大きさで押し切ったのです。

では、長さの影響を取り除いて「向きの一致度」だけを取り出すには? 公式を $\cos\theta$ について解くだけです。

$$\cos\theta = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \, \|\mathbf{b}\|}$$

この量を**コサイン類似度(cosine similarity)**と呼びます。右辺をよく見ると、$\frac{\mathbf{a}}{\|\mathbf{a}\|} \cdot \frac{\mathbf{b}}{\|\mathbf{b}\|}$、つまり**正規化してから内積を取ったもの**に他なりません。前節の単位ベクトルは、このための道具でした。

動物園で再判定してみましょう。

$$\cos(\mathbf{猫}, \mathbf{犬}) = \frac{30}{\sqrt{33}\sqrt{29}} \approx 0.97, \qquad \cos(\mathbf{猫}, \mathbf{ゾウ}) = \frac{43}{\sqrt{33}\sqrt{91}} \approx 0.78, \qquad \cos(\mathbf{猫}, \mathbf{車}) \approx 0.31$$

猫に最も似ているのは犬。順位が直観どおりに戻りました。コサイン類似度は最大でも 1(向きが完全一致)なので、0.97 という値は「ほぼ同じ向き」と読めます。

## 2.4 内積 = 類似度スコアという直観

ここまでを一度、言葉で整理します。

- **内積** = 向きの一致度 × 両者の大きさ。「似ていて、かつ強い」に反応する
- **コサイン類似度** = 向きの一致度のみ。「似ている」だけを純粋に測る

どちらが正しいということはなく、用途によって使い分けます。大事なのは、これ以降この本では、内積を単なる計算手順ではなく**類似度スコア(similarity score)**として読む、という習慣です。$\mathbf{a} \cdot \mathbf{b}$ という式を見たら、「$\mathbf{a}$ と $\mathbf{b}$ の相性を採点している」と読んでください。

そして1行だけ予告しておきます。ラスボスの式に入っていた $QK^T$ ——あの正体は、**この「内積 = 類似度」の計算を大量に一斉に実行するもの**です。論文がこの仕組みを "dot-product attention"(内積による注意)と名付けている、その dot-product とは、いまあなたが手に入れた内積のことです。続きは第4章で。

## 2.5 [コード] forループで内積を手実装する

定義を本当に理解したかどうかの検算をしましょう。やり方はいつもどおり、自分の手で実装して、ライブラリの答えと突き合わせることです。

内積の定義は「掛けて、足す」でした。それをそのまま forループで書きます。

```python
# 第1巻 第2章 2.5: forループで内積を手実装し、np.dot と一致確認する
import numpy as np


def dot_loop(a, b):
    """forループによる内積。定義「掛けて、足す」をそのまま書いたもの。"""
    assert a.shape == b.shape  # 次元が違うベクトル同士の内積は定義されない
    total = 0.0
    for i in range(a.shape[0]):
        total += a[i] * b[i]
    return total


a = np.array([2.0, 1.0, 3.0])
b = np.array([1.0, 5.0, -2.0])

print("dot_loop :", dot_loop(a, b))   # 1.0
print("np.dot   :", np.dot(a, b))     # 1.0
assert np.allclose(dot_loop(a, b), np.dot(a, b))
print("ok: 一致しました")
```

いくつか補足します。まず、NumPy には内積がはじめから用意されていて、`np.dot(a, b)` で計算できます。関数名の dot は、もちろん dot product の dot です。自作の `dot_loop` と `np.dot` の答えが一致するか、`assert` で確認しています。等号 `==` ではなく `np.allclose` を使っているのは、小数の計算には微小な誤差がつきものだからです。「十分近ければ合格」とみなすこの比較は、本シリーズの全巻で使い続けます。冒頭の `assert a.shape == b.shape` は、定義の確認その2(次元が同じでないと内積は定義されない)をコードに翻訳したものです。

一致しました。では、自作版があるのだから `np.dot` は要らないのでは? ——そうはいかないことを、次の実験が教えてくれます。3次元では差がわからないので、思い切って**100万次元**のベクトルで勝負させてみましょう。スクリプト全体は次のとおりです(前半の `dot_loop` は再掲。`code/ch02/dot_product.py` として保存してあります)。

```python
# 第1巻 第2章 2.5: forループで内積を手実装 → np.dot と一致確認 → 速度比較
import time

import numpy as np


def dot_loop(a, b):
    """forループによる内積。定義「掛けて、足す」をそのまま書いたもの。"""
    assert a.shape == b.shape  # 次元が違うベクトル同士の内積は定義されない
    total = 0.0
    for i in range(a.shape[0]):
        total += a[i] * b[i]
    return total


# --- 一致確認(小さなベクトル) ---
a = np.array([2.0, 1.0, 3.0])
b = np.array([1.0, 5.0, -2.0])

result_loop = dot_loop(a, b)
result_np = np.dot(a, b)

print("dot_loop :", result_loop)   # 1.0
print("np.dot   :", result_np)     # 1.0
assert np.allclose(result_loop, result_np)

# --- 一致確認(大きな乱数ベクトル) ---
rng = np.random.default_rng(42)
n = 1_000_000
x = rng.standard_normal(n)
y = rng.standard_normal(n)

assert np.allclose(dot_loop(x, y), np.dot(x, y))

# --- 速度比較 ---
t0 = time.perf_counter()
dot_loop(x, y)
t1 = time.perf_counter()
np.dot(x, y)
t2 = time.perf_counter()

time_loop = t1 - t0
time_np = t2 - t1
print(f"forループ: {time_loop * 1000:.1f} ms")
print(f"np.dot  : {time_np * 1000:.3f} ms")
print(f"速度比  : 約 {time_loop / time_np:.0f} 倍")

print("ok: すべての assert を通過しました")
```

`rng = np.random.default_rng(42)` は乱数の種を固定する作法で、これも全巻共通です(誰が実行しても同じ乱数列になり、結果を再現できます)。`time.perf_counter()` は時刻を秒で返すので、処理の前後で差を取れば所要時間が測れます。

筆者の環境(ふつうのノートPC)での実測は、こうなりました。

```
forループ: 79.8 ms
np.dot  : 0.127 ms
速度比  : 約 627 倍
```

答えは同じ。なのに速度は**数百倍**違います(実行のたび・環境によって数字は揺れますが、桁はこのあたりに落ち着きます)。約80ミリ秒という時間は、1回なら一瞬です。しかし先の巻で私たちは、この種の計算を訓練ループの中で何百万回と繰り返すことになります。数百倍の差は、「数分で終わる実験」と「数日かかる実験」の差です。

なぜこれほど差がつくのでしょうか。Python の forループは、1周ごとにインタプリタが「次は何をするんだっけ」と確認しながら進む、律儀だが腰の重い仕組みです。100万周なら、その確認作業が100万回入ります。一方 `np.dot` は、「100万個まとめて掛けて足せ」という指示を一度だけ受け取り、内部の高速な実装(Cで書かれ、CPUの並列機能も使う)が一息に処理します。

ここから、本シリーズで守り続ける鉄則が生まれます。**ループを書きたくなったら、配列まるごとの演算で書けないか考える。**この書き換えを**ベクトル化(vectorization)**と呼びます。今日のは最初の体験にすぎません。第4章では行列積で同じ実験をして、もっと劇的な差を見ることになります。

## まとめ

- 内積は「対応する成分を掛けて、足す」: $\mathbf{a} \cdot \mathbf{b} = \sum_i a_i b_i$。結果はベクトルではなくスカラーで、同じ次元同士でしか定義されない
- 自分自身との内積の平方根がノルム(長さ): $\|\mathbf{a}\| = \sqrt{\mathbf{a} \cdot \mathbf{a}}$。ノルムで割れば長さ1の単位ベクトルになる(正規化)
- 内積の幾何的な顔: $\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\|\|\mathbf{b}\|\cos\theta$。内積は「向きの一致度 × 大きさ」に反応し、長さの影響を除いた $\cos\theta$ がコサイン類似度
- 以後、内積は**類似度スコア**として読む。これがラスボスの "dot-product attention" の dot-product
- forループと `np.dot` は同じ答えを返すが、速度は数百倍違う。ループより配列演算——ベクトル化の習慣はここから始まる

**ラスボスとの距離**: 式(1)の $QK^T$ のうち、「掛け合わせて類似度を測る」という心臓部の演算(dot-product)が読めるようになりました。残るは肩の $T$(第3章)と、大文字同士の掛け算の意味(第4章)です。

## 演習

**演習1** 次の3つのベクトルについて、すべてのペア(3通り)の内積を手計算し、最も「似ている」ペアを答えてください。負の内積になるペアがあれば、なぜ負になるのか成分を見て説明してください。

$$\mathbf{p} = (1,\ 2,\ 0), \qquad \mathbf{q} = (2,\ 1,\ 1), \qquad \mathbf{r} = (-1,\ 0,\ 3)$$

<details><summary>略解</summary>

$\mathbf{p} \cdot \mathbf{q} = 2 + 2 + 0 = 4$、$\mathbf{p} \cdot \mathbf{r} = -1 + 0 + 0 = -1$、$\mathbf{q} \cdot \mathbf{r} = -2 + 0 + 3 = 1$。最も似ているのは $\mathbf{p}$ と $\mathbf{q}$。$\mathbf{p} \cdot \mathbf{r}$ が負なのは、第1成分の符号が逆($1$ と $-1$)で負の貢献をし、それを打ち消す正の貢献が他の成分にないためです($\mathbf{p}$ の第3成分が 0 なので、$\mathbf{r}$ の大きな第3成分は集計に参加できません)。

</details>

**演習2** $\mathbf{a} = (3, 4)$ のノルムを求め、$\mathbf{a}$ を正規化して単位ベクトルにしてください。できた単位ベクトルのノルムが本当に 1 になることも確認してください。

<details><summary>略解</summary>

$\|\mathbf{a}\| = \sqrt{9 + 16} = 5$。正規化すると $\hat{\mathbf{a}} = (3/5,\ 4/5) = (0.6,\ 0.8)$。そのノルムは $\sqrt{0.36 + 0.64} = \sqrt{1} = 1$ で、確かに単位ベクトルです。

</details>

**演習3** $\mathbf{u} = (1, 1)$ に対して、$\mathbf{v} = (1, 0)$ と $\mathbf{w} = (4, -1)$ のどちらが「似ている」でしょうか。(a) 生の内積で判定した場合と、(b) コサイン類似度で判定した場合のそれぞれについて答え、結果が食い違う理由を説明してください。

<details><summary>略解</summary>

(a) $\mathbf{u} \cdot \mathbf{v} = 1$、$\mathbf{u} \cdot \mathbf{w} = 4 - 1 = 3$ で、内積では $\mathbf{w}$ の勝ち。(b) $\|\mathbf{u}\| = \sqrt{2}$、$\|\mathbf{v}\| = 1$、$\|\mathbf{w}\| = \sqrt{17}$ なので、$\cos(\mathbf{u}, \mathbf{v}) = \frac{1}{\sqrt{2}} \approx 0.71$、$\cos(\mathbf{u}, \mathbf{w}) = \frac{3}{\sqrt{2}\sqrt{17}} \approx 0.51$ で、コサイン類似度では $\mathbf{v}$ の勝ち。食い違う理由は $\mathbf{w}$ の長さです。向きの一致度では $\mathbf{v}$ に及ばないのに、$\|\mathbf{w}\| \approx 4.12$ という大きさで生の内積を稼いでいます(2.3節のゾウと同じ現象)。

</details>

**演習4** 次の4つの単語ベクトルの中から、最も「似ている」ペアを内積で見つけるコードを書いてください。すべてのペア(同じ対を2回数えない・自分自身は除く)の内積を表示し、最大のペアを答えとして出力すること。

```python

# 第1巻 第2章 演習4 略解: 最も「似ている」ベクトル対を内積で探す
import numpy as np

vectors = {
    "猫": np.array([4.0, 4.0, 1.0]),
    "犬": np.array([4.0, 3.0, 2.0]),
    "車": np.array([0.0, 1.0, 5.0]),
    "バス": np.array([0.0, 0.5, 4.0]),
}

names = list(vectors.keys())
best_pair = None
best_score = -np.inf

for i in range(len(names)):
    for j in range(i + 1, len(names)):  # 同じ対を2回数えない・自分自身は除く
        score = np.dot(vectors[names[i]], vectors[names[j]])
        print(f"{names[i]} ・ {names[j]} = {score}")
        if score > best_score:
            best_score = score
            best_pair = (names[i], names[j])

print("最も似ている対:", best_pair, "内積 =", best_score)
assert best_pair == ("猫", "犬")
assert np.allclose(best_score, 30.0)
```

最も似ている対は猫と犬(内積 30)です。ところで、このコードには forループが2重に入っています。ベクトルが4つなら6ペアで済みますが、1000個なら約50万ペア——書けなくはないものの、いかにも筋が悪そうです。この「内積の総当たり」を、ループなしのたった1つの演算で書く方法が第4章に待っています。
