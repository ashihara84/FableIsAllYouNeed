# 序章 ラスボスとの対面

## 0.1 このシリーズが目指すもの

最初に、はっきりさせておきたいことがあります。

この本は、機械学習の教科書ではありません。

機械学習の教科書なら、もっと良いものが世の中にたくさんあります。決定木、サポートベクターマシン、ベイズ推定、強化学習——機械学習という分野の地図を隅々まで描いてくれる本は、すでに揃っています。この本には、それらはひとつも出てきません。

この本は、**たった1本の論文を読むための本**です。

その論文は2017年に発表されました。タイトルは "Attention Is All You Need"。日本語にすれば「注意こそすべて」とでもなるでしょうか。たった15ページのこの論文が提案した **Transformer** というアーキテクチャは、その後の人工知能の歴史を一変させました。ChatGPT の「T」も、GPT の「T」も、Transformer の T です。あなたが今日どこかで AI と会話したなら、その向こう側で動いていたのは、ほぼ間違いなくこの論文の子孫です。

このシリーズ全8巻を終えたとき、あなたは次の2つができるようになっています。

1. この論文を、1行も飛ばさずに読める
2. 論文に書かれたものを、自分の手でゼロから実装し、訓練できる

そのために必要な数学とプログラミングを、**必要になった順に、必要な分だけ**学んでいきます。遠回りはしません。この論文に向かう最短経路だけを通ります。だから、機械学習の常識とされる話題でも、この論文に出てこないものは容赦なく省きます。網羅性は、この本の美徳ではありません。

## 0.2 ラスボスの掲示

では、その論文の中身を見てみましょう。これが、シリーズ全体を通してあなたが倒すことになる相手——いわばラスボスです。論文のセクション 3.2.1 に、こんな式が出てきます。

> $$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1, 式(1)

続くセクション 3.2.2 には、こんな式もあります。

> $$\mathrm{head}_i = \mathrm{Attention}(QW_i^Q,\ KW_i^K,\ VW_i^V)$$
> — 同論文, Section 3.2.2

どうでしょうか。

おそらく、何ひとつ読めないはずです。$Q$ とは何か。$K$ とは、$V$ とは。肩に乗っている小さな $T$ は何を意味するのか。$QK^T$ と2つの文字が並んでいるのは、掛け算なのか。だとしたら何と何の掛け算なのか。$\mathrm{softmax}$ という関数はどこから来たのか。なぜ $\sqrt{d_k}$ などという中途半端なもので割るのか。

読めなくて、まったく構いません。**今は読めない。それを確認するためにここに掲げました。**

そして約束します。この第1巻を終えたとき、あなたはこの式の **行列としての構造** が読めるようになっています。具体的には:

- $Q$、$K$、$V$、$W_i^Q$ が「行列」と呼ばれるものであること
- $K^T$ の $T$ が「転置」という操作であること
- $QK^T$ が「行列積」であり、その正体が **内積の総当たり表** であること
- それぞれの行列の形(shape)がどう繋がって、最後にどんな形が出てくるか

が、すらすら読めるようになります。

一方で、この巻が終わっても読めないものも、先に白状しておきます。$\mathrm{softmax}$ と $\sqrt{d_k}$ です。この2つには確率の道具が必要で、それは第4巻の仕事です。第1巻の終章で、もう一度この式の前に立ったとき、「読める部分」と「まだ読めない部分」の境界線を自分で引けるようになっていること——それがこの巻のゴールです。

各巻はすべて、この構成で進みます。巻の冒頭で論文の一部を掲げ、「今は読めない」と確認する。巻を通して道具を揃える。巻末で同じ箇所をもう一度読み、読めるようになったことを確認する。8巻めの終わりに、あなたは論文を最初から最後まで通読します。それがこのシリーズの卒業式です。

## 0.3 この巻でやること

第1巻のテーマは **線形代数(linear algebra)** です。

身構えなくて大丈夫です。この巻で学ぶことは、突き詰めればたった1行に集約されます。

```python
Y = X @ W + b
```

数を並べたもの(`X`)に、別の数の並び(`W`)を掛けて、さらに別の数の並び(`b`)を足す。これだけです。ニューラルネットワークと呼ばれるものの計算は——どれほど巨大なものでも——ほとんどがこの形の繰り返しでできています。ChatGPT が文章を生成しているとき、その内部で起きていることの大半は、この1行の、気が遠くなるほどの反復です。

この巻では、この1行を構成する部品——ベクトル、内積、行列、行列積、転置——を1つずつ手に入れていきます。各部品は、定義を覚えるだけでなく、**NumPy というライブラリを使って自分の手で実装し、動かして**確かめます。コードが書けるなら、その概念を理解している。逆に、コードに詰まるなら、まだどこかが曖昧だということです。コードは、この本では理解の検算機として働きます。

## 0.4 この巻で扱わないこと

線形代数の教科書を開いたことがある人は、こんな目次を見たことがあるかもしれません。行列式、逆行列、連立一次方程式、固有値と固有ベクトル、対角化——。

**この巻には、どれも出てきません。**

省く理由は単純で、Transformer の計算に登場しないからです。たとえば固有値は、線形代数という分野の宝石のような概念ですが、論文のどの式にも現れません。逆行列も同様です(第3巻で一度だけ「実は逆行列を使う近道もあるが、この道は通らない」という形で名前だけ登場します)。

これは「いつか必要になるかもしれないものを、念のため学んでおく」ことの放棄です。私たちは逆の原則を採ります。**必要になってから、必要なものだけを学ぶ。** この原則はシリーズ全体を貫いていて、巻の順番そのものもこの原則で決まっています(確率より先に機械学習をやるのは、そのためです)。安心してください。この巻で省いたものが原因で、後の巻で立ち往生することはないように設計してあります。

## 0.5 準備

必要なものは2つだけです。

- **Python 3**(3.9 以降)
- **NumPy**

```bash
pip install numpy
```

インストールできたか、確認しておきましょう。

```python
import numpy as np

a = np.array([1, 2, 3])
b = np.array([10, 20, 30])
print(a + b)   # [11 22 33]
```

`[11 22 33]` と表示されたら準備完了です。この `np.array` が何者なのかは、さっそく次の章で説明します。

数学の前提知識は、高校1年程度を想定しています。「ベクトル」「行列」という言葉を聞いたことがなくても構いません。すべてゼロから定義します。逆に、大学で線形代数を履修した人にとっても、この巻は「Transformer の文法としての線形代数」という読み直しになるはずです。shape の規律(第3章)は、履修済みの人ほど新鮮に感じるかもしれません。
