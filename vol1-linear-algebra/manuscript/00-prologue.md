## 0.1 このシリーズが目指すもの

この本は機械学習の教科書ではありません。決定木、サポートベクターマシン、ベイズ推定、強化学習——それらはひとつも出てきません。この本は、**たった1本の論文を読むための本**です。

その論文は2017年発表、タイトルは "Attention Is All You Need"(注意こそすべて)。たった15ページのこの論文が提案した **Transformer** というアーキテクチャが、その後の人工知能を一変させました。ChatGPT の「T」も GPT の「T」も Transformer の T です。あなたが今日どこかで AI と会話したなら、その向こうで動いていたのは、ほぼ間違いなくこの論文の子孫です。

全8巻を終えたとき、あなたは次の2つができるようになっています。

1. この論文を、1行も飛ばさずに読める
2. 論文に書かれたものを、自分の手でゼロから実装し、訓練できる

そのために必要な数学とプログラミングを、**必要になった順に、必要な分だけ**学びます。この論文に向かう最短経路だけを通り、論文に出てこない話題は、機械学習の常識とされるものでも省きます。網羅性は、この本の美徳ではありません。

## 0.2 ラスボスの掲示

シリーズ全体を通してあなたが倒す相手——ラスボスです。論文のセクション 3.2.1 に、こんな式が出てきます。

> $$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1, 式(1)

続くセクション 3.2.2 には、こんな式もあります。

> $$\mathrm{head}_i = \mathrm{Attention}(QW_i^Q,\ KW_i^K,\ VW_i^V)$$
> — 同論文, Section 3.2.2

おそらく何ひとつ読めないはずです。$Q$ とは、$K$ とは、$V$ とは何か。肩の小さな $T$ は何を意味するのか。$QK^T$ は掛け算なのか、だとしたら何と何の。$\mathrm{softmax}$ はどこから来たのか。なぜ $\sqrt{d_k}$ で割るのか。

**今は読めない。それを確認するためにここに掲げました。**

この第1巻を終えたとき、あなたはこの式の **行列としての構造** が読めるようになっています。具体的には:

- $Q$、$K$、$V$、$W_i^Q$ が「行列」と呼ばれるものであること
- $K^T$ の $T$ が「転置」という操作であること
- $QK^T$ が「行列積」であり、その正体が **内積の総当たり表** であること
- それぞれの行列の形(shape)がどう繋がり、最後にどんな形が出てくるか

この巻が終わっても読めないものも先に白状します。$\mathrm{softmax}$ と $\sqrt{d_k}$ です。この2つには確率の道具が必要で、それは第4巻の仕事です。終章でもう一度この式の前に立ったとき、「読める部分」と「まだ読めない部分」の境界線を自分で引けること——それがこの巻のゴールです。

各巻はすべてこの構成です。冒頭で論文の一部を掲げ「今は読めない」と確認し、巻を通して道具を揃え、巻末で同じ箇所を読み直す。8巻めの終わりに、あなたは論文を最初から最後まで通読します。それがこのシリーズの卒業式です。

## 0.3 この巻でやること

第1巻のテーマは **線形代数(linear algebra)** です。学ぶことは、突き詰めればたった1行に集約されます。

```python
Y = X @ W + b
```

数を並べたもの(`X`)に、別の数の並び(`W`)を掛けて、さらに別の並び(`b`)を足す。これだけです。ニューラルネットワークの計算は、どれほど巨大でも、ほとんどがこの形の繰り返しでできています。ChatGPT が文章を生成しているとき、内部で起きていることの大半は、この1行の気の遠くなるような反復です。

この巻では、この1行を構成する部品——ベクトル、内積、行列、行列積、転置——を1つずつ手に入れます。各部品は定義を覚えるだけでなく、**NumPy というライブラリで自分の手で実装し、動かして**確かめます。コードが書けるなら理解できている。コードに詰まるなら、まだどこかが曖昧だということ。この本でコードは理解の検算機として働きます。

## 0.4 この巻で扱わないこと

線形代数の教科書でおなじみの目次——行列式、逆行列、連立一次方程式、固有値と固有ベクトル、対角化。

**この巻には、どれも出てきません。**

Transformer の計算に登場しないからです。固有値は線形代数の宝石のような概念ですが、論文のどの式にも現れません。逆行列も同様です(第3巻で一度だけ「逆行列を使う近道もあるが、この道は通らない」という形で名前だけ登場します)。

私たちは **必要になってから、必要なものだけを学ぶ**。この原則がシリーズ全体を貫き、巻の順番そのものも決めています(確率より先に機械学習をやるのはこのためです)。この巻で省いたものが原因で後の巻で立ち往生しないよう設計してあります。

なお、省くのは「学ぶ価値がないから」ではありません。固有値もベイズ統計も、それぞれの分野では主役級の道具です。興味が湧いたものは、いつでも寄り道して学んでください——この本が約束するのは「寄り道しなくても論文まで届く」ことであって、寄り道の禁止ではありません。通らなかった道の話には、第8巻の卒業式でもう一度戻ります。

## 0.5 準備

必要なものは2つだけです。

- **Python 3**(3.9 以降)
- **NumPy**

```bash
pip install numpy
```

インストールできたか確認しておきましょう。

```python
import numpy as np

a = np.array([1, 2, 3])
b = np.array([10, 20, 30])
print(a + b)   # [11 22 33]
```

`[11 22 33]` と表示されたら準備完了です。この `np.array` が何者なのかは、次の章で説明します。

プログラミングの前提は、Python の基本文法(変数・リスト・関数・for 文)が読み書きできることです。文法そのものはこの本では解説しません。数学の前提は高校1年程度を想定しています。「ベクトル」「行列」を聞いたことがなくても構いません。すべてゼロから定義します。大学で線形代数を履修した人にとっても、この巻は「Transformer の文法としての線形代数」という読み直しになるはずで、shape の規律(第3章)は履修済みの人ほど新鮮かもしれません。
