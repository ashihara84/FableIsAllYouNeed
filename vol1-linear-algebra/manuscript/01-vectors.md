ただし、いきなり定義から始めるのはやめておきます。その手前に、もっと根本的な謎があるからです。

"Attention Is All You Need" は、機械翻訳の論文です。入力は英語の文、出力はドイツ語の文。つまりこの論文は、最初から最後まで**言葉**を相手にしています。ところが、コンピュータが計算できるのは**数**だけです。足し算と掛け算の塊であるコンピュータに、「言葉の意味」をどうやって渡すのか。この章は、その無茶な問いから始めます。

## 1.1 「単語の意味を数で表す」という無茶な発想

まず、素朴な案から検討しましょう。単語に番号を振ればよいのではないでしょうか。

辞書の順に、cat は 1番、dog は 2番、elephant は 3番、table は 4番——。これで確かに、すべての単語が数になりました。コンピュータに渡せます。

しかし、この番号には致命的な欠陥があります。**数の値に、意味が何も乗っていない**のです。cat(1番)と dog(2番)の差は 1。dog(2番)と elephant(3番)の差も 1。では「cat と dog の近さ」と「dog と elephant の近さ」は同じでしょうか。猫と犬はどちらも身近なペットですが、犬と象はだいぶ違う生き物です。番号の引き算は、そういう感覚を何ひとつ反映しません。番号は「区別」はできても「意味」を運べないのです。

では、どうするか。ここで発想を転換します。

**1つの数で表そうとするから無理が出る。たくさんの数の組で表せばいい。**

たとえば、単語を「採点表」にしてみましょう。採点項目を3つ決めます。「生き物らしさ」「大きさ」「凶暴さ」。それぞれ 0 から 1 の数で採点します。

| 単語 | 生き物らしさ | 大きさ | 凶暴さ |
|---|---|---|---|
| cat(猫) | 0.9 | 0.2 | 0.3 |
| tiger(虎) | 0.9 | 0.7 | 0.9 |
| table(机) | 0.0 | 0.5 | 0.0 |

cat は $(0.9,\ 0.2,\ 0.3)$、tiger は $(0.9,\ 0.7,\ 0.9)$、table は $(0.0,\ 0.5,\ 0.0)$。それぞれの単語が「3つの数の並び」になりました。

そして今度は、数に意味が乗っています。cat と tiger は、1つめの成分(生き物らしさ)がどちらも 0.9 で揃っています。一方 table は 0.0 で、2つと大きく離れています。「猫と虎は似ていて、机は仲間はずれ」という私たちの感覚が、数の並び方の中に現れているのです。単語の意味を、**数の並びの中の「位置」**として表す——これがこの章の、そしてこのシリーズ全体を支える作戦です。

実際の論文では、この「採点項目」がいくつあるのでしょうか。論文のセクション 3.1 と 3.4 に答えが書いてあります。**512個**です。論文ではこの 512 という数に $d_{model}$ という名前が付いています。Transformer に入力される時点で、1つの単語は 512 個の数の並びに変換されているのです。

もっとも、論文の 512 個の数は、私たちが今やったように人間が項目を決めて採点したものではありません。「生き物らしさ」のようなラベルも付いていません。あの数たちは**学習によって機械が自分で見つけた**ものです。どうやって見つけるのかは、ずっと先、第6巻の主題です。今の段階では、「単語 = 数の並び」という**形式**だけを受け取っておけば十分です。

さて、「数の並び」「数の並び」と繰り返してきました。そろそろ、この主役に正式な名前を与えましょう。数学では、これを**ベクトル**と呼びます。

## 1.2 ベクトルの定義と記法

**ベクトル(vector)**とは、数を順番に並べたものです。

$$\mathbf{v} = (v_1,\ v_2,\ \dots,\ v_n)$$

並んでいる1つ1つの数 $v_1, v_2, \dots$ を**成分(component)**と呼び、成分の個数 $n$ を**次元(dimension)**と呼びます。たとえば先ほどの cat のベクトル $(0.9,\ 0.2,\ 0.3)$ は、成分が3つなので3次元ベクトルです。

定義の中の「順番に」は飾りではありません。**並び順を入れ替えたら、別のベクトル**です。$(0.2,\ 0.9)$ と $(0.9,\ 0.2)$ は違うベクトルです。採点表の例で考えれば当然で、「大きさ 0.2・凶暴さ 0.9」の生き物と「大きさ 0.9・凶暴さ 0.2」の生き物はまるで別物です。何番目の成分か、には意味が宿っています。

記法をいくつか約束しておきます。このシリーズ全体で使い続ける約束です。

- ベクトルは**太字の小文字**で書きます: $\mathbf{v}$, $\mathbf{x}$, $\mathbf{a}$。教科書によっては $\vec{v}$ と矢印を乗せる流儀もありますが、本シリーズは太字で通します
- ベクトルでないただの1つの数(たとえば 3 や 0.5)は、細字の小文字で書きます: $a$, $c$。こうした「ただの数」を、次の節で**スカラー(scalar)**と呼ぶことになります
- 実数(real number)の全体を $\mathbb{R}$ と書きます。そして、**実数を $n$ 個並べたベクトルの全体**を $\mathbb{R}^n$ と書きます

最後の記号を使うと、「$\mathbf{v}$ は3次元ベクトルである」を

$$\mathbf{v} \in \mathbb{R}^3$$

と書けます。$\in$ は「〜の集まりの一員である」という記号です。読み下せば「$\mathbf{v}$ は、実数3つ組の世界 $\mathbb{R}^3$ の住人である」。論文の単語ベクトルなら、こう書けます。

$$\mathbf{x} \in \mathbb{R}^{512} \quad (= \mathbb{R}^{d_{model}})$$

ここで、512次元と聞いて身構えた人のために、1つ言っておきたいことがあります。

2次元のベクトルは、平面上の点(あるいは矢印)として絵に描けます。3次元なら空間の点です。では512次元は? ——描けません。誰にも描けませんし、思い浮かべられる人もいません。しかし、それで困ることは何もありません。**次元とは「成分の個数」のことだ**、と機械的に捉えてください。512次元ベクトルとは「数が512個並んだもの」、それ以上でも以下でもありません。これから定義する計算のルールは、2次元でも512次元でもまったく同じ形をしています。私たちは2次元の絵で直観を作り、その直観を512次元にそのまま輸出する、という方法で進みます。

なお、ベクトルの書き方には、成分を横に並べる流儀と縦に並べる流儀があります。この区別は行列が登場すると俄然重要になるのですが、それは第3章の仕事です。この章では、横に並べて書いておきます。

## 1.3 足し算とスカラー倍 — 幾何的には矢印の継ぎ足しと伸縮

単語をベクトルにしたのは、眺めるためではありません。**計算するため**です。では、ベクトルの「計算」とは何でしょうか。

この節では、もっとも基本の2つだけを定義します。足し算と、スカラー倍です。実は、この章の最後に見せる「意味の算術」は、この2つだけでできています。

### 足し算

2つのベクトルの足し算は、**同じ位置の成分どうしを足す**ことと定めます。

$$(a_1,\ a_2) + (b_1,\ b_2) = (a_1 + b_1,\ a_2 + b_2)$$

たとえば $\mathbf{a} = (2,\ 1)$、$\mathbf{b} = (1,\ 3)$ なら、

$$\mathbf{a} + \mathbf{b} = (2+1,\ 1+3) = (3,\ 4)$$

定義から明らかに、**足せるのは次元が同じベクトルどうしだけ**です。3次元と2次元は足せません。何番目の成分どうしを足すのか、対応が付かないからです。「形が合うものしか演算できない」——この小さな注意は、第3章で **shape の規律**という、シリーズ全体を貫く習慣に育ちます。

この足し算は、絵に描くと気持ちのよい意味を持ちます。2次元ベクトル $(2,\ 1)$ を、原点から点 $(2,\ 1)$ へ向かう**矢印**だと思ってください。すると $\mathbf{a} + \mathbf{b}$ は、「まず矢印 $\mathbf{a}$ をたどり、その先端から続けて矢印 $\mathbf{b}$ をたどった到達点」になります。矢印の**継ぎ足し**です。

```text
 x2(第2成分)
  4 |              ◎ a+b = (3, 4)
    |             /
  3 |            /
    |           /   ← a の先端から b = (1, 3) を継ぎ足す
  2 |          /
    |         /
  1 |        ● a = (2, 1)
    |   __--
  0 +--------------------- x1(第1成分)
    0    1    2    3
```

図1.1: ベクトルの足し算は矢印の継ぎ足し。原点から $\mathbf{a}=(2,1)$ へ進み、そこから $\mathbf{b}=(1,3)$ ぶん進むと、$\mathbf{a}+\mathbf{b}=(3,4)$ に到達する。

「成分ごとに足す」という事務的な定義と、「矢印を継ぎ足す」という幾何的な像。同じ1つの操作の、2つの顔です。横方向の移動量どうし($2+1$)、縦方向の移動量どうし($1+3$)を別々に合算している、と考えれば、2つの顔が同じものだと納得できるはずです。

なお、足す順番は結果に影響しません: $\mathbf{a} + \mathbf{b} = \mathbf{b} + \mathbf{a}$。成分ごとの足し算がただの数の足し算である以上、当然そうなります。当たり前に見えるこの性質を、わざわざ書き留めたのには理由があります。第4章で登場する行列積では、**この当たり前が成り立たない**のです。今のうちに「演算の順序は、いつでも入れ替えてよいとは限らない」という警戒心の置き場所だけ作っておいてください。

### スカラー倍

もう1つの演算は、ベクトルを「ただの数」で掛け算することです。ベクトルの世界では、ただの1つの数を**スカラー(scalar)**と呼びます。スカラー $c$ をベクトルに掛けるには、**全成分を一斉に $c$ 倍**します。

$$c \cdot (v_1,\ v_2) = (c\, v_1,\ c\, v_2)$$

たとえば $3 \cdot (2,\ 1) = (6,\ 3)$。

幾何的には、これは矢印の**伸縮**です。$3$ 倍すれば、向きはそのままで長さが3倍に伸びます。$0.5$ 倍なら半分に縮みます。そして $-1$ 倍は、長さを変えずに**向きを正反対にひっくり返し**ます。

$-1$ 倍が手に入ると、引き算はもう新しい演算ではなくなります。

$$\mathbf{a} - \mathbf{b} = \mathbf{a} + (-1)\,\mathbf{b}$$

つまり「$\mathbf{b}$ を逆向きにして継ぎ足す」だけです。成分で書けば、同じ位置の成分どうしの引き算になります。$(3,\ 4) - (1,\ 3) = (2,\ 1)$。

ベクトルの演算は、当面これで全部です。「ベクトルどうしの掛け算はないのか」と思った人は、よい勘をしています。あります。しかも、それこそがこのシリーズの心臓部なのですが——例によって、**困ってから**導入します(第2章)。この章の残りでは、いま定義した演算を、紙の上ではなくコンピュータの上で動かします。

## 1.4 [コード] NumPy入門 — list と ndarray の違い

ここまでの話を、そのまま Python に持ち込んでみましょう。Python で「数の並び」といえば、まず list です。1.3 の足し算を list でやってみます。

```python
x_list = [1.0, 2.0, 3.0]
y_list = [10.0, 20.0, 30.0]

print(x_list + y_list)   # [1.0, 2.0, 3.0, 10.0, 20.0, 30.0]
print(x_list * 2)        # [1.0, 2.0, 3.0, 1.0, 2.0, 3.0]
```

期待した $(11,\ 22,\ 33)$ は出てきません。`+` は2つの list を**連結**し、`* 2` は list を**2回繰り返し**ます。Python の list は「ものの並び」であって、数学のベクトルではないのです。文字列でも辞書でも何でも並べられる入れ物ですから、「成分ごとに足す」という数学の約束を知らないのは当然とも言えます。

ここで登場するのが **NumPy** です。NumPy が提供する **ndarray**(n-dimensional array、n次元配列)は、まさに「数学のベクトルとして振る舞う数の並び」です。`np.array` に list を渡すと ndarray が作れます。

```python
import numpy as np

x = np.array([1.0, 2.0, 3.0])
y = np.array([10.0, 20.0, 30.0])

print(x + y)    # [11. 22. 33.]
print(2 * x)    # [2. 4. 6.]
print(-1 * x)   # [-1. -2. -3.]
print(x - y)    # [ -9. -18. -27.]
```

`+` は成分ごとの足し算、スカラー `2` との `*` は全成分の2倍。1.3 で定義したとおりの演算が、定義したとおりの記号で動きます。引き算も成分ごとです。

ndarray には、自分の「形」を答える `shape` という属性があります。

```python
print(x.shape)   # (3,)
```

`(3,)` は「成分が3つ」、つまり3次元ベクトルだという意味です(末尾のカンマは「数が1つだけ入った tuple」という Python の記法で、深い意味はありません)。今はただの長さ確認にしか見えませんが、この `shape` こそ、第3章以降この本を最後まで貫く背骨になります。**わからなくなったら shape を見る**——この習慣の最初の一歩です。

1つだけ、先回りして注意しておきます。ndarray どうしの `*` も動いてしまいます。

```python
print(x * y)   # [ 10.  40.  90.]
```

これは成分ごとの掛け算です。それ自体は正当な演算で、後の巻でも使うのですが、**第2章で学ぶ「内積」とは別物**です。「ベクトルの掛け算」と聞いて多くの文脈で意図されるのは内積の方なので、`*` を見たら「ああ、成分ごとの方だな」と区別する癖をつけてください。

以上を1本のプログラムにまとめ、`assert` による検算を付けたものがこちらです。`assert` は「条件が成り立たなければその場で停止する」文で、本シリーズでは「実行して何も起きなければ、本文の主張はすべて正しい」という検算機として全巻で使います。数値の比較には `np.allclose`(全成分がほぼ等しいか)を使います。

```python
# code/ch01/numpy_intro.py
import numpy as np

# --- list は「数学のベクトル」ではない ---
x_list = [1.0, 2.0, 3.0]
y_list = [10.0, 20.0, 30.0]

assert x_list + y_list == [1.0, 2.0, 3.0, 10.0, 20.0, 30.0]  # 連結される
assert x_list * 2 == [1.0, 2.0, 3.0, 1.0, 2.0, 3.0]          # 繰り返される

# --- ndarray は成分ごとに演算する ---
x = np.array([1.0, 2.0, 3.0])
y = np.array([10.0, 20.0, 30.0])

assert np.allclose(x + y, [11.0, 22.0, 33.0])    # 足し算(1.3)
assert np.allclose(2 * x, [2.0, 4.0, 6.0])       # スカラー倍(1.3)
assert np.allclose(-1 * x, [-1.0, -2.0, -3.0])   # 反転
assert np.allclose(x - y, x + (-1) * y)          # 引き算 = (-1)倍の足し算

# 足す順番は入れ替えてよい(第4章の行列積では成り立たない)
assert np.allclose(x + y, y + x)

# --- shape: 配列の形 ---
assert x.shape == (3,)            # 3次元ベクトル

v = np.zeros(512)                 # 全成分 0 の 512次元ベクトル
assert v.shape == (512,)          # 論文の単語ベクトルと同じ次元

# --- 1.3 の手計算例の検算 ---
a = np.array([2.0, 1.0])
b = np.array([1.0, 3.0])
assert np.allclose(a + b, [3.0, 4.0])
assert np.allclose(3 * a, [6.0, 3.0])
assert np.allclose((a + b) - b, a)   # b を足して b を引けば元に戻る

# --- 要素ごとの積(内積ではない。内積は第2章) ---
assert np.allclose(x * y, [10.0, 40.0, 90.0])

print("第1章: すべての assert を通過しました")
```

途中に出てくる `np.zeros(512)` は、全成分が 0 の512次元ベクトルを作ります。中身はまだ空っぽですが、shape は論文の単語ベクトルと同じです——あなたはもう、$\mathbb{R}^{512}$ の住人をコードで作れるのです。

## 1.5 予告: king − man + woman ≈ queen

章の締めくくりに、この先の旅で出会う光景を1つだけ、先にお見せします。

2013年、word2vec と呼ばれる手法が発表されました。大量の文章を機械に読ませて、単語ごとのベクトルを学習させる手法です(「学習させる」が何を意味するかは、第2〜3巻でじっくりやります)。そうして得られたベクトルたちが、奇妙な性質を示しました。king(王)のベクトルから man(男)のベクトルを引き、woman(女)のベクトルを足すと——

$$\mathbf{v}_{king} - \mathbf{v}_{man} + \mathbf{v}_{woman} \approx \mathbf{v}_{queen}$$

**queen(女王)のベクトルのすぐ近くに着地した**のです。「王から男らしさを引いて、女らしさを足すと、女王になる」。誰も教えていないのに、単語の意味の関係が、ベクトルの足し算と引き算——まさにこの章の 1.3 で定義した、あの2つの演算——で計算できてしまった。**意味の算術**です。

なぜそんなことが起きうるのか、雰囲気だけなら、手作りの採点表でも確かめられます。今度は採点項目を「王族度・男性度・女性度・人間度」の4つにして、4つの単語を採点してみます。

| 単語 | 王族度 | 男性度 | 女性度 | 人間度 |
|---|---|---|---|---|
| king | 0.9 | 0.9 | 0.1 | 1.0 |
| man | 0.1 | 0.9 | 0.1 | 1.0 |
| woman | 0.1 | 0.1 | 0.9 | 1.0 |
| queen | 0.9 | 0.1 | 0.9 | 1.0 |

$\mathbf{v}_{king} - \mathbf{v}_{man} + \mathbf{v}_{woman}$ を成分ごとに計算すると、

$$(0.9-0.1+0.1,\ \ 0.9-0.9+0.1,\ \ 0.1-0.1+0.9,\ \ 1.0-1.0+1.0) = (0.9,\ 0.1,\ 0.9,\ 1.0)$$

queen の行と、ぴったり一致します。king − man という引き算が「王族度 +0.8、男性度 0」という**差分**(王と男の違い、すなわち「王族であること」)を取り出し、それを woman に継ぎ足した、と読むこともできます。意味が成分に分かれて宿っているなら、意味の操作が成分の算術になる——作戦どおりです。

ただし、白状しておくべきことが2つあります。

1つ。この採点表は、答えから逆算して私たちが手で作ったものです。word2vec や Transformer のベクトルは、誰も項目を設計していないのに、学習だけからこれに似た構造を獲得します。**どうやってそんなベクトルを手に入れるのか**——この伏線は、**第6巻第3章で回収します**。そこまでの5巻ぶんの道具が、文字どおりすべて必要になります。

2つ。冒頭の式の記号は $=$ ではなく $\approx$(ほぼ等しい)でした。計算結果は queen のベクトルに「一番近かった」のであって、一致したわけではありません。すると、こう問わざるを得ません。ベクトルとベクトルが「近い」「似ている」とは、何を計算することなのか?

私たちはまだ、この問いに答える道具を持っていません。困りました。——困ったので、次の章で導入します。それが**内積**、ラスボス $QK^T$ の心臓部です。

## まとめ

- 単語に1つの番号を振っても意味は運べない。**たくさんの数の組の中の「位置」**として表すのが、この論文(と現代のAI)の作戦である。論文では1単語 = 512個の数($d_{model} = 512$)
- **ベクトル**は数を順番に並べたもの。1つ1つの数が**成分**、成分の個数が**次元**。$n$ 次元ベクトルの全体を $\mathbb{R}^n$ と書き、ベクトルは太字小文字 $\mathbf{v}$ で表す
- 足し算は**成分ごと**(幾何的には矢印の継ぎ足し)、スカラー倍は**全成分を一斉に伸縮**(負なら反転)。引き算は $(-1)$ 倍の足し算にすぎない
- Python の list は連結する「ものの並び」、NumPy の **ndarray** は成分ごとに演算する「数学のベクトル」。形は `shape` で確かめる
- king − man + woman ≈ queen という**意味の算術**は、この章の演算だけで書ける。ベクトルの入手法は第6巻第3章で、「≈(近い)」の測り方は次章で

**ラスボスとの距離**: 論文の入力——「1単語 = $d_{model} = 512$ 次元のベクトル」——が読めるようになりました。$Q$, $K$, $V$ は、このベクトルたちから作られます。

**問1(手計算)** $\mathbf{a} = (1,\ 2)$、$\mathbf{b} = (3,\ -1)$ とする。次を計算せよ。
(1) $\mathbf{a} + \mathbf{b}$ (2) $2\mathbf{a}$ (3) $\mathbf{a} - 2\mathbf{b}$ (4) ベクトル $(5,\ 0,\ 2,\ -1)$ の次元

**問4(意味の算術)** 1.5 の採点表の4つのベクトルを NumPy で定義し、`king - man + woman` が `queen` と一致することを `np.allclose` で確かめよ。また、4つめの成分(人間度)が計算の前後で 1.0 のまま動かない理由を、引き算の意味から説明せよ。

<details><summary>略解</summary>

```python
import numpy as np
king  = np.array([0.9, 0.9, 0.1, 1.0])
man   = np.array([0.1, 0.9, 0.1, 1.0])
woman = np.array([0.1, 0.1, 0.9, 1.0])
queen = np.array([0.9, 0.1, 0.9, 1.0])
assert np.allclose(king - man + woman, queen)
```

人間度は king も man も 1.0 なので、king − man の時点で差分が 0 になる(王と男は「人間であること」については違いがない)。そこに woman の人間度 1.0 が足されるだけなので、結果も 1.0 のまま。king − man という引き算は「2語の意味の**違い**だけ」を取り出している。

</details>
