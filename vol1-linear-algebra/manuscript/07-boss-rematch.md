# 終章 ラスボス再戦 — QK^T が読める

> [目次](../TOC.md) ・ [← 前の章](06-batch-and-linear.md) ・ [次の章 →](../../vol2-calculus/manuscript/00-prologue.md)

第6章までで、この巻の部品はすべて揃いました。ベクトル(第1章)、内積(第2章)、行列と転置(第3章)、行列積(第4章)、線形変換としての $W$(第5章)、そして `X @ W + b` という背骨(第6章)です。

序章で私たちは、この巻を終えたとき論文の式の**行列としての構造**が読めるようになっている、と約束しました。やり方は予告どおり、**巻頭とまったく同じ引用をもう一度掲げ、今度は1記号ずつ読んでいく**。全8巻、毎巻の巻末で繰り返す儀式の第1回です。

## 7.1 式(1)を読む — 全記号に shape を付けて

序章 0.2 に掲げた式を、一言一句変えずに再掲します。

> $$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
> — Vaswani et al., "Attention Is All You Need", Section 3.2.1, 式(1)

序章では、$Q$ が何者かも、肩の $T$ の意味も、$QK^T$ が掛け算かどうかすらわかりませんでした。いまは左から、各記号に第3章の shape 注釈を付けて読みます。

**$Q$ `(n, d_k)`** — 第3章の言葉では「行ベクトルの束」。$d_k$ 次元のベクトルが $n$ 本、行として積んであります。$n$ 本の「問い合わせ(query)」だから $Q$。名前だけ先取りし、中身にはまだ立ち入りません。

**$K$ `(m, d_k)`** — 「鍵(key)」が $m$ 本。$Q$ と列数(次元)が同じ $d_k$ である点が後で効きます。

**$K^T$ `(d_k, m)`** — 肩の $T$ は第3章の**転置**。行と列が入れ替わり、shape は `(m, d_k)` から `(d_k, m)` へ。序章で正体不明だった $T$ が、もう作業指示として読めます。

**$QK^T$ `(n, m)`** — 行列を2つ並べたら**行列積**(第4章)。「$(n, d_k)$ かける $(d_k, m)$、内側の $d_k$ が一致、結果は $(n, m)$」。$Q$ と $K$ を $d_k$ で揃えたのは、この積を成立させるためでした。中身も読めます。$(i, j)$ 成分は左の行 $i$ と右の列 $j$ の内積(第4章 4.1)で、$K^T$ の列 $j$ は転置前の $K$ の行 $j$ ですから、

$$ (QK^T)[i, j] = (Q \text{ の行 } i) \cdot (K \text{ の行 } j) $$

内積は第2章で確かめたとおり**「似ている」を数にしたもの**——類似度スコアです。つまり $QK^T$ とは、**$n$ 本の query と $m$ 本の key の全ペアの内積を一斉に計算した、類似度の総当たり表 `(n, m)`** です。

——ここで種明かしを。第4章 4.6 で、単語ベクトルを行に積んだ $X$ に `X @ X.T` を計算し、各マスが「単語ペアの類似度」になることを観察しました。**あれは $Q$ と $K$ がどちらも同じ $X$ のときの $QK^T$——attention のスコア計算そのもの**でした。単語が自分自身の仲間と類似度を取り合う形なので、これを**自己注意(self-attention)**と呼びます。Transformer の心臓部を、あなたは名前を知る前に手で計算していたのです。

残りは shape だけなら流せます。$\sqrt{d_k}$ で割るのはスカラー倍(第1章)で shape は `(n, m)` のまま。$\mathrm{softmax}$ の中身はまだ読めませんが、「shape を変えない関数」と先取りすれば出力も `(n, m)`。最後の **$V$ `(m, d_v)`**(「値(value)」が $m$ 本)との積は「$(n, m)$ かける $(m, d_v)$、内側の $m$ が一致、結果は $(n, d_v)$」。

$$ \underbrace{\mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)}_{(n,\ m)} \underbrace{V}_{(m,\ d_v)} \;\to\; \underbrace{\mathrm{Attention}(Q, K, V)}_{(n,\ d_v)} $$

入口から出口まで shape が一度も詰まらず流れました。序章では1文字も読めなかった式が、いまは**行列の配管図**として端から端まで読めます。

## 7.2 3.2.2 の射影を読む — $XW^Q$ は第5章の「役割への変換」

もう1つの引用も、序章とまったく同じものを再掲します。

> $$\mathrm{head}_i = \mathrm{Attention}(QW_i^Q,\ KW_i^K,\ VW_i^V)$$
> — 同論文, Section 3.2.2

ここでの $Q, K, V$ は射影前の入力で、shape は `(n, d_model)` などです(論文では $d_{model} = 512$)。読みどころは $W_i^Q$ `(d_model, d_k)`——第5章で導入した**「あとで中身を調整できる変換」としての $W$** です。第5章 5.5 で、同じ入力 $X$ を $XW^Q$, $XW^K$, $XW^V$ と3つの役割に変換する形を shape だけ読みましたが、この式はまさにそれです。

$$ \underbrace{Q}_{(n,\ d_{model})} \underbrace{W_i^Q}_{(d_{model},\ d_k)} \;\to\; \underbrace{QW_i^Q}_{(n,\ d_k)} $$

512次元のベクトルが $W_i^Q$ を通って $d_k$ 次元(論文では64次元)の「query という役割」に姿を変え、その結果が 7.1 の式(1)の $Q$ の席に座る。2式が shape で連結されました。

添字の $i$ はヘッド(head)の番号です。同じ形の変換を中身の違う $W_i$ で複数セット並走させる——それが multi-head の「multi」ですが、なぜ複数必要かは守備範囲外(第7巻)。いまは「$i$ 番ごとに別の $W$ がある」と読めれば十分です。

ここまでの読みが正しいか、理解の検算機としてのコードで確かめます。この巻最後の出番です。

```python
import numpy as np

rng = np.random.default_rng(42)

n, m = 4, 6            # query 4本、key/value 6本
d_model, d_k, d_v = 512, 64, 64

X_q = rng.standard_normal((n, d_model))   # 射影前の入力(query側)
X_kv = rng.standard_normal((m, d_model))  # 射影前の入力(key/value側)
W_Q = rng.standard_normal((d_model, d_k))
W_K = rng.standard_normal((d_model, d_k))
W_V = rng.standard_normal((d_model, d_v))

Q = X_q @ W_Q          # (n, d_model) @ (d_model, d_k) -> (n, d_k)
K = X_kv @ W_K         # (m, d_model) @ (d_model, d_k) -> (m, d_k)
V = X_kv @ W_V         # (m, d_model) @ (d_model, d_v) -> (m, d_v)

scores = Q @ K.T / np.sqrt(d_k)           # (n, d_k) @ (d_k, m) -> (n, m)

assert Q.shape == (n, d_k)
assert K.shape == (m, d_k)
assert scores.shape == (n, m)             # 類似度の総当たり表
assert (scores @ V).shape == (n, d_v)     # softmax は省略(shape を変えないため)
print("式(1)の配管、全部通りました")
```

`softmax` だけ素通しですが、shape の流れは論文の式と寸分違わず一致します。**この assert が全部通ること——それが「行列の部分が読める」ことの、実行可能な定義です。**

## 7.3 まだ読めないものの棚卸し

序章 0.2 で、この巻のゴールを「読める部分」と「まだ読めない部分」の**境界線を自分で引けるようになること**と言いました。読める部分は確認したので、次は向こう側をごまかさず棚卸しします。

| まだ読めないもの | 何が足りないか | どの巻で読めるようになるか |
|---|---|---|
| $\mathrm{softmax}$ | 数の並びを「確率分布」に変える、という考え方 | 第4巻 |
| $\sqrt{d_k}$ で割る理由 | 内積の**分散**が次元とともに育ってしまう、という事実 | 第4巻 |
| $W$ の中身はどう決まるのか | 「学習」——損失を定義し、微分でWを動かす仕組み | 第2〜3巻 |
| なぜ「類似度の表」で翻訳ができるのか | 言語をベクトルにする方法と、attention が解いた問題 | 第6〜7巻 |

**softmax と $\sqrt{d_k}$**(第4巻)。ともに確率の道具です。softmax は「スコア表を、足すと1になる重みの表に変える関数」で、その「足すと1」を語るには確率分布という言葉が要ります。$\sqrt{d_k}$ は内積のばらつきが次元とともに育つことへの対処です。

**$W$ の中身**(第2〜3巻)。この巻を通して $W$ はずっと乱数のままでした(第6章の `linear`)。乱数では $QW^Q$ は「でたらめな役割への変換」にすぎません。良い $W$ を**探す**仕組みが「学習」です。

**なぜこれで翻訳ができるのか**(第6〜7巻)。一番遠い問いです。類似度の表が作れることと、それで翻訳できることの間にはまだ深い谷があります。単語をどうベクトルにするのか(序章で予告した king − man + woman ≈ queen の種明かしを含めて)、attention 以前の手法は何に苦しんでいたのか——その仕事です。

読めないものが4つ残るのに、なぜこれが「勝利」なのでしょうか。序章では、この式の**どこが読めないのかすら言えません**でした。いまは読めない箇所を4つ固有名詞で指させ、それぞれに「何が足りないか」と「どこで手に入るか」が言えます。正体不明の壁が、住所付きの宿題に変わった——これがこの巻の獲得物です。巻頭付録の論文読解マップを開き、第1巻の行に印を付けてください。

## 7.4 次巻予告 — 学習とは坂を下ること

最後に残った最大の謎は、棚卸し表の3行目です。**$W$ の中身は、どう決まるのか。**

第5章で $W$ を「あとで中身を調整できる変換」と呼び、第6章では乱数のまま `X @ W + b` を組み上げました。では「調整する」とは何でしょうか。$W$ の成分は論文の標準設定だけでも数千万個。そのつまみを、どちらにどれだけ回せばよいのか。総当たりでは宇宙の寿命があっても足りません。

答えの形だけ予告します。**いまの $W$ の「悪さ」を1つの数(損失)で測り、その数が最も急に減る方向へ、全部のつまみを少しずつ同時に回す。** これを何万回も繰り返す。霧の中の山で、足元の傾きだけを頼りに谷底へ降りる方法です。「最も急に減る方向」を教える道具が**微分**、とりわけ**勾配**で、第2巻「微分と最適化 — 学習とは坂を下ること」で手に入れます。

ラスボスの式は、もう配管図としては読めます。次は、その配管に流す $W$ を機械が自分で見つける仕組みです。坂を下りに行きましょう。
