> ステータス: ドラフト v1(2026-06-10)

## ラスボス(この巻の冒頭に原文掲示する論文箇所)

- Section 3.2.1 式(1):
  `Attention(Q, K, V) = softmax(QK^T / √d_k) V`
- Section 3.2.2 の射影:
  `head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)`

巻頭で「今は1文字も読めない。この巻を終えると、**行列としての構造**(Q・K・Vが何者か、`QK^T` が何を計算しているか、shapeがどう流れるか)が読める」と宣言する。`softmax` と `√d_k` は**読めないまま残る**ことも先に予告する(4巻の仕事)。

## [第1章 ベクトル — 数を並べると意味が生まれる](manuscript/01-vectors.md)

- 1.1 「単語の意味を数で表す」という無茶な発想(動機。論文の入力は512個の数の並び)
- 1.2 ベクトルの定義と記法: 成分、次元、太字小文字、R^n
- 1.3 足し算とスカラー倍 — 幾何的には矢印の継ぎ足しと伸縮
- 1.4 **[コード]** NumPy入門: list と ndarray の違い、`shape`、要素ごとの演算
- 1.5 予告: king − man + woman ≈ queen(意味の算術。種明かしは第6巻)
- 演習: 手計算 + NumPy

## [第2章 内積 — 「似ている」を数にする](manuscript/02-dot-product.md)

- 2.1 内積の定義: 掛けて、足す。それだけ
- 2.2 ノルム(長さ)と単位ベクトル
- 2.3 角度とコサイン類似度 — 内積の幾何的意味
- 2.4 **内積 = 類似度スコア**という直観(これが `QK^T` の心臓部、と1行だけ予告)
- 2.5 **[コード]** forループで内積を手実装 → `np.dot` と一致確認 → 速度比較(ベクトル化の最初の体験)
- 演習: どのベクトル対が「似ている」か内積で判定する

## [第3章 行列 — ベクトルを束ねる](manuscript/03-matrices.md)

- 3.1 行列の3つの見方: 数の長方形 / 行ベクトルの束 / 列ベクトルの束
- 3.2 **shapeという規律**: つねに (行, 列) を書く。本シリーズ全体で続く習慣をここで仕込む
- 3.3 転置 — 行と列を入れ替える(`K^T` の T はこれ。書体の意味もここで解説)
- 3.4 行列 × ベクトル: 「行との内積を縦に並べたもの」として導入
- 3.5 **[コード]** 文章 = 単語ベクトルを行に積んだ行列(shape: 単語数 × 次元)。論文の入力形式そのものであることを示す
- 演習: shapeを声に出して読む練習(以後全巻の儀式)

## [第4章 行列積 — すべての内積を一斉に](manuscript/04-matmul.md)

- 4.1 定義: C[i,j] = (Aの行i)・(Bの列j)。行列積とは**内積の総当たり表**である
- 4.2 shapeの連鎖: (m,n) @ (n,p) → (m,p)。掛けられる条件は内側の一致
- 4.3 非可換性: AB ≠ BA(具体例で)
- 4.4 結合法則と計算順序 — (AB)C = A(BC) だが計算量は違う
- 4.5 **[コード]** 3重ループで行列積を手実装 → `@` と一致確認 → ベンチマーク(なぜライブラリを使うのか)
- 4.6 **[コード]** ラスボスを先に触る: 単語ベクトルの行列 X に対して `X @ X.T` を計算してみる
  - 出てきた正方行列の各マスが「単語ペアの類似度」であることを観察する(attentionとは言わない。言うのは終章)
- 演習: shape当てクイズ(掛けられるか? 結果のshapeは?)

## [第5章 線形変換 — 行列は関数である](manuscript/05-linear-maps.md)

- 5.1 視点の転換: 行列は「ベクトルを入れるとベクトルが出てくる関数」
- 5.2 図で見る変換: 回転、拡大縮小、射影(コードより図が伝わる箇所。無理にコードを入れない)
- 5.3 変換の合成 = 行列積(関数合成の順序と積の順序)
- 5.4 記号 W の導入: 「あとで中身を調整できる変換」(なぜ調整するのか・どう調整するのかは2〜3巻の仕事、と明言)
- 5.5 同じ入力を3つの役割に変換する: `XW^Q`, `XW^K`, `XW^V`(論文3.2.2の形を、意味は伏せてshapeだけ読む)
- 演習: 2次元の点の集合に行列を掛けて変形を描画する

## [第6章 バッチと X @ W + b — ニューラルネットの背骨](manuscript/06-batch-and-linear.md)

- 6.1 データを行に積む: 1件ずつではなく、まとめて変換する(バッチ処理)
- 6.2 ブロードキャスティングと `+ b`: shapeが合わないのに足せる仕組み
- 6.3 `X @ W + b` のshapeを読む: (バッチ, 入力次元) @ (入力次元, 出力次元) + (出力次元,)
- 6.4 3次元以上の配列(テンソル)の入り口 — 「行列の束」。本格的な出番は第7巻(multi-head)と予告
- 6.5 **[コード]** 全結合層を「ただの行列演算」として実装する。学習はしない。Wは乱数のまま
  - ここで作る `linear(X, W, b)` が、第5巻でニューラルネットの部品としてそのまま再登場する
- 演習: 多層に重ねたときのshapeの流れを追う

## [終章 ラスボス再戦 — QK^T が読める](manuscript/07-boss-rematch.md)

- 7.1 論文の式(1)を再掲し、全記号にshape注釈を付けて読む
  - Q: (n, d_k)、K: (m, d_k)、`QK^T`: (n, m) = **全ペアの内積 = 類似度の総当たり表**(4.6の正体)
- 7.2 3.2.2の射影を読む: `XW^Q` は5章の「役割への変換」、shapeの流れも全部追える
- 7.3 まだ読めないものの棚卸し:
  - `softmax` と `√d_k` → 第4巻
  - そもそもWの中身はどう決まるのか → 第2〜3巻
  - なぜ「類似度の表」を作ると翻訳ができるのか → 第6〜7巻
- 7.4 次巻予告: 学習とは坂を下ること(第2巻: 微分と最適化)
