## [第1章 「学習する」とは何か](manuscript/01-what-is-learning.md)

- 1.1 ルールを書く vs データから決める: プログラミングの発想の転換
- 1.2 モデル = パラメータ付き関数。f(x; W) という書き方
- 1.3 教師あり学習: (入力, 正解) のペアから W を決める
- 1.4 機械学習の全体地図は描かない宣言: この本は教師あり学習・回帰 → (4巻で)分類、しか通らない
- 演習: 身の回りの問題を「入力 → 出力」の形に定式化してみる

## [第2章 線形回帰 — 最初のモデル](manuscript/02-linear-regression.md)

- 2.1 もっとも単純なパラメータ付き関数: y = X @ w + b(第1巻6章の `linear` がそのまま登場)
- 2.2 **[コード]** データを散布図にし、直線を「手で」当てはめてみる(w, b をスライダー的に動かす)
- 2.3 「いい感じの直線」を客観的に決めるには、「いい感じ」を数にする必要がある → 次章
- 演習: 手動フィットで最良と思う (w, b) を記録(後で勾配降下の結果と比べる)

## [第3章 損失関数 — 「悪さ」を1つの数にする](manuscript/03-loss.md)

- 3.1 残差: 予測と正解のズレ
- 3.2 MSE(平均二乗誤差): なぜ二乗か(符号消し、大外れへのペナルティ。確率論的正当化は第4巻に持ち越すと明言)
- 3.3 損失は W の関数である: L(w, b) の地形(第2巻4章の等高線がここで再登場)
- 3.4 **[コード]** (w, b) 平面上に MSE の地形を描く — 椀型であることを観察
- 演習: 損失地形の上に第2章の手動フィットをプロットする

## [第4章 学習 = 損失最小化 — 勾配降下で線形回帰を解く](manuscript/04-training.md)

- 4.1 ∂L/∂w, ∂L/∂b を手で導出(第2巻の連鎖律の最初の実戦)
- 4.2 **[コード]** 線形回帰のフルスクラッチ実装(NumPyのみ): forward → loss → gradient → update のループ
  - この4拍子が第8巻のTransformer訓練ループまで一切変わらないことを予告
- 4.3 学習曲線: loss の推移を眺める習慣
- 4.4 補足: 実は解析解もある(が、逆行列が要るのでこのシリーズでは通らない、と最短測地線原則を再確認)
- 演習: learning rate・初期値を変えて学習曲線を比較

## [第5章 ミニバッチ — データを少しずつ食べる](manuscript/05-minibatch.md)

- 5.1 全データで勾配 vs 1件ずつ vs ミニバッチ: 三つの流儀
- 5.2 ミニバッチSGD: ノイズはあるが速くて十分(以後シリーズの標準)
- 5.3 epoch、step、shuffle という語彙の整備
- 5.4 **[コード]** バッチサイズを変えて学習曲線のノイズと速度を観察
- 5.5 ここで論文 5.1 を覗き見: "batched together by approximate sequence length" — batch という言葉がもう読める
- 演習: バッチサイズと learning rate の関係を実験

## [第6章 汎化 — 訓練データへの最適化は目的ではない](manuscript/06-generalization.md)

- 6.1 丸暗記の罠: 訓練データで完璧 = 良いモデル、ではない
- 6.2 訓練/検証/テスト分割: なぜ3つに分けるのか
- 6.3 **[コード]** 多項式回帰で過学習を起こす: 次数を上げると訓練lossは下がり続け、検証lossが反転する(本巻のハイライト図)
- 6.4 過学習への対処の見取り図: データを増やす、モデルを縮める、正則化(dropout は第5巻で、と予告)
- 演習: 検証lossが最小になる次数を探す

## [終章 ラスボス再戦 — "training" が読める](manuscript/07-boss-rematch.md)

- 7.1 Section 5 冒頭・5.1 を再読: training regime、training data、batching — 全部この巻でやった営みの名前だった
- 7.2 世界観の確認: モデル = パラメータ付き関数、学習 = 損失最小化、評価 = 汎化性能
- 7.3 まだ読めない部分: label smoothing(第4巻で cross-entropy を得てから第8巻)、dropout(第5巻)

## [エピローグ 突然の宿題 — 分類できない](manuscript/08-epilogue-cliffhanger.md)

- E.1 新しい問題: 出力が数値ではなくラベル(スパムか否か)
- E.2 応急処置: シグモイドで出力を 0〜1 に潰す(道具として先出し。「なぜこの形か」は次巻)
- E.3 **[コード]** シグモイド + MSE で勾配降下 → **勾配がほぼゼロになり学習が止まる**のを観測する
- E.4 「二乗誤差は分類の損失として間違っている。では正しい損失とは?」 — 答えは次巻。**ここで巻を閉じる**
