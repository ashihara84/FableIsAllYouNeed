# 第3章 損失関数 — 「悪さ」を1つの数にする

前章で、あなたは直線を「手で」当てはめました。散布図を眺め、$w$ と $b$ を少しずつ動かして、いちばんしっくりくる直線を探す。最後に、自分の選んだ $(w, b)$ を記録したはずです。

ここで困ったことが起きます。隣の人と答えが違うのです。あなたは $(1.8,\ 2.5)$ と言い、隣の人は $(2.1,\ 0.4)$ と言う。どちらも散布図に重ねれば「いい感じ」に見えます。さて、どちらが良い直線でしょうか。

「いい感じ」のままでは、この口論に決着がつきません。前章の終わりに約束した通り、**「いい感じ」を数にする**必要があります。直線の「悪さ」を1つの数で表せれば、話は単純です——数の小さい方が勝ち。さらに先を言えば、機械に直線を選ばせることもできるようになります。機械は「いい感じ」を理解しませんが、**数を小さくすること**なら得意だからです(第2巻で、私たちはそのための道具——勾配降下法——をすでに持っています)。

この章では、その「悪さの数」を作ります。名前は最後に明かしますが、実はこの数、第2巻第6章で一度だけ顔を見せています。あのとき「この量の名前はまだ伏せておきます。第3巻で命名します」と予告したもの——その回収がこの章です。

## 3.1 残差: 予測と正解のズレ

まず、前章と同じデータを手元に再現しておきます。正解の直線 $y = 2x + 1$ にノイズを乗せて作った、50点の合成データでした。

<!-- TODO(consistency): 第2章のデータ生成コードと一致していることを検収時に突き合わせる(1変数回帰、seed 42、N=50、x∈[0,10]、真の直線 y=2x+1、ノイズ std=1.0) -->

```python
import numpy as np

rng = np.random.default_rng(42)
N = 50
X = rng.uniform(0.0, 10.0, size=(N, 1))     # (50, 1)
noise = rng.normal(0.0, 1.0, size=(N, 1))   # (50, 1)
y = 2.0 * X + 1.0 + noise                   # (50, 1)  正解の直線: w=2.0, b=1.0
```

モデルも前章のままです。直線 $\hat{y} = wx + b$。記号の上に乗っている小さな帽子($\hat{\ }$)は「予測値」の印で、正解の $y$ と区別するために付けます(以後この帽子は頻繁に登場します)。

悪さを測る最小単位から始めましょう。データ1点に注目します。$i$ 番めの点の入力が $x_i$、正解が $y_i$ です。いまの直線がこの点に対して出す予測は $\hat{y}_i = w x_i + b$。すると、この1点での「ズレ」は引き算で書けます。

$$r_i = \hat{y}_i - y_i$$

この $r_i$ を**残差**(residual)と呼びます。散布図の上では、点 $i$ から直線まで縦に引いた線分の長さ(に符号を付けたもの)です。予測が大きすぎれば正、小さすぎれば負、ぴったりなら $0$。1点の悪さとしては、これで文句なしでしょう。

問題は、点が50個あることです。残差も50個出てきます。shape で言えば `(50, 1)` のベクトルです。私たちが欲しいのは「この直線の悪さ」という**たった1つの数**でした。50個の数を、どうやって1つに潰しましょうか。

素朴な案は「平均する」です。50個の残差の平均を悪さとする。簡潔ですし、一見うまくいきそうです。ところが、この案には致命的な穴があります。**残差には符号がある**ことを思い出してください。正のズレと負のズレが、平均の中で打ち消し合ってしまうのです。

どれくらい致命的か、極端な例で確かめます。すべての点に対して、ただ $y$ の平均値だけを答える「水平線」を考えてみてください。傾きを完全に無視した、明らかにひどい予測です。ところが、この水平線の残差の平均は——平均からのズレを足し合わせるのですから——**厳密に $0$** になります。

```python
y_pred_flat = np.full_like(y, y.mean())     # 全予測を y の平均で固定した「水平線」
residual = y_pred_flat - y                  # (50, 1)
assert np.isclose(residual.mean(), 0.0)     # 残差の平均は厳密に 0(符号が打ち消し合う)
```

残差の平均で測ると、このひどい水平線は「悪さゼロ、完璧な予測」と判定されます。上に外した分と下に外した分が、帳簿の上で相殺されてしまった。私たちが測りたかったのは「ズレの大きさ」であって、「ズレの収支」ではありません。符号を、消す必要があります。

## 3.2 MSE(平均二乗誤差): なぜ二乗か

符号を消す方法は、すぐに2つ思いつきます。絶対値 $|r_i|$ を取るか、二乗 $r_i^2$ するか。どちらも負の数を正に変えてくれます。

機械学習の世界が標準として選んだのは、二乗の方です。残差を全部二乗してから平均する。

$$L = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)^2$$

この量を**平均二乗誤差**(mean squared error)、略して **MSE** と呼びます。名前がそのまま作り方です——誤差(残差)を、二乗して、平均する。コードにすると1行です。

```python
def mse(y_pred, y):
    """平均二乗誤差: (N, 1), (N, 1) -> スカラー"""
    return np.mean((y_pred - y) ** 2)
```

性質を2つ、手で確かめておきましょう。まず、予測が完璧なら $0$ になること。そして、さきほどの水平線がちゃんと「悪い」と判定されること。

```python
assert mse(y, y) == 0.0                     # ズレゼロなら損失ゼロ
assert mse(y_pred_flat, y) > 25.0           # 水平線の MSE はおよそ 31.8。悪い予測は悪いと言える
```

二乗はゼロ以上の数しか作りませんから、MSE は必ず $0$ 以上で、$0$ になるのは全点を撃ち抜いたときだけです。「小さいほど良く、最小値は完璧」——悪さの数として、欲しかった性質が揃っています。

ところで、なぜ絶対値ではなく二乗なのでしょうか。符号を消すだけなら絶対値でも足りたはずです。

二乗には、絶対値にはない癖があります。**大きく外した点を、不釣り合いに重く罰する**のです。残差 $1$ の二乗は $1$ ですが、残差 $4$ の二乗は $16$。ズレが4倍になると、罰は16倍になります。数値で見比べてみましょう。

```python
r1 = np.array([[1.0], [1.0], [1.0], [1.0]])  # 全員が 1 ずつ外す
r2 = np.array([[0.0], [0.0], [0.0], [4.0]])  # 1人だけ 4 外す(絶対値の合計は同じ 4)
assert np.isclose(np.abs(r1).mean(), np.abs(r2).mean())  # 平均絶対誤差では同点
assert np.mean(r2**2) == 4 * np.mean(r1**2)              # MSE は大外れを 4 倍重く罰する
```

絶対値で測れば、この2つの外し方は同点です(合計のズレはどちらも $4$)。しかし MSE は「1人だけ大きく外す」方を4倍悪いと判定します。つまり MSE で選ばれる直線は、**1点の大失敗を嫌い、全点をそこそこ近くに保とうとする**直線です。

もうひとつ、二乗を選ぶ実利的な理由があります。次章で私たちはこの $L$ を**微分**します(坂を下るには傾きが要るからです)。二乗関数 $r^2$ はどこでも滑らかで、微分すると $2r$ というこれ以上ないほど素直な形になります。一方、絶対値 $|r|$ は $r=0$ の折れ目で微分が定義できません。坂を下る道具で攻めるなら、地面は滑らかな方がいい。

ここまでが、この巻で語れる「なぜ二乗か」のすべてです。正直に言うと、これらは「二乗は都合がいい」という説明であって、「二乗が**正しい**」という説明ではありません。実は MSE には、もっと深いところから「この状況ではこれが正しい損失だ」と導き出せる正当化——確率の言葉による正当化——が存在します。しかしそれを語るには、尤度という道具が必要です。**この問いは第4巻で回収します。** いまは「符号を消せて、大外れを嫌い、微分しやすい。だから標準になった」で前に進みます。

さて、約束していた命名です。MSE のように、モデルの悪さを1つの数にする関数を、機械学習では**損失関数**(loss function)と呼びます。損失関数の値が**損失**(loss)です。第2巻第6章でパラメータ調整の的にした「データとのズレの合計」、あれの正式名がこれでした。論文の世界でも、訓練の話には必ずこの言葉が(あるいはこの言葉を前提とした言葉が)ついて回ります。覚えるべき語感はひとつ——**損失は小さいほど良い。学習の目的語は、常に損失です。**

## 3.3 損失は W の関数である: L(w, b) の地形

この節が、この章の核心です。ここまでの話を、視点を90度回して見直します。

MSE の式をもう一度見てください。今度は $\hat{y}_i = w x_i + b$ を代入した形で書きます。

$$L(w, b) = \frac{1}{N} \sum_{i=1}^{N} \bigl( (w x_i + b) - y_i \bigr)^2$$

この式には4種類の文字が出てきます。$x_i$、$y_i$、$w$、$b$。ここで問います。**$L$ は、何の関数でしょうか。**

つい「$x$ の関数」と答えたくなります。モデル $\hat{y} = wx + b$ はたしかに $x$ の関数でした。しかし損失の立場から見ると、景色が逆転します。データ $x_i, y_i$ は、もう集め終わって手元にある——**変わらない定数**です。一方、$w$ と $b$ は、私たちがこれから選ぶ——**自由に動かせる変数**です。だから、

$$L\ は\ (w, b)\ の関数です。$$

左辺をわざわざ $L(w, b)$ と書いたのはそのためです。$w$ と $b$ を1組決めるたびに、悪さが1つの数として返ってくる。データはその計算の中に焼き込まれた定数にすぎません。

この視点の転換、思い出のある形ではないでしょうか。第2巻第6章6.1で、私たちは一度この転換をやっています。「動かすのは入力 $x$ ではなく、パラメータ $W$ の方だ」——あのときの転換が、いま損失関数という正式な舞台で再演されています。モデルにとってデータは入力ですが、**学習にとってはパラメータが入力**なのです。

そして、この見方が新しい風景を開きます。$L(w, b)$ は2変数関数です。2変数関数なら、第2巻第4章で散々付き合いました。横軸に $w$、縦軸に $b$ を取った平面の各点に、高さ $L$ が決まっている——つまり $L(w, b)$ は**地形**です。等高線が描けます。あの章の等高線が、ここで再登場するのです。

地形の言葉に翻訳すると、この巻の登場人物は全員、地図の上に居場所を持ちます。

- $(w, b)$ 平面の**1点** = 直線1本(モデルの候補1つ)
- その点の**標高** = その直線の悪さ(損失)
- **良い直線を選ぶ** = 地形の低いところを探す

あなたが前章で手動フィットとして記録した $(w, b)$ も、この地図上の1点です。隣の人の答えも別の1点。どちらが良いかは、標高を比べれば決着します。そして「最良の直線を探す」という問題は、「**この地形の谷底はどこか**」という問題に化けました。谷底を探す道具なら持っています——第2巻の勾配降下法です。それは次章の仕事として、まずはこの地形がどんな形をしているのか、自分の目で見ておきましょう。

## 3.4 [コード] (w, b) 平面上に MSE の地形を描く — 椀型であることを観察

地形を描く手順は素朴です。$(w, b)$ 平面に格子(グリッド)を切り、格子点1つ1つで損失を計算して、標高表を作る。まず、損失関数を「$(w, b)$ を受け取る関数」としてコードに写します。3.3 の主張をそのまま形にしたものです。

```python
def L(w, b):
    """データ (X, y) を固定し、(w, b) を入力とする損失関数。スカラー -> スカラー"""
    return mse(w * X + b, y)

assert L(2.0, 1.0) < L(0.0, 0.0) < L(-2.0, 5.0)  # 正解の直線に近いほど損失は小さい
```

関数の中で `X` と `y` を外から掴んでいて、引数は `w` と `b` だけ。「データは定数、パラメータが変数」という3.3の視点が、関数のシグネチャにそのまま現れています。試しに3点ほど標高を測ると、正解の直線 $(2, 1)$ 付近で約 $0.60$、原点 $(0, 0)$ で約 $165$、見当違いの $(-2, 5)$ では約 $422$。たしかに正解に近いほど低い。

では、格子を切って標高表を作ります。$w$ を $0$ から $4$ まで、$b$ を $-3$ から $5$ までの範囲で、$81 \times 81 = 6561$ 点です。

```python
ws = np.linspace(0.0, 4.0, 81)              # (81,) 刻み幅 0.05
bs = np.linspace(-3.0, 5.0, 81)             # (81,) 刻み幅 0.1
landscape = np.empty((len(bs), len(ws)))    # (81, 81)  行が b、列が w
for i, b in enumerate(bs):
    for j, w in enumerate(ws):
        landscape[i, j] = L(w, b)
```

この標高表を等高線図にするコードがこちらです(描画は手元で試してください。この巻のコード一式では、図を描く部分だけは実行検証の対象外にしてあります)。

```python
import matplotlib.pyplot as plt

W_grid, B_grid = np.meshgrid(ws, bs)        # ともに (81, 81)
fig, ax = plt.subplots(figsize=(6, 5))
cs = ax.contour(W_grid, B_grid, landscape, levels=np.geomspace(0.6, 400, 12))
ax.clabel(cs, inline=True, fontsize=8)
ax.set_xlabel("w")
ax.set_ylabel("b")
ax.set_title("L(w, b)")
plt.show()
```

図3.1: $(w, b)$ 平面上の MSE の等高線。等高線は入れ子の閉曲線(つぶれた楕円)で、中心の1点に向かって標高が下がっていく。

何が見えるはずか、言葉にしておきます。等高線は、中心を共有する**入れ子の輪**になっています。輪の中心がただ1つの谷底で、そこから離れるほど、どの方向に進んでも標高は上がる一方。途中に第二の谷も、丘も、平らな棚もありません。立体で想像するなら、**椀**(ボウル)です。$L$ は $w$ についても $b$ についても二乗の式なので、断面はどこで切っても放物線——それを2方向に組み合わせた、滑らかな椀型の地形です。

目で見たことを、目分量で終わらせずに assert で固定します。図に見えるはずの性質を、3つ数値で検証しましょう。

```python
# 性質1: 地形のどこを見ても損失は正(ノイズがあるので 0 にはならない)
assert landscape.min() > 0.0

# 性質2: 谷底は正解の直線 (w, b) = (2.0, 1.0) のすぐそば
i_min, j_min = np.unravel_index(landscape.argmin(), landscape.shape)
w_best, b_best = ws[j_min], bs[i_min]
assert abs(w_best - 2.0) <= 0.1
assert abs(b_best - 1.0) <= 0.5

# 性質3: 椀型 — どの行(b 固定)も、どの列(w 固定)も「下って、上る」一回きり。
# 途中に別のくぼみ(局所的な谷)は一つもない。
def is_valley(values):
    """1次元配列が「単調に下って、単調に上る」形かを判定する"""
    k = values.argmin()
    return np.all(np.diff(values[: k + 1]) < 0) and np.all(np.diff(values[k:]) > 0)

for i in range(len(bs)):
    assert is_valley(landscape[i, :])       # 横方向に切っても谷は1つ
for j in range(len(ws)):
    assert is_valley(landscape[:, j])       # 縦方向に切っても谷は1つ

print("ok: すべての assert を通過しました")
```

実行して `ok` が出れば、図の印象はあなたの環境でも事実です(コード全体は `code/ch03/mse_landscape.py` にあります)。3つの性質を、それぞれ一言ずつ補足します。

**性質1** ——谷底ですら標高 $0$ ではありません(実測でおよそ $0.58$)。データにはノイズが乗っているので、どんな直線も全点は撃ち抜けないのです。損失 $0$ は目標ではない、という感覚はこの先ずっと大事になります(第6章で「訓練データで損失を下げきること自体が罠になる」という話をします)。

**性質2** ——格子上の谷底は $(w, b) = (2.0,\ 0.8)$ でした。データを生成した正解 $(2.0,\ 1.0)$ の**すぐそばですが、ぴったりではありません**。これもノイズの仕業です。手持ちの50点に対する最良の直線は、データを作った真の直線と少しだけずれる。このずれの意味も、第6章で再訪します。

**性質3** ——これがこの節の主役です。地形を縦横どこで輪切りにしても、谷は1つしかない。第2巻第3章で、凸でない関数の勾配降下が**局所解**——本当の谷底ではないくぼみ——に捕まる体験をしたのを覚えているでしょうか。この地形には、そのくぼみが存在しません。**どこから坂を下り始めても、たどり着く先は同じ谷底**。線形回帰 + MSE は、勾配降下の練習場として、これ以上ないほど安全な地形なのです(先に言っておくと、この安全さは線形回帰の特権です。後の巻でニューラルネットの地形に踏み込むと、くぼみだらけの世界に戻ります)。

地形は見えました。谷底の座標も、格子の総当たりでなら見つかりました。しかし正直に言えば、総当たりは力技です。格子を細かくすれば計算は爆発しますし、パラメータが2個だから6561点で済んだものの、これが100万個になったら格子は宇宙の原子より多くなります。坂の傾きだけを頼りに、総当たりなしで谷底へ歩いて下りる——次章、いよいよ学習です。

## まとめ

- **残差** $r_i = \hat{y}_i - y_i$ は1点ごとのズレ。ただし符号があるので、平均すると打ち消し合い、ひどい予測が「悪さゼロ」に化けることがある
- **MSE** $L = \frac{1}{N}\sum (\hat{y}_i - y_i)^2$ は残差を二乗して平均した「悪さの数」。二乗の理由は、符号が消える・大外れを重く罰する・微分が滑らか。**「なぜ正しいか」の確率論的な正当化は第4巻で回収する**
- モデルの悪さを1つの数にする関数が**損失関数**。学習の目的語は常に損失(第2巻第6章で伏せていた名前の回収)
- **損失はパラメータの関数** $L(w, b)$。データは定数、パラメータが変数。$(w, b)$ 平面の1点 = 直線1本、標高 = 悪さ、良い直線探し = 谷底探し(第2巻第4章の等高線の再登場)
- 線形回帰 + MSE の地形は**椀型**: 谷はただ1つで、局所解がない。どこから下りても同じ谷底に着く

**ラスボスとの距離**: 論文 Section 5 の "training" という営みの目的語——**何を**最小化するのか——が手に入りました。残るは「どうやって」(次章)と「どのデータで」(第5・6章)です。

**問1** (a) $(3+1+0-1-3)/5 = 0$。(b) $(9+1+0+1+9)/5 = 4$。(c) MSE。残差の平均は符号の打ち消しで $0$ になっており、ズレの大きさを表していない。

**問2** 平均絶対誤差: Aは $2$、Bは $8/4 = 2$ で同点。MSE: Aは $4$、Bは $64/4 = 16$ でAの勝ち。MSE は二乗のせいで大外れ($8$)を不釣り合いに重く罰するため、1点の大失敗を抱えるBを嫌う。

**問3** $(1.8,\ 2.5)$ の例なら損失は約 $1.28$、谷底は約 $0.58$ なので、比はおよそ $2.2$ 倍。点は谷底 $(2.0,\ 0.8)$ から見て左上(傾き不足を切片で補った位置)にある。手動フィットの多くは「傾きを少し外し、切片で帳尻を合わせる」配置になり、椀の浅い斜面の途中に乗る。等高線で見ると、自分の点を通る輪と谷底の間に何本の輪があるかで「あと何段下れるか」が読める。

**問4** $w$ を $0.5$ 増やすと、予測は各点で $0.5 x_i$ 動く——$x_i$ が最大 $10$ なので最大 $5$ も動き、二乗でさらに増幅される。一方 $b$ を $0.5$ 増やしても、予測は全点一律 $0.5$ しか動かない。同じ距離でも $w$ 方向の移動は損失を激しく変えるため、等高線は $w$ 方向に詰まり、$b$ 方向に間延びする。(この「方向によって坂の急さが違う」地形は、第2巻6.4で見たジグザグの原因そのものです。)
