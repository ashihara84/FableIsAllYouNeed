# 第2章 線形回帰 — 最初のモデル

> [目次](../TOC.md) ・ [← 前の章](01-what-is-learning.md) ・ [次の章 →](03-loss.md)

前章で、モデルとはパラメータ付き関数 $f(x;\ W)$ であり、教師あり学習とは(入力, 正解)のペアからパラメータを決める営みだ、と宣言しました。ただし具体的な $f$ の中身はまだ白紙です。この章では最初のモデルを1つ手で持ちます。

名前は**線形回帰**(linear regression)です。パラメータはたった2個です。そしてその正体は、第1巻であなたがすでに実装した、あの関数です。

## 2.1 もっとも単純なパラメータ付き関数: y = X @ w + b

この章から数章のあいだ、私たちはずっと次の問題を考えます。

> ある試験について、20人の学生の「勉強時間」と「点数」のデータがある。
> これから受験する学生が「6時間」勉強したとき、何点くらい取れそうか予測したい。

入力 $x$ は勉強時間、出力 $y$ は点数です。出力が数値ですから回帰の問題で、データは(勉強時間, 点数)のペアの集まり——教師あり学習の設定そのものです。

ではモデル $f(x;\ W)$ をどんな形の関数にすればよいでしょうか。データを眺める前から、ひとつだけ信じてよさそうなことがあります。「勉強するほど、点数は上がる(だろう)」。この素朴な信念をもっとも単純な数式にすると、直線になります。

$$\hat{y} = wx + b$$

記号を確認します。$w$ は**傾き**で「勉強1時間あたり何点上がるか」、$b$ は**切片**で「勉強時間ゼロのときの点数」です。中学校で習った $y = ax + b$ の $a$ を $w$ に書き換えただけの式です。

左辺の $\hat{y}$ は「ワイハット」と読みます。データに入っている**正解**の $y$ と、モデルが出力する**予測**とを区別するための帽子です。正解は $y$、予測は $\hat{y}$ です。この2つのズレを問題にする日がすぐ来るので、最初から書き分けます。

この直線を前章の記法で書き直します。

$$f(x;\ w, b) = wx + b$$

パラメータが $w$ と $b$ の2個ある、パラメータ付き関数です。これが線形回帰のモデルであり、「モデル=パラメータ付き関数」の最小の実例です。Transformerのパラメータは約6,500万個ですが、こちらは2個です。しかし「パラメータの値しだいで関数の振る舞いが変わり、その値をデータから決めたい」という構図は、まったく同じです。

### 20人ぶんを一度に: 行列の形に直す

予測したい相手は1人とはかぎりません。20人ぶんの予測を出したいとき、どう書けばよいでしょうか。第1巻6章で同じ問いに答えたとおり、**データは行に積む**のでした。20人の勉強時間を縦に積んで行列 $X$ `(20, 1)` を作ります。1人あたりの入力は数1個なので行の幅は1です。同様に $w$ を `(1, 1)` の行列、$b$ を `(1,)` のベクトルとして用意すると、20人ぶんの予測は1行で書けます。

$$\hat{y} = X @ w + b$$

shapeを読み上げます。

$$\underbrace{(20,\ 1)}_{X} \ @\ \underbrace{(1,\ 1)}_{w} + \underbrace{(1,)}_{b} \ \to\ (20,\ 1)$$

内側の1が消えて $(20, 1)$、そこに $b$ が各行にブロードキャスティングで足され、20人それぞれの行に同じ $w$・$b$ で $wx + b$ が計算されます。

この式、見覚えがあるはずです。第1巻6章で実装した、あの3行です。

```python
def linear(X, W, b):
    """全結合層: (バッチ, d_in) @ (d_in, d_out) + (d_out,) -> (バッチ, d_out)"""
    return X @ W + b
```

一字一句、あのままです。線形回帰のモデルとは、$d_{in} = d_{out} = 1$ の——考えうるかぎり最小の——**全結合層そのもの**です。ニューラルネットの背骨と呼んだあの部品の、いちばん小さな個体を、最初のモデルに選んだことになります。

第1巻でこの関数を作ったとき、「この巻では学習はしません。$W$ と $b$ は乱数のままです」と言い残しました。中身を決める道具がなかったからです。いま、その道具を揃える巻に入っています。この章ではまず、道具を使わずに——**素手で**——$w$ と $b$ を決めようとしてみます。素手の限界を知ることが、次章への最短経路だからです。

## 2.2 [コード] データを散布図にし、直線を「手で」当てはめてみる

本物の試験データは手元にないので、この本では**合成データ**(synthetic data)を使います。私たちが出題者——神様の立場——になってデータを作ってしまうのです。「真の規則」として 点数 = 7.0 × 勉強時間 + 20.0 を定め、そこに1人ずつ、ばらつき(平均0、標準偏差6.0の乱数)を足します。現実の学生も、体調や山勘で真の実力から±数点はぶれるものです。

神様の立場には教科書として大きな利点があります。**答えを知った上で、学習の過程を採点できる**のです。合成データなら $w = 7,\ b = 20$ が正解だと私たちだけが知っています。学習する側(モデル)は $X$ と $y$ しか見ない——このルールで進めます。

シリーズの規約通り、乱数の種は42に固定します。**このデータは次章以降もずっと使い続ける**ので、種を変えないでください。

```python
def make_data():
    """勉強時間 -> 点数 の合成データ(20人)。真の規則: 7.0 * x + 20.0 + ばらつき(標準偏差 6.0)。"""
    rng = np.random.default_rng(42)
    n = 20
    X = rng.uniform(0, 9, size=(n, 1))   # 勉強時間(0〜9時間) (20, 1)
    noise = rng.normal(0, 6.0, size=(n, 1))
    y = 7.0 * X + 20.0 + noise           # 点数 (20, 1)
    return X, y
```

種を42に固定したので、誰の環境でも同じ20人が生まれます。最初の5人はこんな顔ぶれです。

| 学生 | 勉強時間(時間) | 点数 |
|---|---|---|
| 1 | 6.97 | 67.7 |
| 2 | 3.95 | 43.6 |
| 3 | 7.73 | 81.4 |
| 4 | 6.28 | 63.0 |
| 5 | 0.85 | 23.4 |

数表のままでは傾向が見えにくいので、横軸に勉強時間、縦軸に点数をとって20人を点として打ちます。この図を**散布図**(scatter plot)と呼びます。

図2.1: 勉強時間と点数の散布図。20個の点が左下から右上へ、おおむね帯状に並ぶ。0.6時間で19点の学生から、8.8時間で79点の学生まで。ただし点は一直線上には乗っておらず、帯の幅のぶんだけ上下にばらついている。

散布図からは2つのことが読み取れます。第一に、「勉強するほど点数が上がる」はデータでも正しそうで、直線を当てはめる方針は筋が良さそうです。第二に、点は完全な直線上には**乗っていません**。つまりどんな $w, b$ を選んでも全員を貫く直線は存在せず、私たちにできるのは「なるべく良い」直線を選ぶことだけです。

### 素手でフィットする

候補の $(w, b)$ をひとつ決め、20人ぶんの予測 $\hat{y}$ を計算し、正解 $y$ とのズレを眺める——これを納得がいくまで繰り返します。ズレを返す道具を1つ作っておきます。

```python
def try_line(w_val, b_val, X, y):
    """候補 (w, b) を1つ試し、予測とズレ(y - y_hat)を返す。"""
    w = np.array([[float(w_val)]])       # (1, 1)
    b = np.array([float(b_val)])         # (1,)
    y_hat = linear(X, w, b)              # (20, 1) @ (1, 1) + (1,) -> (20, 1)
    return y_hat, y - y_hat              # ズレ: 正なら点が直線の上、負なら下
```

ズレ `y - y_hat` が**正**なら正解が予測より上——その学生の点は**直線より上**にあります。**負**なら直線より下です。

最初の候補は、いかにも先生が言いそうな「1時間につき10点、勉強しなければ0点」説です。**候補A: $(w, b) = (10,\ 0)$**。

```text
[-2.   4.1  4.2  0.2 14.9 -8.5  2.6  1.  19.  10.4 22.8 -7.5 -0.5 -7.1
 11.7 20.6  4.3 13.2 -7.3  6.8]
```

プラスが14人、マイナスが6人。プラスが多い=多くの点が直線より上で、**直線が全体に低すぎる**。さらに +22.8、+20.6、+19.0 と大きなズレが目立ち、誰かと調べると勉強時間 3.3、2.1、1.2 時間——**勉強時間が短い学生たち**です。この直線は彼らに11〜33点しか与えていませんが、実際は30〜56点を取っています。「勉強しなければ0点」説は左端で大きく外しており、切片 $b$ を持ち上げる必要がありそうです。

そこで思い切って $b$ を上げ、上げたぶん傾きを緩めて、**候補B: $(w, b) = (4,\ 40)$**。

```text
[ -0.2 -12.2  10.5  -2.1 -20.    4.2   3.7   3.4 -14.1  -5.3   2.9   2.6
  -5.7  -2.7  -4.3  -7.1  -5.7 -23.3  -2.6   1. ]
```

今度はマイナスが13人です。やりすぎです。特に −23.3 と −20.0 は、また勉強時間の短い学生(0.6時間と0.9時間)です。さっき低すぎた左端が今度は高すぎる——切片の上げすぎです。一方、右端の学生(8.8時間)は +4.2 と予測が足りておらず、傾きを寝かせすぎた兆候です。

2回の失敗から、$b$ は0と40のあいだ、$w$ は4と10のあいだです。**候補C: $(w, b) = (7,\ 20)$** を試します。

```text
[-1.1 -4.1  7.3 -0.9 -2.6 -2.1  3.2  2.2  2.5  2.6 12.8 -2.4 -3.1 -4.9
  3.7  6.8 -0.7 -5.  -4.9  3.9]
```

見違えました。ほとんどの学生がズレ±5以内、符号もプラス9人・マイナス11人とほぼ半々——直線が点の帯のまんなかを通っている証拠です。+12.8 がひとり残っていますが、これは3.3時間の勉強で56点を取った例の学生です(山勘が当たった日もあるでしょう)。

良さそうですが、本当に最良でしょうか。もうひと声、**候補D: $(w, b) = (6.5,\ 25)$** も試します。

```text
[-2.6 -7.1  6.2 -2.8 -7.1 -2.7  1.6  0.7 -1.9 -0.4  9.5 -3.3 -5.2 -6.2
  0.7  2.8 -3.2 -9.8 -6.2  1.7]
```

こちらも悪くありません。全員がズレ±10以内で、候補Cの +12.8 のような大物がいない。一方、ズレ±5を超える学生は、候補Cの3人に対して候補Dは8人。大物がいないのはD、小さくまとまる人が多いのはC。

`linear`・`make_data`・`try_line` と4候補の比較を含む全文と動作確認は `code/ch02/manual_fit.py`(`python3` で全 assert 通過)です。散布図と4本の候補直線を重ねた図は同ファイルの matplotlib コードで再現できます。

図2.2: 散布図に4本の候補直線を重ねたもの。候補A($w=10, b=0$)は点の帯の下を、候補B($w=4, b=40$)は左で帯の上を通って外れている。候補C($w=7, b=20$)と候補D($w=6.5, b=25$)はどちらも帯のまんなかを通っていて、この2本は目ではほとんど区別がつかない。

ここまでが「手で当てはめる」体験です。Jupyterのようなノートブック環境なら、`ipywidgets` で $w$ と $b$ をスライダーにして直線が点の帯に吸い付くまでぐりぐり動かす遊びもできます(やっていることは同じです)。ぜひあなた自身の候補も試してから、先へ進んでください。章末の演習1つめがまさにそれです。

## 2.3 「いい感じの直線」を客観的に決めるには、「いい感じ」を数にする必要がある

候補Cと候補D、**どちらが良い直線でしょうか**。

目視では決められません。著者の周りでも「Cは大ハズレがひとりいるのが気になる」派と「Dのほうが全体にずれている気がする」派に割れました。どちらにも一理ある——つまり「いい感じの直線」の「いい感じ」が人によって違うのです。

しかも、いまはまだ恵まれた状況です。学生は20人しかおらず、ズレの数列を端から目で読む力技が通用しました。これが2万人なら2万個の数を見比べることになり、さらに第1巻を思い出せば、入力が1個ではなく512個の数になる世界が控えています。目視は最初の20人ですでに限界です。

ここで前巻の道具箱を開けます。第2巻で手に入れた勾配降下法——「下りたい関数があるなら、傾きの逆へ一歩ずつ進め」でした。第2巻6章ではパラメータ付き関数をデータに合わせて調整するところまでやりました。今回も $(w, b)$ という2個のパラメータを調整したいのですから、出番に見えます。

ところが勾配降下は今のままでは**起動すらできません**。あのアルゴリズムは「下げたい数」を返す関数があって初めて動くのでした。微分できるのは数を返す関数だけです。「いい感じ」「帯のまんなかを通ってる感じ」——感じのままでは傾きが定義できず、坂が存在しません。

必要なものは、はっきりしました。

> $(w, b)$ を入れると、その直線の「悪さ」を**1つの数**で返してくれる関数。

この関数さえあれば、2つのことが一挙に手に入ります。第一に、候補Cと候補Dの論争が即座に終わる——悪さの数が小さいほうが良い直線、以上です。第二に、「悪さを最小にする $(w, b)$ を探せ」という問題になった瞬間、第2巻の勾配降下がそのまま接続できます。坂さえあれば、下る道具はもう持っているのですから。

では、20個のズレをどうやって1つの数にまとめるのでしょうか。「全部足せばいい」と思った人は、章末の演習4で軽い衝撃を受けます。この「まとめ方」には工夫が要る——それが次章の主題です。

## まとめ

- **線形回帰**は、直線 $\hat{y} = wx + b$ をデータに当てはめる、パラメータ2個($w$: 傾き、$b$: 切片)の最小のモデル。予測を $\hat{y}$、正解を $y$ と書き分ける
- データを行に積めば $\hat{y} = X @ w + b$、つまり $(20, 1)\ @\ (1, 1) + (1,) \to (20, 1)$。これは第1巻6章の `linear` ——$d_{in} = d_{out} = 1$ の全結合層——そのもの
- データは種42の合成データ(真の規則 $7x + 20$ + 標準偏差6のばらつき)。**次章以降も同じ20人を使い続ける**
- $(w, b)$ を手で動かすと、ズレの符号と大きさから修正の方向は読み取れる。しかし「最良の1本」は目視では選べない
- 勾配降下を接続するにも候補を比較するにも、「悪さ」を**1つの数**にする関数が必要——次章でそれを作る

**ラスボスとの距離**: 論文 Section 5 冒頭の "the training regime for our models" のうち、"models" の最小の実例を自分の手で持ちました。パラメータ2個の `linear` も、6,500万個のTransformerも、「パラメータ付き関数」という意味では同じ"model"です。ただし "training" はまだできません——良し悪しを測る数を、私たちはまだ持っていないからです。

## 演習

**問1** 候補C $(w, b) = (7,\ 20)$ で、勉強時間 4.05 時間の学生(データの10人め、実際の点数 51.0)の予測点数を**手で**計算し、ズレ $y - \hat{y}$ を求めてください。この学生は直線の上と下、どちらにいますか。

**問2** ズレの数列には、修正のヒントが埋まっています。次のそれぞれのパターンが見えたとき、$(w, b)$ をどう直すべきか答えてください。
(a) ズレが20人ほぼ全員、同じくらいの大きさのプラスである。
(b) 勉強時間が長い学生ほど、ズレが大きなプラスになっていく。

**問3** `try_line` で自分の候補をいくつも試し、**最良と思う $(w, b)$ を1組、数値で記録してください**。メモはこの巻のあいだ捨てないこと。第4章で、勾配降下法が見つけ出す答えとあなたの目を比べます。

**問4** 「ズレを全部足した合計が0に近いほど良い直線だ」という案を思いついたとします。候補E $(w, b) = (10,\ 5)$ について `try_line` でズレを求め、`diff.sum()` を計算してください。合計はいくつになりますか。そして候補Eは、図2.2に重ねてみると良い直線ですか。この案のどこが壊れているのでしょうか。

<details>
<summary>略解</summary>

**問1** $\hat{y} = 7 \times 4.05 + 20 = 48.35$。ズレは $51.0 - 48.35 = +2.65 \approx +2.6$。ズレが正なので、この学生は直線より**上**にいます(予測より良い点を取った)。

**問2** (a) 全員が同じくらい上にいる=直線が全体にそのぶん低い。切片 $b$ をズレのぶんだけ上げる($w$ はそのまま)。(b) 右へ行くほど直線が点に置いていかれている=傾きが足りない。$w$ を上げる。実は勾配降下法も、本質的にはこの2つの読み取りを自動で行います(第4章)。

**問3** 答えは1つではありませんし、ここで「正解」を確定させる道具がまだないのがこの章の結論でした。参考までに著者の記録は $(w, b) = (6.7,\ 23)$ 付近。「真の規則は $(7,\ 20)$ なのだから、それが最良では?」と思った人は良い着眼です——が、ばらつきが足されたあとの**この20人**にとっての最良は、真の規則から少しずれます。どこへずれるかは、第4章で勾配降下が教えてくれます。

**問4** 合計は約 $+3.3$ で、ほぼ0です。ところが図に重ねると、候補Eは明らかに急すぎる直線で、個々のズレには −13.5 や +17.8 といった大物が並んでいます。プラスのズレとマイナスのズレが**打ち消し合って**、合計だけが0に見えるのです。「上に外す」と「下に外す」はどちらも悪さなのに、足し算は両者を相殺してしまう——符号をどう始末するかが、次章の最初の仕事になります。

</details>
