# 第6章 汎化 — 訓練データへの最適化は目的ではない

前の章までで、私たちの訓練ループは一応の完成を見ました。モデルを用意し(第2章)、悪さを MSE という1つの数にし(第3章)、勾配降下で下り(第4章)、データはミニバッチで少しずつ食べる(第5章)。学習率やバッチサイズを調整して損失をどこまで下げられるか競う、そんな腕前も身についてきたはずです。

ここで立ち止まって考えたい問いがあります。**訓練データでの損失をゼロにできたら、それは最高のモデルでしょうか。** この章で、その直感は一度壊れます。訓練データでの損失は、下げすぎたせいでモデルが悪くなることがある。この現象を自分の手で起こし、数字で目撃するのがこの章の目的です。

## 6.1 丸暗記の罠: 訓練データで完璧 = 良いモデル、ではない

思考実験から始めましょう。訓練データ $(x_1, y_1), \ldots, (x_n, y_n)$ に対して、次のような「モデル」を考えます。

- 入力 $x$ が訓練データの中にあれば、対応する $y$ をそのまま返す
- なければ、適当な値(たとえば 0)を返す

つまり、訓練データを**丸ごと記憶しただけの表**です。このモデルの訓練データでの MSE はゼロ、完璧です。損失最小化というゲームのルール上、これは満点の解答です。

しかしこのモデルが役に立たないことは明らかでしょう。明日やってくる新しい $x$ はほぼ確実に表に載っておらず、そこではただ 0 と答えるだけの置物です。

ここに、この章の中心となる区別があります。**私たちがモデルに望んでいるのは、訓練データへの正解ではなく、まだ見ぬデータへの正解です。** 訓練データは過去の記録にすぎず、モデルの使い道は未来の入力にあります。まだ見ぬデータに対しても良い予測ができる能力を**汎化**(generalization)と呼びます。そして「訓練データでは完璧なのに、まだ見ぬデータでは役立たず」という丸暗記型の失敗を**過学習**(overfitting)と呼びます。

比喩を1つ。過去問の答えを丸暗記した人は過去問を解かせれば満点ですが、本番では見たことのない問題に手も足も出ません。過去問を「解き方を理解するため」に使った人は、本番で得点できます。訓練データでの損失は**勉強の進み具合**であって、**本番の成績ではない**のです。

「丸暗記の表なんて極端だ、私たちの $y = Xw + b$ はそんなことをしない」と思うかもしれません。線形回帰のパラメータはたった数個で、丸暗記する容量がありません。しかしこの章の後半で見るように、**モデルに記憶容量を少し多めに与えるだけで、勾配降下は誰に命じられなくても丸暗記を始めます**。損失最小化に忠実であればあるほど、そうなる。罠は、ルールそのものに仕掛けられているのです。

では、「まだ見ぬデータへの正解率」はどうすれば測れるのでしょうか。まだ見ぬデータは、定義からして手元にないのに。

## 6.2 訓練/検証/テスト分割: なぜ3つに分けるのか

答えは単純です。**手元のデータの一部を、わざと見せずに隠しておく**のです。

手元に 1,000 件のデータがあるなら、たとえば 800 件だけで訓練し、残り 200 件には勾配降下を一切触れさせません。訓練後に、隠しておいた 200 件の損失を測れば、それは「モデルが見たことのないデータでの成績」の見積もりになります。訓練に使う部分を**訓練データ**(training data)、最終評価のために隠しておく部分を**テストデータ**(test data)と呼びます。

これで2分割。問題は、**なぜもう1つ要るのか**です。

### モデルを「選ぶ」という、もう1つの最適化

学習率を変えると結果が変わり、バッチサイズでも変わりました。この章ではモデルの複雑さ(後述する多項式の次数)も変えます。これらは勾配降下が決めてくれる $w$ や $b$ とは違い、**私たちが手で決めるしかない設定値**です。こうした値を**ハイパーパラメータ**(hyperparameter)と呼びます。

ハイパーパラメータの決め方として自然に思いつくのは、「候補を10個試して、テストデータでの損失が一番小さいものを選ぼう」。一見何も悪くなさそうですが、この瞬間に起きたことをよく見てください。10個の候補からテストデータの成績が最も良いものを選ぶ——これは**テストデータを使った最適化**です。勾配降下が訓練データの損失を下げる方向にパラメータを動かすのと同じ構図で、私たちはテストデータの損失を下げる方向にハイパーパラメータを動かしました。違いは、最適化を実行したのが計算機か人間か、だけです。

最適化に使われたデータは、もう「見たことのないデータ」ではありません。10個の候補には、実力ではなく**偶然テストデータと相性が良かっただけ**のものが混ざり、最小値を選ぶ操作はそういう「まぐれ当たり」を優先的に拾います。選ばれたモデルのテスト成績は本当の実力より良く見える——成績表が楽観の側に汚れるのです。

比喩を1つ。合否判定のために取っておいた模試を教材としても使ってしまったら、その点数はもう合否判定に使えません。**評価に使ったデータは、選択に使ったその瞬間から、評価用としての価値を失う**のです。

### だから3つに分ける

そこで、データを3つに分けます。

| 分割 | 役割 | 誰が使うか |
|---|---|---|
| 訓練データ(training data) | パラメータ $w, b$ を決める | 勾配降下 |
| 検証データ(validation data) | ハイパーパラメータを選ぶ | 私たち |
| テストデータ(test data) | 選び終わったモデルの最終成績を1回だけ測る | 誰も触らない |

検証データは「モデル選びのために、汚れる覚悟で使うデータ」です。だからこそテストデータは金庫にしまっておき、すべての選択が終わったあとに**一度だけ**開ける。テストの成績を見て「やっぱりあっちのモデルにしよう」と思った瞬間、テストデータは検証データに堕ち、手元から「まだ見ぬデータ」は消えます。

私たちのラスボス論文もこの規律を守っています。論文の実験では、newstest2013 を開発セット(development set——検証データの別名です)としてモデルの設定選びに使い、最終評価は newstest2014 という別のデータセットで報告しています(論文 6.2 節)。「development set」がモデル選び専用のデータを指すこと、それがなぜ最終評価と分けられているのか——あなたはもう読めるはずです。

道具立てが揃いました。訓練データで学習し、検証データで成績を測る。この2本の物差しを持って、いよいよ過学習を自分の手で起こしに行きます。

## 6.3 [コード] 多項式回帰で過学習を起こす: 次数を上げると訓練lossは下がり続け、検証lossが反転する

### 直線では足りないとき

第2章からずっと、私たちのモデルは直線でした。しかし世の中のデータは曲がっており、直線を当てはめれば大きな残差が残ります。

表現力を上げる古典的で簡単な方法があります。入力 $x$ をそのまま使うのではなく、$x, x^2, x^3, \ldots, x^d$ という**べき乗の列に広げてから**線形回帰にかけるのです。

$$y = w_1 x + w_2 x^2 + \cdots + w_d x^d + b$$

これを**多項式回帰**(polynomial regression)と呼び、$d$ を多項式の**次数**(degree)と呼びます。

新しい学習アルゴリズムは要りません。データ $n$ 件の入力を広げた行列を $\Phi$ `(n, d)` と書くと(各行が1件分、$k$ 列目が $x^k$)、モデルは

$$\mathbf{y} = \Phi \mathbf{w} + b$$

で、$\mathbf{w}$ `(d,)` に対しては完全に線形です。第4章で作った「forward → loss → gradient → update」の4拍子が、$X$ を $\Phi$ に差し替えるだけでそのまま動きます。曲線を学習するのに機械は1ミリも変わらない——これが多項式回帰の気持ちよさです。

次数 $d$ は勾配降下が決めてくれないハイパーパラメータです。$d$ を大きくするほどモデルは曲がれるようになり、表現力が上がります。表現力は上げるほど良いのか、実験で確かめましょう。

### 実験の設計: 神の視点

普段データの「本当の姿」は誰にもわかりませんが、実験では私たちがデータを自作するので正解を知っています。この神の視点が実験の強みです。

- **正解の関数**: $y = 0.5 + 1.0x - 2.0x^2 + 3.0x^3$。つまり**正解は3次多項式**です
- そこに標準偏差 0.4 の**ノイズ**を足して観測データとする(現実のデータは必ず汚れている)
- 訓練データ 20 件、検証データ 100 件
- 次数 $d = 0, 1, 2, \ldots, 9$ の10個のモデルをそれぞれ訓練し、訓練lossと検証lossを記録する

正解が3次だと知っていれば予想が立ちます。$d$ が3より小さいモデルは曲がり方が足りず、3を超えたモデルは余った自由度で何か余計なことを始めるはずです。

### 1つだけ下ごしらえ: 列のスケールを揃える

実務的な罠を1つ片付けます。$x$ が $[-1, 1]$ のとき、$x^9$ はほとんどの点で 0 付近に潰れ、$\Phi$ の列ごとに数のスケールがバラバラになります。スケールの違う方向が混ざると、損失の地形は細長い谷になります——第2巻4章の等高線で見た、勾配降下が苦手なあの地形です。

そこで $\Phi$ の各列から平均を引き、標準偏差で割って、列ごとのスケールを揃えます。これを**標準化**(standardization)と呼びます。規律が1つ。**平均と標準偏差は、訓練データだけから計算する**こと。検証データも混ぜると、検証データの情報がほんの少し訓練手続きに漏れます。6.2節「評価用のデータを訓練側の操作に使ってはいけない」のミニチュア版です。

### コード

核心は、4拍子をそのまま使い、$X$ を $\Phi$ に差し替える点と、標準化の統計量を訓練データだけから計算する点です。

```python
def poly_features(x, degree):
    if degree == 0:
        return np.zeros((len(x), 0))  # 列ゼロ: モデルは b だけになる
    return np.stack([x**k for k in range(1, degree + 1)], axis=1)

for degree in degrees:
    Phi_train = poly_features(x_train, degree)
    Phi_val = poly_features(x_val, degree)
    if degree > 0:
        # 列ごとの標準化。統計量は訓練データだけから計算する(6.2の規律)
        mu = Phi_train.mean(axis=0)
        sigma = Phi_train.std(axis=0)
        Phi_train = (Phi_train - mu) / sigma
        Phi_val = (Phi_val - mu) / sigma
    w, b = fit_gd(Phi_train, y_train)   # 第4章と同じ4拍子
    train_loss = mse(Phi_train @ w + b, y_train)
    val_loss = mse(Phi_val @ w + b, y_val)
```

検証パートは本章の主張をそのまま assert にしています。

```python
# (1) 訓練lossは次数を上げても増えない(下がり続ける)
assert np.all(np.diff(train_losses) <= 1e-6)
# (2) 検証lossは途中で反転する: 最小は端(次数9)ではなく途中にある
best_degree = int(np.argmin(val_losses))
assert 0 < best_degree < 9
# (3) この実験では、検証lossはデータの正体(3次)を言い当てる
assert best_degree == 3
# (4) 反転は明確: 次数9の検証lossは最小値の2倍を超えて悪化している
assert val_losses[9] > 2 * val_losses[best_degree]
# (5) その間も訓練lossは下がっている(訓練lossでは過学習を検出できない)
assert train_losses[9] < train_losses[best_degree]
```

全文と動作確認は `code/ch06/polynomial_overfitting.py`(`python3` で全 assert 通過)。`fit_gd` は `steps=1000000` という大きな反復回数を使います。次数が高いと標準化してもなお損失の谷は細長く収束が遅いため、「途中でやめたから訓練lossが下がりきらなかった」という言い訳を消すべく十分すぎるほど回しています(手元で20秒ほど)。

整理して並べ直します。

| 次数 | 訓練loss | 検証loss |
|---:|---:|---:|
| 0 | 2.6435 | 3.3417 |
| 1 | 0.6367 | 0.6812 |
| 2 | 0.3142 | 0.2978 |
| **3** | 0.0726 | **0.2210** ← 最小 |
| 4 | 0.0584 | 0.3091 |
| 5 | 0.0584 | 0.3097 |
| 6 | 0.0560 | 0.2856 |
| 7 | 0.0553 | 0.2785 |
| 8 | 0.0527 | 0.3693 |
| 9 | 0.0494 | 0.6715 |

**訓練lossの列**は、2.6435 から 0.0494 まで**一度も増えることなく**下がり続けています(4次と5次は表示桁では同じですが、わずかに下がっています。assert (1) が全行を検算済み)。偶然ではありません。次数 $d$ のモデルにできることは次数 $d+1$ のモデルにも全部できます($w_{d+1} = 0$ とすればよい)。だから表現力を上げて損失最小化を真面目にやる限り、訓練lossは原理的に悪くなりようがない。訓練lossだけを見ている人には、9次のモデルが文句なしの最優秀です。

**検証lossの列**はまったく違う物語を語ります。0次から3次までは訓練lossと足並みを揃えて下がり、**3次の 0.2210 で底を打ち、そこから先は二度と 3次を下回りません**。8次・9次で明確に悪化し、9次では 0.6715——最小値の3倍です。教科書どおりの滑らかなU字ではなく途中に凸凹がありますが(本物の実験データはいつもこう)、**下がって、底を打ち、反転する**という大きな形ははっきり見えます。

最大の皮肉がここにあります。検証lossが3倍に悪化していくその間も、**訓練lossは下がり続けている**(assert (5))。訓練lossをどれだけ熱心に見つめても、過学習は**絶対に**見えません。過学習を検出できる物差しは検証lossだけです。データを分けた理由が、この2つの列の対比に凝縮されています。

もう1つ、検証lossが最小になった次数は3——**データを作った本当の関数の次数と一致**しました(assert (3))。検証データという物差しは、正解を知らされていないのに、データの裏にある構造の複雑さを言い当てたのです。

### 何が起きていたのか

なぜ高すぎる次数は検証データで失敗するのでしょうか。

訓練データは 20 点しかありません。9次多項式はパラメータを10個($w_1, \ldots, w_9$ と $b$)持ち、20 点をかなり自由になぞれます。一方データには標準偏差 0.4 のノイズ——**二度と再現されない偶然のゆらぎ**——が乗っています。余った表現力を持て余したモデルは、損失最小化の命令に忠実に従い、本当の3次のカーブだけでなく**ノイズの凹凸までなぞり始めます**。訓練の20点の間を縫って激しくうねる曲線を想像してください。訓練点の上では正確で、点の間では大暴れ。検証データの100点は容赦なくその「間」に落ちるので、検証lossが跳ね上がるのです。

これが6.1節の丸暗記の、なめらかな曲線版です。誰もモデルに「丸暗記しろ」とは言っていません。**損失最小化を忠実にやった結果として**ノイズの暗記が起きた。冒頭の問い——訓練lossをゼロにできたら最高のモデルか?——への答えがこれです。ノイズ入りのデータで訓練lossをゼロに近づけることは、ノイズを暗記することと同義なのです。実際このデータのノイズの分散は $0.4^2 = 0.16$ なので、検証lossはどんな天才モデルでも 0.16 程度より下には行けません。3次モデルの 0.2210 はその理論的な床にかなり近く、これ以上を欲張った瞬間に暗記が始まる、ぎりぎりの位置にいます。

反対側の端、0次や1次の失敗は逆向きです。曲がる能力が足りず、本当の構造すら捉えられていない。これを**学習不足**(underfitting)と呼びます。次数というハイパーパラメータの選択は、学習不足と過学習のあいだの尾根を歩くことであり、その足元を照らすライトが検証lossです。

## 6.4 過学習への対処の見取り図: データを増やす、モデルを縮める、正則化

過学習を起こせるようになったので、次は鎮め方です。対処は大きく3系統あり、ここでは見取り図だけを描きます——詳細は必要になった巻でそれぞれ扱います。

| 対処 | 考え方 | 効き目の理由 |
|---|---|---|
| データを増やす | 暗記しきれない量を見せる | ノイズは点ごとに独立なので、点が増えるほど「ノイズをなぞる曲線」は引けなくなる |
| モデルを縮める | 暗記する容量を与えない | パラメータが少なければ、ノイズまで覚える余裕がない |
| 正則化 | 容量は残したまま、暴れたら罰する | 損失に「複雑さへのペナルティ」を足し、暗記を割に合わなくする |

**データを増やす**のは、使えるならば最強の手です。9次モデルが暴れられたのは訓練点が20個しかなく、点と点の「隙間」が広かったからでした。点を10倍に増やせば隙間は埋まり、ノイズも互いに打ち消し合って、モデルはなぞるべき本当の構造しか残されません。効き目は演習の問2で確かめてください。弱点はデータがタダではないこと。手元のデータが増やせないとき、残りの2系統の出番です。

**モデルを縮める**のは、この章で実際にやったことです。次数を9から3に下げる——検証lossを物差しにして適切な容量のモデルを選ぶことは、過学習への対処そのものでした。

**正則化**(regularization)は、いちばん現代的な系統です。容量を削るのではなく、容量を悪用したら損をする仕掛けを訓練に埋め込みます。たとえば「損失にパラメータの大きさのペナルティを足す」(暴れる曲線は大きな $w$ を必要とするため暗記のコストが上がる)、あるいは訓練手続きそのものにわざと邪魔を入れる、など。ここでは「そういう系統がある」とだけ覚えてください。

ラスボス論文には、この3つめの系統の専用セクションがあります。

> *"We employ three types of regularization during training:"*
> — Vaswani et al., "Attention Is All You Need", Section 5.4
>
> 訳: 訓練中、私たちは3種類の正則化を用いる。

regularization という単語が「何のための装置か」は、もう読めるはずです——Transformer ほどの巨大モデルは放っておけば必ず過学習するので、論文は対策をわざわざ一節を割いて宣言しているのです。ただし中身はまだ読めません。筆頭の **dropout** は「訓練手続きにわざと邪魔を入れる」型の正則化で、ニューラルネットの構造を知らないと説明できないため**第5巻で導入します**。もう1つの label smoothing は cross-entropy(第4巻)の上に立つ道具なので、第8巻まで持ち越しです。ここでは「正則化という棚があり、論文はその棚から3つ取り出している」という見取り図だけ持って先へ進みましょう。

## まとめ

- モデルの目的は訓練データへの正解ではなく、**まだ見ぬデータへの正解**(汎化)。訓練lossは勉強の進み具合であって、本番の成績ではない
- 「まだ見ぬデータでの成績」は、手元のデータの一部を隠しておくことで測る。**評価に使ったデータは、選択に使った瞬間に評価用の価値を失う**ため、訓練(パラメータを決める)/検証(ハイパーパラメータを選ぶ)/テスト(最後に1回だけ測る)の3つに分ける
- 多項式回帰の実験で、**次数を上げると訓練lossは単調に下がり続けるが、検証lossは3次で底を打って反転する**ことを観測した。過学習は訓練lossには決して映らず、検証lossにしか映らない
- 過学習の正体は、損失最小化に忠実なモデルが、再現されない**ノイズまで暗記**すること。逆に容量不足は学習不足と呼ぶ
- 対処は3系統: **データを増やす、モデルを縮める、正則化**。dropout(第5巻)と label smoothing(第4巻→第8巻)は正則化の棚に並ぶ

**ラスボスとの距離**: 論文の training data / development set という語の役割分担と、Section 5.4 の節題 "Regularization" が「過学習への対策の宣言」であることが読めるようになりました。

## 演習

**問1(検証lossが最小になる次数を探す)** 6.3節のコードを実行し、検証lossが最小になる次数を確認してください。それはデータを生成した「本当の次数」と一致していますか。また、`true_f` を2次多項式(たとえば $y = 0.5 + 1.0x - 2.0x^2$)に変えて再実験すると、検証lossが最小になる次数はどう動くと予想しますか。実験して確かめてください(assert (3) の `== 3` は適宜書き換えること)。

<details><summary>略解</summary>

実行すると `検証lossが最小になる次数: 3` と表示され、生成に使った3次と一致します。検証lossは正解を知らされていないのに、データの裏の構造の複雑さを言い当てました。`true_f` を2次に変えて再実行すると、今度は検証lossの最小は2次に移ります(手元の実行では2次の 0.1843 が最小。データの引きによっては隣の次数にずれることもあります)。検証データという物差しが「データに見合った容量」を指すこと自体は変わりません。

</details>

**問2(データを増やす)** 6.4節の表の1行目「データを増やす」の効き目を確かめます。`n_train` を 20 から 200 に変えると、表(次数 × 訓練loss・検証loss)はどう変わるでしょうか。特に9次の検証lossがどうなるか予想してから、実行してください(assert は一部成り立たなくなるので、コメントアウトして観察に集中して構いません)。

<details><summary>略解</summary>

訓練データ200件では、検証lossの「反転」がほぼ消えます。手元の実行では検証lossは4次の 0.1924 が最小、9次でも 0.1980 とほとんど悪化しません(20件のときは 0.2210 → 0.6715 と3倍でした)。点の隙間が埋まり、ノイズが打ち消し合うため、9次モデルにはもう暴れる余地がないのです。同じモデル・同じ訓練手続きでも、データ量だけで過学習は劇的に和らぐ——「データを増やすのは最強の手」の意味がこの数字です。なお訓練lossの絶対値は20件のときより大きくなります(0.0494 → 0.1516)。点が増えるほど全点を完璧になぞることは難しくなるからで、これは悪化ではなく健全化です。

</details>

**問3(考察: テストデータはどこへ行った?)** この章の実験では、訓練データと検証データしか登場せず、テストデータを作りませんでした。いま、検証loss最小の3次モデルを選び終えたとします。「このモデルの性能は、まだ見ぬデータに対して MSE 0.2210 です」と報告してよいでしょうか。6.2節の論理で説明してください。

<details><summary>略解</summary>

報告してはいけません。0.2210 という数字は、10個の候補から検証lossが最小のものを**選んだ結果**です。選択は検証データへの最適化なので、この数字には「偶然この検証データと相性が良かった」分の楽観バイアスが乗っています(まぐれ当たりを優先的に拾うのが「最小を選ぶ」という操作でした)。まだ見ぬデータでの成績として報告するには、選択が終わったあとに、どの選択にも使っていない第3のデータ——テストデータ——で測り直す必要があります。この実験でそれをするなら、`rng.uniform` でもう1セット `x_test, y_test` を生成し、選ばれた3次モデルだけを評価すればよいのです。

</details>
