## E.1 新しい問題 — 出力が数値ではなくラベル

この巻で予測してきたものは、すべて**数値**でした。直線や多項式を当てはめ、予測値と正解のズレを二乗して測る。入力も数値、出力も数値。この設定を回帰(regression)と呼ぶのでした。

今度の依頼は、こうです。

> 「届いたメールがスパムかどうかを、判定してほしい」

入力はこれまでどおり数値で構いません。メール1通から「怪しい単語がどれだけ出てくるか」「感嘆符がどれだけ多いか」という2つの数を取り出すことにします(どちらも前処理で平均0あたりに標準化済みとします)。問題は出力です。**出力は数値ではなく、「スパム」か「スパムでない」かの2択**です。この2択の正解をラベル(label)と呼び、ラベルを当てる問題を分類(classification)、選択肢が2つなら二値分類(binary classification)と呼びます。

実物のメールから特徴量を作るのは本筋ではないので、合成データで進めます。通常メール100通とスパム100通、合わせて200件です。

```python
import numpy as np
rng = np.random.default_rng(42)

# 特徴量2つ(標準化済みのつもりの値): x1=怪しい単語の出現量、x2=感嘆符の多さ
n = 100
X_normal = rng.normal(loc=[-2.0, -2.0], scale=1.0, size=(n, 2))  # 通常メール
X_spam   = rng.normal(loc=[ 2.0,  2.0], scale=1.0, size=(n, 2))  # スパム
X = np.vstack([X_normal, X_spam])                # (200, 2)
y = np.concatenate([np.zeros(n), np.ones(n)])    # (200,) 0=通常, 1=スパム
```

$X$ `(200, 2)` が特徴量、$\mathbf{y}$ `(200,)` がラベルです。ラベルの2択は「スパムでない = 0、スパム = 1」と数値に直してあります。散布図で眺めると(`dead_gradient.py` 参照)、左下に通常メールの塊、右上にスパムの塊が見え、2つの塊の間には十分な隙間があります。

図E.1: スパム判定の合成データ200件の散布図。左下の通常メールと右上のスパムが、直線1本で分けられそうに見えます。

道具箱を棚卸ししておきます。モデルはパラメータ付き関数 $f(\mathbf{x}; \mathbf{w}, b)$(第1章)、「いい感じ」は損失関数で数にする(第3章)、あとは forward → loss → gradient → update の4拍子で坂を下る(第4章)。回帰を解いた道具は、ひとつ残らず手元にあります。これで解けるはずです。

## E.2 応急処置 — シグモイドで出力を 0〜1 に潰す

ひとつだけ、部品のつなぎ目が合いません。モデルの出力です。

線形モデル $\mathbf{x} \cdot \mathbf{w} + b$ の出力は、どんな実数にもなれます。$\mathbf{x}$ を大きくすれば 10 でも出てきますし、負の側にもいくらでも行けます。正解ラベルは 0 か 1 しかないのに、です。「このメールのスパム度は 3.2」「スパム度 −0.8」とは何でしょうか。解釈に困る出力を、正解の 0 や 1 と二乗誤差で比べるのは、座りが悪い。

欲しいのは、**出力を必ず 0 と 1 の間に押し込める仕掛け**です。0〜1 に収まってさえいれば、「0.5 以上ならスパム、未満なら通常」という素朴な約束で判定に使えます。

そういう関数が、あります。シグモイド関数(sigmoid function)です。

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

入力 $z$ はスカラーで、どんな実数でも受け付けます。

```python
def sigmoid(z):
    """どんな実数も 0〜1 の間に潰す(この形の理由は次巻)"""
    return 1.0 / (1.0 + np.exp(-z))

assert np.isclose(sigmoid(0.0), 0.5)
```

値をいくつか挙げると、$\sigma(-10) = 0.000045$、$\sigma(-5) = 0.0067$、$\sigma(0) = 0.5$、$\sigma(5) = 0.9933$、$\sigma(10) = 0.999955$。グラフは左下から右上へゆるやかに立ち上がる S 字です。$z$ をどれだけ大きく・小さくしても、出力は 1 と 0 に近づくだけで届きません。しかも単調増加なので、線形モデルの「大きいほどスパム寄り」という順序はそのまま保たれます。今回の要望には誂えたようにぴったりです。

ここで正直に言っておきます。**なぜ関数の形がこれでなければならないのか。出てきた 0〜1 の数を何と解釈すべきなのか。この巻では、どちらにも答えません。**この数の「正体」を語る言葉を、私たちはまだ1つも持っていないからです。この2つの問いは第4巻で回収します。いまは「実数を 0〜1 に潰してくれる、都合のいい滑らかな関数」という**道具**として使ってください。

道具と割り切れば、モデルは1行で完成します。線形(第1巻第6章の `linear` そのもの)の出力をシグモイドに通すだけです。損失は第3章の MSE をそのまま流用します。0〜1 の出力と 0 か 1 の正解、どちらも数値ですから、ズレの二乗は問題なく計算できます。

```python
def predict(X, w, b):
    """線形(第1巻)の出力をシグモイドで 0〜1 に潰す。(n,) を返す"""
    return sigmoid(X @ w + b)

def mse_loss(X, y, w, b):
    """損失は第3章の MSE をそのまま流用"""
    return np.mean((predict(X, w, b) - y) ** 2)

def accuracy(X, y, w, b):
    """0.5 以上をスパムと判定したときの正解率"""
    return np.mean((predict(X, w, b) >= 0.5) == (y == 1))
```

パラメータは $\mathbf{w}$ `(2,)` とスカラー $b$ の3個。$X$ `(200, 2)` を入れると 0〜1 の予測が `(200,)` で出てきます。`accuracy` は損失とは別の、人間向けの成績表です(学習が使うのはあくまで損失です)。

## E.3 学習が止まる — 勾配がほぼゼロになる

あとは第4章と同じ4拍子で回すだけ——ですが、勾配の用意だけ今回は違うやり方をします。第4章では $\partial L / \partial \mathbf{w}$ を連鎖律で手で導出しました。今回は導出せず、第2巻第1章の**数値微分**(中心差分)で勾配を測ります。手抜きに見えて、観測実験としてはむしろ強い。数値微分は式を使わず「パラメータを実際に少しだけ動かして、損失がどれだけ変わるかを測る」だけなので、この後で奇妙なことが起きても「勾配の式を導出し間違えたのでは?」という疑いを最初から消せます。

```python
def numerical_grad(loss_fn, X, y, w, b, h=1e-5):
    """第2巻第1章の中心差分。式を使わず、実際に少し動かして傾きを測る"""
    grad_w = np.zeros_like(w)
    for i in range(len(w)):
        e = np.zeros_like(w); e[i] = h
        grad_w[i] = (loss_fn(X, y, w + e, b) - loss_fn(X, y, w - e, b)) / (2 * h)
    grad_b = (loss_fn(X, y, w, b + h) - loss_fn(X, y, w, b - h)) / (2 * h)
    return grad_w, grad_b

def train(w0, b0, lr=0.5, steps=2000):
    """第4章と同じ4拍子: forward → loss → gradient → update"""
    w = np.array(w0, dtype=float); b = float(b0)
    history = [mse_loss(X, y, w, b)]
    for _ in range(steps):
        grad_w, grad_b = numerical_grad(mse_loss, X, y, w, b)
        w -= lr * grad_w; b -= lr * grad_b
        history.append(mse_loss(X, y, w, b))
    return w, b, history
```

初期値は控えめに、すべてゼロから回します(`train(w0=[0.0, 0.0], b0=0.0)`)。

```
実験1(初期値ゼロ):
  loss: 0.2500 -> 0.0037   正解率: 99.5%
```

損失は 0.25 から 0.0037 へ一直線に下がり、200件中199件を正しく判定しています(残り1件は2つの塊の境目に落ちた、運の悪い1通です)。分類問題、解けました。めでたしめでたし——

——と巻を閉じる前に、第4章の演習で身につけた習慣を思い出してください。**初期値を変えて、学習曲線を比べる**のでした。今度は $\mathbf{w} = (-8, -8)$ から始めます。2つの特徴量の効き方を、本来とは正反対に思い込んでいる初期値です。回帰の感覚なら、出発点が悪いだけで、坂を下ればいずれ同じ谷底に着くはずです。

```python
# 実験2: 初期値 w = (-8, -8), b = 0 から学習
w0_bad = np.array([-8.0, -8.0])
grad_w0, grad_b0 = numerical_grad(mse_loss, X, y, w0_bad, 0.0)
w_B, b_B, hist_B = train(w0=w0_bad, b0=0.0)
```

```
実験2(初期値 w = (-8, -8)):
  開始時の loss: 0.994979(ほぼ最悪)  正解率: 0.5%
  開始時の勾配: grad_w = [-5.3e-06, -1.2e-05], grad_b = -2.1e-05
  2000ステップ後の loss: 0.994978   正解率: 0.5%
```

学習が、起きていません。

数字を順に見てください。開始時の損失は 0.994979。全件外れたときの損失がほぼ 1 ですから、**ほぼ最悪の場所**です。正解率は 0.5%、200件中199件を間違え、実験1と正解・不正解がきれいに裏返った最悪のスパムフィルタです。そして2000ステップ後の損失は 0.994978。小数第6位が1つ動いただけ。学習曲線(図E.2)を描けば、その異様さは一目でわかります。

図E.2: 実験1と実験2の学習曲線。実験1は序盤で滑り台のように下りきるのに対し、実験2は 1.0 のすぐ下に**水平線**として貼りついたまま、2000ステップの間ぴくりとも動きません。

故障でしょうか。いいえ。答えは print された**勾配の値そのもの**に出ています。`grad_w = [-5.3e-06, -1.2e-05]`、つまり −0.0000053 と −0.000012 です。学習率 0.5 を掛けても、1ステップで $\mathbf{w}$ が動く量は 0.00001 にも届きません。$\mathbf{w}$ は $(-8, -8)$ から谷底まで何単位も移動しなければならないのに、です。勾配降下法は仕様どおり動いています。**勾配が、ほぼゼロなのです。**

第2巻からずっと、私たちは「坂を下れば良い場所に着く」という約束で歩いてきました。いま、モデルはほぼ最悪の場所に立ち、直しどころは満載のはずです。ところがその場所で——導出した式ではなく、実際に動かして測った実測値で——足元の傾きはほぼ真っ平ら。最悪の場所が、平原の真ん中にあるのです。

データ200件分の平均だから薄まって見えるだけでは、と思うかもしれません。1点だけ取り出して顕微鏡で覗きます。スパムの塊の中心 $(2, 2)$ にいちばん近い「典型的なスパム」を選び、それを正反対に判定するパラメータでの勾配を測ります。

```python
# 実験3: 顕微鏡 — 「自信を持って間違えている」1点での勾配
i = int(np.argmin(np.abs(X[n:] - [2.0, 2.0]).sum(axis=1))) + n
x_one, y_one = X[i:i+1], y[i:i+1]   # 正解ラベルは 1(スパム)
w_wrong, b_wrong = np.array([-3.0, -3.0]), 0.0  # 出力を 0(非スパム)の端へ振り切らせる
pred_one = predict(x_one, w_wrong, b_wrong)[0]
grad_w1, grad_b1 = numerical_grad(mse_loss, x_one, y_one, w_wrong, b_wrong)

# 比較対照: シグモイドを外した素の線形回帰 + MSE
def linear_mse_loss(X, y, w, b):
    return np.mean((X @ w + b - y) ** 2)
grad_w2, grad_b2 = numerical_grad(linear_mse_loss, x_one, y_one, w_wrong, b_wrong)
```

```
実験3(自信を持って間違えている1点):
  正解ラベル: 1(スパム)   モデルの出力: 0.000005(ほぼ0 = 非スパムと確信)
  この1点での勾配: grad_w = [-2.2e-05, -2.1e-05], grad_b = -1.0e-05
  (比較)シグモイドを外すと: grad_w = [-54.0, -52.6], grad_b = -26.3
```

正解はスパム(1)。モデルの出力は 0.000005、「スパムではない」側の端に振り切れています。**自信満々に、間違えている**のです。それなのに、この1点が発する勾配は 0.00002 そこそこ。一方、シグモイドを外した素の線形回帰なら、同じ点・同じパラメータで勾配は −54。桁が**6つ**違います。

第4章で導出した回帰の勾配の式は、残差に比例していました。大きく外すほど強く引き戻される——勾配降下の学習が成り立つのは、この性質があってこそでした。ところがシグモイドを通した途端、関係が壊れています。**自信を持って大きく間違えている点ほど、勾配が小さい。**いちばん反省すべき点が、いちばん黙っているのです。

なぜこんなことが起きるのか。その種明かしはこの巻ではしません。意地悪ではなく、種明かしを理解するための道具が、この巻にはまだ存在しないのです。

## E.4 間違っているのは損失関数 — 答えは次巻

容疑者を整理しましょう。

- **データは無実です。** 実験1で、同じデータがほぼ完璧に解けています。解けない問題ではありません。
- **勾配降下法も無実です。** 「勾配の逆向きに進む」を忠実に実行しただけです。実測した勾配がほぼゼロなら、ほぼ動かない——仕様どおりです。
- **残るのは、損失関数です。** 第3章で MSE を選んだ理由は「符号が消える」「大外れに重いペナルティ」でした。あれは回帰のための議論で、**「分類でも正しい」とは一度も確かめていません**。回帰でうまくいった道具を、検品せずに分類へ持ち込んだ。その結果が、最悪の場所に広がる平原です。

そこで、この巻の最後の問いを、はっきり書いておきます。

> **二乗誤差は、分類の損失関数として間違っている。**
> **では、分類にとって「正しい損失関数」とは、何か。**

この問いに、この巻は答えません。机の上には、未解決の宿題が3つ積まれたままです。

1. シグモイドは、なぜあの形でなければならないのか(E.2)
2. シグモイドが出す 0〜1 の数は、何と解釈すべきなのか(E.2)
3. 分類の正しい損失関数とは何か(E.3)

3つはばらばらに見えて、実はひとつながりです。ある1本の糸をたぐると、3つが同時にほどけます。その糸の名前を口にすることすら、ここでは答えの一部を漏らすので、やめておきます。
