# 第4章 学習 = 損失最小化 — 勾配降下で線形回帰を解く

> [目次](../TOC.md) ・ [← 前の章](03-loss.md) ・ [次の章 →](05-minibatch.md)

材料はすべて揃いました。本章で「モデル」「損失」「坂の下り方」の3点セットを初めてひとつに組み上げます。組み上がったものの名前が**学習(training)**——この巻のタイトルにある営みの正体です。

第2章で最初のモデル線形回帰を手に入れ、散布図に直線を「目で」当てはめました。第3章では当てはまりの悪さを1つの数にする損失関数 MSE を定義し、$(w, b)$ 平面に広がる損失の地形がきれいな椀型をしていることを確かめました。そして第2巻で、椀型の底へ降りる方法——勾配を計算し、その逆向きに一歩ずつ進む勾配降下法——も学びました。

やることは2段階です。まず損失の坂の傾き $\partial L/\partial w$ と $\partial L/\partial b$ を**手で**導きます(第2巻で練習した連鎖律の最初の実戦です)。次にその勾配を使って坂を下る訓練ループを NumPy だけで書き、第2章の手動フィットと勝負させます。

## 4.1 ∂L/∂w, ∂L/∂b を手で導出 — 第2巻の連鎖律、最初の実戦

最小化したい相手を正確に書き下します。データは第2章と同じ $n = 20$ 人ぶんの $(x_i, y_i)$——勉強時間と点数です。モデルは線形回帰

$$\hat{y}_i = w x_i + b$$

で、損失は第3章で定義した MSE です。

$$L(w, b) = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)^2$$

記法の注意をひとつしておきます。第2章ではこのモデルを、第1巻6章の `linear` をそのまま使って $X$ `(20, 1)` と $w$ `(1, 1)` の行列の形 `X @ w + b` で書きました。いまは変数が1つなので $w$ の中身はただの数が1個です。手計算ではスカラー $w$ と書くのが素直なので本章ではそうします。中身は一字も変わっていません。

勾配降下に必要なのは $\partial L / \partial w$ と $\partial L / \partial b$ です(偏微分は第2巻4章——「他を止めて、1変数ずつ」でした)。$L$ は2乗や総和が入れ子になっていて一見手強そうです。こういうときの定石を、私たちは第2巻5章で手に入れています。**入れ子の関数は計算グラフに分解して、辺ごとの微分を掛け算でつなぐ**——連鎖律です。

データ1点ぶんの計算をグラフに分解しましょう。第3章の残差 $r_i = \hat{y}_i - y_i$ を中継点に使うと、$w$ から $L$ までの道はこうつながっています。

$$w \;\longrightarrow\; \hat{y}_i = w x_i + b \;\longrightarrow\; r_i = \hat{y}_i - y_i \;\longrightarrow\; s_i = r_i^2 \;\longrightarrow\; L = \frac{1}{n}\sum_i s_i$$

辺は4本です。1本ずつ、第2巻1章の基本ルールだけで微分します。

1. $L = \frac{1}{n}\sum_i s_i$ を $s_i$ で微分: $s_i$ の係数は $\frac{1}{n}$ なので $\dfrac{\partial L}{\partial s_i} = \dfrac{1}{n}$
2. $s_i = r_i^2$ を $r_i$ で微分: $\dfrac{\partial s_i}{\partial r_i} = 2 r_i$
3. $r_i = \hat{y}_i - y_i$ を $\hat{y}_i$ で微分: $y_i$ はデータ(定数)なので $\dfrac{\partial r_i}{\partial \hat{y}_i} = 1$
4. $\hat{y}_i = w x_i + b$ を $w$ で微分: $x_i$ もデータ(定数)なので $\dfrac{\partial \hat{y}_i}{\partial w} = x_i$

連鎖律の出番です。$w$ からデータ点 $i$ を通って $L$ に至る1本道に沿って、辺の微分を全部掛けます。

$$\frac{1}{n} \cdot 2 r_i \cdot 1 \cdot x_i = \frac{2}{n} r_i x_i$$

ここで第2巻5章のもうひとつの規則——**道が複数あるときは道ごとの寄与を足す**、です。$w$ は20人ぶんすべての予測に使われているので $w$ から $L$ への道は20本あります。全部足して、

$$\frac{\partial L}{\partial w} = \frac{2}{n} \sum_{i=1}^{n} r_i x_i = \frac{2}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)\, x_i$$

$b$ も同じ道筋で、違いは最後の辺だけ——$\hat{y}_i = w x_i + b$ を $b$ で微分すると $1$ なので $x_i$ が消えて、

$$\frac{\partial L}{\partial b} = \frac{2}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)$$

導出は完了です。身構えた割に出てきた式は拍子抜けするほど短い——**「残差に $x_i$ を掛けて平均して2倍」と「残差を平均して2倍」**です。それだけです。

式の意味を読み取っておきましょう。$(w, b) = (0, 0)$ から出発すると予測はすべて 0 で、データはその上にあります。残差 $r_i = \hat{y}_i - y_i$ は軒並み負です。すると $\partial L/\partial b = \frac{2}{n}\sum r_i$ も負で、更新則 $b \leftarrow b - \eta \cdot \partial L/\partial b$ は $b$ を**増やします**。予測が低すぎるなら直線を上へ——勾配の式は直観と同じ向きを指しています。数式が信用できるか迷ったら、こうやって符号の向きを言葉に翻訳するのが効きます。

もうひとつの検算手段も私たちは持っています。第2巻1章の数値微分です。$(L(w+h) - L(w-h)) / 2h$ を計算していま導いた式と突き合わせる——「手で導いた勾配は数値微分と照合する」という習慣の出番で、次節のコードの冒頭で実際にやります。

## 4.2 [コード] 線形回帰のフルスクラッチ実装 — forward → loss → gradient → update のループ

学習のアルゴリズムを言葉で書くと、たった4行になります。

1. **forward**: 現在の $(w, b)$ で予測を出す — $\hat{y} = wx + b$(第2章のモデル)
2. **loss**: 予測の悪さを1つの数にする — $L = \mathrm{MSE}(\hat{y}, y)$(第3章の損失)
3. **gradient**: 損失の坂の傾きを求める — $\partial L/\partial w,\ \partial L/\partial b$(4.1節の式)
4. **update**: 傾きの逆向きに一歩進む — $w \leftarrow w - \eta\, \partial L/\partial w$(第2巻3章の勾配降下)

この4つをこの順番でひたすら繰り返します。本書はこれを**4拍子**と呼びます。1拍目で予測し、2拍目で採点し、3拍目で反省し、4拍目で直します。

いちばん大事な予告をしておきます。**この4拍子は、第8巻で Transformer を訓練するその日まで一切変わりません。** モデルが直線から何億パラメータのネットワークに化けても、損失が MSE から cross-entropy(第4巻)に替わっても、勾配を手で導く代わりに自作の autograd(第5巻)が計算するようになっても、ループの拍子そのものは今日書くこの4つのままです。今日のコードは使い捨ての練習ではなく、最終巻まで使い続ける骨格の初版です。

核心は訓練ループの4拍子です。

```python
# --- 4.1 の検算: 手で導いた勾配を数値微分と突き合わせる(第2巻1章の習慣) ---
w0, b0 = 0.0, 0.0
r = (w0 * X + b0) - y               # 残差 (20,)
grad_w0 = 2.0 * np.mean(r * X)      # 手導出: ∂L/∂w = (2/n) Σ r_i x_i
grad_b0 = 2.0 * np.mean(r)          # 手導出: ∂L/∂b = (2/n) Σ r_i
num_w = (mse((w0 + h) * X + b0, y) - mse((w0 - h) * X + b0, y)) / (2 * h)  # 数値微分
num_b = (mse(w0 * X + (b0 + h), y) - mse(w0 * X + (b0 - h), y)) / (2 * h)

# --- 訓練ループ: 4拍子 ---
w, b = 0.0, 0.0     # 初期値
lr = 0.01           # learning rate(第2巻3章)。0.03 を超えると発散する(演習1)
for step in range(num_steps):
    y_hat = w * X + b                        # 1. forward : 現在の (w, b) で予測
    loss = mse(y_hat, y)                     # 2. loss    : 悪さを1つの数にする
    grad_w = 2.0 * np.mean((y_hat - y) * X)  # 3. gradient: 4.1 で手導出した式
    grad_b = 2.0 * np.mean(y_hat - y)
    w = w - lr * grad_w                      # 4. update  : 坂を下る(第2巻3章)
    b = b - lr * grad_b
    history.append(loss)
```

全文と動作確認は `code/ch04/train_linear_regression.py`(`python3` で全 assert 通過)です。データ生成 `make_data`(第2章と同一の20人)、勾配の検算、第2章の手動フィットとの勝負までを含みます。実行結果はこうなります。

上から順に読みます。

**勾配の検算。** 4.1節で手導出した式と数値微分が $(w, b) = (0, 0)$ で小数4桁まで一致しました。連鎖律の初陣は成功です。値の符号にも注目——$(-670.54, -113.62)$ はどちらも負、つまり「$w$ も $b$ も増やせ」で、4.1節の符号の読みどおりです。

**loss の推移。** 出発点で 3530 あった損失が1歩で 464、3歩で 103 です。最初の数歩の下がり方は劇的です。これは第3章の地形で腑に落ちます。出発点 $(0, 0)$ は椀の縁の高いところにあり、椀の縁ほど坂は急——勾配が大きいので1歩が大きいのです。ただしそのあとが長く、step 10 で 95 まで降りたのに 21.14 に落ち着くまでさらに2000歩かかります。第3章の問4で見たとおり、この椀は $w$ 方向に急で $b$ 方向に緩いのでした。急な $w$ の坂はあっという間に降りきり、残りは $b$ 方向の浅い谷をゆっくり進むのです。step 2000 以降は小数6桁が動かなくなる——収束です。

**学習結果。** $(w, b) = (6.7221, 22.0922)$ です。第2章の演習の答え合わせをしましょう。第2章で決着のつかなかった論争から——候補C $(7, 20)$ の損失は 22.05、候補D $(6.5, 25)$ は 24.54 で**Cの勝ち**です(第3章の演習で地形に重ねた人はもう知っていたはずです)。筆者の記録 $(6.7, 23)$ の損失は 21.77、勾配降下の答えは 21.14 です。目で当てはめた直線も健闘したけれど、機械が4拍子を5000回刻んだ答えには届きません。そして勾配降下側の損失が「これ以上どの方向に動かしても下がらない」椀の底であることを、私たちは地形の形から知っています。

気づいた人がいるかもしれません。このデータの真の規則は $y = 7x + 20$ でした。なのに学習結果は $(6.7221, 22.0922)$ で $(7, 20)$ ではありません。学習が失敗したのか? いいえ。勾配降下が最小化したのは「真の規則とのズレ」ではなく、**ばらつき込みの手持ちデータ20人に対する MSE** です。その仕事は完璧にやり遂げています(4.4節で証明します)。第2章の演習3で「この20人にとっての最良は真の規則から少しずれる。どこへずれるかは第4章で」と予告した答えがこれです。手持ちデータへの最適化と、まだ見ぬデータへの強さは別物ではないか——この疑念は正しく、第6章の主題になります。いまは頭の隅に置いてください。

もう一度ループ本体を眺めてください。`for` の中身は実質4行です。"training" という、論文が定義すらせずに使っている言葉の正体は、この4行の反復でした。

## 4.3 学習曲線 — loss の推移を眺める習慣

前節のコードでは毎ステップの loss を `history` に記録しました。これを横軸 step、縦軸 loss でプロットしたものを**学習曲線(learning curve)**と呼びます。

なぜ描くのでしょうか。訓練ループの中身は人間の目には速すぎて見えません。1000歩ぶんの試行錯誤を一目で見渡せる窓は、この曲線だけです。学習が順調なのか、止まっているのか、暴走しているのか——診断はすべてこの曲線から始まります。モデルが巨大になり訓練が何時間にも伸びる後の巻では、この曲線を眺めることが訓練の日常そのものになります。**ループを書いたら学習曲線を描く**。今日から習慣にしてください。

描画コード(matplotlib)の全文は `code/ch04/plot_learning_curves.py` です。図4.1(lr=0.01 の学習曲線、線形/対数スケール)に加え、図4.2(learning rate を変える・演習1)と図4.3(初期値を変える・演習2)を描きます。手元で実行して同じ形が出ることを確かめてください。

図4.1: lr=0.01 の学習曲線。左は線形スケール、右は縦軸を対数にしたもの。

左の図に見えるはずのもの——最初の数歩で崖のように落ち、あとはほぼ水平な床に張りつく、L字型の曲線です。これが「順調な学習」の典型的なシルエットです。

ただし左の図には弱点があります。step 500 と step 2000 の差(23.10 と 21.14)は、縦軸が 3530 から始まるグラフでは線の太さに埋もれて見えません。「もう収束したのか、まだじわじわ下がっているのか」が読めません。そこで右の図——**縦軸を対数にする**と、床に張りついて見えた部分が引き伸ばされ、step 2000 あたりまで着実に下がり続けてから水平になる様子が読めます。学習曲線は対数スケールでも眺める、と覚えてください。後の巻でも損失は桁で変わるので、この習慣はずっと使います。

なお今回の曲線は一度も増えずに滑らかに下がりました(コードの検算1で assert もしました)。これは「全データで勾配を計算する」「地形が椀型」という今の設定がくれた特権です。learning rate を欲張ると曲線が暴れ上がること(演習1)、第5章でデータを少しずつ食べるようになると曲線がギザギザになること——きれいな曲線との別れは、すぐそこです。

## 4.4 補足: 実は解析解もある — が、この道は通らない

白状しておくべきことがあります。第2巻2章で「傾き = 0 を解けばいい、が通用しない関数が世の中のほとんど」と学びました。実は線形回帰 + MSE はその例外です。$\partial L/\partial w = 0$ と $\partial L/\partial b = 0$ は連立方程式として**そのまま解けて**、答えが1つの式で書けます。これを**解析解(closed-form solution)**と呼びます。一般の(変数が多い)線形回帰では解は

$$\mathbf{w}^* = (X^T X)^{-1} X^T \mathbf{y}$$

という形になります。式の中の $(\cdot)^{-1}$ は**逆行列(inverse matrix)**——第1巻の序章で「第3巻で一度だけ名前だけ登場する」と予告した、あの道具です。予告どおり、ここがその一度で、予告どおり名前だけで通り過ぎます。

1000歩もループを回さなくても一発の式で答えが出ます。ならばなぜこの近道を通らないのでしょうか。理由は2つあります。

第一に、**この近道は線形回帰 + MSE という特権的な組み合わせの外では消滅する**からです。モデルが少しでも非線形になると(早ければ本巻のエピローグで、その瞬間に立ち会います)、「傾き = 0」は手で解けなくなり、解析解は存在しなくなります。ニューラルネットワークにも Transformer にも解析解はありません。あの世界で通用する道は勾配降下だけです。つまり4.2節の4拍子は解析解が使えないときの妥協案ではなく、**この先のすべての地形で通用する唯一の本道**なのです。

第二に、最短測地線の原則です。逆行列を真面目に学ぶにはそれだけで1章ぶんの寄り道が要り、そして Transformer の訓練と推論には逆行列は一度も登場しません。論文に出てこない道具のために章を費やさない——第1巻で行列式や固有値を切り捨てたときと同じ判断を、ここでも下します。

とはいえ「解析解と一致する」ことは勾配降下の最高の検算になります。都合よく、変数が1つの今回に限っては解析解は逆行列なしの式に潰れます($\bar{x}, \bar{y}$ は平均。導出はしません——検算に使うだけです)。

$$w^* = \frac{\sum_i (x_i - \bar{x})(y_i - \bar{y})}{\sum_i (x_i - \bar{x})^2}, \qquad b^* = \bar{y} - w^* \bar{x}$$

突き合わせる核心はこの2行です。

```python
x_mean, y_mean = X.mean(), y.mean()
w_star = np.sum((X - x_mean) * (y - y_mean)) / np.sum((X - x_mean) ** 2)
b_star = y_mean - w_star * x_mean
```

全文と動作確認は `code/ch04/closed_form_check.py`(`python3` で全 assert 通過)です。勾配降下の答え(4.2の4拍子)と解析解を突き合わせます。実行結果はこうです。

```
勾配降下: w = 6.722065, b = 22.092189
解析解  : w = 6.722065, b = 22.092189
ok: 勾配降下は解析解と一致しました
```

小数6桁まで一致しました。5000回の小さな一歩は、一発の式と同じ場所——椀の底のただ一点に正確に到着していました。4.2節の `assert np.allclose([w, b], [6.7221, 22.0922], atol=1e-3)` に書いた謎の数値の種明かしも、これで済みました。あれは解析解(を4桁に丸めたもの)だったのです。

## まとめ

- 学習とは損失最小化であり、その実体は **forward → loss → gradient → update の「4拍子」** の反復である。この4拍子は第8巻の Transformer の訓練ループまで一切変わらない
- MSE の勾配は連鎖律で手導出できる: $\partial L/\partial w = \frac{2}{n}\sum r_i x_i$、$\partial L/\partial b = \frac{2}{n}\sum r_i$(残差を平均して2倍、が骨格)。手導出した勾配は数値微分と照合する
- 勾配降下は第2章の手動フィット(loss 21.77)を破り(loss 21.14)、その着地点は解析解と小数6桁まで一致した
- **学習曲線**を描くのは訓練の日常。縦軸は対数スケールでも眺める
- 線形回帰には逆行列を使う解析解もあるが、その近道はニューラルネット以降は消滅する。このシリーズは通らない(最短測地線)

**ラスボスとの距離**: 論文 Section 5 の冒頭 *"This section describes the training regime for our models."* の "training" が指す営み——4拍子のループ——が、自分の手の中で動くようになりました。

**問1(learning rate を変える)** 4.3節の `train` を使い、`lr` を 0.001 / 0.01 / 0.05 に変えて100ステップずつ訓練し、学習曲線を重ねて描いてください(図4.2のコード)。実行前に3本の曲線の形を予想すること。余力があれば、収束と発散の境目の `lr` を実験で挟み撃ちにしてみてください。

<details><summary>略解</summary>

第2巻3章の「小さすぎると遅い、大きすぎると発散」が、本物の損失で再現されます。lr=0.001 は滑らかに下がるが遅く、100歩で loss 95.2(5000歩かけても 23.1 までしか届かない)。lr=0.01 は約2000歩で 21.1356 に収束。lr=0.05 は1歩ごとに椀の反対側のより高い場所へ跳び移り、loss は 3530 → 1.9万 → 11万 → … と爆発、30歩で10の25乗を超えます。対数スケールだと、発散は「右肩上がりの直線」として一目でわかります。境目を挟み撃ちにすると、このデータでは lr=0.029 は収束し 0.030 は発散——境界はおよそ 0.0296 にあります(この値がデータから決まる仕組みは、本シリーズでは深追いしません。「lr の上限はデータとモデルで変わる」という体感だけ持ち帰ってください)。

</details>

**問2(初期値を変える)** 初期値を $(0, 0)$、$(10, -5)$、$(-3, 8)$ に変えて訓練し(lr=0.01、5000ステップ。図4.3のコード)、学習曲線と最終的な $(w, b)$ を比べてください。実行前に「着地点は変わるか?」を予想すること。

<details><summary>略解</summary>

3本の曲線は出発点の高さ(loss 3529.87 / 191.36 / 4744.77)こそ違いますが、すべて同じ着地点 $(6.7221, 22.0922)$ に到着します。理由は第3章で見た地形です。MSE の地形は椀型で、くぼみが1つしかないので、どこから転がしても同じ底に着く。第2巻3章の演習で体験した「局所解にハマる」事故は、この地形では起こり得ません。ただしこれも線形回帰 + MSE の特権で、後の巻の地形では初期値が結果を左右するようになります。

</details>

**問3(手計算で1歩)** データが3点 $(0, 1), (1, 3), (2, 5)$、初期値 $(w, b) = (0, 0)$、lr $= 0.1$ とします。4.1節の式で $\partial L/\partial w$ と $\partial L/\partial b$ を手計算し、1歩更新した後の $(w, b)$ を求めてください。電卓可。

<details><summary>略解</summary>

予測はすべて 0 なので残差は $r = (-1, -3, -5)$。
$\partial L/\partial w = \frac{2}{3}(0 \cdot (-1) + 1 \cdot (-3) + 2 \cdot (-5)) = \frac{2}{3}(-13) = -\frac{26}{3} \approx -8.67$。
$\partial L/\partial b = \frac{2}{3}(-1 - 3 - 5) = -6$。
更新: $w \leftarrow 0 - 0.1 \times (-8.67) = 0.867$、$b \leftarrow 0 - 0.1 \times (-6) = 0.6$。どちらも増える向き——予測が低すぎるのだから妥当です。なおこの3点は一直線 $y = 2x + 1$ 上にあるので、回し続ければ loss は 0 に向かいます。

</details>

**問4(わざと壊す)** 4.2節のコードの update の符号を間違えて `w = w + lr * grad_w`(`b` も同様)としたら、学習曲線はどうなるでしょうか。予想してから実行してください。

<details><summary>略解</summary>

loss は1歩目から増え続け、発散します。勾配は「最も急な登り」の向きなので(第2巻4章)、符号を間違えると全力で坂を登るアルゴリズムになるからです。4拍子の最後のマイナス記号1つに、「下る」という意思のすべてが宿っています。

</details>
