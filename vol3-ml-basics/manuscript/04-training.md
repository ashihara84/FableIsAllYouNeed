# 第4章 学習 = 損失最小化 — 勾配降下で線形回帰を解く

材料は、すべて揃いました。

第2章で、私たちは最初のモデル——線形回帰を手に入れ、散布図に直線を「目で」当てはめました。第3章では、その当てはまりの悪さを1つの数にする道具——損失関数 MSE を定義し、$(w, b)$ 平面の上に広がる損失の地形が、きれいな椀型をしていることを自分の目で確かめました。そして第2巻で、私たちはすでに知っています。椀型の地形の底へ降りていく方法を。勾配を計算し、その逆向きに一歩ずつ進む——勾配降下法です。

つまり、いま私たちの手元には「モデル」「損失」「坂の下り方」の3点セットがあります。本章では、これらを初めてひとつに組み上げます。組み上がったものの名前は、**学習(training)**。この巻のタイトルにある「学習する」という営みの正体が、本章で目の前で動き出します。

やることは2段階です。まず、損失の坂の傾き $\partial L/\partial w$ と $\partial L/\partial b$ を**手で**導きます(第2巻で練習した連鎖律の、最初の実戦です)。次に、その勾配を使って坂を下る訓練ループを NumPy だけで書き、第2章の手動フィットと勝負させます。

## 4.1 ∂L/∂w, ∂L/∂b を手で導出 — 第2巻の連鎖律、最初の実戦

最小化したい相手を、まず正確に書き下します。データは第2章と同じ $n = 20$ 人ぶんの $(x_i, y_i)$——勉強時間と点数です。モデルは線形回帰

$$\hat{y}_i = w x_i + b$$

で、損失は第3章で定義した MSE です。

$$L(w, b) = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)^2$$

ひとつ記法の注意です。第2章ではこのモデルを、第1巻6章の `linear` をそのまま使って $X$ `(20, 1)` と $w$ `(1, 1)` の行列の形 `X @ w + b` で書きました。いまは変数が1つなので、$w$ の中身はただの数が1個です。手計算ではスカラー $w$ と書くのが素直なので、本章ではそうします。中身は一字も変わっていません。

さて、勾配降下に必要なのは $\partial L / \partial w$ と $\partial L / \partial b$ です(偏微分は第2巻4章——「他を止めて、1変数ずつ」でした)。$L$ の式は2乗や総和が入れ子になっていて、一見手強く見えます。こういうときの定石を、私たちは第2巻5章で手に入れています。**入れ子の関数は、計算グラフに分解して、辺ごとの微分を掛け算でつなぐ**——連鎖律です。あのとき練習用の小さな関数で素振りしたことを、いま本物の損失関数で実戦します。

データ1点ぶんの計算を、グラフに分解しましょう。第3章の残差 $r_i = \hat{y}_i - y_i$ を中継点に使うと、$w$ から $L$ までの道はこうつながっています。

$$w \;\longrightarrow\; \hat{y}_i = w x_i + b \;\longrightarrow\; r_i = \hat{y}_i - y_i \;\longrightarrow\; s_i = r_i^2 \;\longrightarrow\; L = \frac{1}{n}\sum_i s_i$$

辺は4本。1本ずつ微分します。どれも第2巻1章の基本ルールだけでできる微分です。

1. $L = \frac{1}{n}\sum_i s_i$ を $s_i$ で微分: $s_i$ の係数は $\frac{1}{n}$ なので $\dfrac{\partial L}{\partial s_i} = \dfrac{1}{n}$
2. $s_i = r_i^2$ を $r_i$ で微分: $\dfrac{\partial s_i}{\partial r_i} = 2 r_i$
3. $r_i = \hat{y}_i - y_i$ を $\hat{y}_i$ で微分: $y_i$ はデータ(定数)なので $\dfrac{\partial r_i}{\partial \hat{y}_i} = 1$
4. $\hat{y}_i = w x_i + b$ を $w$ で微分: $x_i$ もデータ(定数)なので $\dfrac{\partial \hat{y}_i}{\partial w} = x_i$

連鎖律の出番です。$w$ からデータ点 $i$ を通って $L$ に至る1本道に沿って、辺の微分を全部掛けます。

$$\frac{1}{n} \cdot 2 r_i \cdot 1 \cdot x_i = \frac{2}{n} r_i x_i$$

ここで第2巻5章のもうひとつの規則を思い出してください。**道が複数あるときは、道ごとの寄与を足す**。$w$ は20人ぶんのデータ点すべての予測に使われているので、$w$ から $L$ への道は20本あります。全部足して、

$$\frac{\partial L}{\partial w} = \frac{2}{n} \sum_{i=1}^{n} r_i x_i = \frac{2}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)\, x_i$$

$b$ も同じ道筋です。違いは最後の辺だけ——$\hat{y}_i = w x_i + b$ を $b$ で微分すると $1$ なので、$x_i$ が消えて、

$$\frac{\partial L}{\partial b} = \frac{2}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)$$

これで導出は完了です。あれだけ身構えた割に、出てきた式は拍子抜けするほど短い——**「残差に $x_i$ を掛けて平均して2倍」と「残差を平均して2倍」**。それだけです。

式を眺めて、意味を読み取っておきましょう。たとえば $(w, b) = (0, 0)$ から出発すると、予測はすべて 0 で、データはその上にあります。残差 $r_i = \hat{y}_i - y_i$ は軒並み負。すると $\partial L/\partial b = \frac{2}{n}\sum r_i$ も負で、更新則 $b \leftarrow b - \eta \cdot \partial L/\partial b$ は $b$ を**増やします**。予測が低すぎるなら直線を上へ——勾配の式は、私たちの直観と同じ向きを指しています。数式が信用できるかどうか迷ったら、こうやって符号の向きを言葉に翻訳してみるのが効きます。

もうひとつの検算手段も、私たちはすでに持っています。第2巻1章の数値微分です。$(L(w+h) - L(w-h)) / 2h$ を計算して、いま導いた式と突き合わせる——「手で導いた勾配は、数値微分と照合する」という習慣の出番です。次節のコードの冒頭で、実際にやります。

## 4.2 [コード] 線形回帰のフルスクラッチ実装 — forward → loss → gradient → update のループ

いよいよ組み立てです。学習のアルゴリズムを言葉で書くと、たった4行になります。

1. **forward**: 現在の $(w, b)$ で予測を出す — $\hat{y} = wx + b$(第2章のモデル)
2. **loss**: 予測の悪さを1つの数にする — $L = \mathrm{MSE}(\hat{y}, y)$(第3章の損失)
3. **gradient**: 損失の坂の傾きを求める — $\partial L/\partial w,\ \partial L/\partial b$(4.1節の式)
4. **update**: 傾きの逆向きに一歩進む — $w \leftarrow w - \eta\, \partial L/\partial w$(第2巻3章の勾配降下)

この4つを、この順番で、ひたすら繰り返す。本書はこれを**4拍子**と呼ぶことにします。forward、loss、gradient、update。1拍目で予測し、2拍目で採点し、3拍目で反省し、4拍目で直す。

先に、この章でいちばん大事な予告をしておきます。**この4拍子は、第8巻で Transformer を訓練するその日まで、一切変わりません。** モデルが直線から何億パラメータのネットワークに化けても、損失が MSE から cross-entropy(第4巻)に替わっても、勾配を手で導く代わりに自作の autograd(第5巻)が計算してくれるようになっても、ループの拍子そのものは今日書くこの4つのままです。今日のコードは使い捨ての練習ではなく、シリーズ最終巻まで使い続ける骨格の初版です。

では書きます。NumPy のみ、全文掲載・全文実行可能です。

```python
# 第3巻 第4章 4.2: 線形回帰のフルスクラッチ訓練(NumPyのみ)
# forward -> loss -> gradient -> update の「4拍子」。
# この4つの拍は、第8巻の Transformer の訓練ループまで一切変わらない。
import numpy as np


def make_data():
    """第2章と同一の合成データ(20人)。真の規則 y = 7x + 20 + ばらつき(標準偏差6)。"""
    rng = np.random.default_rng(42)
    X = rng.uniform(0, 9, size=(20, 1))       # 勉強時間 (20, 1)
    noise = rng.normal(0, 6.0, size=(20, 1))
    y = 7.0 * X + 20.0 + noise                # 点数 (20, 1)
    return X.ravel(), y.ravel()               # 本章はスカラー w で進めるので (20,) に潰す


def mse(y_hat, y):
    """第3章で定義した平均二乗誤差。"""
    return np.mean((y_hat - y) ** 2)


X, y = make_data()
assert np.allclose(X[:3], [6.96560444, 3.94990596, 7.72738128])  # 第2章と同じ20人

# --- 4.1 の検算: 手で導いた勾配を数値微分と突き合わせる(第2巻1章の習慣) ---
w0, b0 = 0.0, 0.0
r = (w0 * X + b0) - y               # 残差 (20,)
grad_w0 = 2.0 * np.mean(r * X)      # 手導出: ∂L/∂w = (2/n) Σ r_i x_i
grad_b0 = 2.0 * np.mean(r)          # 手導出: ∂L/∂b = (2/n) Σ r_i

h = 1e-6                            # 数値微分: (L(θ+h) - L(θ-h)) / 2h
num_w = (mse((w0 + h) * X + b0, y) - mse((w0 - h) * X + b0, y)) / (2 * h)
num_b = (mse(w0 * X + (b0 + h), y) - mse(w0 * X + (b0 - h), y)) / (2 * h)
assert np.allclose(grad_w0, num_w, atol=1e-4)
assert np.allclose(grad_b0, num_b, atol=1e-4)
print("勾配の検算 OK: 手導出 ({:.4f}, {:.4f}) = 数値微分 ({:.4f}, {:.4f})".format(
    grad_w0, grad_b0, num_w, num_b))

# --- 訓練ループ: 4拍子 ---
w, b = 0.0, 0.0     # 初期値
lr = 0.01           # learning rate(第2巻3章)。0.03 を超えると発散する(演習1)
num_steps = 5000
history = []        # 学習曲線用に loss を記録(4.3)

for step in range(num_steps):
    y_hat = w * X + b                        # 1. forward : 現在の (w, b) で予測
    loss = mse(y_hat, y)                     # 2. loss    : 悪さを1つの数にする
    grad_w = 2.0 * np.mean((y_hat - y) * X)  # 3. gradient: 4.1 で手導出した式
    grad_b = 2.0 * np.mean(y_hat - y)
    w = w - lr * grad_w                      # 4. update  : 坂を下る(第2巻3章)
    b = b - lr * grad_b
    history.append(loss)
    if step in (0, 1, 2, 3, 10, 100, 500, 1000, 2000):
        print("step {:>4}: loss = {:.6f}".format(step, loss))

print("step {:>4}: loss = {:.6f}".format(num_steps - 1, history[-1]))
print("学習結果: w = {:.4f}, b = {:.4f}".format(w, b))

# --- 検算 1: loss は一度も増えずに下がり続けたか ---
assert history[-1] < history[0]
for i in range(len(history) - 1):
    assert history[i + 1] <= history[i] + 1e-12
# --- 検算 2: 収束先(この値の種明かしは 4.4) ---
assert np.allclose([w, b], [6.7221, 22.0922], atol=1e-3)

# --- 第2章の演習の答え合わせ: 手動フィットと勝負する ---
w_hand, b_hand = 6.7, 23.0  # 第2章の演習3で筆者が記録した値
loss_hand = mse(w_hand * X + b_hand, y)
loss_gd = mse(w * X + b, y)
assert loss_gd < loss_hand  # 勾配降下の勝ち
print("手動フィット (w={}, b={}): loss = {:.6f}".format(w_hand, b_hand, loss_hand))
print("勾配降下                 : loss = {:.6f}".format(loss_gd))
print("ok: すべての assert を通過しました")
```

上から順に読んでいきます。

**勾配の検算。** 4.1節で手導出した式と数値微分が、$(w, b) = (0, 0)$ の地点で小数4桁まで一致しました。連鎖律の実戦、初陣は成功です。値の符号にも注目してください。$(-670.54, -113.62)$ ——どちらも負、つまり「$w$ も $b$ も増やせ」。4.1節の符号の読みどおりです。

**loss の推移。** 出発点では 3530 あった損失が、1歩で 464、3歩で 103。最初の数歩の下がり方は劇的です。これは第3章で見た地形を思い出すと腑に落ちます。出発点 $(0, 0)$ は椀の縁の高いところにあり、椀の縁ほど坂は急——勾配が大きいので、1歩が大きいのです。ただし、そのあとが長い。step 10 で 95 まで降りたのに、21.14 に落ち着くまでさらに2000歩かかっています。第3章の問4で見たとおり、この椀は $w$ 方向に急で $b$ 方向に緩い。急な $w$ の坂はあっという間に降りきり、残りの行程は $b$ 方向の浅い谷をゆっくり進むのです。step 2000 以降は小数6桁が動かなくなります。収束です。

**学習結果。** $(w, b) = (6.7221, 22.0922)$。ここで第2章の演習の答え合わせをしましょう。あなたが散布図に目で直線を当てはめて記録した $(w, b)$ は、いくつだったでしょうか。まず、第2章で決着のつかなかった論争から。候補C $(7, 20)$ の損失は 22.05、候補D $(6.5, 25)$ は 24.54——**Cの勝ち**でした(第3章の演習で地形に重ねた人は、もう知っていたはずです)。そして筆者の記録は $(6.7, 23)$ で、その損失は 21.77。勾配降下の答えは 21.14 です。目で当てはめた直線もかなり健闘したけれど、機械が4拍子を5000回刻んだ答えには届かない——そして勾配降下側の損失が「これ以上どの方向に動かしても下がらない」椀の底であることを、私たちは地形の形から知っています。

ところで、気づいた人がいるかもしれません。このデータの真の規則は $y = 7x + 20$ でした。なのに学習結果は $(6.7221, 22.0922)$ で、$(7, 20)$ ではありません。学習が失敗したのでしょうか? いいえ。勾配降下が最小化したのは「真の規則とのズレ」ではなく、**ばらつき込みの手持ちデータ20人に対する MSE** です。その仕事は完璧にやり遂げています(4.4節で証明します)。第2章の演習3で「この20人にとっての最良は、真の規則から少しずれる。どこへずれるかは第4章で」と予告した、その答えがこれです。手持ちのデータへの最適化と、まだ見ぬデータへの強さは別物ではないか——この疑念は正しく、第6章の主題になります。いまは頭の隅に置いておいてください。

最後にもう一度、ループ本体を眺めてください。`for` の中身は実質4行です。"training" という、論文が定義すらせずに使っている言葉の正体は、この4行の反復でした。

## 4.3 学習曲線 — loss の推移を眺める習慣

前節のコードでは、毎ステップの loss を `history` に記録しました。これを横軸 step、縦軸 loss でプロットしたものを**学習曲線(learning curve)**と呼びます。

なぜわざわざ描くのでしょうか。訓練ループの中身は、人間の目には速すぎて見えません。1000歩ぶんの試行錯誤を一目で見渡せる窓は、この曲線だけです。学習が順調なのか、止まっているのか、暴走しているのか——診断はすべてこの曲線から始まります。モデルが巨大になり訓練が何時間にも伸びる後の巻では、この曲線を眺めることが訓練の日常そのものになります。**ループを書いたら学習曲線を描く**。今日から習慣にしてください。

描くコードはこうです(matplotlib を使います。図は掲載のみ——手元で実行して同じ形が出ることを確かめてください)。後半の2つの図は章末の演習で使います。

```python
# 第3巻 第4章 4.3 と演習: 学習曲線を描く(本文掲載用。実行には matplotlib が必要)
# learning rate と初期値を変えて、学習曲線の形を見比べる。
import matplotlib.pyplot as plt
import numpy as np


def make_data():
    """第2章と同一の合成データ(20人)。"""
    rng = np.random.default_rng(42)
    X = rng.uniform(0, 9, size=(20, 1))       # 勉強時間 (20, 1)
    noise = rng.normal(0, 6.0, size=(20, 1))
    y = 7.0 * X + 20.0 + noise                # 点数 (20, 1)
    return X.ravel(), y.ravel()               # 本章はスカラー w で進めるので (20,) に潰す


def train(X, y, w=0.0, b=0.0, lr=0.01, num_steps=5000):
    """4.2 の4拍子。loss の推移(学習曲線)を返す。"""
    history = []
    for _ in range(num_steps):
        y_hat = w * X + b                        # forward
        loss = np.mean((y_hat - y) ** 2)         # loss
        grad_w = 2.0 * np.mean((y_hat - y) * X)  # gradient
        grad_b = 2.0 * np.mean(y_hat - y)
        w = w - lr * grad_w                      # update
        b = b - lr * grad_b
        history.append(loss)
    return w, b, history


X, y = make_data()

# --- 図4.1: 学習曲線(lr=0.01)。左: そのまま、右: 縦軸を対数に ---
_, _, history = train(X, y, lr=0.01, num_steps=3000)
fig, axes = plt.subplots(1, 2, figsize=(10, 4))
axes[0].plot(history)
axes[0].set_xlabel("step")
axes[0].set_ylabel("loss (MSE)")
axes[0].set_title("学習曲線(線形スケール)")
axes[1].plot(history)
axes[1].set_yscale("log")  # 平坦に見える部分の続きが見える
axes[1].set_xlabel("step")
axes[1].set_ylabel("loss (MSE)")
axes[1].set_title("学習曲線(縦軸を対数に)")
fig.tight_layout()

# --- 図4.2: learning rate を変える(演習1)---
plt.figure(figsize=(6, 4))
for lr in [0.001, 0.01, 0.05]:
    _, _, history = train(X, y, lr=lr, num_steps=100)
    plt.plot(history, label="lr = {}".format(lr))
plt.yscale("log")
plt.xlabel("step")
plt.ylabel("loss (MSE)")
plt.legend()
plt.title("learning rate と学習曲線")

# --- 図4.3: 初期値を変える(演習2)---
plt.figure(figsize=(6, 4))
for w0, b0 in [(0.0, 0.0), (10.0, -5.0), (-3.0, 8.0)]:
    w, b, history = train(X, y, w=w0, b=b0, lr=0.01, num_steps=5000)
    plt.plot(history, label="init w={}, b={} -> ({:.3f}, {:.3f})".format(w0, b0, w, b))
plt.yscale("log")
plt.xlabel("step")
plt.ylabel("loss (MSE)")
plt.legend()
plt.title("初期値と学習曲線(着地点はすべて同じ)")

plt.show()
```

図4.1: lr=0.01 の学習曲線。左は線形スケール、右は縦軸を対数にしたもの。

左の図に見えるはずのもの——最初の数歩で崖のように落ち、あとはほぼ水平な床に張りつく、L字型の曲線です。これが「順調な学習」の典型的なシルエットです。

ただし、左の図には弱点があります。step 500 と step 2000 の差(23.10 と 21.14)は、縦軸が 3530 から始まるグラフでは線の太さに埋もれて見えません。「もう収束したのか、まだじわじわ下がっているのか」が読めないのです。そこで右の図——**縦軸を対数にする**と、床に張りついて見えた部分が引き伸ばされ、step 2000 あたりまで着実に下がり続けてから水平になる様子が読めます。学習曲線は対数スケールでも眺める、と覚えてください。後の巻でも損失の動きは桁で変わるので、この習慣はずっと使います。

なお、今回の曲線は一度も増えずに滑らかに下がりました(コードの検算1で assert もしました)。これは「全データで勾配を計算する」「地形が椀型」という今の設定がくれた特権です。learning rate を欲張ると曲線が暴れ上がること(演習1)、そして第5章でデータを少しずつ食べるようになると曲線がギザギザになること——きれいな曲線との別れは、すぐそこです。

## 4.4 補足: 実は解析解もある — が、この道は通らない

最後に、白状しておくべきことがあります。

第2巻2章で「傾き = 0 を解けばいい、が通用しない関数が世の中のほとんど」と学びました。実は、線形回帰 + MSE は、その例外です。$\partial L/\partial w = 0$ と $\partial L/\partial b = 0$ は連立方程式として**そのまま解けて**、答えが1つの式で書けます。これを**解析解(closed-form solution)**と呼びます。一般の(変数が多い)線形回帰では、解は

$$\mathbf{w}^* = (X^T X)^{-1} X^T \mathbf{y}$$

という形になります。式の中の $(\cdot)^{-1}$ は**逆行列(inverse matrix)**——第1巻の序章で「第3巻で一度だけ、名前だけ登場する」と予告した、あの道具です。予告どおり、ここがその一度です。そして予告どおり、名前だけで通り過ぎます。

1000歩もループを回さなくても、一発の式で答えが出る。ならばなぜ、この近道を通らないのでしょうか。

理由は2つあります。第一に、**この近道は線形回帰 + MSE という特権的な組み合わせの外では消滅する**からです。モデルが少しでも非線形になると(早ければ本巻のエピローグで、その瞬間に立ち会います)、「傾き = 0」は手で解けなくなり、解析解は存在しなくなります。ニューラルネットワークにも、Transformerにも、解析解はありません。あの世界で通用する道は勾配降下だけです。つまり4.2節の4拍子は、解析解が使えないときの妥協案ではなく、**この先のすべての地形で通用する、唯一の本道**なのです。

第二に、最短測地線の原則です。逆行列を真面目に学ぶには、それだけで1章ぶんの寄り道が要ります。そして Transformer の訓練と推論には、逆行列は一度も登場しません。論文に出てこない道具のために章を費やさない——第1巻で行列式や固有値を切り捨てたときと同じ判断を、ここでも下します。

とはいえ「解析解と一致する」ことは、私たちの勾配降下の最高の検算になります。都合のいいことに、変数が1つの今回に限っては、解析解は逆行列なしの式に潰れます($\bar{x}, \bar{y}$ は平均です。導出はしません——検算に使うだけです)。

$$w^* = \frac{\sum_i (x_i - \bar{x})(y_i - \bar{y})}{\sum_i (x_i - \bar{x})^2}, \qquad b^* = \bar{y} - w^* \bar{x}$$

突き合わせてみましょう。

```python
# 第3巻 第4章 4.4: 解析解との答え合わせ
# 1変数の線形回帰に限り、解析解は逆行列なしの式で書ける。
# 勾配降下(4.2)が同じ答えに着いていることを確認する。
import numpy as np


def make_data():
    """第2章と同一の合成データ(20人)。"""
    rng = np.random.default_rng(42)
    X = rng.uniform(0, 9, size=(20, 1))       # 勉強時間 (20, 1)
    noise = rng.normal(0, 6.0, size=(20, 1))
    y = 7.0 * X + 20.0 + noise                # 点数 (20, 1)
    return X.ravel(), y.ravel()               # 本章はスカラー w で進めるので (20,) に潰す


def train(X, y, w=0.0, b=0.0, lr=0.01, num_steps=5000):
    """4.2 の4拍子を関数に畳んだだけ。中身は一字も変わっていない。"""
    for _ in range(num_steps):
        y_hat = w * X + b                        # forward
        grad_w = 2.0 * np.mean((y_hat - y) * X)  # gradient(loss は更新に不要)
        grad_b = 2.0 * np.mean(y_hat - y)
        w = w - lr * grad_w                      # update
        b = b - lr * grad_b
    return w, b


X, y = make_data()

# 勾配降下の答え
w_gd, b_gd = train(X, y)

# 解析解(1変数の特例。多変数では逆行列が必要になる)
x_mean, y_mean = X.mean(), y.mean()
w_star = np.sum((X - x_mean) * (y - y_mean)) / np.sum((X - x_mean) ** 2)
b_star = y_mean - w_star * x_mean

print("勾配降下: w = {:.6f}, b = {:.6f}".format(w_gd, b_gd))
print("解析解  : w = {:.6f}, b = {:.6f}".format(w_star, b_star))
assert np.allclose([w_gd, b_gd], [w_star, b_star], atol=1e-6)
print("ok: 勾配降下は解析解と一致しました")
```

```
勾配降下: w = 6.722065, b = 22.092189
解析解  : w = 6.722065, b = 22.092189
ok: 勾配降下は解析解と一致しました
```

小数6桁まで一致しました。5000回の小さな一歩は、一発の式と同じ場所——椀の底のただ一点に、正確に到着していたのです。4.2節の `assert np.allclose([w, b], [6.7221, 22.0922], atol=1e-3)` に書いた謎の数値の種明かしも、これで済みました。あれは解析解(を4桁に丸めたもの)だったのです。

## まとめ

- 学習とは損失最小化であり、その実体は **forward → loss → gradient → update の「4拍子」** の反復である。この4拍子は第8巻の Transformer の訓練ループまで一切変わらない
- MSE の勾配は連鎖律で手導出できる: $\partial L/\partial w = \frac{2}{n}\sum r_i x_i$、$\partial L/\partial b = \frac{2}{n}\sum r_i$(残差を平均して2倍、が骨格)。手導出した勾配は数値微分と照合する
- 勾配降下は第2章の手動フィット(loss 21.77)を破り(loss 21.14)、その着地点は解析解と小数6桁まで一致した
- **学習曲線**を描くのは訓練の日常。縦軸は対数スケールでも眺める
- 線形回帰には逆行列を使う解析解もあるが、その近道はニューラルネット以降は消滅する。このシリーズは通らない(最短測地線)

**ラスボスとの距離**: 論文 Section 5 の冒頭 *"This section describes the training regime for our models."* の "training" が指す営み——4拍子のループ——が、自分の手の中で動くようになりました。

**問3(手計算で1歩)** データが3点 $(0, 1), (1, 3), (2, 5)$、初期値 $(w, b) = (0, 0)$、lr $= 0.1$ とします。4.1節の式で $\partial L/\partial w$ と $\partial L/\partial b$ を手計算し、1歩更新した後の $(w, b)$ を求めてください。電卓可。

<details><summary>略解</summary>

予測はすべて 0 なので残差は $r = (-1, -3, -5)$。
$\partial L/\partial w = \frac{2}{3}(0 \cdot (-1) + 1 \cdot (-3) + 2 \cdot (-5)) = \frac{2}{3}(-13) = -\frac{26}{3} \approx -8.67$。
$\partial L/\partial b = \frac{2}{3}(-1 - 3 - 5) = -6$。
更新: $w \leftarrow 0 - 0.1 \times (-8.67) = 0.867$、$b \leftarrow 0 - 0.1 \times (-6) = 0.6$。どちらも増える向き——予測が低すぎるのだから妥当です。なおこの3点は一直線 $y = 2x + 1$ 上にあるので、回し続ければ loss は 0 に向かいます。

</details>
