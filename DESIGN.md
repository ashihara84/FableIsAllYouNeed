> claude.ai での設計議論からの引き継ぎ文書(2026-06-10)。
> Claude Code でのコンテンツ執筆・実装作業はこのファイルを正とする。

## プロジェクト概要

- **目的**: 「Attention Is All You Need」を読解し、Transformerをフルスクラッチ実装できるようになるための教科書シリーズ(全8巻)
- **裏テーマ(実験)**: 同じ仕様から異なるモデル世代で執筆し、質の向上を検証する。Opus版に対する Fable 版という位置づけ。モデル世代ごとの「版」概念(ヴィンテージ)として運用する
- **想定読者**: 高校〜大学初年級の数学知識 + プログラミング経験あり。機械学習の前提知識ゼロ
- **対象**: ゼミ学生 + 社内エンジニア
- **タイトル**: "Fable Is All You Need"(仮・第一候補)。原論文のパロディ + 「Fableさえあれば良い教科書が書ける」という検証可能な主張。fable = 寓話 の二重の意味も序文で言及する。

## 全巻共通の設計原則

1. **コードで示した方がわかりやすい概念は、コードで例を示す**。コードは目的ではなく説明の手段であり、毎巻必須ではない。数式や図の方が伝わる箇所に無理にコードを入れない
2. **論文をラスボスとして各巻の冒頭に置く**:
   - 巻頭で論文の該当箇所を原文掲示し「今は読めない。この巻を終えると読める」と宣言
   - 巻末で同じ箇所を再読し、読めるようになったことを確認する
3. **Transformerへの最短測地線のみを通る**。CNN・強化学習・ベイズ統計・SVM等の古典MLは意図的に省く。序文で「機械学習の教科書ではなく、この論文を読んで、Transformerを自分で実装できるようになるための教科書」と宣言する
4. **需要駆動の順序**: 道具は必要になってから導入する(ML入門→確率、の順序はこの原則による)
5. **巻頭付録「論文読解マップ」**: 論文の各セクション・各数式 ↔ 巻・章の対応表を全巻共通で掲載。読者が常に現在地を確認できる

## 巻構成(確定版)

### 第1巻: 線形代数 — すべては行列演算である
- ベクトル、行列、内積、行列積、転置
- NumPyで全演算を手実装
- ゴール: NNの計算は全部 `X @ W + b` だと腹落ちする。論文の `QK^T` が読める

### 第2巻: 微分と最適化 — 学習とは坂を下ること
- 偏微分、勾配、連鎖律、勾配降下法
- 2変数関数の最小値探しを可視化、連鎖律を計算グラフとして実装
- 論文の optimizer / learning rate warmup の前提

### 第3巻: 機械学習入門 — 「学習する」とは何か
- 題材は**回帰に限定**(1〜2巻の道具だけで完結させるため)
- 線形回帰 + MSE をフルスクラッチ実装
- 教師あり学習、損失関数、訓練/検証/テスト分割、過学習、汎化
- 「モデル = パラメータ付き関数、学習 = 損失最小化」の世界観を確立
- **巻末クリフハンガー**: 分類問題を出し、シグモイド + MSE で勾配が死ぬことをコードで観測させて終わる
  - シグモイドは「出力を0〜1に潰す関数」として道具的に先出し(確率論的正当化は4巻に持ち越し)

### 第4巻: 確率と情報量 — 分類の損失関数を求めて
- 3巻の宿題に答える巻として設計
- 確率分布、期待値、分散、尤度、最尤推定
- 「尤度最大化 = cross-entropy最小化」の導出がクライマックス
- 巻中盤で**ロジスティック回帰を実装し、3巻の分類問題を解決**
- softmax、KL、perplexity
- **√d_k の伏線**: 内積の分散の議論をここで仕込む(論文の scaled dot-product の "scaled" の根拠)

### 第5巻: ニューラルネットと誤差逆伝播 — 自動微分を自作する
- MLP、活性化関数、backprop
- ミニautograd(micrograd相当)の自作
- residual connection、layer norm、dropout もここで先行導入
- 論文のアーキテクチャ図の部品の大半がこの巻で揃う

### 第6巻: 言語をベクトルにする — TransformerまでのNLP
- トークン化(BPE)、埋め込み、言語モデルという問題設定
- n-gram → RNN/LSTM → seq2seq → attention付きseq2seq
- この巻の役割は歴史ではなく**動機**: RNNの並列化不能・長距離依存の苦手さを実装で体感させ、「attentionだけでいい」というタイトルの意味を刺す

### 第7巻: Attention Is All You Need 精読
- 論文のセクション順に1行ずつ読解
- scaled dot-product attention、multi-head、positional encoding、masking、encoder-decoder構造
- 各コンポーネントを単体実装 + テスト
- ゴール: 「論文の式番号 ↔ 自分のコードの行」が1対1対応する状態

### 第8巻: 全部組み立てて訓練する
- フルTransformerの組み立て、訓練ループ
- Adam + warmup、label smoothing、データ準備
- 小規模コーパスでの訓練(nanoGPT規模)、評価
- 最終章のみ: GPT/BERTへの分岐の概観(現代への橋)

## 設計判断の記録(なぜこうしたか)

- **3巻(ML入門)を4巻(確率)より先にした理由**: 確率を先にやると抽象的すぎて脱落者が出る。「cross-entropyが必要になった、だから確率をやる」という需要駆動にした。これによりロジスティック回帰が「ML入門の例題その2」から「確率の巻のクライマックス」に昇格し、各巻の主役が明確になった(3巻=学習という概念、4巻=cross-entropyの導出)
- **難易度カーブ**: 数学・数学・息継ぎ・数学・実装・実装・精読・実装。脱落ポイントと想定される2〜3巻の境目が軽くなる
- **古典MLを全部切る理由**: シリーズの差別化。網羅性ではなく1本の論文への最短経路
