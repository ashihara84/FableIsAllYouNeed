# 第5章 RNN — 記憶を持つネットワーク

> [目次](../TOC.md) ・ [← 前の章](04-ngram.md) ・ [次の章 →](06-seq2seq.md)

前章の結末を思い出してください。n-gram は数えるだけで文章を生成しましたが、$n$ を増やすと組合せ爆発でデータが足りなくなり、テストの perplexity は無限大に発散しました。教訓は「数える」のではなく「一般化する」必要がある、でした。

一般化する道具は、もう持っています。第3章でトークンは埋め込みによってベクトル $\mathbf{x}$ になり、第5巻ではベクトルを入力にとるネットワークと、それを訓練する autograd を自作しました。ならば、直前の文字たちの埋め込みを MLP に入れて次の文字を当てさせればよさそうです。

ところが関門があります。MLP の入力は**固定長** `(B, d)` で、長さがまちまちの文をそのまま受け取れません。「直前 $n-1$ 文字に切り詰める」なら動きますが、それは n-gram の割り切りをニューラルの靴に履き替えただけで、遠くの文脈が見えないという前章の壁は残ります。

文頭から**ここまでの全部**を、可変長のまま見たい。この需要に答えたのが、この章の主役 RNN です。RNN は n-gram に勝ちます——それを確認した直後に、私たちは2つの「痛み」を実測します。この痛みこそ、巻頭に掲げたラスボス(論文の Introduction と Background)の正体です。

## 5.1 発想: 隠れ状態 h を持ち回り、1トークンずつ読む — h = 「ここまでの要約」

いまこの本を読んでいるあなたの頭の中には、全文字の暗記ではなく「あらすじ」——どんな概念が出て、いま何の話か、の要約——があります。新しい一文を読むたび、そのあらすじを少しだけ更新する。**固定サイズの要約を持ち歩き、1単位読むごとに更新する**わけです。

これを形式化します。要約の置き場所として、**隠れ状態**(hidden state)$\mathbf{h}$ `(d_h,)` を用意し、「1トークン読んで要約を更新する」規則を、既知の道具——線形変換と活性化関数——で書きます。

$$\mathbf{h}_t = \tanh(\mathbf{x}_t W_x + \mathbf{h}_{t-1} W_h + \mathbf{b})$$

shape で読みます。$\mathbf{x}_t$ `(d_e,)` は位置 $t$ のトークンの埋め込み(第3章)。$W_x$ `(d_e, d_h)` がそれを要約の言葉に翻訳します。$\mathbf{h}_{t-1}$ `(d_h,)` は1つ前までの要約で、$W_h$ `(d_h, d_h)` がそれを新しい要約へ持ち越します。2つの寄与とバイアス $\mathbf{b}$ `(d_h,)` を足して `(d_h,)`、tanh は要素ごとなので $\mathbf{h}_t$ も `(d_h,)`。つまり「**新しい要約 = tanh(いま読んだもの + 古い要約)**」です。バッチ処理では $\mathbf{x}_t$ が $X_t$ `(B, d_e)`、$\mathbf{h}_t$ が $H_t$ `(B, d_h)` になるだけです。

自分の出力 $\mathbf{h}$ が次の自分の入力に戻る——この再帰のかたちから、この仕組みは**再帰型ニューラルネットワーク**(recurrent neural network、RNN)と呼ばれます。式は1行ですが、観察点が2つあります。

第一に、**パラメータが $t$ に依りません**。$W_x, W_h, \mathbf{b}$ の1組を、位置1でも位置1000でも使い回します。だから RNN は、何文字の入力でも同じパラメータ数で読める——冒頭の「固定長の関門」はこれで突破です。

第二に、活性化が tanh です(第5巻1章で紹介した、値域 $(-1, 1)$ のあの関数)。$\mathbf{h}$ は何百回も更新を重ねるので、値域を縛って爆発を防げるのは好都合です。ただし「両端で飽和する」体質も連れてきます——この代金は 5.4 で支払います。

予測は、各位置の要約から作ります。

$$\mathrm{logits}_t = \mathbf{h}_t W_{out} + \mathbf{b}_{out} \quad \text{(} W_{out} \texttt{(d\_h, V)} \text{、結果は } \texttt{(V,)} \text{)}$$

これを softmax に通せば $P(\text{次のトークン} \mid \text{ここまで全部})$ です。n-gram の条件部は「直前 $n-1$ 個」でしたが、RNN の条件部は「文頭からの全部(の要約)」。視野の割り切りが、定義の上では消えました。

時間方向に展開して描くと、構造がよく見えます。

```
 x_1        x_2        x_3              x_L        各 (d_e,)
  │          │          │                │
  ▼          ▼          ▼                ▼
h_0 ──→ ◯ ──→ h_1 ──→ ◯ ──→ h_2 ──→ … ──→ ◯ ──→ h_L   各 (d_h,)
        │           │                    │
        ▼           ▼                    ▼
    logits_1    logits_2             logits_L      各 (V,)
```

図5.1: 時間方向に展開した RNN。◯ はすべて**同じ** $W_x, W_h, \mathbf{b}$ による更新。$h_0$ はゼロベクトル(何も読んでいない状態)。横一列の $h$ のバケツリレーが、この章のすべての主題です。

## 5.2 [コード] RNN 言語モデルを実装・訓練する — n-gram より良い perplexity を確認

道具は第5巻5章で自作した行列版 autograd(`tensor_autograd.py` の `Tensor`)です。1つだけ tanh が足りないので、同じ流儀で自分の章のコードに足します。

```python
def tanh(t):
    """RNN の活性化関数。微分は 1 - tanh^2(最大でも 1、両端では 0 に潰れる)。"""
    out = Tensor(np.tanh(t.data), (t,))

    def _backward():
        t.grad += (1.0 - out.data ** 2) * out.grad

    out._backward = _backward
    return out
```

docstring の「微分は最大でも1」は、いまは検収項目の1つですが、5.4 で主役になります。

題材は**文字レベル**の言語モデルです。コーパスはイソップ寓話の再話11編、計4,622文字・語彙29文字(全文は `code/ch05/rnn_lm.py`)。第2章の「文字は系列が長すぎる」弱点は承知の上で、語彙が小さくモデルを小さく保て、長い系列はこの章の目的——系列方向の痛みの実測——にむしろ好都合だからです。

データは100文字の塊に切り、7つに1つを検証用に抜きます(第3巻6章の分割の規律)。末尾の1編を丸ごと検証に回すと初見の単語だらけになるため、検証の塊はコーパス全体から満遍なく取ります。

パラメータは、埋め込み行列 $E$ `(V, d_e)`、5.1 の $W_x$ `(d_e, d_h)`・$W_h$ `(d_h, d_h)`・$\mathbf{b}$ `(d_h,)`、予測ヘッドの $W_{out}$ `(d_h, V)`・$\mathbf{b}_{out}$ `(V,)` の一式です(初期化は `rnn_lm.py` の `init_params`。shape を声に出して読んでください——第1巻3章の儀式です)。

心臓部は、5.1 の式をそのままコードにした forward です。

```python
def forward_chunk(params, X_ids, Y_ids):
    """長さ L のひと区切りを 1 トークンずつ読み、各位置の損失の平均を返す。
    X_ids, Y_ids: (B, L) の整数配列。Y は X を 1 文字ずらしたもの(次の文字が正解)。
    この for ループこそが RNN の心臓部であり、同時に痛み1の現場でもある。"""
    B, L = X_ids.shape
    V = params["E"].data.shape[0]
    d_h = params["W_h"].data.shape[0]
    h = Tensor(np.zeros((B, d_h)))                      # h_0: 何も読んでいない状態
    loss = Tensor(0.0)
    for t in range(L):                                  # ← 逐次。t は t-1 を待つ
        x_t = Tensor(onehot(X_ids[:, t], V)) @ params["E"]          # (B, d_e)
        h = tanh(x_t @ params["W_x"] + h @ params["W_h"] + params["b"])  # (B, d_h)
        logits = h @ params["W_out"] + params["b_out"]               # (B, V)
        loss = loss + softmax_cross_entropy(logits, Y_ids[:, t])
    return loss * (1.0 / L)
```

`onehot(...) @ E` は第3章 3.2 の「lookup はただの行列演算」です。各位置で logits を出し、正解(1文字先)との cross-entropy を取り、平均する。backward は**1行も書きません**——`loss.backward()` を呼べば、この for ループが作った計算グラフを autograd が逆走してくれます(第5巻の成果)。

訓練は第3巻4章以来の4拍子(forward → loss → gradient → update)のままで、モデルが RNN になっても変わりません。$d_e = 24$、$d_h = 48$、バッチ16本・長さ32文字の区切りで600ステップ回します(終盤200ステップは学習率を $1/5$ に絞って仕上げ)。比較台の n-gram も同じ訓練/検証データで評価します。検証側には未見の並びが必ず出て perplexity が無限大に発散するため(前章4.3の壁)、出現回数に下駄 $k = 0.1$ を履かせる平滑化でゼロだけ避けてあります。

`code/ch05/rnn_lm.py` を実行した結果が表5.1 です(乱数 seed 42、実行約2秒)。

| モデル | 検証 perplexity(平均分岐数) |
|---|---|
| あてずっぽう(一様分布) | 29.00 |
| bigram(数える + 平滑化) | 9.23 |
| trigram(数える + 平滑化) | 7.28 |
| **RNN 言語モデル** | **6.85** |

表5.1: 同一データでの検証 perplexity。値は「次の1文字を選ぶ実質の択数」(第4巻7章)。コードでは `assert ppl_rnn < ppl_trigram < ppl_bigram` でこの順位を固定してある。

RNN が勝ちました。数字の差以上に、勝ち方に注目してください。trigram は「見たことのある並び」の表を引き、初見には平滑化の穴埋めでしのぎます。RNN には表がありません。埋め込みと重み行列を通して、見たことのない並びにも滑らかに確率を割り振る——前章4.4で求めた「数えるのではなく一般化する」が、初めて実現した瞬間です。

生成もさせてみます(前章と同じ、機械が文を書く体験です)。

```
ther, you hes walf the word agopr bis lased, wnlf be the mog sblghe gaiened ...
```

まだ寝言です。それでも、単語の長さの分布、冠詞 the の頻出、寓話の語彙の断片(walf、word)——4,622文字しか読んでいないモデルとしては健闘しています。

ここで章を終えれば「RNN は良いものだ」で済みます。しかし本巻の目的は、Transformer が解決した問題を自分の足で踏むこと。いまから2つ、踏みます。

## 5.3 痛み1: 並列化できない — $h_t$ は $h_{t-1}$ を待つ

`forward_chunk` の `for t in range(L)` ——この for ループは消せません。$\mathbf{h}_t$ を計算するには $\mathbf{h}_{t-1}$ が**完成している**必要があるからです。位置1から $L$ までの計算は、原理的に1列に並んで順番を待つしかありません。

なぜ「痛み」なのか。第1巻4章のベンチマークでは、3重ループの行列積と `@` 1発の差は数千倍でした。教訓は「forループを行列演算に潰して**一斉に**やる」ことが現代の数値計算の生命線だ、ということ。このコードでもバッチ方向にはその武器が効き、16本の系列の $\mathbf{h}_t$ は `(16, d_h)` の行列として1発で更新されます。しかし**時間方向には、この武器が原理的に使えません**。

実測します。系列を長くすれば仕事が増えて遅くなるのは当たり前なので、**仕事の総量を固定**します。1ステップで処理するトークン数 $B \times L = 512$ を保ったまま、細切れの度合い $L$ だけを変えます。

```python
for B, L in [(64, 8), (32, 16), (16, 32), (8, 64), (4, 128)]:
    times[L] = one_step_time(B, L)   # forward + backward 1回の時間(7回測って最小値)
```

`code/ch05/rnn_pains.py` を実行した結果が表5.2です。

| B | L | 1ステップの時間 | 対 L=8 比 |
|---|---|---|---|
| 64 | 8 | 1.03 ms | ×1.0 |
| 32 | 16 | 1.53 ms | ×1.5 |
| 16 | 32 | 2.80 ms | ×2.7 |
| 8 | 64 | 5.32 ms | ×5.2 |
| 4 | 128 | 9.10 ms | ×8.8 |
| (参考)同じ512トークンを1発の行列積で | | 0.46 ms | ×0.4 |

表5.2: 総トークン数 512 を固定し、系列長 $L$ だけを変えたときの訓練1ステップの所要時間。処理する仕事の総量は全行で同じ。

**仕事の総量は1行目から5行目まで完全に同じ**なのに、$L$ を16倍にすると時間は約9倍——ほぼ $L$ に比例して伸びます。原因は行列積の性質です。`@` が速いのは「大きな塊を一斉に」処理するからで、同じ仕事を $L$ 回の小さな呼び出しに刻むと一斉の旨味が消えます。参考行が決定打です。時間方向の依存がなく512トークンを1枚の行列で流せたなら、0.46 ms——$L=128$ の RNN の**20倍速**です。

これは CPU の NumPy での話で、数千個の演算器で「一斉に」を極限まで追求した GPU では、この待ちの損失はさらに容赦なく効きます。論文が扱う機械翻訳の系列は、128文字どころではありません。

巻頭のラスボスの1文目を見てください。

> *"This inherently sequential nature precludes parallelization within training examples, which becomes critical at longer sequence lengths, as memory constraints limit batching across examples."*
> — Vaswani et al., "Attention Is All You Need", Section 1 Introduction
>
> 訳: この本質的に逐次的な性質は、訓練事例の内部での並列化を不可能にする。メモリ制約が事例をまたいだバッチ化を制限するため、これは系列長が長くなるほど深刻になる。

"This inherently sequential nature" の "This" は、$\mathbf{h}_t$ が $\mathbf{h}_{t-1}$ から作られるあの更新規則です。"precludes parallelization **within** training examples" ——事例の**内部**、つまり時間方向で並列化できない。表5.2 の右肩上がりがそれです。続く "batching across examples" は私たちのバッチ方向 $B$、唯一残された逃げ道ですが、長い系列ほど1事例あたりのメモリが膨らみ、束ねられる本数 $B$ が減る——逃げ道もメモリで頭打ちになる、と論文は言っています。序章では一語も読めなかった文が、いまはあなたの実測値の要約として読めるはずです。

## 5.4 痛み2: 長距離依存 — 第5巻6.1の再演が系列方向で起きる

2つ目の痛みは、速度ではなく**記憶**の話です。

$\mathbf{h}$ は `(d_h,)` の固定長ベクトルで、1文字読むたびに $W_h$ と tanh を通して**丸ごと作り直され**ます。100文字前の情報は、100回の作り直しを生き延びた分しか残りません。要約とはそういうもので、新しいことを書き入れるには古いことを薄めるしかないのです。

学習の立場で見ると、より正確に記述できます。位置 $t$ の損失が $d$ 文字前の入力 $\mathbf{x}_{t-d}$ から学ぶには、勾配が連鎖律(第2巻5章)で $d$ 個の更新ステップを逆走します。1ステップ逆走するたび、勾配には「tanh の微分」と「$W_h$」が掛かります。tanh の微分は最大でも1、飽和帯域ではほぼ0です(5.2 の docstring のあの性質)。1未満の係数を $d$ 回掛ければ、指数的に減衰します。

この景色には見覚えがあるはずです。**第5巻6.1**——10層の MLP で、出口の勾配が入口に届く頃には消えていた、あの勾配消失(対数軸で一直線に落ちるグラフ)です。図5.1 をもう一度見てください。時間方向に展開した RNN は、長さ $L$ の系列に対しては**重みを共有した $L$ 層の深いネットワーク**にほかなりません。深さ方向の病気が、系列方向で再演されます。しかも MLP なら層数は設計者が減らせましたが、**系列の長さは課題が決めます**。文が長いことを、私たちは変えられません。

実測します。損失を**最後の1位置だけ**に置き、そこから各位置の入力 $\mathbf{x}_{t}$ に届く勾配のノルムを測ります。「$d$ 文字前の入力は、いまの予測の改善にどれだけ寄与できるか」の直接測定です。

```python
h = Tensor(np.zeros((B, d_h)))
xs = []                                              # 各位置の入力ベクトルを覚えておく
for t in range(L):                                   # L = 64
    x_t = Tensor(onehot(X[:, t], V)) @ params["E"]
    xs.append(x_t)
    h = tanh(x_t @ params["W_x"] + h @ params["W_h"] + params["b"])
logits = h @ params["W_out"] + params["b_out"]       # 損失は最後の1位置だけに置く
loss = softmax_cross_entropy(logits, targets)
loss.backward()
# 距離 d の入力への勾配ノルム = np.linalg.norm(xs[L - d].grad, axis=1).mean()
```

| 損失からの距離 $d$ | $\|\partial \mathrm{loss} / \partial \mathbf{x}\|$ | 対 $d=1$ 比 |
|---|---|---|
| 1 | 1.98e-02 | 1.000 |
| 2 | 1.99e-02 | 1.004 |
| 4 | 1.64e-02 | 0.828 |
| 8 | 9.89e-03 | 0.500 |
| 16 | 4.91e-03 | 0.248 |
| 32 | 1.08e-03 | 0.055 |
| 64 | 9.93e-05 | 0.005 |

表5.3: 最後の位置の損失から、距離 $d$ だけ離れた入力に届く勾配のノルム。距離8で半分、距離32で5%、距離64で0.5%。対数で見ればほぼ直線——第5巻6.1の図と同じ、指数的減衰の形。

距離64の入力に届く学習信号は、直前の入力の **0.5%** です。コーパスの「狼少年」では、村人が助けに来ない理由は何十文字も前の「嘘の叫び」にあります。文頭の主語が文末の動詞の形を決める、代名詞の指し先が数文前にある——言語はこうした**長距離依存**(long-range dependency)に満ちています。しかし勾配がほぼゼロなら、その対応関係は**学習されません**。遠い手がかりの使い方を、モデルは覚えたくても覚えられないのです。

ラスボスの2文目が、これです。

> *"...the number of operations required to relate signals from two arbitrary input or output positions grows in the distance between positions... This makes it more difficult to learn dependencies between distant positions."*
> — 同論文, Section 2 Background
>
> 訳: 任意の2つの位置の信号を関係づけるのに必要な演算数が、位置間の距離とともに増える。(中略)これにより、離れた位置どうしの依存関係を学習することが難しくなる。

RNN で位置 $t-d$ と位置 $t$ を関係づけるには、$d$ 回の更新という**中継**を通すしかありません。演算数は距離に比例して増え("grows in the distance")、中継のたびに信号も勾配も薄まり、遠い依存関係は学習困難になる("more difficult to learn dependencies between distant positions")。表5.3 は、この2文の実測版です。

第5巻6.3では深さ方向の救済——residual connection、「+1 の素通りの迂回路」——を学びました。時間方向にも同じ発想の救済が考案されています。それが次節です。

## 5.5 LSTM: ゲートで記憶を守る改良(概観のみ)— それでも2つの痛みは残る

痛み2の原因は「毎ステップ、要約を**丸ごと作り直す**から」でした。ならば作り直しをやめ、覚えておくべきものは素通りさせ、書く・消す・読むを**選択的に**行えばいい——この到達点が **LSTM**(Long Short-Term Memory)です。

LSTM は、隠れ状態 $\mathbf{h}$ に加えて**セル状態**(cell state)$\mathbf{c}$ `(d_h,)` を持ち回ります。$\mathbf{c}$ はベルトコンベアで、原則そのまま次の時刻へ流れ、その流れに3つの**ゲート**(gate)——シグモイドが出す 0〜1 を要素ごとに掛ける蛇口——が介入します。

```
        忘却ゲート f_t          入力ゲート i_t
        「どこを消す?」        「どこに書く?」
             │ (0〜1)               │ (0〜1)
             ▼                      ▼
c_{t-1} ──→ [× f_t] ──────→ [ + i_t ⊙ 新情報 ] ──→ c_t ──→ (次の時刻へ素通り)
                                                    │
                                              [tanh]·[× o_t]   出力ゲート o_t
                                                    │          「どこを読む?」
                                                    ▼
                                                   h_t
```

図5.2: LSTM の直観図。主役は上段のベルトコンベア $\mathbf{c}$。更新は $\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t$ ——掛け直しではなく**加算**で、ゲートが開いている要素はそのまま生き残る。

注目すべきは $\mathbf{c}$ の更新が**加算**であることです。第5巻6.3 の residual connection が深さ方向に作った「素通りの迂回路」を、時間方向に作った——そう読めます。忘却ゲートが開いている($\mathbf{f}_t \approx 1$ の)要素では、勾配は減衰せず過去へ流れます。記憶を守るために、守る場所と捨てる場所をデータから学習する。論文の Introduction 冒頭にも、当時の最先端としてその名前が刻まれています。

> *"Recurrent neural networks, long short-term memory and gated recurrent neural networks in particular, have been firmly established as state of the art approaches in sequence modeling..."*
> — 同論文, Section 1 Introduction
>
> 訳: 再帰型ニューラルネットワーク、とりわけ LSTM とゲート付き再帰ネットワークは、系列モデリングにおける最先端の手法としての地位を確立してきた。

本シリーズでは、LSTM の内部(3つのゲートの式)は**実装しません**。ここまでの概観が、Transformer への最短測地線上で必要なすべてで、ゲートの式は論文のどこにも登場しないからです。

実装しない代わりに、LSTM が2つの痛みを治したかを確認します。

**痛み1は、まったく残ります。** $\mathbf{c}_t$ も $\mathbf{h}_t$ も、$t-1$ の完成を待つことに変わりはありません。むしろゲートが増えた分、1ステップの計算量は素の RNN の約4倍。逐次性という病根には指一本触れていません。

**痛み2は、緩和されますが解決しません。** ゲートは 0〜1 の値で、完全に開きっぱなしでない限り信号は距離とともにやはり目減りします。固定長の要約に文全体を詰め込む構造も同じです。実用上、LSTM が扱える依存の距離は素の RNN より大きく伸びました——しかし「**任意の距離を、等しいコストで**」には届きません。距離との戦いを、戦いやすくしただけです。

改良の道は、ここで頭打ちが見えています。論文の著者たちが選んだのは、さらなる改良ではなく**除去**でした——結末は第7章で見ます。次章ではまず、この RNN を2本つないで翻訳に挑み、3つ目の痛みを踏みます。

## まとめ

- RNN は固定長の隠れ状態 $\mathbf{h}$(=ここまでの要約)を持ち回り、$\mathbf{h}_t = \tanh(\mathbf{x}_t W_x + \mathbf{h}_{t-1} W_h + \mathbf{b})$ で1トークンずつ更新する。パラメータは全位置で共有され、可変長の入力を読める
- 文字レベル言語モデルとして訓練すると、検証 perplexity 6.85 で trigram(7.28)に勝つ。表引きと穴埋めではなく「一般化」が起きている(第4章の教訓の回収)
- **痛み1(並列化できない)**: $\mathbf{h}_t$ は $\mathbf{h}_{t-1}$ を待つ。仕事の総量を固定しても訓練時間は系列長 $L$ にほぼ比例(実測: $L$ 16倍で約9倍)。第1巻4章の「行列積で一斉に」が時間方向には効かない
- **痛み2(長距離依存)**: 学習信号は距離とともに指数的に減衰(実測: 距離64で直前の0.5%)。第5巻6.1の勾配消失の、系列方向での再演。遠い依存関係は学びたくても学べない
- LSTM はゲートと加算のセル(時間方向の residual)で痛み2を緩和したが、逐次性はそのまま、距離との戦いも本質的に残る

**ラスボスとの距離**: Introduction の "This inherently sequential nature precludes parallelization within training examples" と Background の "grows in the distance between positions" が、表5.2・表5.3 という自分の実測値の要約として読めるようになりました。

## 演習

**問1(実測: 系列長と訓練時間)** `rnn_pains.py` の表5.2 に $(B, L) = (2, 256)$ の行を追加し、時間が $L$ にほぼ比例して伸び続けることを確かめてください。また、各行について「1トークンあたりの時間(= 1ステップの時間 ÷ 512)」を計算し、$L$ とともにどう変わるか観察してください。

<details><summary>略解</summary>

$L=256$ では対 $L=8$ 比でおよそ17〜20倍になります(環境により揺れますが、比例傾向は崩れません)。仕事の総量が同じなので、1トークンあたりの時間も同じ比率で増えます。原因は「一斉に」の喪失です。$L$ が増えるほど行列積1回あたりの塊($B$ 行)が小さくなり、呼び出し回数だけが増える——細切れの固定費が支配的になります。

</details>

**問2(実測: 訓練後の勾配ノルム)** 表5.3 は訓練**前**(初期化直後)のパラメータでの測定でした。`rnn_lm.py` で600ステップ訓練した**後**のパラメータで同じ測定を行うと、減衰は消えるでしょうか。予想してから実測してください。

<details><summary>略解</summary>

消えません。個々の数値は変わりますが(訓練で $W_h$ のスケールが調整されるため)、距離に対する指数的減衰の形は残ります。減衰の原因は「tanh の微分 ≤ 1 と $W_h$ を距離の回数だけ掛ける」という**構造**にあり、訓練はその構造を変えられないからです。

</details>

**問3(考察: 記憶の運び手はどの項か)** 更新式から $\mathbf{h}_{t-1} W_h$ の項を取り除く($W_h = 0$ に固定する)と、この言語モデルは実質的に何と同等になるでしょうか。

<details><summary>略解</summary>

$\mathbf{h}_t = \tanh(\mathbf{x}_t W_x + \mathbf{b})$ となり、$\mathbf{h}_t$ は**直前1トークンだけ**の関数になります。つまり $P(\text{次} \mid \text{直前の1文字})$——ニューラル版の bigram です。文頭からの記憶を運んでいたのは $\mathbf{h}_{t-1} W_h$ の項だった、と裏から確認できます。そして皮肉なことに、この項こそが痛み1(待ち)と痛み2(減衰)の発生源でもあります。記憶と痛みは、RNN では同じ場所から生まれているのです。

</details>
