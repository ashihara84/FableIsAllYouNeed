# 第3章 誤差逆伝播 — 連鎖律を逆向きに流す

前章の最後で、私たちは地獄を見ました。たった2層のMLPの $\partial L / \partial W_1$ を手で展開しただけで、添字だらけの式が紙を埋め尽くしたのです。あの導出は間違ってはいません。しかし、層が3つになったら? 隠れ層の幅が512になったら? パラメータを1つ更新するたびにあの作業をやり直すのなら、学習どころではありません。

ところで、思い出してほしいことがあります。第2巻5章で、私たちはすでに答えの原型を握っていました。計算をグラフに描き、forward で値を流し、その値を使って逆向きに微分を流す——あの小さな `forward_chain` / `backward_chain` です。あのとき扱った値はスカラーでした。この章でやることは、ただひとつ。**あの仕組みを、行列が流れるグラフにそのまま持ち込む**ことです。

この章はコードをほとんど書きません。紙と図が主役です(自動化のコードは次章でたっぷり書きます)。そのかわり、章を終えたとき、前章の地獄が「6行の機械的な手続き」に畳み込まれる瞬間を目撃することになります。

## 3.1 計算グラフの再登場(第2巻5章): forward で値を、backward で勾配を流す

第2巻5章の道具を、棚から下ろしましょう。覚えていることを3つ、確認します。

1. **計算グラフ**(computational graph): 計算の手順を、演算のノードと値の流れる辺で描いた図
2. **局所微分**: 各ノードが、グラフ全体を見ずに、自分の入出力だけで計算できる微分。微分とは「グラフの辺をたどる掛け算」であり、パスが複数あれば足す
3. **2段構え**: 先に前向きに値を流し(中間値を覚えておき)、その値を使って逆向きに微分を流す。逆向きの1往復で、すべての変数の偏微分が同時に手に入る

第2巻5章では、この2段構えに「正式な名前が付きます」と予告していました。ここで回収します。前向きに値を流す計算を**順伝播**(forward pass)、逆向きに微分を流す計算を**逆伝播**(backward pass)と呼びます。そして、逆伝播で損失の勾配を求める方法の全体が、章題の**誤差逆伝播法**(backpropagation)——通称 backprop です。なぜ「誤差」なのかは、3.3節で流れるものの正体を見たときにわかります。

では、第2巻のあのコードと、いま目の前にあるMLPとの間には、何が足りないのでしょうか。

違いは2つあります。第一に、**流れる値がスカラーから行列になりました**。前章の forward では、辺の上を $(N, d_h)$ のような行列が流れています。「局所微分」は何になるのでしょうか。行列を行列で微分する——考えただけで前章の地獄が再来しそうです。第二に、第2巻のグラフは演算が3つの一本道でしたが、MLPのグラフには $W_1, \mathbf{b}_1, W_2, \mathbf{b}_2$ という4つのパラメータがぶら下がっていて、その**全部**の勾配が欲しい。

この2つの違いを埋めるのが、この章の残りの仕事です。まず、スカラーのままでもう一段だけ見方を強くします(3.2節)。その見方を持って、行列の世界に入ります(3.3節)。

## 3.2 鍵となる見方: 各ノードは「自分の局所勾配 × 上流から来た勾配」を下流に渡すだけ

第2巻5章のルールは「**パスごとに**局所微分を掛け、合流したら足す」でした。実はこのルール、そのままでは深いネットワークに使えません。なぜでしょうか。

パスの数を数えてみるとわかります。第2巻5章の演習で、$x^{100}$ には入力から出力へのパスが100本ありました。層を重ねたネットワークでは事情はさらに深刻で、あるパラメータから損失へのパスの本数は、途中の層の幅を掛け合わせた数だけあります。幅512の層を5つ通れば $512^5$ 本——列挙した瞬間に負けです。パスを「数え上げる」発想を捨てる必要があります。

そこで、視点をパスからノードに移します。これがこの章でいちばん大事な、鍵となる見方です。

> 各ノードは、**上流から来た勾配**に**自分の局所勾配**を掛けて、**下流に渡す**。それだけ。

言葉を決めておきます。グラフの出力(損失 $L$)に近い側を**上流**、入力に近い側を**下流**と呼びます。逆伝播では、勾配は上流から下流へ——forward と逆向きに——流れていきます。各ノードに届く「上流から来た勾配」とは、$\partial L / \partial (\text{自分の出力})$ のことです。ノードはそれを受け取り、自分の局所勾配(第2巻5章の局所微分のことです。以後、勾配の言葉で統一します)を掛けて、自分の入力たちへ配る。配られた側のノードも、同じことをする。これが端まで続くだけです。

このリレーが、なぜ「パスごとに掛けて足す」と同じ答えになるのでしょうか。勾配がノードからノードへ渡されるたびに局所勾配が1つ掛かるので、端に届いたときには、通ってきた辺の局所勾配が全部掛かっています——これがパスの積。そして合流点では届いた勾配が足される——これがパスの和。つまりこのリレーは、**すべてのパスの「掛けて足す」を、列挙せずに同時に実行している**のです。途中までの掛け算の結果が後続のノードたちに使い回されるので、無駄もありません。

ノードの「キャラクター」を3つ覚えると、逆伝播は手で追えるようになります。上流から勾配 $g$ が届いたとして:

- **加算ノードは分配器**。$z = x + y$ の局所勾配はどちらの入力についても $1$ ですから、届いた $g$ を**そのまま両方の入力にコピーして配ります**。
- **乗算ノードは入れ替え器**。$z = x \cdot y$ の局所勾配は「相方の値」でした(第2巻5章)。つまり $x$ 側には $g \cdot y$ を、$y$ 側には $g \cdot x$ を渡す。**forward で入ってきた値を、入れ替えて掛けて返す**わけです。
- **ReLU はゲート**。$\mathrm{ReLU}(x) = \max(0, x)$ の局所勾配は、$x > 0$ なら $1$、$x \le 0$ なら $0$。forward で通した成分は $g$ を素通しし、forward で $0$ に潰した成分は勾配も遮断します。

もうひとつ、ノードではありませんが大事な役者がいます。**分岐点**です。同じ値が2か所以上で使われていたら、backward ではそこに2方向から勾配が届きます。ルールは第2巻5章のとおり、**足す**。forward で値を配った場所には、backward で勾配が集まって返ってくる——「配ったら、集める」と覚えてください。この対称性は、すぐ次の節でバイアス $\mathbf{b}$ の勾配を出すときに効きます。

小さな数値例で、リレーを一周してみましょう。$L = (a + b) \cdot c$、値は $a = 2$、$b = 3$、$c = 4$ とします。forward は $a + b = 5$、$L = 20$。backward は出力から始めます。出発点はいつも $\partial L / \partial L = 1$ です。

1. 乗算ノード(入れ替え器): $c$ 側へ相方 $5$ を掛けて $1 \times 5 = 5$。$(a+b)$ 側へ相方 $4$ を掛けて $1 \times 4 = 4$
2. 加算ノード(分配器): 届いた $4$ をそのまま $a$ と $b$ にコピー

よって $\partial L/\partial a = 4$、$\partial L/\partial b = 4$、$\partial L/\partial c = 5$。検算すると $L = ac + bc$ なので $\partial L/\partial a = c = 4$、$\partial L/\partial c = a + b = 5$。一致です。どのノードも、グラフの全体像を一度も見ていないことに注意してください。**各ノードは自分の入出力しか知らないのに、リレーの結果として正しい勾配が端まで届く**。この局所性こそ、次章で backprop を自動化できる理由そのものです。

見方は手に入りました。いよいよ行列です。

## 3.3 2層MLPの backprop を図 + 行列で導出: δ の伝播、∂L/∂W = 入力^T @ δ(第1巻の転置がこんな所で効く)

前章のネットワークを、shape 込みで再掲します。入力 $X$ `(N, d_in)`、正解 $Y$ `(N, d_out)` として、

$$Z_1 = X W_1 + \mathbf{b}_1, \qquad H = \mathrm{ReLU}(Z_1), \qquad \hat{Y} = H W_2 + \mathbf{b}_2$$

パラメータは $W_1$ `(d_in, d_h)`、$\mathbf{b}_1$ `(d_h,)`、$W_2$ `(d_h, d_out)`、$\mathbf{b}_2$ `(d_out,)`。中間値は $Z_1, H$ `(N, d_h)`、$\hat{Y}$ `(N, d_out)` です。損失は第3巻のMSEを全成分で平均したもの、$L = \frac{1}{N d_{out}} \sum_{i,k} (\hat{Y}_{ik} - Y_{ik})^2$ とします。計算グラフはこうです。

```
X ──→ [@W1 + b1] ──Z1──→ [ReLU] ──H──→ [@W2 + b2] ──Ŷ──→ [MSE] ──→ L
           ↑                                 ↑               ↑
         W1, b1                            W2, b2            Y
```

図3.1: 2層MLPの計算グラフ(forward)。行列が左から右へ流れる。パラメータと正解 $Y$ は、途中のノードに横からぶら下がっている。

欲しいものをはっきりさせておきます。勾配降下の更新 $W \leftarrow W - lr \cdot \mathrm{grad}_W$ に使うのですから、欲しいのは4つのパラメータの勾配です。ここで、行列の世界での羅針盤をひとつ。**勾配は、対象と必ず同じ shape になります**。$\partial L / \partial W_2$ は「$W_2$ の各成分を動かしたら $L$ がどれだけ動くか」を成分ごとに並べたものですから、$W_2$ と同じ `(d_h, d_out)` です。導出の途中で shape が合わなくなったら、それはどこかで間違えた印です。

逆伝播の途中で各ノードに届く上流勾配には、伝統的に $\delta$(デルタ)という名前が付いています。これから $\delta$ が図3.1を右から左へ流れていく様子を、ノード1つぶんずつ追います。

**ステップ0 — MSEノード。** 出発点 $\partial L/\partial L = 1$ を受けたMSEノードが、$\hat{Y}$ に向けて最初の $\delta$ を発行します。$L$ を $\hat{Y}$ の1成分 $\hat{Y}_{ik}$ で微分すると(他の成分は定数扱い。第2巻4章の偏微分です)、$\frac{2}{N d_{out}}(\hat{Y}_{ik} - Y_{ik})$。全成分まとめて書けば

$$\delta_2 = \frac{\partial L}{\partial \hat{Y}} = \frac{2}{N d_{out}} (\hat{Y} - Y) \qquad \text{shape: } (N,\ d_{out})$$

$\hat{Y}$ と同じ shape です。羅針盤、ひとつめの確認。そして中身を見てください。$\delta_2$ は、定数倍を除けば**予測の誤差 $\hat{Y} - Y$ そのもの**です。これから下流へ流れていくのは、誤差(を各層の言葉に翻訳したもの)なのです。誤差**逆伝播**法という名前は、この景色から来ています。

**ステップ1 — 2層めの linear ノード。** $\hat{Y} = H W_2 + \mathbf{b}_2$ に、上流から $\delta_2$ が届きました。このノードの仕事は3つです。$W_2$ への勾配、$\mathbf{b}_2$ への勾配、そして下流の $H$ へ渡す勾配。1つずつ、スカラーの問いに落として出します。

(a) $W_2$ の勾配。成分 $W_2[j,k]$ は forward のどこで使われたでしょうか。行列積の定義(第1巻4章)に戻ると、$W_2[j,k]$ はサンプル $i$ ごとに $H[i,j] \cdot W_2[j,k]$ という形で $\hat{Y}[i,k]$ に寄与しています。つまり局所勾配は $\partial \hat{Y}[i,k] / \partial W_2[j,k] = H[i,j]$。そして $W_2[j,k]$ は $N$ 個のサンプル**全部**で使われた——分岐点です。3.2節のルールで、上流勾配 × 局所勾配をサンプルにわたって足します。

$$\frac{\partial L}{\partial W_2[j,k]} = \sum_{i=1}^{N} H[i,j]\ \delta_2[i,k]$$

この式の形に、見覚えはないでしょうか。「$H$ の列 $j$ と $\delta_2$ の列 $k$ の内積」です。それを全部の $(j, k)$ について並べたもの——**内積の総当たり表**。第1巻4章で、行列積の正体はこれだと学びました。総当たりにしたい列たちを行に立てるのが転置でしたから、

$$\boxed{\ \frac{\partial L}{\partial W_2} = H^T \delta_2\ } \qquad (d_h, N)\ @\ (N, d_{out}) \to (d_h, d_{out})$$

shape は $W_2$ と一致。**第1巻の転置が、こんな所で効きました。** 第1巻3章で「行の束を列の束に組み替える操作」として導入した $T$ は、ここでは「サンプルの束だった $H$ を特徴の束に組み替え、バッチ $N$ 件分の寄与を1本の内積に束ねる」仕事をしています。覚え方も第1巻流で構いません。手元にあるのは入力 $H$ `(N, d_h)` と $\delta_2$ `(N, d_out)`。この2つから $W_2$ の shape `(d_h, d_out)` を作れる行列積は、$H^T \delta_2$ しかないのです。shape を声に出して読む規律が、公式の暗記を不要にしてくれます。

(b) $\mathbf{b}_2$ の勾配。forward で $\mathbf{b}_2$ は、ブロードキャスティング(第1巻6章)によって $N$ 行すべてにコピーして足されました。コピーは分岐、足し算は分配器。3.2節の「配ったら、集める」です。$N$ 行に配ったのだから、backward では $N$ 行分の勾配を集めます。

$$\frac{\partial L}{\partial \mathbf{b}_2} = \sum_{i=1}^{N} \delta_2[i,:] \qquad \text{(} \delta_2 \text{ の列ごとの和。shape: } (d_{out},) \text{)}$$

(c) 下流へ渡す勾配。$H[i,j]$ は、行 $i$ の出力 $\hat{Y}[i,1], \ldots, \hat{Y}[i,d_{out}]$ のすべてに、それぞれ $W_2[j,k]$ 倍で寄与しています。これも分岐なので足して、$\partial L / \partial H[i,j] = \sum_k \delta_2[i,k]\ W_2[j,k]$。これは「$\delta_2$ の行 $i$ と $W_2$ の行 $j$ の内積」、つまりまた総当たり表で、

$$\frac{\partial L}{\partial H} = \delta_2 W_2^T \qquad (N, d_{out})\ @\ (d_{out}, d_h) \to (N, d_h)$$

また転置です。今度は $W_2$ 側に付きました。**forward で $W_2$ を掛けて進んだ道を、backward では $W_2^T$ を掛けて戻る**——きれいな対称性です。

**ステップ2 — ReLUノード。** ReLU は要素ごとの演算なので、ゲートの規則(3.2節)が成分ごとに働くだけです。$Z_1$ に届く勾配を $\delta_1$ と名付けると、

$$\delta_1 = \frac{\partial L}{\partial Z_1} = (\delta_2 W_2^T) \odot \mathbb{1}[Z_1 > 0] \qquad \text{shape: } (N,\ d_h)$$

$\odot$ は要素ごとの積(element-wise product。NumPy の `*`)、$\mathbb{1}[Z_1 > 0]$ は $Z_1$ の正の成分だけ $1$、他は $0$ の行列です。forward で生き残った成分の勾配だけが、ゲートを通過します。

**ステップ3 — 1層めの linear ノード。** $Z_1 = X W_1 + \mathbf{b}_1$ に $\delta_1$ が届きました。ここで気づいてほしいのです——**もう導出することがありません**。このノードはステップ1のノードと同じ種類で、3.2節で見たとおり、ノードは自分の入出力しか見ません。上流に何があろうと、レシピは同じです。

$$\frac{\partial L}{\partial W_1} = X^T \delta_1, \qquad \frac{\partial L}{\partial \mathbf{b}_1} = \delta_1 \text{ の列ごとの和}$$

shape は順に $(d_{in}, N)\ @\ (N, d_h) \to (d_{in}, d_h)$ と $(d_h,)$。どちらもパラメータと一致します(下流の $X$ へ渡す $\delta_1 W_1^T$ も作れますが、$X$ はデータであって学習対象ではないので、今日はここで止まります)。

以上を1枚にまとめます。これが2層MLPの backprop の全貌です。

```
        grad_W1 = Xᵀ@δ1          δ1 = (δ2@W2ᵀ)⊙ゲート      grad_W2 = Hᵀ@δ2
        grad_b1 = δ1の列和                                  grad_b2 = δ2の列和
           ↑                          ↑                        ↑
X ←──── [@W1 + b1] ←──δ1──── [ReLU] ←──δ2@W2ᵀ──── [@W2 + b2] ←──δ2──── [MSE] ←── 1
```

図3.2: 同じグラフを逆向きに流れる $\delta$。各 linear ノードは、届いた $\delta$ から「重みの勾配・バイアスの勾配・下流へのメッセージ」の3点セットを機械的に作る。

$$
\begin{aligned}
\delta_2 &= \tfrac{2}{N d_{out}}(\hat{Y} - Y) &&(N, d_{out})\\
\mathrm{grad}_{W_2} &= H^T \delta_2 &&(d_h, d_{out})\\
\mathrm{grad}_{\mathbf{b}_2} &= \textstyle\sum_i \delta_2[i,:] &&(d_{out},)\\
\delta_1 &= (\delta_2 W_2^T) \odot \mathbb{1}[Z_1 > 0] &&(N, d_h)\\
\mathrm{grad}_{W_1} &= X^T \delta_1 &&(d_{in}, d_h)\\
\mathrm{grad}_{\mathbf{b}_1} &= \textstyle\sum_i \delta_1[i,:] &&(d_h,)
\end{aligned}
$$

6行です。前章2.4節で何枚もの紙を埋めた $\partial L / \partial W_1$ の展開が、この6行に畳まれました。あのとき書き下した添字の海は、いま $\delta_1$ という1個の行列の中に折り畳まれています。地獄の原因は、数学が難しかったことではなく、**全体を一度に展開しようとしたこと**でした。グラフは1ノードずつ、局所的に処理します。そして使った道具を数えてみてください——連鎖律(第2巻5章)、転置と行列積(第1巻3・4章)、ブロードキャスティング(第1巻6章)。**新しい数学は、ひとつもありません。**

このレシピの強さは、一般性にあります。層が3つでも100でも、各 linear 層がやることは「届いた $\delta$ から3点セットを作る」の繰り返しです。損失をMSEから別のものに替えても、変わるのはステップ0の1行だけで、層側のレシピは1文字も変わりません(第4巻の cross-entropy でこれを実演するのは第5章です)。

ひとつ白状しておくと、この導出はまだ「私たちがそう導いた」にすぎません。本当に正しいのか? 裏を取る道具を、私たちは第2巻1章から持ち歩いています。数値微分です。章末の演習で、この6行に検算の裏を取らせます。

## 3.4 なぜ「逆向き」なのか: forward-mode との比較を直観レベルで(パラメータが多く出力が1個だから逆が得)

最後に、ずっと棚上げにしてきた問いに答えます。第2巻5章の終わりで「なぜ『逆向き』だと1回で済むのか、という掘り下げは第5巻でやります」と予告した、その回収です。

連鎖律は掛け算の列でした。掛け算の列は、どちらの端から畳んでも値は同じです。だとすると、入力側から——forward と同じ向きに——微分を運ぶこともできるはずです。実際この方式には forward-mode という名前があり、ちゃんと機能します。$W_1$ のある1成分を選び、「この成分が動いたら $Z_1$ がどう動くか、すると $H$ が、$\hat{Y}$ が、$L$ が……」と感度を前向きに運んでいけば、最後にその成分の勾配が1個得られます。

問題は問いの形です。forward-mode の1回のパスが答えるのは「**この1個のパラメータ**は、出力にどう効くか」。では、パラメータは何個あるでしょうか。第1巻6章の演習で数えました——論文のFFN、たった2層で約210万個です。forward-mode では210万回のパスが要ります。

reverse-mode(逆向き)の1回のパスが答えるのは、逆の問いです。「**この1個の出力**に、すべてのパラメータはそれぞれどう効くか」。では、出力は何個あるでしょうか。**1個です。** 損失 $L$ というスカラー、ただひとつ。だから backward は1回で済むのです。第2巻5章のコードで「backward を1回呼ぶと $\partial z/\partial x$ と $\partial z/\partial y$ が同時に出た」のは、この性質のいちばん小さな現れでした。3.3節でも、たった1往復で4つのパラメータの勾配が全部出ています。

直観的には、こう言えます。出力側から畳むと、途中までの掛け算の結果——まさに $\delta$ ——が、それより下流の**すべての**パラメータで使い回せます。$\delta_1$ を一度作れば、$W_1$ の210万成分がそれを共有する。入力側から畳むと、この共有が効きません(逆に、入力が1個で出力が大量にある計算なら forward-mode が得です。ニューラルネットの学習は、その正反対の形をしているというだけです)。

ただし、逆向きはタダではありません。図3.2を見直すと、backward は forward の中間値($H$ は grad の材料に、$Z_1$ はゲートの判定に)を使っています。つまり**逆伝播は、順伝播の途中結果をすべて覚えておくことを要求します**。第2巻5章の `forward_chain` が中間値を律儀に返していたのは、このためでした。そして次章で作る `Value` クラスの設計——値と勾配と「親と演算」を覚える箱——は、この要求にそのまま応えるものです。

材料は揃いました。レシピは6行、ノードの規則は3つ、向きは逆、中間値は覚えておく。あとは、これを人間がやるのをやめるだけです。

## まとめ

- 第2巻5章の計算グラフが正式な名前を得た: forward で値を流す**順伝播**、backward で勾配を流す**逆伝播**。逆伝播で勾配を求める方法が**誤差逆伝播法**(backprop)
- 鍵となる見方: 各ノードは「上流から来た勾配 × 自分の局所勾配」を下流に渡す**だけ**。加算は分配器、乗算は入れ替え器、ReLU はゲート、分岐の合流は足す。パスの列挙は不要になり、局所性が次章の自動化を可能にする
- linear 層のレシピは全層共通の3点セット: $\mathrm{grad}_W = \text{入力}^T @\ \delta$、$\mathrm{grad}_b = \delta$ の列和、下流へは $\delta\ @\ W^T$。第1巻の転置が、バッチ $N$ 件の寄与を内積に束ねる役で効く
- $\delta$ は各層に届く上流勾配で、出力層では(定数倍を除き)予測誤差 $\hat{Y} - Y$ そのもの。勾配は常に対象と同じ shape——導出の検算に使える羅針盤
- 「逆向き」が得なのは、出力(損失)が1個でパラメータが膨大だから。backward 1回で全勾配が出る。代償は、順伝播の中間値をすべて記憶しておくこと

**ラスボスとの距離**: 論文の式(2) $\mathrm{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$ は、今日 backprop を導出したネットワークそのものの形です。論文のFFNを**学習させる**ための勾配が、もう紙の上で出せます。

## 演習

**問1**(ノードの規則で手計算)$L = (a \cdot b + c)^2$、値は $a = 2$、$b = -1$、$c = 5$ とします。計算グラフを描き、3.2節のノードの規則(分配器・入れ替え器、2乗の局所勾配は $2u$)だけを使って、出力側から順に $\partial L/\partial a$、$\partial L/\partial b$、$\partial L/\partial c$ を求めてください。式を展開して偏微分した結果と一致することも確かめること。

**問2**(shape で backprop を読む)第1巻6章の演習に登場した3層ネットワーク——入力 $X$ `(64, 300)`、$W_1$ `(300, 100)`、$W_2$ `(100, 100)`、$W_3$ `(100, 10)`、各層の間に ReLU——について、出力に届く上流勾配 $\delta_3$ `(64, 10)` から始めて、$\mathrm{grad}_{W_3}$、$\delta_2$、$\mathrm{grad}_{W_2}$、$\delta_1$、$\mathrm{grad}_{W_1}$ を3.3節のレシピで書き下し、それぞれの shape を答えてください(数値は不要。式と shape だけで構いません)。

**問3**(数値微分で検算 — この章の総仕上げ)3.3節の6行を NumPy に書き写し、第2巻1章の数値微分(中心差分)で検算してください。手順: (1) 小さな2層MLP($N=8$、$d_{in}=3$、$d_h=4$、$d_{out}=2$ 程度)の forward と、6行の backward を実装する。(2) 各パラメータの各成分を $\pm h$ だけ動かして損失を測り直す「行列版の数値微分」を書く。(3) 4つの勾配すべてが `np.allclose` で一致することを assert で確かめる。

**問1** グラフは $a, b \to [\times] \to u$、$u, c \to [+] \to v$、$v \to [\text{2乗}] \to L$。forward: $u = -2$、$v = 3$、$L = 9$。backward: 出発点 $\partial L/\partial L = 1$。2乗ノードの局所勾配は $2v = 6$ なので $v$ に $6$ が届く。加算ノードは分配器なので $u$ と $c$ にそのまま $6$ ずつ。乗算ノードは入れ替え器なので $a$ には $6 \times b = -6$、$b$ には $6 \times a = 12$。よって $\partial L/\partial a = -6$、$\partial L/\partial b = 12$、$\partial L/\partial c = 6$。検算: $\partial L/\partial a = 2(ab+c) \cdot b = 2 \times 3 \times (-1) = -6$ など、すべて一致します。

**問2** 上流から順に($Z_\ell$ を各層の linear 出力、$H_\ell = \mathrm{ReLU}(Z_\ell)$ とします):

- $\mathrm{grad}_{W_3} = H_2^T \delta_3$: $(100, 64)\ @\ (64, 10) \to (100, 10)$
- $\delta_2 = (\delta_3 W_3^T) \odot \mathbb{1}[Z_2 > 0]$: $(64, 10)\ @\ (10, 100) \to (64, 100)$
- $\mathrm{grad}_{W_2} = H_1^T \delta_2$: $(100, 64)\ @\ (64, 100) \to (100, 100)$
- $\delta_1 = (\delta_2 W_2^T) \odot \mathbb{1}[Z_1 > 0]$: $(64, 100)\ @\ (100, 100) \to (64, 100)$
- $\mathrm{grad}_{W_1} = X^T \delta_1$: $(300, 64)\ @\ (64, 100) \to (300, 100)$

どの勾配も対応する $W$ と同じ shape、どの $\delta$ も対応する $Z$ と同じ shape になっています。バッチの 64 は $\delta$ の中だけを流れ、$\mathrm{grad}_W$ では転置との行列積に吸われて消える——この対比が「転置がバッチの寄与を束ねる」の意味です。

**問3** 解答例の全文です。

```python
import numpy as np


def numerical_grad(loss_fn, P, h=1e-5):
    """中心差分による勾配(第2巻1章の数値微分を、配列の全成分に1つずつ適用した版)"""
    grad = np.zeros_like(P)
    for idx in np.ndindex(P.shape):
        orig = P[idx]
        P[idx] = orig + h
        loss_plus = loss_fn()
        P[idx] = orig - h
        loss_minus = loss_fn()
        P[idx] = orig                        # 動かした成分は必ず元に戻す
        grad[idx] = (loss_plus - loss_minus) / (2 * h)
    return grad


rng = np.random.default_rng(42)
N, d_in, d_h, d_out = 8, 3, 4, 2
X = rng.standard_normal((N, d_in))
Y = rng.standard_normal((N, d_out))
W1 = rng.standard_normal((d_in, d_h))
b1 = rng.standard_normal(d_h)
W2 = rng.standard_normal((d_h, d_out))
b2 = rng.standard_normal(d_out)


def forward():
    """2層MLP + MSE。backward が局所勾配の計算に使う中間値も返す(第2巻5章と同じ2段構え)"""
    Z1 = X @ W1 + b1                         # (N, d_h)
    H = np.maximum(Z1, 0.0)                  # ReLU (N, d_h)
    Y_hat = H @ W2 + b2                      # (N, d_out)
    loss = np.mean((Y_hat - Y) ** 2)         # スカラー
    return Z1, H, Y_hat, loss


def backward(Z1, H, Y_hat):
    """3.3 の6行。各 linear 層のレシピ: grad_W = 入力.T @ δ、grad_b = δ の列和、下流へ δ @ W.T"""
    delta2 = 2 * (Y_hat - Y) / Y.size        # ∂L/∂Ŷ: MSE ノードの局所勾配 (N, d_out)
    grad_W2 = H.T @ delta2                   # (d_h, d_out)
    grad_b2 = delta2.sum(axis=0)             # (d_out,)
    delta1 = (delta2 @ W2.T) * (Z1 > 0)      # ReLU のゲート: 0/1 のマスク (N, d_h)
    grad_W1 = X.T @ delta1                   # (d_in, d_h)
    grad_b1 = delta1.sum(axis=0)             # (d_h,)
    return grad_W1, grad_b1, grad_W2, grad_b2


def loss_fn():
    return forward()[3]


Z1, H, Y_hat, loss = forward()
grad_W1, grad_b1, grad_W2, grad_b2 = backward(Z1, H, Y_hat)

# 勾配は必ずパラメータ自身と同じ shape(更新 W ← W − lr * grad_W に使うため)
assert grad_W1.shape == W1.shape and grad_b1.shape == b1.shape
assert grad_W2.shape == W2.shape and grad_b2.shape == b2.shape

# 数値微分との照合: 4つのパラメータすべて、全成分が一致する
assert np.allclose(grad_W1, numerical_grad(loss_fn, W1))
assert np.allclose(grad_b1, numerical_grad(loss_fn, b1))
assert np.allclose(grad_W2, numerical_grad(loss_fn, W2))
assert np.allclose(grad_b2, numerical_grad(loss_fn, b2))

print("ok: すべての assert を通過しました(backprop と数値微分が一致)")
```

数値微分は backprop の導出を何ひとつ知らないのに、4つの勾配の全成分が一致します。`backward` の本体が、図3.2の $\delta$ の流れと1行ずつ対応していることも確認してください。なお、ReLU の折れ目($Z_1$ の成分がちょうど $0$ 近傍)では数値微分が不正確になりえますが、乱数データで成分が $\pm h$ の幅に落ちることはまずありません。次章では、この「数値微分と照合する」テストを、自作 autograd の全演算に対して張りめぐらせます。
