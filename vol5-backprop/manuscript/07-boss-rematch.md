# 終章 ラスボス再戦 — アーキテクチャ図の箱が読める

第6章までで、この巻の部品はすべて揃いました。活性化関数と ReLU(第1章)、MLP(第2章)、誤差逆伝播(第3章)、自作の autograd(第4章)、autograd による学習(第5章)、そして residual connection・layer norm・dropout(第6章)。

序章で、私たちはひとつの約束をしました。この巻を終えたとき、**論文のアーキテクチャ図(図1)の、attention 以外の箱が全部読める**ようになっている、という約束です。いまから、それを確かめます。

やり方はいつもどおり。**巻頭に掲げたものとまったく同じ引用を、もう一度掲げる。そして今度は、1記号ずつ、1文ずつ読んでいく。** 第1巻から続けてきた儀式の、これが第5回です。ただし今回は趣が少し違います。第1巻のラスボスは1本の式でした。今回は式が1本と、英語の文が2つ。数式だけでなく**論文の地の文**まで読みにいきます。

## 7.1 式(2)を読む — ReLU を挟んだ2層 linear

序章 0.2 に掲げた引用の1つ目を、一言一句変えずに再掲します。

> $$\mathrm{FFN}(x) = \max(0,\ xW_1 + b_1)W_2 + b_2$$
> — Vaswani et al., "Attention Is All You Need", Section 3.3, 式(2)

序章でこの式を見たとき、$\max(0,\ \cdot)$ が何の操作なのかも、なぜ $W$ が2つあるのかも、わかりませんでした。いまは内側から、1つずつ読めます。例によって shape 注釈を付けながら。論文の設定では $d_{model} = 512$、$d_{ff} = 2048$ です。

**$x$ `(1, 512)`** — 入力です。$d_{model} = 512$ 次元の行ベクトルが1本。実際には $n$ 本まとめて `(n, 512)` で流しますが、まずは1本で読みます。

**$xW_1 + b_1$ `(1, 2048)`** — 第1巻以来の背骨、`X @ W + b` です。$W_1$ `(512, 2048)` との積で、512次元のベクトルが2048次元に**広がります**。$b_1$ `(2048,)` はバイアス。第2章 2.2 の「隠れ層の幅は設計変数だ」への論文の解答がこれです: $d_{ff} = 2048$、つまり $d_{model}$ のちょうど4倍。

**$\max(0,\ \cdot)$ `(1, 2048)`** — 第1章の **ReLU** です。成分ごとに、負なら0、正ならそのまま。shape は変わりません。序章では正体不明だったこの記法は、いまや「折れ線で何でも作る部品」(第1章 1.4)として読めます。これが入っているおかげで、線形 ∘ 線形 = 線形(第1章 1.2)の罠を回避し、線形では割れない関係を表現できます。

**$(\cdots)W_2 + b_2$ `(1, 512)`** — もう一度 linear。$W_2$ `(2048, 512)` との積で、2048次元から元の512次元に**戻ります**。$b_2$ `(512,)`。

まとめましょう。式(2)の全体は、

$$ \underbrace{x}_{(1,\ 512)} \;\to\; \underbrace{xW_1 + b_1}_{(1,\ 2048)} \;\to\; \underbrace{\max(0,\ \cdot)}_{(1,\ 2048)} \;\to\; \underbrace{(\cdots)W_2 + b_2}_{(1,\ 512)} $$

**linear → ReLU → linear。隠れ層が1つの、ただの MLP です。** 第2章で定義し、第5章で autograd の上に組んで訓練ループを回したあのコード——`linear → relu → linear`——そのものです。論文が "Feed-Forward Networks" と仰々しく名付けたこの箱の中身を、あなたはすでに自分の手で書き、自分の autograd で学習させています。

512 → 2048 → 512 という「一度4倍に広げてから戻す」形も読めます。広げた2048次元は、ReLU で折り曲げながら特徴を混ぜ合わせるための作業場です。入口と出口が同じ512次元なのは、次の 7.2 で読む residual connection——$x$ と足し算するには shape が揃っていなければならない——のため。式どうしが shape で噛み合っていることが見えます。

1つだけ、まだ読めない言葉を残しておきます。論文のセクション名にある "Position-wise"(位置ごとに)です。この式が `(n, 512)` の**各行に同じ $W_1, W_2$ を適用する**ことを指しているのは shape から読めますが、その「行」が何を表すのか——文章の中の単語の**位置**——は、言語を数値にする方法(第6巻)を待つ必要があります。

## 7.2 LayerNorm(x + Sublayer(x)) を読む — 第6章と1対1対応

引用の残り2つも、序章とまったく同じものを再掲します。今度は数式ではなく、論文の地の文です。

> *"We employ a residual connection around each of the two sub-layers, followed by layer normalization. That is, the output of each sub-layer is LayerNorm(x + Sublayer(x))."*
> — 同論文, Section 3.1
>
> *"We apply dropout to the output of each sub-layer, before it is added to the sub-layer input and normalized."*
> — 同論文, Section 5.4

訳: 「2つの sub-layer のそれぞれの周りに residual connection を設け、その後に layer normalization を適用する。すなわち、各 sub-layer の出力は LayerNorm(x + Sublayer(x)) である。」「各 sub-layer の出力には、それが sub-layer の入力に加算され正規化される前に、dropout を適用する。」

序章では residual も LayerNorm も dropout も全部が未知の単語でした。いまは1語ずつ、第6章の実装に対応させながら読めます。

**"a residual connection around each of the two sub-layers"** — 第6章 6.2 の $x + f(x)$ です。$\mathrm{Sublayer}(x)$ が $f(x)$、つまり「迂回される側」の変換。理由も言えます。backprop のとき、足し算のノードは上流の勾配をそのまま素通しします(6.2 で確かめた「$+1$ が効く」)。層を深く積んでも勾配が消えずに入口まで届くための配管でした。第6章 6.1 で勾配消失を自分の目で観測したあなたには、これは「おまじない」ではなく「対策」として読めます。

**"followed by layer normalization"** — 第6章 6.3 の layer norm。足した結果を、サンプルごとに平均0・分散1に整え、スケール $\gamma$ とシフト $\beta$ で調整する。まとめて $\mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$。論文のこの式は、第6章 6.5 で書いた実装と記号まで同じ形です。

**Section 5.4 の dropout の文**は、適用の**順序**を定めています。"before it is added ... and normalized"——加算と正規化の**前に**、sub-layer の出力へ dropout をかける。第6章 6.4 の言葉で言えば「訓練時だけランダムに消す」のは $\mathrm{Sublayer}(x)$ の出力であって、迂回路を通る $x$ 本体ではありません。3つの文を1行に合成すると、こうなります。

$$ \mathrm{LayerNorm}\big(x + \mathrm{Dropout}(\mathrm{Sublayer}(x))\big) $$

これは第6章 6.5 で深い MLP に組み込んで効果を比較した、あのコードの構造と1対1対応です。論文の図1で "Add & Norm" と書かれた箱の中身は、この1行です。

ここで、論文の "each of the **two** sub-layers" という言い回しに目を留めてください。encoder の各ブロックには sub-layer が2つある、と論文は言っています。1つはいま読めるようになった Feed-Forward。そしてもう1つが——Multi-Head Attention。読める箱と読めない箱の境界線が、論文の文の中にそのまま引かれています。次の節で、この境界線を図1の全体にわたって引きます。

## 7.3 図1の部品チェックリスト — シリーズの現在地

序章で眺めた図1を、もう一度開いてください。あのときは「箱がたくさんある」としか言えませんでした。いまは箱を1つずつ指さして、読める・読めないを宣言できます。棚卸しです。

**読める箱:**

| 箱 | 正体 | どこで読めるようになったか |
|---|---|---|
| Feed Forward | ReLU を挟んだ2層 linear(式2) | 本巻 第1・2・5章 |
| Add & Norm | $\mathrm{LayerNorm}(x + \mathrm{Dropout}(\mathrm{Sublayer}(x)))$ | 本巻 第6章 |
| Dropout | 訓練時だけランダムに消す正則化(図には箱として描かれませんが、5.4節がすべての sub-layer 出力への適用を定めています) | 本巻 第6章 |
| Softmax(出力部) | スコアの並びを確率分布に変える関数 | 第4巻 |
| Linear(出力部) | `X @ W + b`。シリーズの背骨 | 第1巻(学習の仕組みは第2〜3巻) |

**まだ読めない箱:**

| 箱 | 何が足りないか | どの巻で読めるようになるか |
|---|---|---|
| Multi-Head Attention | attention そのもの。式(1)の配管は第1巻で読めましたが、なぜ類似度の表で文が処理できるのか、なぜヘッドが複数要るのかはまだです | 第7巻 |
| Positional Encoding | 語順——「位置」を数値で表す方法 | 第7巻 |
| Embedding(Input / Output) | そもそも単語をベクトルにする方法 | 第6巻 |

地図の上で現在地を確認しましょう。図1は、大きく見れば「入口(Embedding + Positional Encoding)→ 本体(sub-layer の積み重ね)→ 出口(Linear + Softmax)」という3階建てです。**出口は全部読めます。本体も、2種類の sub-layer のうち1つ(Feed Forward)と、すべての sub-layer を包む配管(Add & Norm、Dropout)が読めます。残っているのは、本体の心臓部が1つと、入口がまるごと。** 序章の約束——attention 以外の箱が全部読める——は、果たされました。

巻頭付録の論文読解マップを開いて、第5巻の行に印を付けてください。第1巻の終章で初めて灯りをともしてから4巻ぶん、読める領域はここまで広がりました。

そしてもうひとつ、この巻にはチェックリストに載らない獲得物があります。第4章で自作した autograd です。図1のどの箱を実装するときも、もう勾配を手で導出する必要はありません。第2章 2.4 の地獄が、私たちの最後の手導出でした。これから先の巻で書くすべての部品は、`loss.backward()` の一声で学習可能になります。ラスボスと戦うための武器が、この巻で恒久装備になりました。

## 7.4 次巻予告 — そもそも言語をどうやって数値にするのか

まだ読めない箱は3種類。次は Multi-Head Attention に挑みたくなるところですが、その前に、避けて通れない問いがあります。

図1の**一番下**、すべての矢印の出発点を見てください。"Inputs" と書かれた場所から、Embedding の箱に矢印が入っています。この論文は翻訳のモデルです。つまり Inputs とは、**文章**です。

ここまでのシリーズで、入力 $X$ はいつも数値でした。家賃を予測する物件の面積、XOR の平面上の点。数値だったからこそ `X @ W + b` に流せました。しかし「私は猫が好きだ」という文は、行列ではありません。**言語を、どうやって `(n, 512)` の数値の行列にするのか。** この問いに答えない限り、図1には何も流せません。7.1 で保留にした "Position-wise" の「位置」も、7.3 の表に残った Embedding と Positional Encoding も、すべてこの問いの先にあります。

第6巻「言語をベクトルにする — Transformer までの NLP」で、この問いに答えます。単語を数値にする方法(トークン化と埋め込み)、「言語モデル」という問題設定、そして attention 以前の手法たち——RNN や seq2seq——が何に苦しんでいたのかを、実装で体感します。そこまで行って初めて、この論文のタイトル "Attention Is All You Need"——**attention だけ**でいい——が、何に対する宣言なのかが刺さるようになります。

箱の前に、箱に流す中身です。言語を数値にしに行きましょう。
