# 序章 ラスボスとの対面

## 0.1 ここまでの到達点 — 式(1)は読めた。だが図1には、まだ箱がある

おかえりなさい。第5巻です。

新しい巻に入る前に、前巻の終わりに起きたことを思い出しておきましょう。第4巻の終章で、このシリーズにとって初めての出来事がありました。論文の式(1)——

$$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

——の**すべての記号が読める**ようになったのです。$Q$、$K$、$V$、$QK^T$、転置は第1巻。softmax と $\sqrt{d_k}$ は第4巻。第1巻の序章で「おそらく、何ひとつ読めないはずです」と掲げられたあの式を、あなたはいま、shape から分散の議論まで含めて説明できます。シリーズの折り返し地点として、これ以上ない戦果です。

では、次はどこへ向かうのか。

論文を開いて、3ページめの**図1**を見てください。Transformer のアーキテクチャ図——2つの塔が並び、中にいくつもの箱が積み上がっている、あの有名な図です。箱のラベルを上から読み上げてみましょう。Softmax。Linear。**Add & Norm**。**Feed Forward**。Add & Norm(また出てきました)。Multi-Head Attention。Positional Encoding。Embedding。

このうち、いま読めるのはどれでしょうか。Softmax は第4巻で倒しました。Linear は第1巻からの長い付き合いです。Multi-Head Attention の箱も、中身の式(1)はもう読めます(「Multi-Head」の部分が何なのかは、まだです)。

残るは **Feed Forward** と、2回も登場する **Add & Norm**。さらに論文の本文には、図には描かれていないもうひとつの部品——**Dropout**——が待ち構えています。

先に宣言しておきます。**この巻を終えると、図1の attention 以外の箱が、全部読めるようになります。**

## 0.2 ラスボスの掲示 — 部品は読めるのに、意図が読めない

では、今回のラスボスに会いに行きましょう。3箇所から引きます。

> $$\mathrm{FFN}(x) = \max(0,\ xW_1 + b_1)W_2 + b_2$$
> — Vaswani et al., "Attention Is All You Need", Section 3.3, 式(2)
>
> *"We employ a residual connection around each of the two sub-layers, followed by layer normalization. That is, the output of each sub-layer is LayerNorm(x + Sublayer(x))."*
> — 同論文, Section 3.1
>
> *"We apply dropout to the output of each sub-layer, before it is added to the sub-layer input and normalized."*
> — 同論文, Section 5.4

英文のほうを訳しておきます。Section 3.1 は「2つの各サブ層のまわりに residual 接続を施し、その後に層正規化を行う。すなわち、各サブ層の出力は LayerNorm(x + Sublayer(x)) である」。Section 5.4 は「各サブ層の出力に dropout を適用する。サブ層の入力に加算され正規化される前に、である」。

さて、今回の「読めなさ」は、これまでの巻とは手触りが違います。

式(2)をよく見てください。$xW_1 + b_1$——読めますよね。第1巻からずっと使ってきた、あの linear そのものです。その結果に $W_2$ を掛けて $b_2$ を足す。linear をもう1回。つまりこの式は、**行列計算としての骨格なら、すでに読めてしまう**のです。

読めないのは、真ん中に鎮座する $\max(0,\ \cdot)$ だけ。「0 と比べて大きいほうを取る」——たったそれだけの、拍子抜けするほど単純な操作です。しかし、なぜそんなものが式の中心にいるのでしょうか。そもそも、なぜ linear を2回重ねるのでしょうか。第1巻5.3で確認したことを思い出すと、これはむしろ不可解です。**線形変換の合成は1つの線形変換にまとめられる**のでした。$W_1$ と $W_2$ を2枚用意しても、1枚の行列と同じことしかできないはず——では、この式は何のために2枚使っているのか。

実は、その答えの鍵こそが $\max(0,\ \cdot)$ です。部品はぜんぶ知っているのに、設計の意図がまったく読めない。それが式(2)の読めなさです。

Section 3.1 と 5.4 の文章は、もっと率直に読めません。residual connection(迂回路?)、sub-layer(層の下に何かあるのか?)、layer normalization(層を……正規化?)、dropout(何かを落とす?)。単語が軒並み初耳です。そしてこれらの部品は、図1で attention の箱を上下から挟んでいる、あの「Add & Norm」の正体でもあります。

この巻を通して、これらの部品を1つずつ——なぜ必要なのか、という需要から——手に入れていきます。

## 0.3 この巻のもう一つの主役 — もう手で微分したくない

ところで、この巻には、上のラスボス引用のどこにも写っていない、もう1人の主役がいます。

ここまでのシリーズで、勾配が必要になるたびに、私たちが何をしてきたかを思い出してください。第3巻では、線形回帰の損失 $L$ をパラメータで偏微分して、$\partial L / \partial w$ を**手で**導出しました。第4巻では、sigmoid + log loss の勾配を**手で**導出し、$(y - t)\,x$ という美しい形に辿り着きました。softmax + cross-entropy でも、また**手で**導出しました。

毎回、手でやってきたのです。そして正直に言えば、毎回それなりにしんどかったはずです。

この巻で作るニューラルネットワークは、linear と活性化関数を交互に、何層も重ねます。パラメータは $W_1, b_1, W_2, b_2, \ldots$ と層の数だけ増えていきます。この構造に対して $\partial L / \partial W_1$ を手で導出したらどうなるか——第2章で、一度だけ、実際にやってもらいます。先に予告しておくと、地獄です。そしてその地獄を見たあとで、約束します。**それがあなたの最後の手導出になります。**

もう限界です。微分を、自動化します。

仕組みの土台は、**誤差逆伝播(backpropagation)**。第2巻5章で手に入れた連鎖律と計算グラフが、この巻の第3章で、ついに主役として帰ってきます。そして第4章で、この巻の最大の山に挑みます——**自動微分(automatic differentiation)のエンジンを、自分の手で作る**のです。

身構えたかもしれませんが、規模を先に言っておくと、100行ちょっとです。値と勾配を覚える小さなクラスに、足し算・掛け算・$\max(0,\ \cdot)$ などの演算を1つずつ生やしていく。完成したエンジンに第3巻の回帰と第4巻のロジスティック回帰を解かせると、あなたが汗をかいて手導出したあの勾配たちが、**すべて自動で**出てきます(第5章)。これは、PyTorch という有名なライブラリで `loss.backward()` と書いたときに起きていることの、正体そのものでもあります。構造は同じで、規模が違うだけです。

ラスボスの部品(FFN、residual、layer norm、dropout)を揃える巻であると同時に、シリーズの残り3巻すべてを支える計算エンジンを鍛造する巻。それが第5巻です。

## 0.4 この巻で扱わないこと

恒例の白状コーナーです。今回は3つあります。

1つめ。**CNN(畳み込みニューラルネットワーク)を扱いません。** ニューラルネットの教科書なら、まず間違いなく1章を割く話題です。しかし、論文の Abstract にはこう書いてあります——"dispensing with recurrence and convolutions entirely"(再帰と畳み込みを完全に排して)。畳み込みは、この論文が「捨てる」と宣言した道具です。捨てられる側を学んでから捨てるほど、私たちの道のりは長くありません。

2つめ。**RNN(再帰型ニューラルネットワーク)も、この巻では扱いません。** こちらは事情が少し違います。RNN は同じ Abstract で捨てられているにもかかわらず、第6巻でしっかり登場します。「attention だけでいい」というタイトルの意味を理解するには、RNN の痛み——並列化できない、遠くを覚えられない——を自分で体感する必要があるからです。つまり RNN は、必要になってから学びます。いつもの需要駆動です。この巻で作るネットワークは、時間や順序の概念を持たない、いちばん素直な多層パーセプトロンだけです。

3つめ。**重みの初期化の理論を、一般論としては展開しません。** 深いネットワークでは「最初の乱数の選び方」が思いのほか効くのですが、Xavier 初期化に代表される体系的な理論には立ち入らず、第6章で**使う分だけ**触れます。そこで行う計算は、実は第4巻7章で $\sqrt{d_k}$ を導いたのと同じ、分散の計算です。あの道具がもう一度だけ働きます。

## 巻頭付録 論文読解マップについて

シリーズ共通の巻頭付録、**論文読解マップ**(`paper-map.md`)で現在地を確認しておきましょう。

第4巻までに「読める」側へ移ったのは、式(1)の全記号、Section 5 の training と batching の語彙、そして learning rate の式の意図でした。地図の上では、Section 3.2(attention の心臓部)と Section 5 の一部に旗が立っている状態です。

今回は、Section 3 の残りの平野部——3.3 の FFN と、3.1 の後半の LayerNorm(x + Sublayer(x))——に加えて、Section 5.4 の dropout に旗を立てにいきます。第5巻が担当するのは、地図の中の次の行です。

| 論文の箇所 | 内容 | この巻の章 |
|---|---|---|
| 3.3 式(2) | $\mathrm{FFN}(x) = \max(0,\ xW_1 + b_1)W_2 + b_2$、ReLU、$d_{ff} = 2048$ | 第1・2章 |
| 3.1 | $\mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$(residual + layer norm) | 第6章 |
| 5.4 | dropout($P_{drop} = 0.1$) | 第6章 |

この巻を終えると、図1のアーキテクチャ図は「読める箱」が多数派になります。残るは Multi-Head Attention、Positional Encoding、Embedding——いずれも第6巻以降の獲物です。

それでは、始めましょう。最初の問いはこれです——第1巻からずっと頼ってきた `X @ W + b` に、実は**どうしてもできないこと**があります。それは何でしょうか。第1章では、まずその限界に正面からぶつかります。
